# Show-o와 이산 확산 통합 모델

> Transfusion은 연속 및 이산 표현을 혼합합니다. Show-o(Xie et al., 2024년 8월)는 반대 방향으로 접근합니다: 텍스트 토큰은 인과적 다음 토큰 예측을 사용하고, 이미지 토큰은 MaskGIT의 정신을 따르는 마스킹된 이산 확산을 사용합니다. 둘 다 하이브리드 어텐션 마스크를 가진 하나의 트랜스포머 안에 통합됩니다. 결과적으로 VQA, 텍스트-이미지 생성, 인페인팅, 혼합 모달리티 생성이 하나의 백본, 모달리티당 하나의 토크나이저, 하나의 손실 함수(다음 토큰 예측을 마스킹된 예측으로 확장) 아래에서 통합됩니다. 이 강의에서는 Show-o 설계를 살펴보고 — 마스킹된 이산 확산이 병렬적 소수 단계 이미지 생성기인 이유 — Transfusion 및 Emu3와의 차이점을 대조합니다.

**유형:** 학습  
**언어:** Python (표준 라이브러리, 마스킹된 이산 확산 샘플러)  
**선수 조건:** 12단계 · 13단계 (Transfusion)  
**소요 시간:** ~120분

## 학습 목표

- **마스킹된 이산 확산(Masked Discrete Diffusion)** 설명: 토큰을 균일하게 마스킹한 후 트랜스포머가 이를 복원하도록 하는 스케줄.
- **병렬 이미지 디코딩(Show-o, MaskGIT)**과 **자기회귀적 이미지 디코딩(Chameleon, Emu3)**을 속도 및 품질 측면에서 비교.
- **Show-o**가 하나의 체크포인트에서 처리하는 세 가지 작업 이름: **텍스트-이미지 생성(T2I)**, **시각적 질의응답(VQA)**, **이미지 인페인팅**.
- **마스킹 스케줄(코사인, 선형, 절단형)** 중 하나를 선택하고 샘플 품질에 미치는 영향에 대해 논의.

## 문제

Transfusion의 두 손실(two-loss) 훈련은 작동하지만 더 까다로운 동역학을 가집니다 — 연속 확산 손실(continuous diffusion loss)은 이산 NTP(Next-Token Prediction) 손실과 다른 수치적 규모(numerical scale)에 존재합니다. 손실 가중치 균형은 하이퍼파라미터 탐색 문제입니다. 아키텍처는 효과적이지만 복잡합니다.

Show-o의 해결책: 두 모달리티를 모두 이산(discrete)으로 유지(Chameleon과 유사)하되, 순차적으로 생성하는 대신 마스킹된 이산 확산(masked discrete diffusion)을 통해 이미지를 병렬로 생성합니다. 훈련 목표는 다음 토큰 예측(NTP)을 자연스럽게 일반화하는 단일 마스킹된 토큰 예측(masked-token-prediction)이 됩니다.

## 개념

### 마스크된 이산 확산(MaskGIT)

Chang et al. (2022)의 원본 MaskGIT 기법은 우아합니다. 완전히 마스크된 이미지(모든 토큰이 특수 `<MASK>` ID)에서 시작합니다. 각 단계에서 마스크된 모든 토큰을 병렬로 예측한 후, 가장 신뢰도 높은 상위 K개 예측을 유지하고 나머지는 다시 마스크합니다. 약 8-16회 반복 후 모든 토큰이 채워집니다. 단계별 언마스크할 토큰 수는 코사인 스케줄로 조정되며, 이 방법이 잘 작동합니다.

훈련은 간단합니다: [0, 1]에서 균일하게 마스킹 비율을 샘플링하고, 이미지의 VQ 토큰에 적용한 후, 마스크된 토큰을 복원하도록 트랜스포머를 훈련시킵니다. BERT가 텍스트에 적용한 것과 정확히 동일한 방식을 이미지 생성에 확장한 것입니다.

### Show-o: 하나의 트랜스포머, 하이브리드 마스크

Show-o는 MaskGIT을 인과적 언어 모델 트랜스포머 내부에 배치합니다. 어텐션 마스크는 다음과 같습니다:

- 텍스트 토큰: 인과적(표준 LLM).
- 이미지 토큰: 이미지 블록 내에서 전체 양방향(마스크된 토큰이 예측 중 다른 모든 이미지 토큰을 볼 수 있음).
- 텍스트-이미지: 텍스트는 이전 이미지에 어텐션, 이미지는 이전 텍스트에 어텐션.

훈련은 다음을 번갈아 수행합니다:
1. 텍스트 시퀀스에 대한 표준 NTP(Next Token Prediction).
2. T2I 샘플: 텍스트 → 마스크된 이미지 토큰, 마스크 토큰 예측 손실.
3. VQA 샘플: 이미지 → 마스크된 텍스트 토큰(실제로는 NTP).

통합 손실은 `<MASK>` 토큰에 대한 교차 엔트로피이며, 텍스트 NTP(마지막 토큰만 "마스크")와 이미지 마스크 확산(무작위 부분 집합 마스크)을 모두 포괄합니다.

### 병렬 샘플링

Show-o는 이미지 생성에 ~16단계만 소요되며, 이는 ~1000단계(토큰별 자동회귀) 또는 ~20단계(확산)보다 훨씬 빠릅니다. 각 단계에서 마스크된 모든 토큰을 병렬로 예측하고, 상위 K개 신뢰도 예측을 커밋한 후 반복합니다.

비교:
- Chameleon / Emu3(토큰별 자동회귀): 일반적으로 이미지당 1024-4096회 순전파.
- Transfusion(연속 확산): ~20단계, 각 단계마다 전체 트랜스포머 순전파.
- Show-o(마스크된 이산 확산): ~16단계, 각 단계마다 전체 트랜스포머 순전파.

Show-o는 유사한 규모의 Chameleon보다 빠르며, Transfusion과 단계 수가 비슷하지만 단계당 비용이 낮습니다(이산 어휘 로짓 대 연속 MSE 손실).

### 하나의 체크포인트에서 지원하는 작업

Show-o는 추론 시 프롬프트 형식으로 선택되는 4가지 작업을 지원합니다:

- 텍스트 생성: 표준 자동회귀 텍스트 출력.
- VQA: 이미지 입력, 텍스트 출력.
- T2I: 텍스트 입력, 마스크된 이산 확산을 통한 이미지 출력.
- 인페인팅: 일부 토큰이 마스크된 이미지, 마스크된 영역 채우기.

인페인팅 기능은 마스크 예측 훈련에서 무료로 얻어집니다. VQ-토큰 그리드의 특정 영역을 마스크하고, 나머지와 텍스트 프롬프트를 입력으로 사용하여 마스크된 토큰을 예측합니다.

### 마스킹 스케줄

단계별 언마스크할 토큰 수는 품질을 결정합니다. Show-o는 코사인을 권장합니다:

```
mask_ratio(t) = cos(pi * t / (2 * T))   # t = 0..T
```

단계 0에서는 모든 토큰이 마스크됨(비율 1.0). 단계 T에서는 마스크되지 않음. 코사인은 예측이 가장 유익한 중간 범위에 집중합니다. 선형 스케줄도 작동하지만 더 빨리 정체됩니다.

### Show-o2

Show-o2(2025 후속 연구, arXiv 2506.15564)는 Show-o를 확장합니다: 더 큰 LLM 기반, 향상된 토크나이저, 개선된 마스크 스케줄. 동일한 아키텍처 패턴을 유지합니다.

### Show-o의 위치

2026년 분류 체계:

- 이산 토큰 + NTP: Chameleon, Emu3. 간단하지만 추론이 느림.
- 이산 토큰 + 마스크 확산: Show-o, MaskGIT, LlamaGen, Muse. 병렬 샘플링, 토크나이저에 따라 손실 발생.
- 연속 + 확산: Transfusion, MMDiT, DiT. 최고 품질, 더 복잡한 훈련.
- 연속 + VLM 내 흐름 매칭: JanusFlow, InternVL-U. 최신 기술.

작업별 선택: T2I + 인페인팅 + VQA를 하나의 오픈 모델에서 합리적인 속도로 원할 때 Show-o; 품질이 최우선이고 두 손실 배관을 감당할 수 있을 때 Transfusion.

## 사용 방법

`code/main.py`는 Show-o 샘플링을 시뮬레이션합니다:

- 16개의 VQ 토큰으로 구성된 장난감 그리드.
- 프롬프트와 현재 마스킹되지 않은 토큰을 기반으로 로짓(logits)을 예측하는 모의 "트랜스포머".
- 코사인 스케줄(cosine schedule)을 사용한 8단계 병렬 마스킹 샘플링.
- 중간 상태(마스크 패턴 변화 과정)와 최종 토큰을 출력합니다.

실행하여 마스킹이 단계별로 해제되는 과정을 확인하세요.

## 연습 문제

1. 마스킹된 이산 확산은 ~16단계로 샘플링합니다. 왜 1단계가 아닌가? 0단계에서 모든 마스킹을 해제하면 무엇이 잘못되는가?

2. 마스킹된 확산에서는 인페인팅이 무료입니다. Show-o의 인페인팅이 전문 모델보다 뛰어난 제품 사용 사례(실제 또는 가상)를 제안해 보세요.

3. 코사인 스케줄 vs 선형 스케줄: T=8일 때 단계별 비마스킹 토큰 수를 추적해 보세요. 어떤 것이 더 균형 잡혔는가?

4. 512x512 Show-o 이미지는 1024 토큰입니다. 어휘 K=16384에서 모델은 1024 * log2(16384) = 14,336비트(~1.75 KiB) 데이터를 출력합니다. Stable Diffusion은 512*512*24비트 = 6,291,456비트(~768 KiB) 원시 픽셀을 출력합니다. 압축 비율과 이 압축이 얻는 품질은 무엇인가?

5. LlamaGen(arXiv:2406.06525)을 읽어보세요. LlamaGen의 클래스 조건부 자기회귀 이미지 모델은 Show-o의 마스킹 접근법과 어떻게 다른가?

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|---------------------|-----------|
| Masked discrete diffusion | "MaskGIT-style" | 마스킹된 토큰을 예측하도록 학습; 추론 시 가장 확신하는 예측부터 반복적으로 언마스크 |
| Cosine schedule | "Unmask schedule" | 추론 단계에 따른 마스크 비율 감소; 중간 범위에서 확신 증가를 집중시킴 |
| Parallel decoding | "All tokens at once" | 매 단계에서 마스킹된 토큰 전체 시퀀스를 한 번의 순전파로 예측한 후 상위-K 확정 |
| Hybrid attention | "Causal + bidirectional" | 텍스트 토큰에 대해 인과적(causal)이며 이미지 블록 내에서는 양방향(bidirectional)인 마스킹 |
| Inpainting | "Fill-in generation" | 일부 토큰이 마스킹된 이미지를 조건으로 누락 토큰 예측; 학습 목적 함수로부터 자유로움 |
| Commitment rate | "Top-K per step" | 반복당 확정되는 토큰 수; 추론 속도와 품질 간 트레이드오프 제어 |
