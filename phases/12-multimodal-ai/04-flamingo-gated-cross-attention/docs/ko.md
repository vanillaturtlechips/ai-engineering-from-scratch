# Flamingo와 Few-Shot VLMs를 위한 게이티드 크로스 어텐션

> DeepMind의 Flamingo(2022)는 두 가지 혁신을 이루었습니다. 첫째, 단일 모델이 이미지, 비디오, 텍스트의 임의로 교차된 시퀀스를 처리할 수 있음을 보였습니다. 둘째, VLM(비전-언어 모델)이 인-컨텍스트 학습이 가능함을 입증했습니다 — 3개의 예시 (이미지, 캡션) 쌍으로 구성된 few-shot 프롬프트를 제공하면 모델은 그래디언트 단계 없이도 새로운 이미지에 대한 캡션을 생성할 수 있습니다. 이 메커니즘의 핵심은 **게이티드 크로스 어텐션 레이어**로, 이는 고정된 LLM의 기존 레이어 사이에 삽입되며, 학습된 tanh 게이트를 통해 초기화 시 LLM의 텍스트 능력을 보존합니다(게이트는 0에서 시작). 이 레슨에서는 Flamingo의 Perceiver 리샘플러와 게이티드 크로스 어텐션 아키텍처를 탐구합니다 — 이는 Gemini의 교차 입력과 Idefics2의 시각적 토큰의 기반이 되는 기술입니다.

**유형:** 학습  
**언어:** Python (표준 라이브러리, 게이티드 크로스 어텐션 + Perceiver 리샘플러 데모)  
**선수 지식:** Phase 12 · 03 (BLIP-2 Q-Former)  
**소요 시간:** ~120분

## 학습 목표

- **초기화 시 tanh(gate) = 0을 통해 게이트 교차 어텐션(gated cross-attention)이 고정된 LLM의 텍스트 생성 능력을 보존하는 방식을 설명**할 것.
- **Perceiver 리샘플러(Perceiver resampler) 동작 과정**을 설명할 것: N개의 이미지 패치 → 교차 어텐션(cross-attention)을 통해 K개의 고정된 "잠재(latent)" 쿼리 생성.
- **Flamingo가 이미지 배치를 고려한 인과적 마스킹(causal masking)으로 이미지-텍스트 교차 시퀀스를 처리하는 방법**을 설명할 것.
- **Few-shot 멀티모달 프롬프트 구조**를 재현할 것: 3개의 이미지-캡션 예시 후 쿼리 이미지 입력.

## 문제 정의

BLIP-2는 32개의 시각 토큰을 고정된 LLM(Large Language Model)의 입력 계층에 공급합니다. 이는 한 프롬프트당 하나의 이미지에 대해 작동합니다. 하지만 "여기 이미지 A가 있습니다. 캡션을 달아주세요; 여기 이미지 B가 있습니다. 캡션을 달아주세요; 이제 이미지 C가 있습니다. 캡션을 달아주세요"와 같이 텍스트와 *여러* 이미지를 번갈아 입력하려면 어떻게 해야 할까요? LLM의 자기 주의(self-attention) 메커니즘은 단일 스트림에서 이미지 토큰과 텍스트 토큰을 처리해야 하며, 어떤 위치가 어떤 이미지에 주의를 기울여야 하는지 문제가 복잡해집니다.

Flamingo의 해결책: LLM의 입력 스트림을 전혀 변경하지 않습니다. 기존 LLM 블록 사이에 추가적인 교차 주의(cross-attention) 레이어를 삽입합니다. 텍스트 토큰은 여전히 LLM의 인과적 자기 주의(causal self-attention)를 통해 항상 흐르듯이 이동합니다. 몇 개의 LLM 블록마다 텍스트 토큰은 새로운 게이팅(gated) 레이어를 통해 이미지 특징에 교차 주의를 기울입니다. (0으로 초기화된) 게이팅은 학습 시작 단계에서 새로운 레이어가 무효(no-op) 상태임을 의미합니다. 즉, 모델은 사전 학습된 LLM과 정확히 동일하게 동작합니다. 학습이 진행됨에 따라 게이팅이 열리고 시각 정보가 흐르기 시작합니다.

Flamingo가 해결한 두 번째 문제: 프롬프트당 가변적인 수의 이미지(0개, 1개, 또는 여러 개)를 어떻게 처리할 것인가? Perceiver 리샘플러(resampler) — 입력 패치의 수와 관계없이 고정된 수의 시각 잠재 토큰을 생성하는 소규모 교차 주의 모듈 — 가 이를 해결합니다. LLM의 교차 주의 레이어는 프롬프트에 포함된 이미지 수와 무관하게 동일한 형태의 토큰을 처리합니다.

## 개념

### 고정된 LLM

Flamingo는 고정된 Chinchilla 70B LLM으로 시작합니다. 70B 가중치 모두 변경되지 않습니다. 기존 텍스트 자기 주의(self-attention) 및 FFN(Feed-Forward Network)은 정상적으로 작동합니다.

### Perceiver 리샘플러

프롬프트의 각 이미지에 대해 ViT(Vision Transformer)는 N개의 패치 토큰을 생성합니다. Perceiver 리샘플러는 K개의 고정된 학습 가능한 잠재 표현(Flamingo는 K=64 사용)을 가집니다. 각 리샘플러 블록은 두 하위 단계로 구성됩니다:

1. 교차 주의(cross-attention): K개의 잠재 표현이 N개의 패치 토큰을 참조합니다 (Q는 잠재 표현에서, K/V는 패치에서).
2. 잠재 표현 내에서의 자기 주의(self-attention) + FFN.

6개의 리샘플러 블록 이후 출력은 ViT가 생성한 패치 수와 무관하게 1024차원의 K=64 시각 토큰입니다. 224x224 이미지(196개 패치)와 480x480 이미지(900개 패치) 모두 64개의 리샘플러 토큰으로 출력됩니다.

비디오의 경우 리샘플러는 시간적으로 적용됩니다: 각 프레임의 패치는 64개의 잠재 표현을 생성하며, 시간적 위치 인코딩을 통해 모델은 t=0과 t=N을 구분할 수 있습니다. 전체 비디오는 T * 64 시각 토큰이 됩니다.

### 게이팅 교차 주의

고정된 LLM의 모든 M개 레이어 사이(Flamingo는 M=4 사용)에 새로운 게이팅 교차 주의 블록을 삽입합니다:

```
x_after_llm_block = llm_block(x_before)
cross = cross_attn(x_after, resampler_output)
gated = tanh(alpha) * cross + x_after
x_before_next_block = gated
```

- `alpha`는 0으로 초기화된 학습 가능한 스칼라입니다.
- `tanh(0) = 0`이므로 초기화 시 게이팅 브랜치는 0을 기여합니다.
- `alpha`가 0에서 벗어나면 교차 주의 기여도가 부드럽게 증가합니다.
- 잔차 연결(residual connection)은 완전히 열린 게이트라도 LLM의 텍스트 표현을 덮어쓰지 않고 시각 정보를 추가합니다.

이것은 Flamingo에서 가장 중요한 설계 선택입니다: 시각 조건화는 가법적이며, 게이팅되고, 초기화 시 0입니다. 0단계의 Flamingo는 텍스트 전용 입력에 대해 완벽한 Chinchilla 70B입니다.

### 교차 주의 마스킹

"<이미지 A> 캡션 A <이미지 B> 캡션 B <이미지 C> ?"와 같은 프롬프트에서 각 텍스트 토큰은 시퀀스 상 이전에 나온 이미지만 참조해야 합니다. 교차 주의 마스크는 다음을 강제합니다: 위치 `t`의 텍스트 토큰은 이미지 인덱스 `i < i_t`인 리샘플러 토큰만 참조합니다. 여기서 `i_t`는 위치 `t` 이전의 가장 최근 이미지입니다. "마지막 선행 이미지만 참조" 또는 "모든 선행 이미지 참조" 모두 유효한 선택이며, Flamingo는 전자를 선택했습니다.

### 인-컨텍스트 소수 샷 학습

Flamingo 프롬프트 예시:

```
<이미지1> 고양이 사진. <이미지2> 강아지 사진. <이미지3> 새 사진.
```

모델은 완성 패턴을 인식하고 "새"(또는 이미지3에 표시된 내용)를 출력합니다. 경사 하강법 단계 없이 고정된 LLM의 인-컨텍스트 학습 능력이 게이팅 교차 주의를 통해 전달됩니다. 이것이 논문의 핵심이며 중요한 이유입니다.

### 학습 데이터

Flamingo는 다음 세 데이터셋으로 학습되었습니다:

1. 멀티모달 매스시브웹(M3W): 43M 웹 페이지, 이미지와 텍스트가 교차하며 읽기 순서 재구성.
2. 이미지-텍스트 쌍(ALIGN + LTIP): 4.4B 쌍.
3. 비디오-텍스트 쌍(VTP): 27M 짧은 비디오 클립.

OBELICS(2023)는 교차 웹 코퍼스의 오픈 재현판으로, Idefics, Idefics2 및 대부분의 오픈 "Flamingo 유사" 모델이 사용합니다.

### OpenFlamingo 및 Otter

OpenFlamingo(2023)는 오픈 재현판입니다. 아키텍처는 동일(Perceiver 리샘플러 + 고정된 LLaMA 또는 MPT 상의 게이팅 교차 주의). 3B, 4B, 9B 체크포인트. 더 작은 기본 LLM과 적은 데이터로 인해 Flamingo보다 품질이 낮습니다.

Otter(2023)는 OpenFlamingo를 기반으로 MIMIC-IT(다중모달 명령어 데이터셋)에 대한 지시 튜닝을 수행하여 게이팅 교차 주의가 명령어 따르기 작업에도 효과적임을 보여줍니다.

### 후속 모델들

- Idefics / Idefics2 / Idefics3: Hugging Face의 게이팅 교차 주의 계보, 점진적으로 단순화(Idefics2는 리샘플러를 제거하고 적응형 풀링과 직접 패치 토큰 사용).
- Flamingo-to-Chameleon 전환: 2024년까지 많은 팀이 초기 융합(early-fusion)으로 전환(레슨 12.11); Flamingo 스타일의 게이팅 교차 주의는 백본 고정이 필요한 프로덕션에서 여전히 사용.
- Gemini의 교차 입력: 개념적으로 Flamingo의 교차 형식 유연성을 계승하지만 정확한 메커니즘은 비공개.

### BLIP-2와의 비교

| | BLIP-2 | Flamingo |
|---|---|---|
| 시각 브리지 | 입력 시 Q-Former 1회 | 매 M개 레이어마다 게이팅 교차 주의 |
| 시각 토큰 | 이미지당 32개 | 교차 주의 레이어당 이미지당 64개 |
| 고정된 LLM | 예 | 예 |
| 인-컨텍스트 소수 샷 | 약함 | 강함 — 논문의 중심 |
| 교차 입력 | 네이티브 지원 없음 | 예, 설계 목표 |
| 학습 데이터 | 1.3B 쌍 | 1.3B 쌍 + 43M 교차 페이지 |
| 파라미터 수 | 188M 학습됨 | ~10B 학습됨(교차 주의 레이어) |
| 컴퓨팅 | 8개 A100에서 수일 | 수천 개 TPUv4에서 수주 |

예산이 제한된 단일 이미지 VQA에는 BLIP-2를 선택하세요. 교차, 소수 샷 또는 다중 이미지 추론에는 Flamingo/Idefics2를 선택하세요.

## 사용 방법

`code/main.py`는 다음을 보여줍니다:

1. 36개의 가짜 패치 토큰과 8개의 학습 가능한 잠재 변수를 사용한 Perceiver 리샘플러(순수한 Python 크로스 어텐션).
2. `alpha = 0`인 게이티드 크로스 어텐션 단계 → 출력이 입력과 동일(LLM 변경 없음), 이후 `alpha = 2.0` → 시각적 기여가 혼합됨.
3. "(이미지 1) (텍스트 1) (이미지 2) (텍스트 2)" 시퀀스에 대한 2D 어텐션 마스크를 생성하는 인터리브 마스크 빌더.

## Ship It

이 레슨은 `outputs/skill-gated-bridge-diagnostic.md`를 생성합니다. 열린 VLM(비전 언어 모델)의 설정(리샘플러 사용 여부(Y/N), 크로스 어텐션 빈도, 게이트 방식)을 기반으로 Flamingo 계열 요소를 식별하고 동결 전략을 설명합니다. 파인튜닝 후 텍스트 성능이 저하된 원인을 디버깅하는 데 유용합니다(예상 답변: 게이트가 너무 빨리 과도하게 열렸기 때문).

## 연습 문제

1. Flamingo-9B의 시각적 파라미터 수를 계산하세요: 9B LLM + 1.4B 게이팅 크로스-어텐션 레이어 + 64M 리샘플러. 전체 파라미터 중 훈련되는 파라미터의 비율은 얼마인가요?

2. PyTorch에서 게이팅 잔차 `y = tanh(alpha) * cross + x`를 구현하세요. `alpha=0`일 때 초기화 시점에서 `y==x`가 정확히 성립함을 실험적으로 보여주세요.

3. OpenFlamingo 섹션 3.2(arXiv:2308.01390)를 읽고, 각 프롬프트의 이미지 수가 다를 때 배치 내 여러 이미지를 처리하는 방법을 설명하세요. 패딩 전략을 설명하세요.

4. Flamingo의 크로스-어텐션 마스크는 왜 텍스트 토큰이 모든 이전 이미지가 아닌 *가장 최근의 이전 이미지만* 어텐션할 수 있도록 허용하나요? Flamingo 논문 섹션 2.4를 읽고 트레이드오프를 설명하세요.

5. 인-컨텍스트 few-shot: 새로운 Flamingo 변형을 위해 "이미지 → 주 객체 색상" 예시 4개를 포함한 프롬프트를 구성하세요. 예시 수를 0에서 8로 변화시킬 때 예상되는 정확도 패턴을 설명하세요.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|----------------|------------------------|
| Perceiver 리샘플러 | "Fixed-latent cross-attention" | 가변 개수의 입력 패치에서 K개의 고정 토큰을 생성하는 모듈 |
| 게이티드 크로스 어텐션 | "Tanh-gated bridge" | 잔차 레이어 `y = tanh(alpha)*cross + x`, 학습 가능한 alpha, 초기화 0 |
| 인터리브 입력 | "Mixed sequence" | 이미지와 텍스트가 읽기 순서에 자유롭게 혼합된 프롬프트 형식 |
| 프로즌 LLM | "No LLM gradients" | 텍스트 LLM의 가중치는 업데이트되지 않음; 리샘플러 + 크로스 어텐션 레이어만 학습 |
| 퓨샷 | "In-context examples" | 프롬프트에 몇 개의 (이미지, 답변) 쌍을 제공; 모델은 파인튜닝 없이 일반화 |
| OBELICS | "Interleaved web corpus" | 읽기 순서에 이미지와 텍스트가 포함된 1억 4,100만 개의 웹 페이지 오픈 데이터셋 |
| 친칠라 | "70B frozen base" | Flamingo의 프로즌 텍스트 LLM, DeepMind의 Chinchilla 논문에서 유래 |
| 게이트 스케줄 | "How alpha moves" | 학습 중 크로스 어텐션 게이트가 열리는 속도 |
| 크로스 어텐션 빈도 | "Every M layers" | 게이티드 크로스 어텐션 블록이 삽입되는 빈도; Flamingo는 M=4 사용 |
