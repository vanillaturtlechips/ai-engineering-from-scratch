# CLIP에서 BLIP-2로 — 모달리티 브리지로서의 Q-Former

> CLIP은 이미지와 텍스트를 정렬하지만 캡션 생성, 질문 답변, 대화 기능은 없습니다. BLIP-2(Salesforce, 2023)는 32개의 학습 가능한 쿼리 벡터를 사용하는 작은 훈련 가능한 브리지로 이 문제를 해결했습니다. 이 벡터는 크로스-어텐션을 통해 고정된 ViT의 특징들에 주목한 후, 고정된 LLM의 입력 스트림에 직접 연결됩니다. 188M 파라미터의 브리지는 11B LLM과 ViT-g/14를 연결했습니다. 2026년까지의 모든 어댑터 기반 VLM(MiniGPT-4, InstructBLIP, LLaVA의 변종들)은 이 Q-Former의 후손입니다. 이 레슨에서는 Q-Former의 아키텍처를 읽고, 2단계 훈련 방식을 설명하며, 시각적 토큰을 고정된 텍스트 디코더에 입력하는 장난감 버전을 구축합니다.

**유형:** 구축
**언어:** Python (표준 라이브러리, 크로스-어텐션 + 학습 가능한 쿼리 데모)
**선수 지식:** Phase 12 · 02 (CLIP), Phase 7 (트랜스포머)
**소요 시간:** ~180분

## 학습 목표

- 고정된 비전 인코더와 고정된 LLM 사이의 학습 가능한 병목(bottleneck)이 종단간 미세 조정(end-to-end finetuning)보다 비용 및 안정성 측면에서 우수한 이유를 설명.
- 고정된 학습 가능한 쿼리 집합이 외부 이미지 특징에 주목하는 크로스 어텐션(cross-attention) 블록 구현.
- BLIP-2의 2단계 사전 학습 과정 분석: 표현 학습(ITC + ITM + ITG) 후 생성 학습(고정된 디코더로 LM 손실 적용).
- Q-Former를 LLaVA에서 사용된 간단한 MLP 프로젝터(projector)와 비교하고 각 접근 방식이 우수한 경우를 논증.  

## 전문 용어 설명 (괄호 안 영어 표기)
- **병목(bottleneck)**: 모델의 중간 계층에서 특징 차원을 축소하는 구조  
- **종단간 미세 조정(end-to-end finetuning)**: 전체 모델을 동시에 조정하는 방식  
- **크로스 어텐션(cross-attention)**: 서로 다른 모달리티 간 어텐션 메커니즘  
- **표현 학습(representation learning)**: 의미 있는 특징 추출을 위한 학습  
- **ITC(Image-Text Contrastive)**: 이미지-텍스트 대조 학습  
- **ITM(Image-Text Matching)**: 이미지-텍스트 매칭 예측  
- **ITG(Image-Text Grounding)**: 이미지-텍스트 지역 정렬  
- **LM 손실(Language Modeling loss)**: 언어 모델링 목적 함수  
- **Q-Former**: 쿼리 기반 비전-언어 정렬 모듈  
- **MLP 프로젝터(MLP projector)**: 다층 퍼셉트론을 이용한 특징 변환기  

> 번역 규칙 준수: 코드/수식/URL/고유명은 원본 유지, 마크다운 구조 보존, 전문 용어는 한국어(영어) 형식으로 표기.

## 문제 정의

여러분은 이미지당 1408차원의 256개 패치 토큰을 생성하는 고정된 ViT를 보유하고 있습니다. 또한 4096차원 토큰 임베딩을 입력으로 받는 70억(7B) 파라미터의 고정된 LLM을 보유하고 있습니다. 가장 직관적인 연결 방법인 1408차원에서 4096차원으로의 선형 레이어는 작동하지만, 256개의 모든 패치 토큰을 LLM 컨텍스트에 입력하면 이미지당 256개의 추가 토큰이 소모됩니다. 32개 이미지 배치에서는 시각 모달리티만으로 8192개의 토큰이 소비됩니다.

BLIP-2의 질문: 256개 토큰의 이미지 표현을 훨씬 적은 토큰(예: 32개)으로 압축하면서도 LLM이 이미지 캡션 생성, 질문 답변, 추론을 수행할 수 있을 만큼 충분한 정보를 보존할 수 있을까요? 그리고 고정된 백본 모델을 변경하지 않으면서 이 연결 모듈을 훈련시켜 훈련 비용을 연결 모듈의 파라미터 수준으로만 유지할 수 있을까요?

답변: Q-Former입니다. ViT의 패치 토큰에 교차 어텐션(cross-attend)하는 32개의 학습 가능한 "쿼리" 벡터로, LLM이 소비하는 32개 토큰의 시각적 요약을 생성합니다. 총 1억 8800만(188M) 파라미터로 구성되며, LLM을 사용하기 전에 대조(contrastive), 매칭(matching), 생성(generative) 목적 함수로 훈련됩니다.

## 개념

## 학습 가능한 쿼리

Q-Former의 핵심 트릭: LLM의 텍스트 토큰이 이미지 패치에 어텐션(attention)하는 대신, 새로운 32개의 학습 가능한 쿼리 벡터 `Q`를 도입하고 이들이 이미지 패치에 어텐션하도록 합니다. 쿼리는 모델의 파라미터이며 — 훈련 중에 학습되고 모든 이미지에 동일한 32개 쿼리가 사용됩니다.

교차 어텐션(cross-attention) 후, 각 쿼리는 이미지의 압축된 요약을 보유합니다 — "주요 객체 설명", "배경 설명", "객체 개수 세기" 등. 쿼리는 의미적 레이블에 문자 그대로 특화되지 않으며; 다운스트림 손실(downstream loss)을 감소시키는 인코딩을 학습합니다.

## 아키텍처

Q-Former는 두 경로를 가진 소형 트랜스포머(12층, ~100M 파라미터)입니다:

1. 쿼리 경로: 32개 쿼리 벡터가 자기 어텐션(self-attention, 서로 간)을 통과한 후, 고정된 ViT의 패치 토큰에 대한 교차 어텐션을 거치고, FFN(피드포워드 네트워크)을 통과합니다.
2. 텍스트 경로: BERT와 유사한 텍스트 인코더가 쿼리 경로와 자기 어텐션 및 FFN 가중치를 공유합니다. 텍스트 경로에서는 교차 어텐션이 비활성화됩니다.

훈련 시에는 두 경로가 모두 실행됩니다. 쿼리와 텍스트는 공유된 자기 어텐션을 통해 상호작용하며, 이는 쿼리가 텍스트 조건부 작업(ITM, ITG)에 조건화될 수 있음을 의미합니다. VLM 핸드오프 추론 시에는 쿼리만 흐르며 32개의 시각적 토큰을 생성합니다.

## 2단계 훈련

BLIP-2는 두 단계로 사전 훈련됩니다:

**1단계: 표현 학습 (LLM 없음)**  
3가지 손실 함수:
- ITC (이미지-텍스트 대조): 풀링된 쿼리 토큰과 텍스트 CLS 토큰 간의 CLIP 스타일 대조 학습.
- ITM (이미지-텍스트 매칭): 이진 분류기 — 이 이미지-텍스트 쌍이 매칭되는가? 하드 네거티브 마이닝 적용.
- ITG (이미지 기반 텍스트 생성): 쿼리에 조건화된 텍스트의 인과적 언어 모델 헤드. 쿼리가 텍스트 생성 가능한 콘텐츠를 인코딩하도록 강제.

Q-Former만 훈련됩니다. ViT는 고정됩니다. LLM은 관여하지 않습니다.

**2단계: 생성 학습**  
고정된 LLM(OPT-2.7B 또는 Flan-T5-XL 등)을 연결합니다. 32개 쿼리 출력을 작은 선형 레이어를 통해 LLM의 임베딩 차원으로 투영합니다. 이를 텍스트 프롬프트 앞에 추가합니다. 선형 투영과 Q-Former만 훈련하며, 결합된 프롬프트 + 이미지 + 캡션 시퀀스에 대한 LM 손실을 최적화합니다.

2단계 후, Q-Former + 투영 레이어가 전체 시각적 어댑터입니다. 추론 시: 이미지 → ViT → Q-Former → 선형 투영 → 텍스트 앞에 추가 → 고정된 LLM이 출력 생성.

## 파라미터 경제학

BLIP-2는 ViT-g/14(1.1B, 고정) + OPT-6.7B(6.7B, 고정) + Q-Former(188M, 훈련) = 총 8B 파라미터 중 188M만 훈련됩니다. Q-Former는 전체 스택의 약 2.4% 파라미터입니다. 훈련 비용도 이를 반영합니다: A100 몇 개로 며칠 vs 엔드투엔드 훈련 시 몇 주.

성능: BLIP-2는 제로샷 VQA에서 Flamingo-80B와 동등하거나 더 나은 성능을 보이면서도 50배 더 작습니다. 브릿지 방식이 효과적입니다.

## InstructBLIP과 명령어 인식 Q-Former

InstructBLIP(2023)은 Q-Former에 추가 입력(명령어 텍스트 자체)을 도입합니다. 교차 어텐션 시 쿼리는 이미지 패치와 명령어 모두에 접근할 수 있습니다. 쿼리는 명령어별로 특화될 수 있습니다("자동차 개수 세기", "분위기 설명"). 고정되지 않은 작업에서 벤치마크 성능 향상.

## MiniGPT-4와 프로젝터 전용 접근법

MiniGPT-4는 Q-Former를 유지하지만 출력 선형 투영만 훈련하고 나머지는 고정합니다. 저렴하지만 품질 희생 — 쿼리는 BLIP-2의 것이며 사용자 정의가 불가능합니다. 빠른 반복에는 적합하지만 최적 아키텍처는 아닙니다.

## LLaVA가 단순화된 이유

LLaVA(2023, 레슨 12.05)는 Q-Former를 2층 MLP로 대체하여 모든 ViT 패치 토큰을 LLM 공간으로 투영합니다 — 24x24 그리드에 이미지당 576토큰을 LLM에 입력. 압축은 떨어지지만 LLM이 원시 패치에 어텐션할 수 있습니다. 당시엔 논란이 있었으나, 2023년 후반에는 시각적 명령어 데이터(LLaVA-Instruct-150k)로 MLP가 충분한 신호를 보존할 수 있음이 입증되며 주류로 자리잡았습니다. 트레이드오프: LLaVA는 컨텍스트가 빨리 채워지지만 멀티이미지/비디오로 자연스럽게 확장됩니다.

2026년 현재 분야는 양분됨: 토큰 예산이 중요한 경우(긴 비디오, 다수 이미지) Q-Former 생존; 토큰당 품질이 우선인 경우 MLP 프로젝터 우세.

## 게이티드 교차 어텐션: 조상 Flamingo

Flamingo(레슨 12.04)는 BLIP-2 이전에 동일한 교차 어텐션 아이디어를 사용했지만, 단일 브릿지가 아닌 모든 고정된 LLM 레이어에서 적용했습니다. BLIP-2는 입력 레이어만 압축해도 작동함을 보였습니다. Gemini와 Idefics는 두 방식을 결합: 인터리브드 입력 토큰 + 인컨텍스트 퓨샷을 위한 선택적 게이티드 교차 어텐션.

## 2026년의 후손들

- Q-Former: BLIP-2, InstructBLIP, MiniGPT-4 및 토큰 예산 문제로 인한 대부분의 비디오-언어 모델.
- Perceiver 리샘플러: Flamingo 변형(레슨 12.04); Idefics 패밀리, Eagle, OmniMAE.
- MLP 프로젝터: LLaVA, LLaVA-NeXT, LLaVA-OneVision, Cambrian-1.
- 어텐션 풀: VILA, PaliGemma.

네 가지 모두 유효합니다. 결정적 질문은 토큰 예산에 제약이 있는지, 아니면 토큰당 품질이 우선인지입니다.

## 사용 방법

`code/main.py`는 stdlib Q-Former 스타일의 크로스 어텐션(cross-attention)을 구축합니다:

1. 256개의 이미지 패치 토큰(차원 128)을 시뮬레이션합니다.
2. 32개의 학습 가능한 쿼리(차원 128)를 인스턴스화합니다.
3. 스케일링된 닷 프로덕트 크로스 어텐션(scaled-dot-product cross-attention)을 실행합니다(쿼리(Q)는 쿼리 토큰에서, 키(K)/값(V)은 패치 토큰에서 가져옴).
4. 선형 레이어를 통해 LLM 차원(512)으로 투영합니다.
5. 32개의 LLM-준비 완료 시각 토큰을 출력합니다.

모든 연산은 순수 파이썬(중첩된 벡터 루프)으로 구현됩니다. 장난감 수준이지만 올바른 형태를 가집니다. 어텐션 가중치 행렬이 출력되므로 각 쿼리가 어떤 패치에서 정보를 가져왔는지 확인할 수 있습니다.

## Ship It

이 레슨은 `outputs/skill-modality-bridge-picker.md`를 생성합니다. 목표 VLM 구성(비전 인코더 토큰 수, LLM 컨텍스트 예산, 배포 제약 조건, 품질 목표)이 주어졌을 때, Q-Former vs MLP vs Perceiver 리샘플러를 짧은 근거와 각 브리지의 파라미터 수 추정치와 함께 추천합니다.

## 연습 문제

1. PyTorch에서 교차 어텐션(cross-attention) 블록을 구현하세요. 32개의 쿼리(queries)와 256개의 키/값(keys/values)으로 어텐션 가중치 행렬이 32 x 256이고, 소프트맥스(softmax) 후 각 행의 합이 1인지 검증하세요.

2. BLIP-2 1단계에서 Q-Former는 ITC, ITM, ITG 세 가지 손실 함수를 동시에 실행합니다. 각 손실 함수의 순전파(forward) 시그니처를 의사 코드로 작성하세요. 어떤 손실 함수가 텍스트 인코더(text encoder) 경로를 활성화해야 하나요?

3. 파라미터 수 비교: Q-Former(12층, 768 hidden) vs 2층 MLP 프로젝터(1408 → 4096, 두 층). 188M Q-Former의 비용이 훈련 효율성 측면에서 어떤 LLM 규모에서 회수되나요?

4. BLIP-2 논문(arXiv:2301.12597) 3.2절을 읽고 Q-Former 초기화 방법을 설명하세요. BERT-base(랜덤이 아닌)에서 초기화하는 것이 수렴을 가속화하는 이유는 무엇인가요?

5. 1FPS로 샘플링된 60프레임의 10분 동영상에 대해, (Q-Former → 프레임당 32토큰) vs (MLP 프로젝터 → 프레임당 576토큰) 방식의 프레임당 토큰 비용을 계산하세요. 128k 토큰 LLM 컨텍스트 창에 어떤 방식이 적합한가요?

| Q-Former | "쿼리하는 트랜스포머" | 32개의 학습 가능한 쿼리 벡터를 가진 작은 트랜스포머로, 고정된 ViT 특징에 대해 교차 어텐션(cross-attention)을 수행 |
| 학습 가능한 쿼리 | "비전을 위한 소프트 프롬프트" | 교차 어텐션의 쿼리 역할을 하는 고정된 매개변수 집합; 모델별로 학습되며 모든 입력에 공유됨 |
| 교차 어텐션 | "여기의 Q, 저기의 K/V" | 쿼리, 키, 값이 서로 다른 소스에서 오는 어텐션; 쿼리가 ViT 패치에서 정보를 추출하는 방식 |
| ITC | "이미지-텍스트 대조" | Q-Former 풀링된 쿼리와 텍스트 CLS에 적용된 CLIP 스타일 손실 함수(loss function) |
| ITM | "이미지-텍스트 매칭" | 하드 네거티브 마이닝된 쌍에 대한 이진 분류기; 쿼리가 세부적인 불일치를 구분하도록 강제 |
| ITG | "이미지 기반 텍스트 생성" | 쿼리를 조건으로 텍스트를 생성하는 인과적 언어 모델(causal LM) 손실; 쿼리가 텍스트 디코딩 가능한 내용을 인코딩하도록 강제 |
| 2단계 사전 학습 | "표현 학습 후 생성 학습" | 1단계: Q-Former 단독 학습 (ITC/ITM/ITG); 2단계: 고정된 LLM 연결 후 프로젝션 + Q-Former만 학습 |
| 고정된 백본 | "파인튜닝하지 않음" | 비전 인코더와 LLM 가중치는 고정; 오직 브리지(연결부)만 학습 |
| 프로젝션 헤드 | "LLM 차원으로의 선형 변환" | Q-Former 출력을 LLM의 임베딩(embedding) 차원으로 매핑하는 최종 선형 계층 |
| 퍼서버 리샘플러 | "Flamingo의 버전" | 유사한 학습 가능한 쿼리 교차 어텐션; Flamingo에서 단일 브리지가 아닌 모든 계층에 사용됨 |
