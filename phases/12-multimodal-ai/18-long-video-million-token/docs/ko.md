# 백만 토큰 컨텍스트에서의 장문 비디오 이해

> 24FPS의 1시간 4K 비디오는 패치 및 임베딩 시 약 6천만 개의 토큰을 생성합니다. 2시간 분량의 팟캐스트 에피소드 전사본은 3만 개의 토큰입니다. 강력한 풀링으로 압축된 풀 블루레이 영화도 수십만 개의 토큰에 달합니다. Google의 Gemini 1.5(2024년 3월)는 1천만 토큰 컨텍스트로 이 시대를 열었으며, 1시간 분량의 비디오에서 신뢰할 수 있는 니들-인-헤이스택 검색을 수행했습니다. LWM(Liu et al., 2024년 2월)은 링 어텐션의 확장 경로를 보여주었습니다. LongVILA와 Video-XL은 인게이지먼트를 더 확장했습니다. VideoAgent는 원시 컨텍스트를 에이전트 기반 검색으로 대체했습니다. 각 접근 방식은 계산, 검색 성능, 엔지니어링 복잡도 간의 서로 다른 트레이드오프를 나타냅니다. 이 레슨에서는 이들을 나란히 비교합니다.

**유형:** 구축(Build)  
**언어:** Python (표준 라이브러리, 니들-인-헤이스택 시뮬레이터 + 에이전트 기반 검색 라우터)  
**선수 지식:** 12단계 · 17단계 (비디오 시간 토큰)  
**소요 시간:** ~180분

## 학습 목표

- 다양한 FPS(초당 프레임 수)와 풀링(pooling) 조건에서 장편 비디오의 총 시각-토큰 수를 계산.
- 세 가지 확장 경로 설명: 무차별 컨텍스트(Gemini 1.5), 링 어텐션(ring attention, LWM), 토큰 압축(token compression, LongVILA / Video-XL).
- 원시 컨텍스트 비디오 VLM(Video Language Model)과 에이전트 기반 검색 비디오 VLM(VideoAgent)을 정확도(accuracy)와 지연 시간(latency) 측면에서 비교.
- 30분 분량의 비디오에 대해 "바늘 찾기(needle-in-a-haystack)" 테스트 설계 및 특정 분(minute)에서의 재현율(recall) 측정.

## 문제 정의

Qwen2.5-VL 크기의 384 네이티브 해상도 패치로 단일 프레임은 ~729 토큰입니다. 3x3 풀링 시 프레임당 81 토큰이 됩니다. 1 FPS의 30분 클립은 1800 프레임 = 145,800 토큰입니다. 2025년 오픈 VLMs(Visual Language Models)로 처리 가능하지만 마진이 매우 타이트합니다. 2 FPS에서는 291,600 토큰이 발생하며, 이는 가장 큰 컨텍스트 윈도우만 수용할 수 있습니다.

1 FPS의 2시간 영화는 583k 토큰입니다. 이는 대부분의 2026년 오픈 모델 범위를 초과하며, Gemini 2.5 Pro 또는 더 공격적인 풀링이 필요합니다.

세 가지 확장 경로가 도출되었습니다.

## 개념

### 경로 1: 무차별적 컨텍스트 (Gemini 1.5, Claude Opus)

하드웨어를 문제에 투입한다. 컨텍스트를 수백만 토큰으로 확장하고, 모든 것을 단일 순전파(forward pass)에서 처리한다.

Gemini 1.5 Pro는 1M 토큰으로 출시했으며, Gemini 1.5 Ultra는 10M, 2026년 출시 예정인 Gemini 2.5 Pro는 몇 시간 분량의 비디오를 안정적으로 처리한다. 논문(arXiv:2403.05530)은 약 9.5M 토큰까지 99.7%의 "바늘 찾기(needle-in-a-haystack)" 재현율을 기록했다.

엔지니어링: 메모리 계층(로컬 + 글로벌 + 희소)을 갖춘 커스텀 어텐션 구현과 장거리 컨텍스트 효율성을 위한 MoE 전문가 라우팅. 전체 세부 사항은 공개되지 않았으며, 오픈소스도 아니다.

### 경로 2: 링 어텐션 (LWM, LongVILA)

링 어텐션은 긴 시퀀스를 "링" 형태로 장치들에 분배하며, 각 장치는 청크를 보유한다. 전체 시퀀스에 대한 어텐션은 각 장치가 청크를 링 패턴으로 다음 장치에 전송하고, 부분 어텐션을 계산한 후 집계하는 방식으로 발생한다.

LWM(Liu et al., 2024)은 이 방식으로 1M 토큰 컨텍스트 모델을 훈련했다. 훈련 계산량은 컨텍스트에 선형적으로 증가하며, 어텐션의 이차적 비용(quadratic cost)은 링 내 장치들에 분산된다.

LongVILA(arXiv:2408.10188)는 이 패턴을 VLM(비전 언어 모델)에 적용했다. 프레임당 192 토큰의 1400개 프레임 비디오 = 268k 컨텍스트를 8-way 병렬화로 훈련했다.

### 경로 3: 토큰 압축 (Video-XL, LongVA)

무차별적 컨텍스트보다 저렴하다: LLM이 시퀀스를 보기 전에 공격적으로 압축한다.

Video-XL(arXiv:2409.14485)은 시각적 요약 토큰을 사용한다: N 프레임 클립마다 하나의 "요약" 토큰을 생성하며, 이 토큰은 N 프레임에 대해 어텐션을 수행한다. 추론 시 LLM은 클립당 하나의 요약 토큰만 처리하므로 컨텍스트가 크게 축소된다.

LongVA는 "장거리 컨텍스트 전이" 기법으로 LLM 컨텍스트를 200k에서 2M으로 확장한다. 장거리 텍스트 컨텍스트로 훈련한 후, 공유 표현을 통해 장거리 비디오로 전이한다.

토큰 압축은 특정 타임스탬프의 재현율과 확장성을 교환한다. 모델은 일반적으로 발생한 일을 알지만, 정확한 프레임을 놓칠 수 있다.

### 경로 4: 에이전트 기반 검색 (VideoAgent)

전체 비디오를 LLM에 입력하지 않는다. 대신 비디오를 데이터베이스로 취급하고 LLM으로 쿼리한다.

VideoAgent(arXiv:2403.10517):

1. LLM이 질문을 읽는다.
2. LLM이 검색 도구에 관련 클립을 요청한다("고양이가 있는 세그먼트를 보여줘").
3. 도구가 일치하는 클립 타임스탬프를 반환한다.
4. LLM이 VLM을 통해 해당 클립을 읽는다.
5. LLM이 답변을 구성하거나 후속 쿼리를 요청한다.

이는 장거리 비디오에 적용된 LLM-as-agent 패턴이다. 추론 비용이 저렴하지만(관련 클립만 인코딩), 엔지니어링 난이도가 높다(검색 품질이 병목).

### 바늘 찾기 벤치마크

표준 장거리 컨텍스트 테스트: 비디오 내 무작위 지점에 고유한 시각적/텍스트 마커를 삽입한 후, 이를 재현해야 하는 쿼리를 요청한다.

지표: 비디오 길이와 마커 위치에 따른 Recall@k.

Gemini 2.5 Pro는 최대 90분 비디오에서 >99% 재현율을 기록한다. 오픈 72B 모델(Qwen2.5-VL-72B, InternVL3-78B)은 30분에서 ~85-90%, 60분 이후 성능 저하.

VideoAgent는 도구가 우수할 경우 2시간 이상 콘텐츠에서 원시 컨텍스트 모델을 능가할 수 있다.

### 어떤 경로를 선택할까

15분 클립에서 최첨단 정확도: 오픈 72B + 네이티브 컨텍스트(Qwen2.5-VL-72B)가 일반적으로 작동한다.

30분~1시간 콘텐츠: 오픈에서는 LongVILA 또는 Video-XL, 클로즈드에서는 Gemini 2.5 Pro. 품질 기준이 중요하다 — 최앞단은 클로즈드.

2시간 이상 콘텐츠: VideoAgent 또는 유사 검색 패턴. 또는 작은 청크로 요약하고 계층적 요약을 입력한다.

### 2026년 프로덕션 패턴

실제 프로덕션 장거리 비디오 파이프라인은 하이브리드 방식:

1. 전체 비디오에 동적 FPS 샘플링 + 공격적 풀링 적용(100k 토큰 글로벌 표현 획득).
2. 72B VLM에 전달하여 글로벌 요약 생성.
3. 사용자가 상세 질문을 하면, 요약을 인덱스로 사용해 에이전트 기반 검색 실행.

이는 글로벌 이해를 위한 무차별적 컨텍스트와 로컬 디테일을 위한 검색을 결합한다.

## 사용 방법

`code/main.py`:

- 1분부터 3시간까지 다양한 FPS 및 풀링(pooling) 조건에서 동영상의 토큰 예산(token budgets)을 계산합니다.
- 무작위 타임스탬프에 마커(marker)를 삽입하고 질문을 입력한 후 재현율(recall)을 평가하는 "바늘 찾기(needle-in-a-haystack)" 시뮬레이션을 실행합니다.
- 특정 클립(clip)을 선택하여 다운스트림 VLM(Video Language Model)에 전달하는 에이전트 기반 검색 라우터(agentic-retrieval router) 시뮬레이터를 포함합니다.

1. 1FPS(초당 프레임 수), 프레임당 81토큰인 45분 강의. 총 토큰 수는? 어떤 모델의 컨텍스트에 적합한가?

2. 바늘 찾기 테스트 설계: 마커를 주입하는 분(단위)과 정확한 쿼리 형식은?

3. 무차별 컨텍스트 Qwen2.5-VL-72B(80k 컨텍스트)와 VideoAgent(Claude 3.5 + 검색)를 1시간 영상에서 비교. 재현율(recall)에서 승자는? 지연 시간(latency)에서 승자는?

4. 링 어텐션(ring attention)의 메모리 비용은 시퀀스 길이에 선형적으로, 장치 개수에 선형적으로 증가한다. 그 이유와 링 회전(ring-rotation) 단계를 생략하면 어떤 문제가 발생하는지 설명하라.

5. Gemini 1.5 5장 "바늘 찾기" 실험 분석. 100만 토큰 vs 1000만 토큰 경계에서 재현율(recall)에 대해 논문에서 발견한 내용은?

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|---------------------|-----------|
| Brute context | "Just more tokens" | LLM 컨텍스트를 수백만 토큰으로 확장; 모든 것을 한 번에 처리 |
| Ring attention | "LWM-style parallel" | 각 장치가 청크를 보유하고 회전하는 분산 어텐션 패턴 |
| Token compression | "Summary tokens" | LLM 이전에 학습된 압축기를 통해 클립당 토큰 수 감소 |
| Needle-in-haystack | "NIH test" | 무작위 지점에 고유 마커를 삽입하고, 테스트 시 모델이 이를 회상하도록 요청 |
| Agentic retrieval | "LLM as query planner" | LLM이 관련 클립을 검색 도구에 요청하고, VLM을 통해 읽은 후 답변을 구성 |
| VideoAgent | "Retrieval pattern for video" | 표준 에이전트 기반 검색 설계: 질문 -> 도구 -> 클립 -> 답변 |
