# 오픈 웨이트 VLM 레시피: 실제로 중요한 것은 무엇인가

> 2024-2026년 오픈 웨이트 VLM 문헌은 다양한 실험 결과(ablation tables)의 숲과 같습니다. Apple의 MM1은 이미지 인코더, 커넥터, 데이터 혼합의 13가지 조합을 테스트했습니다. Allen AI의 Molmo는 상세한 인간 캡션이 GPT-4V 증류보다 우수함을 입증했습니다. Cambrian-1은 20개 이상의 인코더 비교를 수행했습니다. Idefics2는 5축 설계 공간을 공식화했습니다. Prismatic VLMs는 통제된 벤치마크에서 27가지 훈련 레시피를 비교했습니다. 이 모든 정보 속에서 여러 논문에 걸쳐 일관되게 나타나는 소수의 결과가 있습니다: 이미지 인코더는 커넥터 아키텍처보다 중요하며, 데이터 혼합은 둘보다 더 중요하고, 상세한 인간 캡션은 증류된 합성 데이터보다 우수합니다. 이 레슨은 이러한 실험 결과를 해석하여 여러분이 직접 표를 분석할 필요가 없도록 합니다.

**유형:** 학습 + 실습  
**언어:** Python (표준 라이브러리, 실험 결과 표 파서 + 레시피 선택기)  
**선수 지식:** Phase 12 · 05 (LLaVA 베이스라인)  
**소요 시간:** ~180분

## 학습 목표

- 5축 VLM 설계 공간(image encoder, connector, LLM, 데이터 혼합, 해상도 스케줄)의 구성 요소를 명명할 수 있다.
- MM1 / Idefics2 / Cambrian-1의 애블레이션 테이블을 읽고 특정 벤치마크에 영향을 주는 조절 요소(knob)를 예측할 수 있다.
- 계산 예산과 작업 유형(task mix)이 주어졌을 때 새로운 VLM을 위한 레시피(인코더, 커넥터, 데이터, 해상도)를 선택할 수 있다.
- 동일한 토큰 수에서 GPT-4V 증류보다 상세한 인간 작성 캡션이 더 효과적인 이유를 설명할 수 있다.

## 문제 정의

수백 개의 오픈웨이트 VLM(Visual Language Model)이 존재한다. "좋은" 모델과 "최첨단" 모델 사이의 대부분의 격차는 아키텍처가 아닌 데이터, 해상도 스케줄, 인코더 선택에 기인한다. 모델이 저조한 성능을 보일 때 어떤 파라미터를 먼저 조정해야 하는지 아는 것은 500만 GPU-시간의 실수를 방지하는 데 도움이 된다.

2023년 물결(LLaVA-1.5, InstructBLIP, MiniGPT-4)은 캡션 쌍 사전 학습 + LLaVA-Instruct-150k를 기반으로 실행되었다. 좋은 베이스라인이었다. MMMU 점수 약 35%에서 정체되었다.

2024년 물결(MM1, Idefics2, Molmo, Cambrian-1, Prismatic VLMs)은 철저한 실험을 수행했다. 결과는 놀랍고 실용적이었다.

## 개념

### 5축 설계 공간

Idefics2 (Laurençon et al., 2024)는 다음과 같은 축을 명명했습니다:

1. **이미지 인코더**. CLIP ViT-L/14, SigLIP SO400m/14, DINOv2 ViT-g/14, InternViT-6B. 인코더는 패치 크기, 해상도, 사전 학습 목적에서 차이가 있습니다.
2. **커넥터**. MLP (2-4층), Q-Former (32개 쿼리 + 크로스 어텐션), Perceiver Resampler (64개 쿼리), C-Abstractor (컨볼루션 + 이중 선형 풀링).
3. **언어 모델**. Llama-3 8B / 70B, Mistral 7B, Phi-3, Gemma-2, Qwen2.5. LLM 크기가 주요 파라미터 비용입니다.
4. **학습 데이터**. 캡션 쌍 (CC3M, LAION), 인터리브 (OBELICS, MMC4), 지시 (LLaVA-Instruct, ShareGPT4V, PixMo, Cauldron).
5. **해상도 스케줄**. 고정 224/336/448, AnyRes, 네이티브 동적. 학습 중 점진적 증가 또는 일정 유지.

모든 프로덕션 VLM은 각 축에서 선택을 합니다. MMMU 점수 변동의 대부분은 축 1, 4, 5에 의해 설명됩니다 — 커넥터 선택은 영향이 적습니다.

### 축 1: 인코더 > 커넥터

MM1 섹션 3.2는 다음과 같이 보여줍니다: CLIP ViT-L/14에서 SigLIP SO400m/14로 교체하면 MMMU 점수가 3점 이상 상승했습니다. 커넥터를 MLP에서 Perceiver Resampler로 교체하면 1점 미만 상승했습니다. Idefics2도 재현: SigLIP > CLIP, Q-Former ≈ MLP ≈ Perceiver (동일 토큰 수 기준).

Cambrian-1의 "Cambrian Vision Encoders Match-Up" (Tong et al., 2024)은 20개 이상의 인코더를 비전 중심 벤치마크 (CV-Bench)에서 평가했습니다. 리더보드 상위는 DINOv2와 SigLIP가 혼합, CLIP은 중간, ImageBind와 ViT-MAE는 하위입니다. CLIP ViT-L에서 DINOv2 ViT-g/14까지의 격차는 CV-Bench에서 ~5-7점입니다.

2026년 오픈 VLM의 기본 인코더는 SigLIP 2 SO400m/14 (시맨틱 + 밀집 특징), 때로는 DINOv2 ViT-g/14 특징과 연결 (Cambrian의 "공간 비전 집계기"가 이 방식을 사용).

### 축 2: 커넥터 설계는 무의미

MM1, Idefics2, Prismatic, MM-Interleaved는 모두 동일한 결론에 도달했습니다: 고정된 시각 토큰 수에서 커넥터 아키텍처는 거의 영향이 없습니다. 평균 풀링된 패치에 2층 MLP를 적용하면 동일 토큰 예산에서 32개 쿼리 Q-Former과 1점 이내 성능 차이를 보입니다.

중요한 것은 토큰 수입니다. 더 많은 시각 토큰 = 더 많은 LLM 계산 = 일정 수준까지는 성능 향상, 이후 한계 수익 감소. 이미지당 64개 토큰은 OCR에 부족합니다. 576-1024개 토큰이 대부분의 오픈 VLM에 적합합니다. 2048개 이상은 문서 및 차트에만 유용합니다.

Q-Former vs MLP는 품질 문제가 아닌 비용 문제입니다: Q-Former는 이미지 해상도와 무관하게 토큰을 32-64개로 제한, MLP는 모든 패치 토큰을 출력합니다. 고해상도 입력에서는 Q-Former가 LLM 컨텍스트를 절약, 저해상도에서는 차이가 미미합니다.

### 축 3: LLM 크기가 상한을 결정

LLM을 7B에서 13B로 두 배 증가시키면 모든 VLM 논문에서 MMMU 점수가 2-4점 안정적으로 상승합니다. 70B에서는 대부분의 벤치마크가 포화됩니다. VLM의 멀티모달 추론 상한은 LLM의 텍스트 추론 상한입니다 — 시각 인코더는 데이터만 제공할 뿐 추론은 LLM이 수행합니다.

이것이 Qwen2.5-VL-72B와 Claude Opus 4.7이 MMMU-Pro와 ScreenSpot-Pro를 압도하는 이유입니다: 언어 모델 크기가 큽니다. 7B VLM은 커넥터 설계로 70B VLM을 대체할 수 없습니다.

### 축 4: 데이터 — 상세 인간 캡션이 증류보다 우수

Molmo + PixMo (Deitke et al., 2024)는 2024년 모든 사람이 읽어야 할 결과입니다. Allen AI는 인간 주석자가 1-3분 동안 이미지를 상세히 설명하도록 하여 712K개의 밀집 캡션 이미지를 생성했습니다. 학습 데이터에는 GPT-4V 증류가 전혀 없습니다.

Molmo-72B는 11개 벤치마크에서 Llama-3.2-90B-Vision을 능가했습니다. 차이는 아키텍처가 아닌 캡션 품질입니다. 상세 인간 캡션은 짧은 웹 캡션보다 이미지당 5-10배 더 많은 정보를 포함하며, GPT-4V 증류가 환각하는 부분에서 사실적 근거를 유지합니다.

ShareGPT4V (Chen et al., 2023)와 Cauldron (Idefics2)는 인간 + GPT-4V 캡션으로 동일한 방식을 적용했습니다. 추세는 명확합니다: 2026년 프론티어에서 캡션 밀도 > 캡션 양 > 증류 편의성.

### 축 5: 해상도 및 스케줄

Idefics2의 실험: 384 → 448은 1-2점 상승. 448 → 980 (AnyRes로 이미지 분할)은 OCR 벤치마크에서 3-5점 추가 상승. 고정 해상도 학습은 중간 정확도에서 정체, 해상도 점진적 증가 (224 시작, 448 또는 네이티브 종료)는 더 빠르게 학습하고 더 높은 성능을 달성합니다.

Cambrian-1은 해상도 대 토큰 트레이드오프를 실험했습니다: 고정 계산에서 저해상도-더 많은 토큰 또는 고해상도-더 적은 토큰을 선택할 수 있습니다. 고해상도는 OCR에 유리, 저해상도-더 많은 토큰은 일반 장면 이해에 유리합니다.

2026년 프로덕션 레시피: 1단계는 384 고정 해상도로 학습, 2단계는 OCR 중심 작업을 위해 동적 해상도 (최대 1280)로 학습.

### Prismatic 통제 비교

Prismatic VLM (Karamcheti et al., 2024)은 모든 축을 통제한 논문입니다. 동일한 13B LLM, 동일한 지시 데이터, 동일한 평가 — 한 번에 하나의 축만 변경. 결과:

- 이미지당 시각 토큰 수가 변동의 ~60% 설명.
- 인코더 선택이 ~20% 설명.
- 커넥터 아키텍처가 ~5% 설명.
- 나머지 (데이터 혼합, 스케줄러, 학습률)가 ~15% 설명.

이는 대략적인 분해이지만, "무엇을 먼저 실험해야 하는가"에 대한 가장 명확한 답변입니다.

### 2026년 선택 가이드

증거를 바탕으로 2026년 새 프로젝트를 위한 기본 오픈 VLM 레시피:

- **인코더**: NaFlex를 사용한 네이티브 해상도의 SigLIP 2 SO400m/14, 분할/접지 작업이 필요하면 DINOv2 ViT-g/14 밀집 특징과 연결.
- **커넥터**: 패치 토큰에 2층 MLP. 토큰 제약이 없는 한 Q-Former는 생략.
- **LLM**: Qwen2.5 / Llama-3.1 / Gemma 2, 비용 절감을 위해 7B, 품질 우선으로 70B, 목표 지연 시간에 따라 선택.
- **데이터**: PixMo + ShareGPT4V + Cauldron, 작업별 지시 데이터로 보충.
- **해상도**: 동적 (긴 변 기준 최소 256, 최대 1280 픽셀).
- **스케줄**: 1단계 정렬 (프로젝터만), 2단계 전체 미세 조정, 3단계 작업별 미세 조정.

이 기본값들은 이 강의 끝에 인용된 논문들의 측정 실험에서 유래했습니다.

## 사용 방법

`code/main.py`는 **어블레이션 테이블 파서**이자 **레시피 선택기**입니다. MM1과 Idefics2의 **어블레이션 테이블**(간소화된 버전)을 인코딩하고 다음 쿼리를 지원합니다:

- "예산 X와 작업 Y가 주어졌을 때, 어떤 레시피가 승리하나요?"
- "7B Llama에서 SigLIP을 CLIP으로 교체하면 예상되는 MMMU 델타는 얼마인가요?"
- "80% 신뢰도 답변을 위해 어떤 축을 먼저 어블레이션해야 하나요?"

출력은 예상 **벤치마크 델타**가 포함된 순위별 레시피 목록과 "먼저 어블레이션할" 권장 사항입니다.

## Ship It

이 레슨은 `outputs/skill-vlm-recipe-picker.md`를 생성합니다. 목표 작업 조합, 컴퓨팅 예산, 지연 시간 목표가 주어졌을 때, 각 선택을 정당화하는 실험 결과(ablation study)에 대한 인용을 포함한 완전한 레시피(인코더, 커넥터, LLM, 데이터 조합, 해상도 스케줄)를 출력합니다. 새로운 VLM 프로젝트가 시작될 때마다 엔지니어가 Idefics2 실험 결과 테이블을 다시 만드는 것을 방지합니다.

## 연습 문제

1. MM1 섹션 3.2를 읽어주세요. 50M 이미지 예산에서 고정된 2B LLM에 대해 어떤 인코더가 승리하나요? 13B LLM에서 답이 뒤바뀔까요? 그 이유는?

2. Cambrian-1은 DINOv2 + SigLIP을 연결(concatenate)하는 것이 비전 중심 벤치마크에서 단독 사용보다 우수하지만 MMMU에서는 신호(signal)가 추가되지 않음을 발견했습니다. 어떤 벤치마크에서 성능이 향상되고 어떤 벤치마크는 유지될지 예측하세요.

3. 목표는 2B LLM 기반의 모바일 UI 에이전트입니다. 인코더, 커넥터, 해상도, 데이터 믹스를 선택하고 각 선택을 특정 애블레이션 테이블로 정당화하세요.

4. Molmo는 4B 및 72B 모델을 출시했습니다. 4B 모델은 폐쇄형 7B VLM과 경쟁력이 있으며, 72B 모델은 11/11 벤치마크에서 Llama-3.2-90B-Vision을 능가합니다. 이는 LLM 크기 플래토(plateau) 가설에 대해 무엇을 시사하나요?

5. 7B VLM에서 데이터 믹스 품질과 인코더 품질을 분리하기 위한 애블레이션 테이블을 설계하세요. 최소 몇 번의 학습 실행이 필요한가요? 네 가지 축(axis) 설정을 제안하세요.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|---------------------|-----------|
| Ablation | "하나의 조절 장치 돌리기" | 다른 모든 요소를 일정하게 유지하면서 정확히 하나의 설계 공간 축에서 차이가 나는 여러 번의 실행으로 훈련 |
| Connector | "브릿지" / "프로젝터" | 비전 인코더 출력을 LLM의 토큰 공간으로 매핑하는 학습 가능한 모듈 (MLP, Q-Former, Perceiver) |
| Detailed human caption | "밀집 캡션" | 웹 대체 텍스트보다 풍부한 다문장 인간 작성 설명 (일반적으로 80-300 토큰) |
| Distillation | "GPT-4V 캡션" | 더 강력한 독점 VLM으로 생성된 훈련 데이터; 편리하지만 상속된 환각(hallucination) 문제 발생 가능 |
| AnyRes / dynamic res | "고해상도 경로" | 타일링 또는 M-RoPE를 통해 인코더의 기본 해상도보다 큰 이미지를 입력하는 전략 |
| Resolution ramp | "커리큘럼" | 저해상도로 시작하여 점진적으로 해상도를 높이는 훈련 일정; 정렬 학습 가속화 |
| Vision-centric bench | "CV-Bench / BLINK" | 언어 중심 추론보다 세밀한 시각 인식을 강조하는 평가 |
| PixMo | "Molmo의 데이터" | Allen AI에서 제공하는 712K개의 밀집 캡션이 달린 이미지 데이터셋; 인간 음성을 밀집 캡션으로 변환 |
