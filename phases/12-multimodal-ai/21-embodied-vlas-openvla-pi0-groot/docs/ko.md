# Embodied VLAs: RT-2, OpenVLA, π0, GR00T

> 웹사이트 레시피를 읽고 주방 로봇에서 실행한 첫 번째 모델은 RT-2(Google DeepMind, 2023년 7월)였습니다. RT-2는 행동을 텍스트 토큰으로 이산화하고, 웹 데이터와 로봇 행동 데이터에 VLM(Vision-Language Model)을 공동 미세 조정하며, 웹 규모의 시각-언어 지식이 로봇 제어로 전이됨을 입증했습니다. OpenVLA(2024년 6월)는 오픈 7B 기준 모델을 출시했습니다. Physical Intelligence의 π0 시리즈(2024-2025)는 흐름 매칭(flow-matching) 행동 전문가를 추가했습니다. NVIDIA의 GR00T N1(2025년 3월)은 인간형 로봇을 위한 이중 시스템(System 1 / System 2) 제어를 대규모로 구현했습니다. 시각-언어-행동(VLA) 기본 요소 — 보고, 읽고, 행동하는 단일 모델 — 은 이 단계의 이해 모델과 15단계(Phase 15)의 자율 시스템 사이의 가교입니다.

**유형:** 학습  
**언어:** Python (표준 라이브러리, 행동 토크나이저 + VLA 추론 스켈레톤)  
**선수 지식:** 12단계 · 05 (LLaVA), 15단계 (자율 시스템, 참조됨)  
**소요 시간:** ~180분

## 학습 목표

- **액션 토크나이징** 설명: 이산적 빈 인코딩(RT-2), FAST 효율적 액션 토큰, 연속적 흐름 매칭 액션(π0).
- **웹 + 로봇 데이터 공동 미세 조정**이 새로운 작업으로의 일반 지식 전달을 보존하는 이유 설명.
- **OpenVLA**(오픈 7B Llama+VLM), **π0**(흐름 매칭), **GR00T N1**(듀얼 시스템)을 동일한 로봇 작업에서 비교.
- **Open X-Embodiment** 데이터셋 이름 및 **RT-X 훈련 코퍼스**로서의 역할 설명.

## 문제 정의

자연어 명령으로부터 집안일을 수행하는 로봇은 1970년대부터 연구 대상이었다. 2020년대의 해결책: 비전-언어-행동(VLA) 모델. VQA에 사용되는 동일한 VLM 아키텍처를 활용하지만, 출력이 텍스트 대신 행동(관절 토크, 엔드 이펙터 포즈, 이산 명령)이다.

VLA의 고유한 과제:

1. 행동 공간은 연속적(관절 각도, 힘)이며 고차원적이다(7자유도 팔 + 3자유도 그리퍼 = 30Hz에서 10차원).
2. 로봇 전용 훈련 데이터가 부족하다. Open X-Embodiment에는 ~1M 궤적이 있는 반면, 웹 텍스트-이미지 데이터는 5B+이다.
3. 제어 주파수가 중요하다. 30Hz 제어 루프는 행동당 33ms의 시간 제약을 의미한다.
4. 안전성. 잘못된 행동은 하드웨어, 인간 또는 재산에 손상을 입힐 수 있다.

## 개념

### 액션 토큰화 (RT-2)

RT-2의 트릭: 각 조인트 타겟을 양자화된 텍스트 토큰으로 표현. 정규화된 [-1, 1] 범위를 256개의 빈(bin)으로 이산화하고, 각 빈을 어휘 사전 ID에 매핑. 10-DOF 액션은 각 제어 단계에서 10개의 토큰이 됨.

PaLM-X VLM을 다음 혼합 데이터로 공동 미세 조정:
- 웹 이미지-텍스트 쌍 (캡션 생성, VQA).
- 로봇 데모, 액션을 토큰으로 표현.

모델은 "빨간 큐브를 들어 올려" (언어) → 이미지 (비전) → 10-토큰 액션 시퀀스 (이산화된 조인트 타겟)를 처리. 웹 사전 학습은 일반 지식 전달을 보존: RT-2는 "빠르게 움직이는 물체 쪽으로 이동"을 따를 수 있음. "빠르게 움직이는"이 학습 데이터에 없어도 가능.

RT-2 논문에서 3-5Hz 추론 속도, VLM 자동회귀 디코딩에 의해 제한됨.

### OpenVLA — 오픈 7B 레퍼런스

OpenVLA (Kim et al., 2024년 6월)는 오픈 웨이트인 RT-2의 동등 모델. 7B Llama 백본, DINOv2 + SigLIP 듀얼 비전 인코더, 256개 빈에 대한 액션 토큰화.

Open X-Embodiment (22개 로봇에 걸친 970k 궤적)로 학습. 새로운 로봇에 적응하기 위한 LoRA 미세 조정 지원 포함.

추론: A100에서 양자화 시 4-5Hz. 느린 조작에는 충분하지만 고주파 제어에는 부족.

### FAST 토크나이저 — 더 빠른 액션 디코딩

Pertsch et al. (2024)는 이산 빈 토큰화가 비효율적임을 보여줌 — 대부분의 액션은 빈 공간의 작은 영역에 집중. FAST (주파수 영역 액션 시퀀스 토크나이저)는 DCT를 통해 액션 시퀀스를 압축하고 계수를 양자화.

30단계 액션 궤적은 300개의 이산 빈 토큰 대신 ~10개의 FAST 토큰으로 변환. 품질 저하 없이 추론 속도 3-5배 향상.

### π0과 흐름 매칭 액션

Physical Intelligence의 π0 (Black et al., 2024년 10월)은 이산 액션 토큰을 흐름 매칭 액션 전문가로 대체:
- 작은 액션 트랜스포머가 VLM의 은닉 상태를 읽고 정류 흐름(rectified flow)을 통해 연속적인 50단계 액션 시퀀스를 출력.
- 액션 헤드는 흐름 매칭 손실로 학습; VLM 사전 학습은 변경 없음.
- 추론: ~5번의 디노이징 단계에서 전체 액션 시퀀스 생성, 효과적으로 50Hz 제어.

π0의 주장: 광범위한 조작 작업에서 OpenVLA와 Octo를 능가. 연속 액션 표현은 이산화로 인한 매끄러움 손실을 방지.

π0.5와 π0-FAST는 점진적 업그레이드. π0-FAST는 FAST 토큰화와 흐름 매칭을 결합.

### GR00T N1 — 휴머노이드용 듀얼 시스템

NVIDIA의 GR00T N1 (2025년 3월)은 휴머노이드 로봇(>30 DOF, 전신)을 위해 설계:
- 시스템 2: 장면 + 지시를 읽는 대형 VLM, ~1Hz로 고수준 하위 목표 생성.
- 시스템 1: 하위 목표에 조건화된 저수준 50-100Hz 조인트 명령을 생성하는 소형 액션 헤드 트랜스포머.

분할은 카너먼의 빠른-느린 사고에 매핑: 시스템 2는 계획, 시스템 1은 실행. 이점: 느린 VLM 크기 계획이 빠른 제어를 차단하지 않음; 시스템 1은 지연 시간을 위해 작게 유지.

GR00T N1.7 (2025년 말)은 데이터 확장 개선. GR00T는 Omniverse의 시뮬레이션-실제 데이터로 미세 조정.

### Open X-Embodiment

학습 데이터. RT-X (2023년 10월)는 22개 로봇에서 1M 궤적을 포함하는 22개 데이터셋을 통합. Open X-Embodiment는 모두가 사용하는 코퍼스:
- ALOHA / Bridge V2 / Droid / RT-2 Kitchen / Language Table.
- 각 샘플: (로봇 상태, 카메라 뷰, 지시, 액션 시퀀스).
- 학습 위생: 액션 공간 통합, 조인트 범위 정규화, 카메라 크기 조정.

OpenVLA와 π0은 Open X-Embodiment에서 학습. 특정 로봇과의 도메인 격차는 100-1000개의 작업별 데모에 대한 LoRA 미세 조정으로 해소.

### 공동 미세 조정 vs 로봇 전용

공동 미세 조정은 웹 VQA 데이터와 로봇 궤적을 혼합. 비율이 중요: 너무 많은 VQA는 액션 망각을 유발; 너무 많은 로봇 데이터는 일반 지식 손실.

RT-2의 비율: ~1:1. OpenVLA: ~0.5:1 (웹:로봇). π0: 유사. 정확한 비율은 데이터셋 크기당 튜닝할 하이퍼파라미터.

로봇 전용 학습은 분포 외 지시에서 실패하는 작업별 모델을 생성. 공동 미세 조정은 "데모에 있는 빨간 큐브 들어 올리기"와 "왼쪽에서 세 번째로 큰 물체 들어 올리기 (새로운 표현법)"의 차이.

### 안전 및 액션 제한

모든 프로덕션 VLA는 다음을 포함:
- 하드 조인트 제한 (사양 초과 토크 불가).
- 속도 제한 (소프트 클리핑).
- 작업 공간 경계 (엔드 이펙터가 테이블 벗어날 수 없음).
- 새로운 작업에 대한 인간 개입 승인.

이들은 VLA 외부의 제어 계층 검사로 존재. VLA의 출력은 제안이지 명령이 아님.

## 사용 방법

`code/main.py`:

- 256-빈(bin) 액션 토큰화 및 디토큰화 구현.
- DCT(이산 코사인 변환) + 양자화 기반 FAST 토크나이저 스케치.
- (이산 빈, FAST, 연속 흐름) 간 액션 스텝당 토큰 수 비교.
- RT-2 → OpenVLA → π0 → GR00T의 계보 요약 출력.

## Ship It

이 레슨은 `outputs/skill-vla-action-format-picker.md`를 생성합니다. 로봇 작업(조작, 항법, 인간형 전신 동작)이 주어졌을 때, 이산-빈 + RT-2, FAST + OpenVLA, 흐름-일치 + π0, 또는 이중-시스템 + GR00T 중에서 선택합니다.

## 연습 문제

1. 30Hz 제어 속도의 10-DOF(자유도) 팔. 256개의 빈(bin)으로 이산화 토큰화할 때 초당 몇 개의 토큰이 생성되나요? 7B VLM(비전 언어 모델)이 이를 처리할 수 있을까요?

2. FAST 토큰화는 30단계 궤적을 ~10개의 토큰으로 압축합니다. 궤적에 고주파 운동(예: 드럼 연주)이 포함된 경우 사용자는 무엇을 잃게 되나요?

3. π0의 흐름 매칭(flow-matching) 헤드는 ~5단계로 노이즈를 제거합니다. OpenVLA의 4-5Hz 자기회귀 디코딩과 처리량(throughput)을 비교하세요.

4. GR00T의 시스템 1/시스템 2 분할은 카너먼의 이론에 대응됩니다. 이족 보행에 도움이 될 수 있는 다른 분할(시스템 3?)을 제안하세요.

5. Open X-Embodiment 4장의 데이터셋 선별(curation) 섹션을 읽고, 도메인 누출(domain leakage)을 방지하는 세 가지 선별 규칙을 명시하세요.
