# LLaVA와 시각적 지시 튜닝

> LLaVA (2023년 4월)는 지구상에서 가장 많이 복제된 멀티모달 아키텍처입니다. BLIP-2의 Q-Former를 2층 MLP로 대체하고, Flamingo의 게이팅 크로스-어텐션(gated cross-attention)을 순진한 토큰 연결(naive token concatenation)로 교체했으며, GPT-4가 텍스트 전용 캡션에서 생성한 158k 시각적 지시 턴(visual-instruction turns)으로 학습되었습니다. 2023년부터 2026년 사이에 VLM을 구축한 모든 실무자는 LLaVA의 변형을 구축했습니다. LLaVA-1.5는 AnyRes를 추가했고, LLaVA-NeXT는 해상도를 향상시켰습니다. LLaVA-OneVision은 이미지, 멀티이미지, 비디오를 하나의 레시피로 통합했습니다. 이 레슨은 레시피를 읽고, 프로젝터(projector)를 구현하며, "더 단순한 것이 이긴다"는 이유를 설명합니다.

**유형:** 구축(Build)
**언어:** Python (stdlib, 프로젝터 + 지시 템플릿 빌더)
**선수 지식:** 12단계 · 02 (CLIP), 11단계 (LLM 엔지니어링 — 지시 튜닝)
**소요 시간:** ~180분

## 학습 목표

- ViT 패치 임베딩(차원 1024)을 LLM의 임베딩 차원(차원 4096)으로 매핑하는 2계층 MLP 프로젝터 구축
- LLaVA 2단계 레시피 수행: (1) 558k 캡션 쌍에서의 프로젝터 정렬, (2) 158k GPT-4 생성 턴에서의 시각적 지시 튜닝
- 이미지 토큰 플레이스홀더, 시스템 프롬프트, 사용자/조수 턴이 포함된 LLaVA 형식 프롬프트 구성
- Q-Former의 토큰 예산 우위에도 불구하고 커뮤니티가 Q-Former에서 MLP로 전환한 이유 설명

## 문제

BLIP-2의 Q-Former(레슨 12.03)는 이미지를 32개의 토큰으로 압축합니다. 깔끔하고 효율적이며 벤치마크에 좋습니다. 하지만 두 가지 문제가 있습니다.

첫째, Q-Former는 학습 가능하지만 그 손실 함수는 최종 작업이 아닙니다. 1단계에서는 ITC+ITM+ITG를 학습합니다. 2단계에서는 LM 손실을 학습합니다. 쿼리들은 LLM이 디코딩해야 하는 중간 표현을 학습합니다. 병목 현상에서 정보가 손실됩니다.

둘째, Q-Former는 188M 파라미터를 가지며, LLaVA의 2023년 규모에서는 대상 LLM과 함께 공동 설계해야 했습니다. LLM을 변경하면 Q-Former를 재학습해야 합니다. 비전 인코더를 변경하면 재학습이 필요합니다. 모든 조합은 별도의 R&D 프로젝트였습니다.

LLaVA의 해결책은 단순함이 부끄러울 정도였습니다: ViT의 576개 패치 토큰을 가져와 각각을 2층 MLP(`1024 → 4096 → 4096`)에 통과시킨 후, 576개 모두를 LLM의 입력 시퀀스에 넣습니다. 병목 현상 없음. 이상한 목표에 대한 1단계 사전 학습 없음. 직접 LM 손실로 MLP를 학습하기만 하면 됩니다.

데이터는 어디서 왔을까요? LLaVA의 두 번째 통찰: GPT-4(텍스트 전용)를 사용해 지시 데이터를 생성합니다. GPT-4에 COCO 캡션과 이미지의 바운딩 박스 데이터를 입력하고, 대화, 설명, 복잡한 추론 질문을 생성하도록 요청합니다. 158k개의 지시-응답 턴을 무료로 얻습니다. 인간 주석 없음.

결과: 8개의 A100에서 하루 동안 실행된 VLM이 MMMU에서 Flamingo를 능가하고, 커뮤니티가 확장할 수 있는 오픈 체크포인트를 출시했습니다. 2023년 말에는 50개 이상의 포크가 생성되었습니다.

## 개념

### 아키텍처

LLaVA-1.5 13B:
- 비전 인코더: CLIP ViT-L/14 @ 336 (1단계 동안 고정, 2단계에서 선택적으로 해제).
- 프로젝터: GELU 활성화 함수를 사용하는 2층 MLP, `1024 → 4096 → 4096`.
- LLM: Vicuna-13B (이후 Llama-3.1-8B).

이미지 + 텍스트 프롬프트에 대한 순전파:

```
img -> ViT -> 1024차원 576개 패치
패치 -> MLP -> 4096차원 576개 토큰
프롬프트: 시스템 + "<image>" 플레이스홀더 + 사용자 질문
<image> 토큰을 576개 투영된 토큰으로 대체
전체 시퀀스를 LLM에 입력
응답 디코딩
```

이미지는 LLM 컨텍스트의 576개 토큰을 차지합니다. 2048 컨텍스트에서는 텍스트에 1472개 토큰이 남고, 32k 컨텍스트에서는 반올림 오차 수준입니다.

### 1단계: 프로젝터 정렬

ViT 고정. LLM 고정. 2층 MLP만 훈련. 데이터셋: 558k 이미지-캡션 쌍(LAION-CC-SBU). 손실: 투영된 이미지 토큰에 조건화된 캡션에 대한 언어 모델링.

배치 128로 단일 에포크에 몇 시간 내 완료. 프로젝터는 ViT 공간을 LLM 공간으로 매핑하는 방법을 학습합니다. 작업별 감독 없음.

### 2단계: 시각적 지시 튜닝

프로젝터 훈련 가능 상태 유지. LLM 해제(일반적으로 전체, 때로는 LoRA). 158k 시각적 지시 턴으로 훈련.

지시 데이터 생성이 핵심입니다. Liu et al.은 다음 방법으로 생성:
1. COCO 이미지 선택.
2. 텍스트 설명 추출(5개 인간 캡션 + 바운딩 박스 목록).
3. 3가지 프롬프트 템플릿으로 GPT-4에 전송:
   - 대화: "이 이미지에 대해 사용자와 어시스턴트 간 대화를 생성하세요."
   - 상세 설명: "이미지에 대한 풍부하고 상세한 설명을 제공하세요."
   - 복잡한 추론: "이미지 추론이 필요한 질문을 생성한 후 답변하세요."
4. GPT-4 출력을 (지시, 응답) 쌍으로 파싱.

이 과정은 이미지를 직접 건드리지 않고 텍스트 설명만 사용합니다. GPT-4가 그럴듯한 이미지 내용을 환각하지만, 158k 턴으로 대화 기능을 활성화하는 데 충분했습니다.

### 커뮤니티가 이를 복제한 이유

- 1단계 전용 손실 조정 불필요. 전체 과정에서 LM 손실 사용.
- 프로젝터는 며칠이 아닌 몇 시간 내 훈련 완료.
- LLM 교체 가능(LLaVA-Llama2, LLaVA-Mistral, LLaVA-Llama3). 프로젝터만 재훈련.
- 시각적 지시 데이터 파이프라인은 GPT-4를 사용하며 새 도메인에 대해 재생성이 저렴.

### LLaVA-1.5와 LLaVA-NeXT

LLaVA-1.5(2023년 10월) 추가 사항:
- 학술 작업 데이터(VQA, OKVQA, RefCOCO)를 지시 튜닝에 혼합.
- 개선된 시스템 프롬프트.
- 2048 → 32k 컨텍스트.

LLaVA-NeXT(2024년 1월) 추가 사항:
- AnyRes: 고해상도 이미지를 2x2 또는 1x3 그리드로 분할(336x336 크롭) + 글로벌 저해상도 썸네일. 각 크롭은 576개 토큰; 이미지당 총 약 2880개 시각 토큰. OCR 및 차트 작업 성능 향상.
- ShareGPT4V(고품질 GPT-4V 캡션)와의 혼합으로 개선된 지시 데이터.
- 더 강력한 기본 LLM(Mistral-7B, Yi-34B).

### LLaVA-OneVision

레슨 12.08에서 OneVision을 심층 분석합니다. 요약: 동일한 프로젝터를 사용하지만, 단일 이미지, 다중 이미지, 비디오를 하나의 모델에서 공유 시각 토큰 예산으로 처리하는 커리큘럼으로 훈련.

### Q-Former와의 비교

| | Q-Former(BLIP-2) | MLP(LLaVA) |
|---|---|---|
| 이미지당 시각 토큰 | 32 | 576(기본) 또는 2880(AnyRes) |
| 훈련 가능 파라미터 | 188M + LM | 40M + LM |
| 1단계 손실 | ITC+ITM+ITG | LM 전용 |
| LLM 교체 | 재훈련 필요 | 최소 재훈련으로 교체 |
| 다중 이미지 | 불편 | 자연스러움(연결) |
| 비디오 | 불편 | 자연스러움(프레임별 연결) |
| 토큰 예산 | 작음 | 큼 |

MLP는 단순성과 토큰 유연성에서 우위. Q-Former는 토큰 예산에서 우위. 2023년 말 기준 토큰 예산은 더 이상 제약 요소가 아니었고(LLM 컨텍스트 32k-128k+로 성장), 단순성이 우세했습니다.

### 프롬프트 형식

```
호기심 많은 인간과 인공지능 어시스턴트 간의 대화. 어시스턴트는 인간의 질문에 도움이 되고 상세하며 예의 바른 답변을 제공합니다. USER: <image> 이 이미지를 자세히 설명해주세요. ASSISTANT: 이미지에는 ... |
```

`<image>`는 플레이스홀더 토큰입니다. 토큰화 전 576개 시각 토큰(또는 AnyRes 시 2880개)으로 대체됩니다. 토크나이저는 훈련 시보다 약간 긴 시퀀스를 보지만, 1단계에서 학습했기 때문에 LLM이 새로운 입력을 처리합니다.

### 파라미터 경제성

LLaVA-1.5-7B 구성:
- CLIP ViT-L/14 @ 336: 303M(1단계 고정, 2단계에서 종종 해제).
- 프로젝터(2x 선형): ~22M 훈련 가능.
- Llama-7B: 7B.
- 총계: 7.3B 파라미터. 2단계 훈련 가능: 전체 7B + 22M 프로젝터.

2단계 훈련 비용: 8xA100에서 약 20시간. 이것이 핵심 숫자입니다. 하루, 하나의 노드, 재현 가능. 그래서 LLaVA가 확산되었습니다.

## 사용 방법

`code/main.py`는 다음을 구현합니다:

1. 순수 Python으로 작성된 2계층 MLP 프로젝터(장난감 규모용 dim 16 → 32 → 32).
2. 프롬프트 생성 파이프라인: 시스템 프롬프트 + `<image>`를 N개의 투영된 토큰으로 대체 + 사용자 발화 + 어시스턴트 생성 플레이스홀더.
3. 576토큰 시각 블록이 LLM 컨텍스트에서 차지하는 비율(2k / 32k / 128k 컨텍스트 소비량 대비)을 시각화하는 도구.

## Ship It

이 레슨은 `outputs/skill-llava-vibes-eval.md`를 생성합니다. LLaVA 계열 체크포인트를 입력으로 받아 10개 프롬프트로 구성된 vibes-eval 스위트(캡셔닝 3개, VQA 3개, 추론 2개, 거부 2개)를 실행하고 사람이 읽을 수 있는 스코어카드를 보고합니다. 벤치마크가 아닌, 프로젝터와 LLM이 잘 연결되었는지 확인하는 스모크 테스트입니다.

## 연습 문제

1. `1024 → 4096 → 4096` 구조의 2층 MLP 프로젝터(projector)에 대한 학습 가능 파라미터 수를 계산하시오. GELU와 바이어스(bias)를 사용할 때, 이 프로젝터가 LLaVA-13B 모델의 몇 %를 차지하는지 계산하시오.

2. "거부(refusal)" 사례를 위한 LLaVA 프롬프트를 구성하시오 — 이미지에 개인(private individual)이 포함되어 있다. 예상되는 어시스턴트 응답을 작성하시오. LLaVA가 이 제로샷(zero-shot) 사례를 거부해야 하는 이유와, 거부를 강화하기 위해 필요한 훈련 데이터는 무엇인지 설명하시오.

3. LLaVA-NeXT 블로그의 AnyRes 섹션을 읽고, 1344x672 이미지에 대한 시각적 토큰 수를 계산하시오. 336x336 해상도의 기본 576 토큰과 비교하시오.

4. LLaVA 1단계 프로젝터는 캡션에 대한 LM 손실(loss)로 훈련된다. 1단계를 건너뛰고 바로 2단계(시각적 지시 튜닝)로 진행하면 어떤 문제가 발생하는가? 답변을 위해 Prismatic VLMs의 실험 결과(arXiv:2402.07865)를 인용하시오.

5. LLaVA-Instruct-150k는 GPT-4와 COCO 캡션을 사용하여 지시(instruction)를 생성한다. 새로운 도메인(의료 X-레이, 위성 이미지)에 대해 도메인 지시를 생성하는 4단계 데이터 파이프라인을 설명하시오. 각 단계에서 발생할 수 있는 문제는 무엇인가?

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|----------------|------------------------|
| 프로젝터(Projector) | "MLP 브리지" | ViT 차원을 LLM 차원으로 매핑하는 GELU 활성화 함수를 사용한 2층 MLP |
| 이미지 토큰(Image token) | "<image> 플레이스홀더" | 추론 전에 N개의 투영된 시각적 토큰으로 대체되는 프롬프트 마커 |
| 시각적 지시 튜닝(Visual instruction tuning) | "LLaVA 2단계" | GPT-4가 생성한 (이미지, 지시, 응답) 삼중항으로 훈련 |
| 1단계 정렬(Stage 1 alignment) | "프로젝터 사전 훈련" | ViT와 LLM을 고정하고 캡션에 대한 LM 손실로 프로젝터 훈련 |
| AnyRes | "다중 크롭 타일링" | 고해상도 이미지를 타일 그리드로 분할하고 각 타일의 시각적 토큰을 연결 |
| LLaVA-Instruct | "GPT-4 생성" | COCO 캡션 + GPT-4로 합성된 158k개의 지시-응답 쌍 |
| 비전 인코더 고정(Vision encoder freeze) | "백본 잠금" | 1단계에서 CLIP 가중치가 업데이트되지 않으며, 때로는 2단계에서도 업데이트되지 않음 |
| ShareGPT4V | "더 나은 캡션" | GPT-4V로 생성된 100만 개의 밀집 캡션, 고품질 정렬에 사용 |
| VQA | "시각적 질의응답" | 이미지에 대한 자유 형식 질문에 답변하는 작업 |
| 프리즘 VLMs(Prismatic VLMs) | "설계 공간 논문" | Karamcheti 2024의 프로젝터 및 데이터 선택 사항을 체계적으로 테스트한 실험 논문 |
