# Transfusion: 하나의 트랜스포머에서 자기회귀 텍스트 + 확산 이미지

> 카멜레온과 Emu3는 이산 토큰에 모든 것을 걸었습니다. 그들은 작동하지만, 양자화 병목 현상이 눈에 띕니다 — 이미지 품질이 연속 공간 확산 모델 아래에서 정체됩니다. Transfusion (Meta, Zhou et al., 2024년 8월)은 반대 방향으로 베팅합니다: 이미지를 연속적으로 유지하고, VQ-VAE를 완전히 제거하며, 두 가지 손실 함수로 하나의 트랜스포머를 훈련시킵니다. 텍스트 토큰은 다음 토큰 예측을 수행합니다. 이미지 패치는 흐름 매칭/확산 손실을 받습니다. 두 목적 함수는 동일한 가중치를 최적화합니다. Stable Diffusion 3의 기반 아키텍처(MMDiT)는 가까운 친척입니다. 이 레슨은 Transfusion 논문을 읽고, 장난감용 두 손실 트레이너를 구축하며, 하나의 트랜스포머가 두 작업을 수행할 수 있게 하는 어텐션 마스크를 추적합니다.

**유형:** 구축
**언어:** Python (표준 라이브러리, MNIST 규모의 장난감에 대한 두 손실 트레이너)
**사전 요구 사항:** 12단계 · 11단계 (카멜레온), 8단계 (생성형 AI)
**소요 시간:** ~180분

## 학습 목표

- 하나의 백본에서 두 가지 손실(텍스트 토큰에 대한 NTP(Neural Text Processing), 이미지 패치에 대한 확산 MSE(Mean Squared Error))을 실행하는 트랜스포머를 구성(wire)하기.
- 이미지 패치 간 양방향 어텐션(bidirectional attention)과 텍스트 토큰에 대한 인과적 어텐션(causal attention)을 조합한 마스크가 적합한 선택인 이유 설명하기.
- Transfusion 방식(연속 이미지, 확산 손실)과 Chameleon 방식(이산 이미지, NTP)을 계산 효율성(compute), 품질(quality), 코드 복잡도(code complexity) 측면에서 비교하기.
- MMDiT의 기여점: 각 블록별 모달리티 특화 가중치(modality-specific weights), 잔차 스트림(residual stream)에서의 통합 어텐션(joint attention) 명명하기.

## 문제 정의

이산(discrete) vs 연속(continuous) 이미지 토큰 논쟁은 LLM(Large Language Model)보다 오래된 주제입니다. 연속 표현(원자 픽셀, VAE 잠재 공간)은 세부 정보를 보존합니다. 이산 토큰(VQ 인덱스)은 트랜스포머의 기본 어휘(vocabulary)에 적합하지만 양자화 단계에서 세부 정보가 손실됩니다.

Chameleon / Emu3는 이산 방식을 선택했습니다: 하나의 손실 함수, 하나의 아키텍처. 하지만 이미지 품질은 토크나이저 성능에 의해 제한됩니다.

확산 모델(Diffusion models)은 연속 방식을 선택했습니다: 탁월한 이미지 품질, 하지만 LLM과 별도의 모델, 복잡한 노이즈 스케줄 엔지니어링, 텍스트 생성과의 깔끔한 통합 부재.

Transfusion은 질문합니다: 둘 다 가질 수 있을까요? 이미지를 연속으로 유지하면서 하나의 모델을 훈련시키고, 두 손실 함수를 하나의 그래디언트 단계로 결합할 수 있을까요?

## 개념

### 두 손실 아키텍처

단일 디코더 전용 트랜스포머가 다음 내용을 포함하는 시퀀스를 처리합니다:

- 텍스트 토큰 (이산적, BPE 어휘에서 추출).
- 이미지 패치 (연속적, 16x16 픽셀 블록을 선형 임베딩을 통해 은닉 차원으로 투영 — ViT 인코더 입력과 동일).
- 연속적 패치가 위치한 영역을 표시하는 `<image>` 및 `</image>` 태그.

순전파(forward pass)는 한 번만 실행됩니다. 손실은 토큰당 두 헤드 중 하나를 선택합니다:

- 텍스트 토큰: 어휘-로그잇 헤드에 대한 표준 교차 엔트로피.
- 이미지 패치: 연속적 패치에 대한 확산 손실 — 각 패치에 추가된 노이즈를 예측.

그래디언트는 공유 트랜스포머 본체를 통해 흐릅니다. 두 손실 모두 공유 가중치를 동시에 개선합니다.

### 어텐션 마스크: 인과적 텍스트 + 양방향 이미지

텍스트 토큰은 인과적이어야 합니다 — 텍스트 토큰이 미래 텍스트에 어텐션하는 것을 허용하지 않아야 하며, 그렇지 않으면 교사 강요(teacher forcing)가 깨집니다. 그러나 이미지 패치는 단일 스냅샷을 나타내므로, 동일한 이미지 블록 내에서 서로 양방향 어텐션해야 합니다.

마스크:

```
M[i, j] = 1 if:
  (i가 텍스트이고 j가 텍스트이며 j <= i)   # 텍스트에 대한 인과적
  OR (i가 이미지이고 j가 이미지이며 same_image_block(i, j))   # 이미지 내 양방향
  OR (i가 텍스트이고 j가 이미지이며 j < i_image_end)   # 텍스트는 이전 이미지에 어텐션
  OR (i가 이미지이고 j가 텍스트이며 j < i_image_start)   # 이미지는 선행 텍스트에 어텐션
```

훈련 및 추론 시 블록-삼각 마스크로 구현됩니다.

### 트랜스포머 내부의 확산 손실

확산 손실은 표준적입니다: 이미지 패치에 노이즈를 추가하고, 모델이 노이즈(또는 동등하게 깨끗한 패치)를 예측하도록 합니다. Transfusion의 버전은 흐름 매칭(flow matching)을 사용합니다 — 노이즈에서 깨끗한 상태로의 속도장을 예측합니다.

훈련 중:
1. 각 이미지 패치 x0에 대해 무작위 타임스텝 t를 샘플링합니다.
2. 노이즈 ε을 샘플링하고, xt = (1-t) * x0 + t * ε (흐름 매칭을 위한 선형 보간)을 계산합니다.
3. 트랜스포머는 v_theta(xt, t)를 예측합니다; 손실 = MSE(v_theta(xt, t), ε - x0).
4. 동일한 시퀀스에서 텍스트 NTP 손실과 함께 역전파(backpropagation)를 수행합니다.

추론 시 생성은 다음과 같습니다:
- 텍스트 토큰: 표준 자기회귀적 샘플링.
- 이미지 패치: 이전 텍스트 토큰에 조건화된 확산 샘플링 루프 (일반적으로 10-30 단계).

### MMDiT: Stable Diffusion 3의 변형

Stable Diffusion 3 (Esser et al., 2024년 3월)은 Transfusion과 거의 동시에 MMDiT(Multimodal Diffusion Transformer)를 출시했습니다. 두 아키텍처는 형제 관계입니다.

MMDiT의 주요 차이점:

- 블록별 모달리티 전용 가중치. 각 트랜스포머 블록은 텍스트 토큰과 이미지 패치에 대해 별도의 Q, K, V, MLP 가중치를 가집니다. 어텐션은 공동(교차 모달리티)로 수행되며, 나머지는 모달리티 전용입니다.
- 직교 흐름(rectified flow) 훈련. DDPM보다 샘플링과 수학이 간단한 특정 흐름 매칭 변형입니다.
- 규모. MMDiT는 SD3의 백본입니다 (2B 및 8B 파라미터 변형). Transfusion 논문은 7B까지 확장됩니다.

둘 모두 동일한 핵심 아이디어로 수렴합니다: 하나의 트랜스포머가 텍스트에 대해 NTP를 실행하고 연속적 이미지 표현에 대해 확산을 실행합니다.

### Chameleon 스타일보다 우수한 이유

이미지 생성에서 연속적-확산과 이산적-NTP 간의 품질 차이는 측정 가능합니다. Transfusion 논문에서는 다음을 보고합니다:

- 7B 파라미터에서 동일한 크기의 Chameleon 스타일 모델보다 FID에서 3-5점 우수.
- 토크나이저 훈련 불필요 — 이미지 인코더가 더 간단함 (은닉 차원으로의 선형 투영, ViT 입력 레이어와 동일).
- 추론 시 이미지 패치 노이즈 제거를 병렬화할 수 있음, 자기회귀적 이미지 토큰과 달리.

단점: Transfusion은 이중 손실 모델이므로 훈련 동역학이 더 복잡합니다. 손실 가중치 조정이 필요합니다. NTP와 확산 간의 스케줄 불일치로 인해 한 헤드가 우세해질 수 있습니다.

### 다운스트림에 위치

Janus-Pro(레슨 12.15)는 Transfusion의 아이디어를 개선하여 이해를 위한 비전 인코더와 생성을 위한 인코더를 분리하면서 — SigLIP과 VQ를 각각 사용 — 트랜스포머 본체를 공유합니다. Show-o(레슨 12.14)는 확산 대신 이산적-확산(마스킹 예측)을 사용합니다. 통합 생성 패밀리는 Transfusion 이후 빠르게 분기됩니다.

2026년 프로덕션 VLM에서 이미지를 출력하는 — Gemini 3 Pro, GPT-5, Claude Opus 4.7의 이미지 생성 경로 — 거의 확실히 이 패밀리의 후손을 사용합니다. 세부 사항은 프로프라이어터리입니다.

## 사용 방법

`code/main.py`는 작은 MNIST 유사 문제에 대한 장난감 Transfusion 모델을 구축합니다:

- 텍스트 캡션은 숫자(0-9)를 설명하는 짧은 정수 시퀀스입니다.
- 이미지는 4x4 바이트 그리드입니다.
- 공유 가중치 선형 투영 쌍이 트랜스포머 대체재로 작용합니다; 텍스트에는 NTP 손실, 노이즈 패치에 MSE 손실을 적용합니다.
- 학습 루프는 두 손실을 번갈아 사용하며, 어텐션 마스크(attention mask)는 명시적으로 구성됩니다.
- 생성은 단일 순방향 패스로 텍스트 캡션과 4x4 이미지를 동시에 생성합니다.

트랜스포머는 장난감 수준입니다. 두 손실 파이프라인, 어텐션 마스크 구성, 추론 루프가 실제 핵심 구성 요소입니다.

## Ship It

이 레슨은 `outputs/skill-two-loss-trainer-designer.md`를 생성합니다. 새로운 멀티모달 학습 작업(텍스트 + 이미지, 텍스트 + 오디오, 텍스트 + 비디오)이 주어졌을 때, 두 가지 손실 일정(손실 가중치, 마스크 형태, 공유 vs 모달리티별 블록)을 설계하고 구현 위험 요소를 식별합니다.

## 연습 문제

1. Transfusion 스타일 모델은 70% 텍스트 토큰과 30% 이미지 패치를 학습합니다. 이미지 디퓨전 손실(loss)은 텍스트 NTP 손실(loss)보다 크기가 약 10배 더 큽니다. 이들을 균형 있게 조정하는 손실 가중치(loss weight)는 무엇인가요?

2. 시퀀스 `[T, T, <image>, P, P, P, P, </image>, T]`에 대해 블록-삼각 마스크(block-triangular mask)를 구현하세요. 각 항목을 0 또는 1로 표시하시오.

3. MMDiT는 모달리티별 QKV 가중치를 가집니다. 이는 Transfusion의 완전 공유 트랜스포머(fully-shared transformer) 대비 어떤 파라미터 수 오버헤드(parameter count overhead)를 추가하나요? 7B 파라미터 규모에서 이 오버헤드는 가치가 있나요?

4. 생성: 텍스트 프롬프트가 주어졌을 때, 모델은 50개 토큰에 대해 NTP를 실행한 후 `<image>`를 만나고, 이후 256개 패치에 대해 20번의 디노이즈 단계를 거쳐 디퓨전을 실행합니다. 총 몇 번의 순전파(forward pass)가 발생하나요?

5. SD3 논문 3장을 읽고, 정류 흐름(rectified flow)이 무엇인지와 DDPM보다 더 적은 추론 단계에서 수렴하는 이유를 설명하시오.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|---------------------|-----------|
| Two-loss training | "NTP + diffusion" | 단일 트랜스포머가 텍스트 토큰에 대한 교차 엔트로피와 연속 이미지 패치에 대한 MSE(평균 제곱 오차)를 동일한 그래디언트 단계에서 최적화 |
| Flow matching | "Rectified flow" | 노이즈에서 깨끗한 데이터로의 속도장을 예측하는 디퓨전 변형; DDPM보다 간단한 수학 |
| MMDiT | "Multimodal DiT" | Stable Diffusion 3의 아키텍처: 공동 어텐션(joint attention), 모달리티별 MLP 및 정규화(norms) |
| Block-triangular mask | "Causal text + bidirectional image" | 텍스트에는 인과적(causal), 이미지 영역 내에서는 양방향(bidirectional) 어텐션을 적용하는 어텐션 마스크 |
| Continuous image representation | "No VQ" | 정수 코드북 인덱스가 아닌 실수 벡터로 표현된 이미지 패치 |
| Velocity prediction | "v-parameterization" | 네트워크 출력이 노이즈 자체가 아닌 노이즈와 데이터 사이의 속도장(velocity field) |
