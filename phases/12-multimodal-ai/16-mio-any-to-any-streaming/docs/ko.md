# MIO와 Any-to-Any 스트리밍 멀티모달 모델

> GPT-4o는 대부분의 오픈소스 모델이 복제할 수 없는 제품을 제공합니다: 음성을 듣고, 비디오를 보며, 실시간으로 응답하는 에이전트입니다. 2024년 말 오픈소스 생태계의 대응책은 MIO(Wang et al., 2024년 9월)였습니다. MIO는 텍스트, 이미지, 음성, 음악을 토큰화하고, 인터리빙된 시퀀스 위에 하나의 인과적 트랜스포머를 훈련시키며, 어떤 모달리티에서든 어떤 모달리티로든 생성합니다. AnyGPT(Zhan et al., 2024년 2월)는 개념 증명이었고, MIO는 규모 확장 버전이며, Unified-IO 2(Allen AI, 2023년 12월)는 비전 + 액션 그라운딩이 있는 사촌 모델입니다. 이 레슨은 any-to-any 패턴 — 4개의 토크나이저, 1개의 트랜스포머, 스트리밍 친화적인 디코딩 — 을 다룹니다.

**유형:** 학습  
**언어:** Python (표준 라이브러리, 4개 모달리티 토큰 할당기 + 스트리밍 디코딩 루프)  
**선수 조건:** 12단계 · 11단계 (카멜레온), 6단계 (음성 및 오디오)  
**소요 시간:** ~120분

## 학습 목표

- 충돌 없이 텍스트, 이미지, 음성, 음악 토큰을 호스팅하는 공유 어휘집을 설계한다.
- SEED-Tokenizer(이미지)와 SpeechTokenizer residual-VQ(음성)의 압축 + 재구성 트레이드오프를 비교한다.
- 모든 형식 간(any-to-any) 생성을 구축하는 4단계 커리큘럼(curriculum)을 설명한다.
- 세 가지 공개 any-to-any 레시피와 주요 트레이드오프를 명명한다: MIO, AnyGPT, Unified-IO 2.

## 문제 정의

통합 멀티모달 모델은 주장하기는 쉽지만 대규모로 구축하기는 어렵습니다. 2024년까지의 대부분의 "모든-대-모든(any-to-any)" 시스템은 파이프라인 방식이었습니다: 비전 모델 → 텍스트 표현 → 음성 모델 → 오디오. 각 단계에서 정보가 손실되고, 지연 시간이 추가되며, 훈련이 복잡해집니다. GPT-4o 데모 영상에서는 단일 모델 대안으로 초당 응답(subsecond response)을 보여주었지만, 오픈 시스템들은 몇 개월 뒤처져 있었습니다.

엔지니어링 과제:

- 토크나이저는 모든 모달리티에 대해 존재해야 하며, 재구성을 위해 충분히 무손실 압축해야 하고, 트랜스포머가 처리할 수 있는 속도로 토큰을 생성해야 합니다.
- 단일 어휘 집합은 텍스트(32k+), 이미지(16k+), 음성(4k+), 음악(8k+)을 위한 공간을 할당해야 합니다. 최소 40,000개 이상의 항목이 필요합니다.
- 훈련 데이터는 모든 입력-출력 쌍(텍스트→이미지, 이미지→음성, 음성→이미지 등)을 커버하거나 모델이 조합(composition)할 수 있어야 합니다.
- 추론은 대화형 지연 시간(<500ms 첫 오디오 바이트 전송 시간)을 위해 충분히 빠르게 출력 토큰을 스트리밍해야 합니다.

## 개념

### 네 가지 모달리티를 위한 네 가지 토크나이저

MIO의 토크나이저 스택:

- 텍스트: 표준 BPE, 어휘 집합 ~32000.
- 이미지: SEED-Tokenizer (2023) — 이산 코드북을 사용하는 양자화된 VAE, 4096개 항목, 이미지당 32x32 토큰.
- 음성: SpeechTokenizer residual-VQ (2023) — 16kHz 파형을 8계층 코드북으로 인코딩; 첫 번째 계층은 거친 콘텐츠, 이후 계층은 운율 및 화자 식별 정보 추가.
- 음악: 유사한 residual-VQ (Meta의 MusicGen / Encodec 계열), 4-8개 코드북.

각 모달리티는 정수 토큰을 생성합니다. 토큰은 공유 어휘 집합에서 서로 다른 ID 범위를 가집니다:

```
텍스트:   0..31999
이미지:  32000..36095  (4096개 이미지 토큰)
음성:  36096..40191  (4096개 음성 기본 토큰, 잔여 계층 추가)
음악:  40192..48383  (8192개 음악 토큰)
구분자: 48384..48390  (<image>, <speech>, <music>, </...>, 등.)
```

총 어휘 집합: ~48k. 입력 임베딩과 출력 프로젝션은 전체를 아우릅니다.

### 스트리밍 디코딩

음성 생성은 residual-VQ를 사용합니다. 트랜스포머는 기본(계층 0) 음성 토큰을 예측하고, 병렬로 디코딩되는 잔여 양자화기가 이후 계층을 예측합니다. 각 계층 0 토큰은 16kHz에서 약 50ms 오디오에 해당합니다.

스트리밍 패턴:

1. 사용자가 마이크에 말하면 실시간 오디오 토크나이저가 50ms마다 음성 토큰을 방출합니다.
2. MIO는 토큰이 도착하는 대로 소비합니다(프롬프트 프리필 + 증분 전달).
3. 생성된 출력 토큰이 스트리밍되며, 병렬 음성 디코더가 이를 ~50-150ms 지연으로 오디오 샘플로 변환합니다.
4. 첫 오디오 바이트까지의 시간: MIO 논문에서는 ~300-500ms, GPT-4o의 ~250ms에 근접.

Mini-Omni (arXiv:2408.16725), GLM-4-Voice (arXiv:2412.02612), Moshi (arXiv:2410.00037)는 보완적인 스트리밍 음성-LLM 설계입니다. 특히 Moshi는 단일 GPU에서 160ms 왕복 시간을 달성합니다.

### 네 단계 커리큘럼

MIO의 훈련 커리큘럼:

1. 단계 1 — 정렬. 대규모 모달리티 쌍 코퍼스: 텍스트-이미지, 텍스트-음성, 텍스트-음악. 각 쌍은 자체 토큰 어휘 집합 세그먼트를 사용합니다. 공유 어휘 집합을 훈련합니다.
2. 단계 2 — 교차. 다중 모달리티 교차 문서(이미지 + 동영상이 포함된 블로그, 대본이 있는 팟캐스트 등). 모달리티 간 컨텍스트를 훈련합니다.
3. 단계 3 — 음성 강화. 텍스트 능력을 잃지 않으면서 음성 품질을 높이기 위한 추가 오디오 데이터.
4. 단계 4 — SFT. 모달리티 간 지시 튜닝: VQA, 캡션 생성, 내레이션, 음성-음성 대화.

단계를 생략하면 특정 기능이 저하됩니다: 단계 2를 건너뛰면 모델이 모달리티 간 컨텍스트를 잃고, 단계 3을 건너뛰면 음성 품질이 낮아집니다.

### 시각적 사고 사슬

MIO는 시각적 사고 사슬(chain-of-visual-thought)을 도입했습니다: 모델이 추론 단계로 중간 이미지 토큰을 방출합니다. "고양이가 나무를 오르고 있나요?"에 대해 모델은:

1. 장면을 렌더링하는 `<image>` 토큰을 방출합니다(입력 이미지 또는 스케치에서).
2. 스케치를 분석하는 텍스트를 방출합니다.
3. 최종 답변을 방출합니다.

렌더링된 중간 이미지는 스크래치패드 역할을 합니다. 공간 추론 작업에서 벤치마크가 개선됩니다. 이 아이디어는 텍스트 추론을 위한 사고 사슬(chain-of-thought)과 유사합니다.

### 모든 모달리티 간 경쟁 모델

- AnyGPT (arXiv:2402.12226): 4개 모달리티(텍스트, 이미지, 음성, 음악), 유사한 설계.
- Unified-IO 2 (arXiv:2312.17172): 비전 액션 출력, 깊이, 노멀 추가. 더 많은 작업 다양성, 더 작은 규모.
- NExT-GPT (arXiv:2309.05519): LLM + 모달리티별 확산 디코더. 단일 모델 접근법이 아님.
- CoDi (arXiv:2305.11846): 조합 가능한 확산; 공유 잠재 공간을 통한 모든 모달리티 간 변환.

MIO는 순수 토큰 기반 모든 모달리티 간 모델에 가장 가깝습니다. AnyGPT는 개념적 조상입니다.

### 지연 시간 예산

대화형 제품의 경우 모든 구성 요소의 지연 시간이 중요합니다:

- 마이크 → 오디오 토큰: ~50ms.
- 프리필(오디오 토큰 + 기록): 8B 모델에서 ~100ms.
- 첫 출력 토큰: ~50ms.
- 병렬 잔여-VQ + 음성 디코더: ~100-150ms.

총 첫 오디오 바이트까지의 시간: 최소 ~300ms. GPT-4o는 ~250ms를 주장합니다. Moshi는 160ms를 주장합니다. MIO/AnyGPT는 공개 벤치마크에서 400-600ms 범위입니다.

### 모든 모달리티 간 모델이 여전히 어려운 이유

2026년에도 오픈 모든 모달리티 간 모델은 두 축에서 폐쇄형 모델에 뒤처집니다:

- 음성 품질. residual-VQ 토크나이저는 손실 압축이 있으며, 대화형 음성은 ElevenLabs급 음성에 비해 로봇처럼 들립니다.
- 모달리티 간 추론. "보이는 것에 대해 노래해 주세요"와 같은 요청은 순수 비전 작업보다 더 자주 실패합니다.

이들은 미해결 연구 문제입니다. Qwen3-Omni (레슨 12.20)는 2025년 가장 진보된 오픈 시도입니다.

## 사용 방법

`code/main.py`:

- 4가지 모달리티(텍스트, 이미지, 오디오 클립, 음악)의 어휘 할당을 정의하고 출력합니다.
- 다중 모달 입력(텍스트, 이미지, 오디오 클립, 음악) 목록을 토크나이저 라우터를 통해 라우팅합니다.
- 지연 시간 측정을 포함한 텍스트-음성 변환 응답을 위한 스트리밍 디코딩을 시뮬레이션합니다.
- 인코더, 프리필, 디코더 지연 시간을 기반으로 첫 오디오 바이트까지의 예상 시간을 계산합니다.

## Ship It

이 레슨은 `outputs/skill-any-to-any-pipeline-auditor.md`를 생성합니다. 대화형 제품 사양(입력 모달리티, 출력 모달리티, 지연 시간 목표)이 주어졌을 때, MIO(Multi-Input-Output) 패밀리 설계 선택을 감사하고 지연 시간 예산을 계산합니다.

## 연습 문제

1. 제품이 음성 입력을 받아 음성 출력을 반환합니다. 종단간 지연 시간 예산 목표는 얼마인가요? 시간을 소모하는 구성 요소를 나열하세요.

2. SpeechTokenizer의 residual-VQ는 8개의 코드북을 사용합니다. 잔차 레벨을 순차적(sequential)이 아닌 병렬 디코딩(parallel-decoding)해야 하는 이유와 이로 인한 지연 시간 절감 효과를 제안하세요.

3. 어휘 집합에 32k 텍스트 + 4k 이미지 + 4k 음성이 있습니다. 여기에 8k 음악과 ~10개의 구분자를 추가합니다. 은닉 차원(hidden dim)이 4096일 때 임베딩 행렬(embedding-matrix)의 파라미터 비용은 얼마인가요?

4. Chain-of-visual-thought는 중간 이미지를 생성합니다. 어떤 종류의 질문이 이점을 얻나요? 어떤 종류는 추가 토큰으로 인해 손해를 보나요?

5. Moshi(arXiv:2410.00037)를 읽고, "내적 독백(inner monologue)" 기법을 설명한 후 MIO의 chain-of-visual-thought와 비교하세요.

## 주요 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|-------------------|-----------|
| Any-to-any | "멀티모달 입출력" | 텍스트, 이미지, 음성, 음악을 어떤 방향으로든 입력받고 출력할 수 있는 단일 모델 |
| Residual-VQ | "음성 토크나이저 스택" | 각 레이어가 정보를 추가하는 다중 코덱북 토크나이징 방식; 기본 레이어는 콘텐츠, 이후 레이어는 운율(prosody) |
| SEED-Tokenizer | "이미지 코드" | MIO에서 사용되는 4096개 항목의 코덱북을 가진 이산 이미지 토크나이저 |
| Chain-of-visual-thought | "시각적 메모장" | 모델이 최종 답변 전에 중간 이미지를 생성하며 추론하는 단계 |
| Time-to-first-audio-byte | "TTFAB" | 사용자 음성부터 첫 오디오 출력까지의 지연 시간; 대화 느낌을 위해 <500ms |
| Four-stage curriculum | "학습 레시피" | 정렬(alignment) -> 인터리빙(interleaved) -> 음성 강화(speech-enhanced) -> SFT(supervised fine-tuning) 순서로 진행 |
