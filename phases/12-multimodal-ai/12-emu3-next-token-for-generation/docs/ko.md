# Emu3: 이미지 및 비디오 생성을 위한 다음 토큰 예측

> BAAI의 Emu3 (Wang et al, 2024년 9월)는 확산 모델 대 오토리그레시브 논쟁에 종지부를 찍은 2024년 연구 성과입니다. 텍스트 + VQ 이미지 토큰 + 3D VQ 비디오 토큰으로 구성된 통합 어휘 집합에서 다음 토큰 예측 목적만으로 학습된 단일 Llama 스타일 디코더 전용 트랜스포머는 이미지 생성에서 SDXL을, 인식에서 LLaVA-1.6을 능가합니다. CLIP 손실 없음. 확산 스케줄 없음. 추론 시 품질 향상을 위해 분류자 없는 가이드(Classifier-free guidance)가 사용되지만, 핵심 학습 목적은 티처 포싱(teacher forcing)을 활용한 다음 토큰 예측입니다. Nature에 게재됨. 이 강의에서는 Emu3 논문을 읽으며 "더 나은 토크나이저 + 규모 확장이 전부"라는 주장을 분석하고 확산 모델과 대조합니다.

**유형:** 학습
**언어:** Python (표준 라이브러리, 3D 비디오 토크나이저 수학 + 오토리그레시브 샘플러 스켈레톤)
**선수 지식:** 12단계 · 11단계 (카멜레온)
**소요 시간:** ~120분

## 학습 목표

- 이미지 품질을 위해 확산(diffusion)이 필요하다는 오랜 가정에도 불구하고 Emu3의 단일 손실(next-token) 목적 함수가 작동하는 이유를 설명하시오.
- 3D 비디오 토크나이저에 대해 설명하시오: 시공간(spatiotemporal) VQ 코드북의 구성, 패치가 시간을 아우르는 이유.
- Emu3와 Stable Diffusion XL을 (훈련 연산량, 추론 비용, 품질 한계) 측면에서 비교하시오.
- 동일한 Emu3 모델이 수행하는 세 가지 역할을 제시하시오: Emu3-Gen(이미지 생성), Emu3-Chat(인식), Emu3-Stage2(비디오 생성).

## 문제 정의

2024년까지의 통념: 이미지 생성에는 확산(diffusion)이 필요하다. 주장: 이산적 이미지 토큰은 세부 사항을 재구성하기에 너무 많은 정보를 손실하며, 자기회귀적 샘플링은 수천 개의 토큰에 걸쳐 오류를 누적시킨다. Stable Diffusion, DALL-E 3, Imagen, Midjourney는 모두 어떤 형태의 확산 방식을 사용한다. Chameleon(레슨 12.11)은 소규모에서 이 주장을 부분적으로 반증했지만 SDXL 수준의 품질에는 미치지 못했다.

Emu3는 이 주장을 정면으로 반박했다. 주장: 더 나은 시각적 토크나이저 + 충분한 규모 + 다음 토큰 손실(next-token loss) = 동일한 모델에서 인식(perception)도 수행하는 확산 방식을 능가하는 이미지 생성.

이 주장은 발표 당시 논란이 많았다. 2년이 지난 현재, 오픈소스 통합 생성 모델 계열(Emu3, Show-o, Janus-Pro, Transfusion)은 연구의 기본 경로이며, 프로덕션 최전선 모델들도 어떤 변형을 사용하는 것으로 보인다.

## 개념

### Emu3 토크나이저

핵심 요소는 시각적 토크나이저입니다. Emu3는 8x8 해상도 감소당 사용자 정의 IBQ(Inverse Bottleneck Quantizer, SBER-MoVQGAN 계열) 토크나이저를 훈련시킵니다. 512x512 이미지는 64x64 = 4096개의 토큰이 되며, 코드북 크기는 32768입니다.

이는 Chameleon의 512x512당 1024개 토큰(K=8192)보다 크지만 토큰당 비용은 더 저렴합니다(더 작은 코드북 조회, 더 간단한 코덱). 주요 지표는 30.5dB의 재구성 PSNR로, 32dB의 Stable Diffusion 연속 잠재 공간과 경쟁할 수 있습니다.

비디오의 경우: 3D VQ 토크나이저는 시공간 패치(4x4x4 픽셀)를 하나의 정수로 인코딩합니다. 8 FPS의 4초 클립에는 32프레임이 있으며, 256x256 해상도에 4x 공간 및 4x 시간 축 감소를 적용하면 토큰 수는 (256/4) * (256/4) * (32/4) = 64 * 64 * 8 = 32,768개입니다.

토크나이저 품질은 상한선입니다. Emu3의 기여는 부분적으로 "매우 우수한 토크나이저를 훈련시켰다"는 점입니다.

### 단일 손실 훈련

Emu3는 하나의 목적 함수를 사용합니다: 텍스트 토큰, 2D 이미지 토큰, 3D 비디오 토큰에 걸친 공유 어휘에서 다음 토큰 예측입니다. 훈련 중에는 가중치들이 모달리티별 계수로 곱해져 기여도를 균형 있게 조정하지만, 손실 함수는 동일합니다.

다음 혼합 데이터로 훈련합니다:
- 이미지 생성: `<텍스트 캡션> <이미지> image_tokens </이미지>`
- 이미지 인식: `<이미지> image_tokens </이미지> <질문> text_tokens`
- 비디오 생성: `<텍스트 캡션> <video> video_tokens </video>`
- 비디오 인식: 유사 방식.
- 텍스트 전용: 표준 NTP.

모델은 데이터 분포에서 이미지 토큰과 텍스트 토큰을 언제 출력할지 학습합니다. 생성은 모델이 `<이미지>` 태그 뒤에 이미지 토큰을 예측하는 과정에서 나타납니다.

### 분류기 없는 가이드 및 온도

오토리그레시브 이미지 생성은 추론 시 분류기 없는 가이드(CFG)로 크게 개선됩니다. Emu3는 이를 사용합니다: 전체 캡션과 빈 캡션으로 각각 한 번씩 생성한 후, 로짓을 가이드 가중치(일반적인 3.0-7.0)로 혼합합니다. 이는 확산 모델이 사용하는 CFG 트릭을 오토리그레시브 설정에 적용한 것입니다.

온도는 중요합니다: 너무 높으면 아티팩트가 발생하고, 너무 낮으면 모드 붕괴가 일어납니다. Emu3의 권장 온도는 인식 작업에는 1.0, 이미지 생성에는 0.8입니다.

### 세 가지 역할, 하나의 모델

Emu3는 세 가지 기능적으로 구분된 API로 제공되지만 기본 가중치 세트는 하나입니다:

- Emu3-Gen. 이미지 생성. 텍스트 입력, 이미지 토큰 출력.
- Emu3-Chat. VQA 및 캡션 생성. 이미지(토큰) 입력, 텍스트 출력.
- Emu3-Stage2. 비디오 생성 및 비디오 VQA. 텍스트 또는 비디오 입력, 텍스트 또는 비디오 출력.

작업별 헤드가 없습니다. 단지 다른 프롬프트 템플릿만 사용합니다. 동일한 체크포인트입니다.

### 벤치마크

Emu3 논문(2024년 9월) 기준:

- 이미지 생성: MJHQ-30K FID(5.4 vs 5.6), GenEval 전체(0.54 vs 0.55 — 통계적 동률), Deep-Eval 복합 점수에서 SDXL을 능가하거나 동등합니다.
- 이미지 인식: VQAv2에서 LLaVA-1.6(75.1 vs 72.4)을 능가하고 MMMU에서는 대략 동등합니다.
- 비디오 생성: 4초 클립 품질에서 Sora 시대 공개 벤치마크 모델과 경쟁 가능한 FVD를 기록합니다.

수치가 항상 승리하는 것은 아니지만 — Emu3는 한 부분에서 점수를 교환합니다 — "다음 토큰 예측이 전부"라는 주장은 모달리티 전반에 걸쳐 방어 가능합니다.

### 계산 비용

Emu3는 7B 매개변수 모델로 약 3000억 개의 멀티모달 토큰으로 훈련되었습니다. GPU-시간은 Llama-2-7B 사전 훈련(A100급 실리콘에서 2k-4k GPU-년)과 대략 비슷합니다. Stable Diffusion 3와 같은 확산 모델은 유사한 예산으로 훈련되지만 별도의 텍스트 인코더와 더 복잡한 파이프라인이 필요합니다.

추론 시 Emu3는 이미지당 SDXL보다 느립니다: 30 tok/s 속도로 4096개 이미지 토큰을 처리하면 512x512 이미지당 약 2분이 소요되며, SDXL은 2-5초입니다. 추측 디코딩과 KV-캐시 최적화는 격차를 줄이지만 완전히 없애지는 못합니다. 오토리그레시브 이미지 생성은 계산 집약적이며, 이는 지속적인 트레이드오프입니다.

### 중요성

Emu3의 심층 기여는 개념적입니다. 다음 토큰 예측이 이미지 생성에서 확산 모델과 맞먹는 수준으로 확장된다면, 통합 모델 경로(하나의 손실, 하나의 백본, 모든 모달리티)가 실현 가능합니다. 미래 모델은 별도의 텍스트 인코더, 별도의 확산 스케줄러, 별도의 VAE가 필요하지 않습니다. 하나의 트랜스포머, 모달리티당 하나의 토크나이저, 확장성만 있으면 됩니다.

Show-o, Janus-Pro, InternVL-U는 모두 이 논문을 기반으로 하거나 도전합니다. 2025년까지 미국 연구소보다 BAAI, DeepSeek 등 중국 연구소가 이 방향으로 더 공격적으로 논문을 발표합니다.

## 사용 방법

`code/main.py`는 두 가지 토이 예제를 구축합니다:

- 2D(이미지) vs 3D(비디오) VQ 토크나이저 토큰 수 계산기: (해상도, 패치 크기, 클립 길이, FPS)를 입력으로 받아 이미지 vs 비디오의 토큰 수를 계산합니다.
- 분류자 자유 가이던스(classifier-free guidance)를 적용한 온도(temperature) 기반 자기회귀적 이미지 토큰 샘플러.

CFG 구현은 Emu3의 레시피를 따릅니다 — 조건부(conditional)와 무조건부(unconditional) 로짓(logit)을 가이던스 가중치(guidance weight)로 혼합합니다.

## Ship It

이 레슨은 `outputs/skill-token-gen-cost-analyzer.md`를 생성합니다. 생성 제품 사양(이미지 또는 비디오, 목표 해상도, 품질 등급, 지연 시간 예산)이 주어졌을 때, 토큰 수, 추론 비용을 계산하고 Emu3-family와 diffusion 중 적절한 모델을 선택합니다.

## 연습 문제

1. Emu3는 512x512 이미지에 대해 8x8 축소 시 4096개의 토큰을 생성합니다. 1024x1024 및 2048x2048 이미지에 대한 동등한 토큰 수를 계산하세요. 추론 지연 시간(latency)은 어떻게 변하나요?

2. Emu3 섹션 3.3의 비디오 토크나이저를 읽고, 3D VQ 패치 형태가 8x8x1이 아닌 4x4x4인 이유와 그 형태를 설명하세요.

3. Classifier-free guidance 가중치 5.0과 3.0의 시각적 효과 차이는 무엇인가요? `code/main.py`의 수학적 흐름을 추적해 설명하세요.

4. 300B 토큰 기준 Emu3-7B의 훈련 FLOPs를 계산하고 Stable Diffusion 3과 비교하세요. 어떤 모델이 훈련에 더 많은 비용이 들었나요?

5. Emu3는 FID 점수에서 SDXL을 능가하지만 VQAv2에서는 전문 VLM(Vision-Language Model)에 비해 성능이 낮습니다. 통합 손실(unified-loss) 접근법이 다른 벤치마크에서 전문가 모델과 다른 강점을 보이는 이유를 설명하세요.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|---------------------|-----------|
| 다음 토큰 예측 | "NTP" | 표준 자기회귀 손실: 토큰[0..i]이 주어졌을 때 토큰[i+1] 예측; 토큰화된 모든 모달리티에 적용 가능 |
| IBQ 토크나이저 | "역 병목 양자화기" | 더 큰 코드북(32,768+)과 Chameleon보다 우수한 재구성 성능을 가진 VQ-VAE 클래스 |
| 3D VQ | "시공간 양자화기" | (시간, 행, 열)로 인덱싱된 코드북; 하나의 토큰이 4x4x4 픽셀 큐브를 커버 |
| 분류기 없는 가이던스 | "CFG" | 조건부 및 무조건부 로짓을 가중치 감마로 혼합; 추론 시 이미지 품질 향상 |
| 통합 어휘 | "공유 토큰" | 텍스트 + 이미지 + 비디오가 모두 동일한 정수 공간에서 추출; 모델은 다음에 올 모달리티를 예측 |
| MJHQ-30K | "이미지 생성 벤치마크" | Midjourney 품질의 30,000개 프롬프트 벤치마크; Emu3는 여기서 FID 보고 |
