# 멀티모달 RAG와 크로스 모달 검색

> Vision-native 문서 RAG는 하나의 조각일 뿐입니다. 프로덕션 멀티모달 RAG는 더 넓은 범위를 다룹니다 — 여행 계획("자연광이 있는 조용한 비건 브런치 찾아줘"), 의료 분류("이 사진과 메모에 맞는 부상은 무엇인가요?"), 전자상거래("이 셀카와 비슷한 사이즈의 옷들"), 현장 서비스("엔진 소리와 부품 사진을 바탕으로 진단")와 같은 워크플로우를 위해 텍스트, 이미지, 오디오, 비디오를 넘나들며 검색합니다. 2025년 세 가지 연구(Abootorabi et al., Mei et al., Zhao et al.)는 하위 문제들을 체계화했습니다: 크로스 모달 검색, 검색 융합, 생성 그라운딩, 멀티모달 평가. 이 강의에서는 이 연구들을 읽고 프로덕션 파이프라인을 설계합니다.

**유형:** 구축
**언어:** Python (표준 라이브러리, 융합 기능이 있는 크로스 모달 검색기 + 그라운딩 생성기)
**선수 조건:** 12단계·23단계(ColPali), 11단계(RAG 기초)
**소요 시간:** ~180분

## 학습 목표

- 크로스 모달 검색 설계: 텍스트 → 이미지, 이미지 → 텍스트, 오디오 → 비디오 등
- 세 가지 융합 전략 비교: 점수 융합(score fusion), 어텐션 기반 융합(attention-based fusion), MoE 융합(MoE fusion)
- 생성 그라운딩 설명: 소스가 다양한 모달리티로 구성될 때 "출처 인용(cite your sources)"이 어떻게 구현되는지
- 2025년 세 가지 표준 멀티모달 RAG 조사 논문 및 하위 문제 분류 체계 명명하기

## 문제 정의

단일 모달리티 RAG(Retrieval-Augmented Generation)는 이미 해결된 패턴입니다: 쿼리 임베딩, 청크 임베딩, 검색, LLM에 입력. 다중 모달리티 RAG는 다음을 요구합니다:

1. 여러 검색 헤드(각 모달리티는 호환 가능한 공간에서의 임베딩이 필요함).
2. 모달리티 간 검색 결과 통합.
3. 모달리티 간 출처를 인용하는 생성 기반(grounding).
4. 크로스 모달 신호를 포괄하는 평가 지표.

2025년 설문 조사들은 모두 동일한 분류 체계에 도달합니다.

## 개념

### 크로스 모달 검색

모달리티 A의 쿼리를 기반으로 모달리티 B의 문서를 검색합니다. 세 가지 패턴:

1. 공유 임베딩 공간. CLIP과 CLAP은 텍스트+이미지/텍스트+오디오 임베딩을 공유 공간에서 생성합니다. 모달리티 간 코사인 유사도가 직접 작동합니다. CLIP 훈련 쌍으로 제한됩니다.

2. 모달리티별 인코더 + 번역. 텍스트 인코더 + 이미지 인코더 + 공간 간 매핑을 위한 소규모 번역 모듈. Gupta 등의 Sen2Sen 및 2024년 설계. 유연하지만 복잡성이 추가됩니다.

3. VLM을 인코더로 사용. VLM의 은닉 상태를 검색 표현으로 사용합니다. VLM이 지원하는 모든 모달리티가 작동합니다. 품질은 높지만 비용이 큽니다.

선택: 텍스트+이미지에는 CLIP/SigLIP 2, 텍스트+오디오에는 CLAP, 최전선 품질의 크로스 모달에는 VLM-은닉 상태.

### 융합 전략

10개의 결과(이미지 5개, 텍스트 구절 3개, 오디오 클립 2개)를 검색했습니다. 어떻게 병합할까요?

점수 융합(가장 저렴). 각 모달리티에 자체 검색기가 있고, 각각 점수를 반환합니다. 모달리티 내 점수를 정규화한 후 합산합니다. 간단하고 종종 효과적입니다.

어텐션 기반 융합. 모든 검색 항목을 연결하고 소규모 어텐션 네트워크가 가중치를 부여합니다. 훈련이 필요합니다.

MoE 융합. 게이팅 네트워크가 모달리티별 전문가로 라우팅합니다. 다른 쿼리 유형은 다르게 라우팅됩니다. 시각적 질문은 이미지 가중치를 높입니다.

프로덕션 기본값: 쿼리 우세 모달리티에 약간의 편향을 둔 점수 융합. A/B 테스트에서 도메인별 명확한 승자가 나타나면 MoE로 업그레이드.

### 생성 그라운딩

LLM은 각 주장을 주도한 검색 항목을 인용해야 합니다. 멀티모달 경우:

- 텍스트 소스: 표준 인용 `[1]`.
- 이미지 소스: 짧은 캡션과 함께 `[img 3]`.
- 오디오: `[audio 2 at 0:34]`.

그라운딩 인식 데이터로 생성기를 훈련시킵니다. 훈련 대상의 각 주장은 소스 인덱스로 태깅됩니다. 추론 시 모델이 자연스럽게 인용을 생성합니다.

### 2025년 조사 연구

Abootorabi 등 (arXiv:2502.08826, "Ask in Any Modality"): 멀티모달 RAG를 위한 분류 체계. 검색, 융합, 생성 분야를 포괄. 가장 광범위한 커버리지.

Mei 등 (arXiv:2504.08748, "A Survey of Multimodal RAG"): 하위 작업 벤치마크와 실패 모드에 집중. 평가 설계에 유용.

Zhao 등 (arXiv:2503.18016): 비전 중심 조사. ColPali 계열 작업에 강점.

세 연구를 모두 읽으면 2025년 봄 기준 최신 기술을 파악할 수 있습니다. 대부분의 하위 문제는 여전히 미해결입니다.

### MuRAG — 기초 논문

MuRAG (Chen 등, 2022)는 최초의 멀티모달 RAG입니다. 멀티모달 KB에서 이미지+텍스트를 검색하고 답변을 생성했습니다. VLM 물결 이전에 실현 가능성을 입증했습니다. 현대 시스템(REACT, VisRAG, M3DocRAG)은 이를 기반으로 구축됩니다.

### 프로덕션 여행 계획 예시

쿼리: "조용한 비건 브런치와 자연광이 있는 곳을 찾아줘."

파이프라인:

1. 쿼리 분해. "조용한" → 오디오/리뷰 키워드; "비건 브런치" → 메뉴 항목; "자연광" → 이미지 특징.
2. 모달리티별 검색:
   - 리뷰 텍스트 검색: "비건 브런치, 조용한 분위기."
   - 레스토랑 사진 이미지 검색: "자연광, 넓은 공간."
   - 주변 소리 클립 오디오 검색: "저데시벨, 음악 없음."
3. 점수 융합. 각 레스토랑에 복합 점수가 부여됩니다.
4. 상위 k개 레스토랑 → 모든 증거를 포함한 VLM 생성기 → 인용이 포함된 답변.

이는 텍스트 RAG를 훨씬 넘어섭니다. 각 모달리티는 텍스트만으로는 놓칠 수 있는 신호를 추가합니다.

### 에이전트형 멀티모달 RAG

다중 홉: 첫 검색에서 높은 신뢰도 답변을 반환하지 못하면 LLM이 재구성하고 다시 검색합니다. 14단계 에이전트형 RAG 패턴이 여기에 적용됩니다. 예시:

- 초기 상위 10개 검색 → LLM이 "너무 시끄러움, <40 dB로 필터링" 요청 → 재검색.
- 이미지 검색 → LLM이 메뉴가 있는 이미지 확인 → 메뉴 텍스트 검색 → 답변.

복잡성이 추가되지만 단일 샷 검색으로는 처리할 수 없는 쿼리를 처리합니다.

### 평가

크로스 모달 평가는 아직 미성숙합니다. 일반적인 대체 지표:

- 모달리티별 Recall@k.
- 융합된 상위 k 정확도.
- 인간 평가 종합 만족도.
- 작업별(완료된 예약, 구매 건수).

모든 모달리티를 아우르는 표준 벤치마크는 없습니다. 대부분의 논문은 도메인별 작업으로 평가합니다.

## 사용 방법

`code/main.py`:

- 공유 레스토랑 코퍼스에서 작동하는 세 가지 모의 검색기(텍스트, 이미지, 오디오).
- 구성 가능한 가중치를 사용하여 모달리티 점수를 결합하는 점수 융합.
- 인용과 함께 최종 답변을 생성하는 생성기 스텁.
- 신뢰도가 낮을 경우 쿼리를 재구성하는 간단한 에이전트 루프.

## Ship It

이 레슨은 `outputs/skill-multimodal-rag-designer.md`를 생성합니다. 멀티모달 쿼리 흐름이 포함된 제품 사양을 바탕으로 검색기(retriever), 융합(fusion), 생성기(generator), 평가(evaluation)를 설계합니다.

## 연습 문제

1. 의료-트리아지 멀티모달 RAG를 제안하시오: 쿼리 = 부상 사진 + 텍스트 증상. 어떤 모달리티가 어떤 KB에서 검색하는가?

2. 점수 융합(Score fusion)은 단순한 가중 합이다. MoE 융합이 피하는 이 방식의 실패 모드는 무엇인가?

3. Abootorabi et al.의 분류 체계(3절)를 읽어보시오. 세 가지 표준 하위 문제는 무엇이며, 선택한 제품에 어떻게 매핑되는가?

4. 여행 계획자 멀티모달 RAG를 위한 평가 명세서를 설계하시오. 이미지 재현율, 오디오 재현율, 복합 정확도를 커버하는 지표는 무엇인가?

5. 에이전트형 다중 홉 RAG는 왕복당 지연 시간 비용이 있다. 어떤 쿼리 난이도에서 정확도 향상이 지연 시간을 정당화하는가?

## 핵심 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|-------------------|-----------|
| Cross-modal retrieval | "한 모달리티로 쿼리, 다른 모달리티 검색" | 텍스트 쿼리로 이미지 검색; 이미지 쿼리로 텍스트 검색; 공유 공간 또는 번역기 필요 |
| Score fusion | "점수 결합" | 모달리티별 검색 점수의 가중합; 가장 간단한 결합 방식 |
| MoE fusion | "모달리티 라우팅 전문가" | 게이팅 네트워크가 쿼리별로 신뢰할 모달리티 점수 선택 |
| Grounded generation | "출처 명시" | 답변의 각 주장에 출처 인덱스 태깅 |
| MuRAG | "최초의 멀티모달 RAG" | 2022년 멀티모달 RAG 패턴을 정립한 논문 |
| Agentic multi-hop | "재구성 및 재시도" | LLM이 첫 검색 신뢰도 낮을 때 리트리버 재쿼리 |
