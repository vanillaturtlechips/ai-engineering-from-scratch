# CLIP과 대조적 비전-언어 사전 학습

> OpenAI의 CLIP(2021)은 다음 5년을 이끌 만큼 강력한 아이디어를 증명했습니다: 노이즈가 있는 웹 이미지-캡션 쌍과 대조적 손실만을 사용하여 이미지 인코더와 텍스트 인코더를 동일한 벡터 공간에 정렬합니다. 감독 레이블은 전혀 사용하지 않았습니다. 4억 개의 쌍으로 학습된 결과 임베딩 공간은 제로샷 분류, 이미지-텍스트 검색 기능을 제공하며, 2026년의 모든 VLM(비전 언어 모델)에서 비전 타워로 활용됩니다. SigLIP 2(2025)는 소프트맥스 대신 시그모이드를 사용하여 더 낮은 비용으로 CLIP을 능가했습니다. 이 강의에서는 InfoNCE에서 시그모이드 페어 손실까지의 수학적 과정을 설명하고, 표준 라이브러리 Python으로 학습 단계를 구현합니다.

**유형:** 구현
**언어:** Python (표준 라이브러리, InfoNCE + 시그모이드 손실 구현)
**선수 지식:** Phase 12 · 01 (ViT 패치), Phase 7 (트랜스포머)
**소요 시간:** ~180분

## 학습 목표

- 상호 정보량(mutual information)에서 InfoNCE 손실(loss)을 유도하고 수치적으로 안정된 벡터화된 버전을 구현한다.
- 시그모이드 페어 손실(SigLIP)이 소프트맥스(softmax)가 요구하는 all-gather 오버헤드 없이 배치 32768+로 확장되는 이유를 설명한다.
- 텍스트 템플릿(`a photo of a {class}`)을 구성하고 코사인 유사도(cosine similarity)에 대해 argmax를 수행하여 제로샷(zero-shot) ImageNet 분류를 실행한다.
- CLIP / SigLIP 사전 훈련이 제공하는 4가지 조절 수단(lever)을 명명한다: 배치 크기(batch size), 온도(temperature), 프롬프트 템플릿(prompt template), 데이터 품질(data quality).

## 문제 정의

CLIP 이전의 컴퓨터 비전 모델은 지도 학습(supervised learning) 방식으로 훈련되었습니다. 레이블이 달린 데이터셋(예: ImageNet—120만 개 이미지, 1000개 클래스)을 수집하고, CNN(Convolutional Neural Network)을 훈련시킨 후 배포했습니다. 레이블은 비용이 많이 들고, 레이블러들이 합의할 수 있는 내용에 편향되며, 새로운 작업에 적용하려면 파인튜닝(fine-tuning)이 필요했습니다.

이미지-캡션 웹 데이터에는 10억 개 이상의 느슨하게 레이블된 쌍이 무료로 제공됩니다. 예를 들어 "공원에서 찍은 내 강아지 맥스"라는 대체 텍스트(alt text)가 달린 골든 리트리버 이미지는 감독 신호(supervisory signal)를 포함합니다. 즉, 텍스트가 이미지를 설명합니다. 문제는 이 데이터를 유용한 훈련에 활용할 수 있는가입니다.

CLIP의 해결책: 이미지-캡션 쌍을 매칭 작업으로 처리합니다. N개의 이미지와 N개의 캡션이 주어졌을 때, 각 이미지를 자신의 캡션과 매칭시키고 나머지 N-1개의 방해 요소(distractor)와 구분하도록 학습합니다. 감독 신호는 "이 두 가지는 함께 속한다. 나머지 N-1개는 속하지 않는다"입니다. 클래스 레이블도, 인간 주석도 필요 없이 대조 손실(contrastive loss)만으로 학습합니다.

결과적으로 생성된 임베딩 공간(embedding space)은 CLIP이 훈련된 목적 이상으로 확장됩니다. ImageNet 제로샷(zero-shot) 추론이 가능한 이유는 "고양이 사진"이라는 텍스트가 명시적으로 '고양이' 레이블이 달리지 않은 고양이 이미지 근처에 임베딩되기 때문입니다. 이 아이디어가 2026년 모든 비전-언어 모델(VLM, Vision-Language Model)의 기반이 되었습니다.

## 개념

### 듀얼 인코더

CLIP은 두 개의 타워를 가집니다:

- 이미지 인코더 `f`: ViT 또는 ResNet, 이미지당 D차원 벡터를 출력합니다.
- 텍스트 인코더 `g`: 소형 트랜스포머, 캡션당 D차원 벡터를 출력합니다.

두 타워 모두 출력을 단위 길이로 정규화합니다. 유사도는 `cos(f(x), g(y)) = f(x)^T g(y)`입니다. 두 벡터가 단위 노름이기 때문입니다.

N개의 (이미지, 캡션) 쌍 배치에 대해 `(N, N)` 형태의 유사도 행렬 `S`를 구성합니다:

```
S[i, j] = cos(f(x_i), g(y_j)) / tau
```

여기서 `tau`는 학습된 온도(CLIP은 0.07로 초기화, 로그 공간에서 학습)입니다.

### InfoNCE 손실

CLIP은 행과 열에 대한 대칭적 교차 엔트로피를 사용합니다:

```
loss_i2t = CE(S, labels=identity)     # 각 이미지의 긍정은 해당 캡션
loss_t2i = CE(S^T, labels=identity)   # 각 캡션의 긍정은 해당 이미지
loss = (loss_i2t + loss_t2i) / 2
```

이것이 InfoNCE입니다. CE의 소프트맥스는 각 이미지가 배치 내 다른 모든 캡션보다 자신의 캡션과 더 잘 매칭되도록 강제합니다. "부정 샘플"은 배치의 다른 모든 항목입니다. 더 큰 배치 = 더 많은 부정 샘플 = 더 강한 신호. CLIP은 배치 크기 32k로 학습되었으며, 스케일이 중요합니다.

### 온도

`tau`는 소프트맥스의 예리함을 제어합니다. 낮은 `tau` → 예리한 분포, 강한 부정 샘플 마이닝 효과. 높은 `tau` → 부드러운 분포, 모든 샘플이 기여. CLIP은 `log(1/tau)`를 학습하며, 붕괴를 방지하기 위해 클리핑합니다. SigLIP 2는 초기 `tau`를 고정하고 학습된 바이어스를 대신 사용합니다.

### 시그모이드가 더 잘 확장되는 이유 (SigLIP)

소프트맥스는 전체 유사도 행렬이 동기화되어야 합니다. 분산 학습에서는 모든 임베딩을 모든 레플리카에 올개더(all-gather)한 후 소프트맥스를 수행해야 합니다. 이는 통신 측면에서 세계 크기의 제곱에 비례합니다.

SigLIP은 소프트맥스를 요소별 시그모이드로 대체합니다: 각 쌍 `(i, j)`에 대해 손실은 "이들이 매칭 쌍인가?"라는 이진 분류입니다. 긍정 클래스 라벨은 대각선, 나머지는 부정입니다. 손실 함수는:

```
L = -1/N sum over (i, j) [ y_ij log sigmoid(S[i,j]) + (1-y_ij) log sigmoid(-S[i,j]) ]
```

`y_ij = 1` if `i == j`, else 0. 각 쌍의 손실은 독립적입니다. 올개더가 필요 없습니다. 각 GPU는 로컬 블록을 계산하고 합산합니다. SigLIP 2는 배치 크기 32k-512k로 저렴하게 확장되지만, CLIP은 비례적으로 더 많은 통신이 필요합니다.

### 제로샷 분류

N개의 클래스 이름이 주어졌을 때, 각 클래스에 대해 텍스트 템플릿을 생성합니다:

```
"a photo of a {class}"
```

텍스트 인코더로 각 템플릿을 임베딩합니다. 이미지 인코더로 이미지를 임베딩합니다. 코사인 유사도 최대값 = 예측 클래스. 대상 클래스에 대한 학습이 필요 없습니다.

프롬프트 템플릿이 중요합니다. CLIP 원본 논문은 클래스당 80개의 템플릿(평문, 예술, 사진, 그림 등)을 사용하고 임베딩을 평균화했습니다. +3 ImageNet 포인트. 현대 사용법은 일반적으로 하나 또는 두 개의 템플릿을 선택합니다.

### 선형 프로브와 파인튜닝

제로샷은 베이스라인입니다. 선형 프로브(대상 클래스를 위해 고정된 CLIP 특징 위에 선형 계층 하나 학습)는 인-도메인 작업에서 제로샷을 능가합니다. 전체 파인튜닝은 인-도메인에서 선형 프로브를 능가하지만 제로샷 전이에는 악영향을 줄 수 있습니다. 세 가지 트레이드오프를 가진 세 가지 영역.

### SigLIP 2: NaFlex 및 밀집 특징

SigLIP 2(2025)는 다음을 추가합니다:
- NaFlex: 단일 모델이 가변 종횡비와 해상도를 처리합니다.
- 분할 및 깊이 추정을 위한 더 나은 밀집 특징, VLM에서 고정된 백본으로 사용 목표.
- 다국어: CLIP이 영어만 지원했던 반면 100+ 언어로 학습.
- 1B 파라미터 규모에서 CLIP은 400M에서 최고를 기록했습니다.

2026년 오픈 VLM에서 SigLIP 2 SO400m/14는 기본 비전 타워입니다. CLIP은 특정 LAION-2B 학습 분포가 쿼리 패턴과 일치하는 순수 이미지-텍스트 검색의 기본값으로 남아 있습니다.

### ALIGN, BASIC, OpenCLIP, EVA-CLIP

ALIGN(Google, 2021): CLIP과 동일한 아이디어, 1.8B 쌍 규모, 90% 노이즈. 노이즈 데이터가 확장 가능함을 증명. OpenCLIP(LAION): LAION-400M / 2B에서 CLIP을 오픈 재현, 여러 규모, 가장 많이 사용되는 오픈 체크포인트. EVA-CLIP: 마스킹된 이미지 모델링에서 초기화; VLM을 위한 강력한 백본. BASIC: Google의 CLIP+ALIGN 하이브리드. 모두 같은 패밀리, 다른 데이터와 튜닝.

### 제로샷 한계

CLIP 계열 모델은 ImageNet 제로샷에서 약 76%로 포화(CLIP-G, OpenCLIP-G). 그 이상은 훨씬 더 큰 데이터(SigLIP 2는 80%+) 또는 아키텍처 변경(지도 학습 헤드, 더 많은 파라미터)이 필요합니다. 벤치마크는 포화 상태이지만, 실제 가치는 다운스트림 VLM이 소비하는 임베딩 공간입니다.

## 사용 방법

`code/main.py`는 다음을 구현합니다:

1. InfoNCE 형태를 NumPy 없이 확인할 수 있도록 하는 장난감 이중 인코더(해시 기반 이미지 특징, 텍스트 문자 특징).
2. 순수 Python으로 구현된 InfoNCE 손실(로그-합-멕스(log-sum-exp)를 통한 수치적 안정성).
3. 비교를 위한 시그모이드 페어 손실.
4. 제로샷 분류 루틴: 텍스트 프롬프트 집합에 대한 코사인 유사도 계산, argmax를 통한 예측.

실행하여 손실 곡선을 관찰하세요. 절대값은 장난감 수준이지만, 실제 CLIP 트레이너가 출력하는 형태와 일치합니다.

## Ship It

이 레슨은 `outputs/skill-clip-zero-shot.md`를 생성합니다. 이미지 집합(경로 통해 제공됨)과 대상 클래스 목록이 주어지면, CLIP 템플릿으로 텍스트 프롬프트를 구성하고, 명시된 체크포인트(예: `openai/clip-vit-large-patch14`)로 양측을 임베딩한 후 유사도 점수와 함께 상위 1개/5개 예측 결과를 반환합니다. 이 스킬은 프롬프트 목록에 없는 클래스에 대해서는 예측을 수행하지 않습니다.

## 연습 문제

1. 4개의 쌍에 대해 InfoNCE를 수동으로 구현하세요. 4x4 유사도 행렬을 구성하고, 소프트맥스를 실행한 후 대각선을 추출하며, 교차 엔트로피를 계산하세요. 이 수동 계산 결과와 Python 구현을 검증하세요.

2. SigLIP은 온도(`tau`) 외에 편향 파라미터 `b`를 사용합니다: `S'[i,j] = S[i,j]/tau + b`. 배치에 클래스 불균형이 심한 경우(행당 긍정보다 부정 샘플이 훨씬 많은 경우) `b`는 어떤 역할을 하나요? SigLIP 섹션 3(arXiv:2303.15343)을 참고하세요.

3. 고양이 vs 개 분류를 위한 제로샷 분류기를 구축하세요. 두 가지 프롬프트 템플릿(`a photo of a {class}`와 `a picture of a {class}`)을 시도하고, 100개의 테스트 이미지에서 정확도를 측정하세요. 템플릿 앙상블이 단일 템플릿보다 성능이 좋은가요?

4. 512-GPU에서 배치 크기 32k로 실행할 때 소프트맥스 InfoNCE와 시그모이드 페어와이즈의 통신 비용을 계산하세요. 어떤 방법이 O(N)으로, 어떤 방법이 O(N²)로 확장되나요? SigLIP 섹션 4를 참고하세요.

5. OpenCLIP 스케일링 법칙 논문(arXiv:2212.07143, Cherti et al.)을 읽으세요. 그림으로부터 데이터 스케일링에 대한 결론을 재현하세요: 고정된 모델 크기에서 ImageNet 제로샷 정확도와 훈련 데이터 크기 사이의 로그-선형 관계는 무엇인가요?

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|----------------|------------------------|
| InfoNCE | "대조 손실(contrastive loss)" | 배치의 유사도 행렬에 대한 교차 엔트로피(cross-entropy); 각 항목의 긍정(positive)은 페어링된 항목, 부정(negative)은 나머지 전체 |
| Sigmoid 손실 | "SigLIP 손실" | 페어별 이진 교차 엔트로피(binary cross-entropy); 소프트맥스(softmax) 없음, 올게더(all-gather) 없음, 분산 학습에서 저렴하게 확장 |
| 온도(temperature) | "타우(tau)" | 소프트맥스(softmax)/시그모이드(sigmoid) 전 로짓(logit)을 스케일링하는 스칼라; 분포의 예리함(sharpness) 제어 |
| 제로샷(zero-shot) | "파인튜닝 없는 분류(no-finetune classification)" | 텍스트 프롬프트를 사용하여 클래스 임베딩 구성 및 코사인 유사도(cosine similarity)로 분류; 대상 클래스에 대한 학습 없음 |
| 프롬프트 템플릿 | "a photo of a ..." | 클래스 이름 주변의 텍스트 구조; 제로샷 정확도에 1-5점 영향 |
| 듀얼 인코더(dual encoder) | "투 타워(two-tower)" | 하나의 이미지 인코더 + 하나의 텍스트 인코더, 공유 D-차원 공간에 출력 |
| 하드 네거티브(hard negative) | "어려운 방해물(tough distractor)" | 긍정(positive)과 충분히 유사하여 모델이 분리하기 위해 노력해야 하는 부정(negative) |
| 선형 프로브(linear probe) | "동결 + 한 레이어(frozen + one layer)" | 동결된 특징(frozen features) 위에 선형 분류기만 학습; 특징 품질 측정 |
| NaFlex | "네이티브 유연한 해상도(native flexible resolution)" | SigLIP 2의 기능: 리사이징 없이 임의의 종횡비와 해상도의 이미지 처리 |
| 온도 스케일링(temperature scaling) | "로그 파라미터화된 타우(log-parametrized tau)" | CLIP은 `log(1/tau)`를 파라미터화하여 그래디언트 동작 개선; 거의 0에 가까운 타우(tau)로의 붕괴 방지 위해 클리핑 적용 |
