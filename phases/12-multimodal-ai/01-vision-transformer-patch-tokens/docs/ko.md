# Vision Transformers와 패치-토큰 기본 요소

> 멀티모달 처리 이전에, 이미지는 트랜스포머가 처리할 수 있는 토큰 시퀀스로 변환되어야 합니다. 2020년 ViT 논문은 16x16 픽셀 패치, 선형 투영, 위치 임베딩으로 이 문제를 해결했습니다. 5년 후 2026년 최첨단 모델들(Claude Opus 4.7의 2576px 네이티브 해상도, Gemini 3.1 Pro, Qwen3.5-Omni)도 여전히 이 방식으로 시작합니다. 인코더는 ViT에서 DINOv2, SigLIP 2로 변경되었고, 레지스터 토큰이 추가되었으며, 위치 인코딩 방식은 2D-RoPE가 되었지만, 기본 요소는 유지되었습니다. 이 레슨은 패치-토큰 파이프라인을 처음부터 끝까지 분석하고 표준 Python 라이브러리로 구현하여, Phase 12의 나머지 부분에서 "시각적 토큰"에 대한 구체적인 정신 모델을 제공합니다.

**유형:** 학습  
**언어:** Python (표준 라이브러리, 패치 토크나이저 + 기하학 계산기)  
**선수 지식:** Phase 7 (트랜스포머), Phase 4 (컴퓨터 비전)  
**소요 시간:** ~120분

## 학습 목표

- HxWx3 이미지를 올바른 위치 인코딩(positional encoding)이 적용된 패치 토큰(patch token) 시퀀스로 변환하기.
- 주어진 (패치 크기(patch size), 해상도(resolution), 은닉 차원(hidden dim), 깊이(depth))에 대해 ViT의 시퀀스 길이(sequence length), 파라미터 수(parameter count), FLOPs(FLOPs) 계산하기.
- 2020년 연구 단계에서 2026년 프로덕션 단계로 ViT를 발전시킨 세 가지 업그레이드(자기 지도 사전 학습(self-supervised pretraining) (DINO / MAE), 레지스터 토큰(register tokens), 네이티브 해상도 패킹(native-resolution packing)) 이름 대기.
- 다운스트림 작업(downstream task)에 대해 CLS 풀링(CLS pooling), 평균 풀링(mean pooling), 레지스터 토큰(register tokens) 중 적절한 방법 선택하기.

## 문제 정의

트랜스포머(Transformer)는 벡터 시퀀스로 동작합니다. 텍스트는 이미 시퀀스(바이트 또는 토큰)입니다. 이미지는 3개의 색상 채널을 가진 2D 픽셀 그리드 — 시퀀스가 아닙니다. 모든 픽셀을 평탄화하면 224x224 RGB 이미지는 150,528개의 토큰이 되며, 이 길이에서의 자기 주의(self-attention)는 시작조차 할 수 없는 문제입니다(시퀀스 길이에 대해 2차 복잡도).

2020년 이전 접근법들은 CNN 특징 추출기를 앞에 결합했습니다: ResNet은 2048차원 벡터의 7x7 특징 맵을 생성하고, 이 49개의 토큰을 트랜스포머에 입력합니다. 이 방법은 작동하지만 CNN의 편향(병진 등변성, 지역적 수용 영역)을 물려받으며, 트랜스포머의 확장성(규모에 대한 적응력)을 잃습니다.

Dosovitskiy et al. (2020)은 직설적인 질문을 던졌습니다: "CNN을 생략하면 어떨까?" 이미지를 고정 크기 패치(예: 16x16 픽셀)로 분할하고, 각 패치를 선형 투영하여 벡터로 변환한 후 위치 임베딩을 추가하고, 이 시퀀스를 기본 트랜스포머에 입력합니다. 당시 이 접근법은 이단적이었다 — 합성곱 없는 비전. 충분한 데이터(JFT-300M, 이후 LAION)로 ImageNet에서 ResNet을 능가하고 계속 개선되었습니다.

2026년에는 ViT(Vision Transformer) 프리미티브가 의심할 여지 없는 기반이 되었습니다. 모든 오픈 웨이트 VLM(Vision-Language Model)의 비전 타워는 그 후손(DINOv2, SigLIP 2, CLIP, EVA, InternViT)입니다. 이제 질문은 "패치를 사용해야 하는가?"가 아니라 "어떤 패치 크기, 어떤 해상도 스케줄, 어떤 사전 학습 목표, 어떤 위치 인코딩을 사용할 것인가?"입니다.

## 개념

### 패치(patch)를 토큰으로

형태 `(H, W, 3)`의 이미지 `x`와 패치 크기 `P`가 주어졌을 때, 이미지를 겹치지 않는 `(H/P) x (W/P)` 그리드로 분할합니다. 각 패치는 `P x P x 3` 픽셀 큐브입니다. 각 큐브를 `3 P^2` 벡터로 평탄화(flatten)한 후, 형태 `(3 P^2, D)`의 공유 선형 투영 `W_E`를 적용하여 각 패치를 모델의 은닉 차원 `D`로 매핑합니다.

ViT-B/16 표준 구성에서:
- 해상도 224, 패치 크기 16 → 그리드 14x14 → 196개의 패치 토큰.
- 각 패치는 `16 x 16 x 3 = 768` 픽셀 값이며, `D = 768`로 투영됩니다.
- 학습 가능한 `[CLS]` 토큰 추가 → 시퀀스 길이 197.

패치 투영은 수학적으로 커널 크기 `P`, 스트라이드 `P`, 출력 채널 `D`의 2D 컨볼루션과 동일합니다. 실제 프로덕션 코드에서는 `nn.Conv2d(3, D, kernel_size=P, stride=P)`로 구현됩니다. "선형 투영" 설명은 개념적이며, 커널 프레임이 효율적입니다.

### 위치 임베딩(positional embeddings)

패치에는 고유한 순서가 없습니다. 트랜스포머는 이를 집합(bag)으로 인식합니다. 초기 ViT는 학습 가능한 1D 위치 임베딩(각 위치에 768차원 벡터, 총 197개)을 추가했습니다. 이 방식은 작동하지만 모델을 학습 해상도에 고정시킵니다. 추론 시 그리드 크기가 변경되면 위치 테이블을 보간해야 합니다.

현대 비전 백본은 2D-RoPE(Qwen2-VL의 M-RoPE, SigLIP 2의 기본값) 또는 분해된 2D 위치를 사용합니다. 2D-RoPE는 패치의 (행, 열) 인덱스에 따라 쿼리 및 키 벡터를 회전시켜, 모델이 회전 각도로부터 상대적 2D 위치를 추론하도록 합니다. 위치 테이블이 필요 없습니다. 모델은 추론 시 임의의 그리드 크기를 처리할 수 있습니다.

### CLS 토큰, 풀링 출력, 레지스터 토큰

이미지 수준 표현은 세 가지 방식으로 얻을 수 있습니다:

1. `[CLS]` 토큰. 패치 시퀀스 앞에 학습 가능한 벡터를 추가합니다. 모든 트랜스포머 블록 이후, CLS 토큰의 은닉 상태가 이미지 표현입니다. BERT에서 유래되었으며, 원본 ViT, CLIP에서 사용됩니다.
2. 평균 풀링(mean pool). 패치 토큰의 출력 은닉 상태를 평균합니다. SigLIP, DINOv2, 대부분의 현대 VLM에서 사용됩니다.
3. 레지스터 토큰. Darcet et al. (2023)은 명시적 싱크 토큰 없이 학습된 ViT가 고노름(high-norm) "아티팩트" 패치를 생성하여 자기 주의(self-attention)를 장악하는 현상을 관찰했습니다. 4–16개의 학습 가능한 레지스터 토큰을 추가하면 이 부하를 흡수하고 분할(segmentation), 깊이(depth) 예측 등의 밀집 예측 품질이 향상됩니다. DINOv2와 SigLIP 2는 모두 레지스터 토큰을 포함합니다.

선택은 다운스트림 작업에 따라 중요합니다. CLS는 분류에 적합합니다. 패치 토큰을 LLM에 입력하는 VLM의 경우 풀링을 생략합니다. 모든 패치가 LLM 입력 토큰이 됩니다. 레지스터 토큰은 전달 전에 폐기됩니다(콘텐츠가 아닌 스캐폴딩 역할).

### 사전 학습: 지도 학습, 대조 학습, 마스킹, 자기 증류

2020년 ViT는 JFT-300M에서 지도 분류로 사전 학습되었습니다. 이후 다음과 같은 방식으로 대체되었습니다:

- CLIP (2021): 4억 개 이미지-텍스트 쌍에 대한 대조 학습. 레슨 12.02.
- MAE (2021, He et al.): 패치의 75%를 마스킹하고 픽셀을 복원합니다. 순수 이미지 기반 자기 지도 학습.
- DINO (2021) / DINOv2 (2023): 학생-교사 자기 증류, 라벨 또는 캡션 없이 학습. 2023년 DINOv2 ViT-g/14는 순수 시각 백본 중 가장 강력하며 "밀집 특징" 사용 사례의 기본값입니다.
- SigLIP / SigLIP 2 (2023, 2025): 시그모이드 손실과 NaFlex를 사용한 CLIP. 2026년 오픈 VLM(Qwen, Idefics2, LLaVA-OneVision)의 주요 비전 타워.

사전 학습 선택은 백본의 용도를 결정합니다: CLIP/SigLIP은 텍스트와의 의미적 매칭에, DINOv2는 밀집 시각 특징에, MAE는 다운스트림 미세 조정의 시작점으로 적합합니다.

### 스케일링 법칙

ViT 스케일링(Zhai et al. 2022)은 모델 크기, 데이터 크기, 계산량에서 예측 가능한 법칙을 따름. 고정된 계산량에서:
- 더 큰 모델 + 더 많은 데이터 → 더 나은 품질.
- 패치 크기는 시퀀스 길이 대 충실도(fidelity)의 조절 장치입니다. 패치 14(DINOv2/SigLIP SO400m의 일반적 크기)는 패치 16보다 이미지당 더 많은 토큰을 제공합니다. OCR 및 밀집 작업에는 유리하지만 속도는 저하됩니다.
- 해상도는 또 다른 주요 조절 장치입니다. 224에서 384, 512로 증가시키면 FLOPs가 제곱으로 증가하지만 거의 항상 성능이 향상됩니다.

ViT-g/14(10억 파라미터, 패치 14, 해상도 224 → 256 토큰)와 SigLIP SO400m/14(4억 파라미터, 패치 14)는 2026년 오픈 VLM의 두 가지 주요 인코더입니다.

### ViT 파라미터 수 계산

전체 계산은 `code/main.py`에 있습니다. ViT-B/16에서 해상도 224일 때:

```
patch_embed = 3 * 16 * 16 * 768 + 768  =  591k
cls + pos    = 768 + 197 * 768          =  152k
block        = 4 * 768^2 (QKVO) + 2 * 4 * 768^2 (MLP) + 2 * 2*768 (LN)
             = 12 * 768^2 + 3k          =  7.1M
12 blocks    = 85M
final LN    = 1.5k
total       ≈ 86M
```

체크포인트를 로드하기 전에 이 방식으로 ViT의 대략적인 크기를 추정합니다. 백본 크기는 모든 다운스트림 VLM에서 VRAM의 최소 요구 사항을 결정합니다.

### 2026년 프로덕션 구성

2026년 대부분의 오픈 VLM이 사용하는 인코더는 SigLIP 2 SO400m/14입니다. 기본 해상도(NaFlex)에서:
- 4억 파라미터.
- 패치 크기 14, 기본 해상도 384 → 이미지당 729개 패치 토큰.
- 이미지 수준 작업에는 평균 풀링 사용. VQA에는 729개 패치 모두 LLM으로 흐름.
- 4개의 레지스터 토큰, LLM 전달 전 폐기.
- 기본 종횡비를 위한 이미지 수준 스케일링이 적용된 2D-RoPE.

이 구성의 모든 결정은 읽을 수 있는 논문에서 비롯되었습니다.

## 사용 방법

`code/main.py`는 패치 토크나이저 및 기하학 계산기입니다. (이미지 H, W, 패치 P, 은닉 D, 깊이 L)을 입력으로 받아 다음을 보고합니다:

- 패칭 후 그리드 형태 및 시퀀스 길이.
- 합성 8x8 픽셀 장난감 이미지에 대한 토큰 시퀀스(평탄화 + 투영 경로 설명).
- 패치 임베딩, 위치 임베딩, 트랜스포머 블록, 헤드별 파라미터 수 분석.
- 목표 해상도에서의 순방향 패스당 FLOPs.
- ViT-B/16 @ 224, ViT-L/14 @ 336, DINOv2 ViT-g/14 @ 224, SigLIP SO400m/14 @ 384 간 비교 표.

실행해 보세요. 파라미터 수를 공개된 수치와 비교해 보세요. 패치 크기와 해상도를 조정해 토큰 수 비용을 체감해 보세요.

## Ship It

이 레슨은 `outputs/skill-patch-geometry-reader.md`를 생성합니다. ViT 구성(패치 크기, 해상도, 은닉 차원, 깊이)이 주어지면, 토큰 수, 파라미터 수, VRAM 추정치와 그 근거를 제공합니다. VLM(비전-언어 모델)을 위한 비전 백본을 선택할 때마다 이 기술을 사용하세요 — "토큰이 폭발적으로 증가해 LLM 컨텍스트가 가득 찼다"는 식의 놀라움을 방지할 수 있습니다.

## 연습 문제

1. Qwen2.5-VL에서 기본 해상도 1280x720 입력과 패치 크기 14일 때 패치-토큰 시퀀스 길이를 계산하시오. CLS-only 표현과 비교하면 어떻게 다른가?

2. 1080p 프레임(1920x1080)에서 패치 크기 14는 몇 개의 토큰을 생성하는가? 5분 길이의 동영상에서 30 FPS 기준 총 시각 토큰 수는 얼마인가? 풀링, 프레임 샘플링, 토큰 병합 중 어떤 방법이 가장 비용을 절감하는가?

3. 순수 파이썬으로 패치 토큰에 대한 평균 풀링을 구현하시오. DINOv2 출력의 196개 토큰에 대한 평균 풀링 결과가 모델의 `forward`에서 풀링된 임베딩을 요청할 때 반환되는 값과 일치하는지 검증하시오.

4. "Vision Transformers Need Registers" (arXiv:2309.16588)의 3절을 읽고, 레지스터가 흡수하는 아티팩트가 무엇이며 이것이 다운스트림 밀집 예측에 중요한 이유를 두 문장으로 설명하시오.

5. `code/main.py`를 수정하여 패치-앤-팩(patch-n'-pack)을 지원하시오: 서로 다른 해상도의 이미지 목록이 주어졌을 때 단일 패킹된 시퀀스와 블록-대각선 어텐션 마스크를 생성하시오. 레슨 12.06에 도달했을 때 검증하시오.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|----------------|------------------------|
| 패치(Patch) | "16x16 픽셀 정사각형" | 입력 이미지의 고정 크기 비중복 영역; 하나의 토큰이 됨 |
| 패치 임베딩(Patch embedding) | "선형 투영(Linear projection)" | 평탄화된 패치 픽셀을 D-차원 벡터로 매핑하는 공유 학습 행렬(또는 stride=P인 Conv2d) |
| CLS 토큰(CLS token) | "클래스 토큰(Class token)" | 최종 은닉 상태가 전체 이미지를 표현하는 학습 가능한 벡터; 2026년에는 선택적 |
| 레지스터 토큰(Register token) | "싱크 토큰(Sink token)" | 사전 학습 중 ViT가 개발하는 고규범(high-norm) 어텐션 아티팩트를 흡수하는 추가 학습 토큰 |
| 위치 임베딩(Position embedding) | "위치 정보(Positional info)" | 시퀀스 순서를 인식하게 하는 위치별 벡터 또는 회전; 2D-RoPE가 현대적 기본값 |
| 그리드(Grid) | "패치 그리드(Patch grid)" | 주어진 해상도와 패치 크기에 대한 (H/P) x (W/P) 2D 패치 배열 |
| NaFlex | "네이티브 유연 해상도(Native flexible resolution)" | SigLIP 2 기능: 단일 모델이 재학습 없이 여러 종횡비와 해상도를 지원 |
| 백본(Backbone) | "비전 타워(Vision tower)" | VLM에서 LLM에 패치-토큰 출력을 제공하는 사전 학습된 이미지 인코더 |
| 풀링(Pooling) | "이미지 수준 요약(Image-level summary)" | 패치 토큰을 하나의 벡터로 변환하는 전략: CLS, 평균, 어텐션 풀링 또는 레지스터 기반 |
| 패치 14 vs 16(Patch 14 vs 16) | "더 세밀한 vs 더 거친 그리드" | 패치 14는 이미지당 더 많은 토큰을 생성하며 OCR에 더 높은 충실도 제공, 느림; 패치 16은 클래식 기본값 |
