# 비디오-언어 모델: 시간적 토큰과 그라운딩

> 비디오는 사진들의 단순한 스택이 아닙니다. 5초짜리 클립에는 인과적 순서, 동작 동사, 이미지 모델이 표현할 수 없는 이벤트 타이밍이 포함됩니다. Video-LLaMA (Zhang et al., 2023년 6월)는 오디오-비주얼 그라운딩을 갖춘 최초의 오픈 비디오-LLM을 출시했습니다. VideoChat과 Video-LLaVA는 이 패턴을 확장했습니다. 2025년까지 Qwen2.5-VL의 TMRoPE는 프론티어 프로프라이어터 모델과의 격차를 해소했습니다. 각 시스템은 시간적 토큰을 다르게 해결했습니다 — 클립당 Q-former, 프레임당 concat-pool, 토큰당 TMRoPE. 이 레슨은 패턴을 읽고, 균일-대-동적 프레임 샘플러를 구축하며, 시간적 그라운딩 태스크에서 평가합니다.

**유형:** 구축  
**언어:** Python (표준 라이브러리, 프레임 샘플러 + 시간적-그라운딩 평가기)  
**선수 지식:** 12단계 · 08 (LLaVA-OneVision)  
**소요 시간:** ~180분

## 학습 목표

- 시간적 위치 인코딩(temporal positional encoding)이 비전 인코더(vision encoder)와 독립적으로 비디오 VLM(Vision-Language Model) 성능을 변화시키는 이유를 설명한다.
- 초당 토큰 수(tokens-per-second) 대 그라운딩 정확도(grounding accuracy) 측면에서 균일(uniform), 동적-FPS(dynamic-FPS), 이벤트 기반(event-driven) 프레임 샘플링을 비교한다.
- Q-former-per-clip(Video-LLaMA) 대 프레임당 풀링(pooled-per-frame, Video-LLaVA) 대 M-RoPE-per-token(Qwen2.5-VL) 설계 방식을 설명한다.
- 4가지 비디오 벤치마크 이름: VideoMME, TempCompass, EgoSchema, Video-MMMU를 말한다.

## 문제

30 FPS의 1분 동영상은 1800프레임입니다. 프레임당 196개의 시각 토큰(224 해상도의 ViT-B 기준)이면 총 352k 토큰이 생성되는데, 이는 2024년 기준 LLM의 최대 컨텍스트 길이보다 큽니다.

세 가지 축소 전략이 존재합니다:

1. 프레임 서브샘플링(콘텐츠에 따라 1-8 FPS로 감소).
2. 각 프레임의 패치 토큰을 공격적으로 풀링(3x3 또는 4x4 이중선형 풀링).
3. 16프레임 클립을 입력받아 64개 토큰을 출력하는 Q-former를 통한 압축.

각 전략의 트레이드오프는 다릅니다. 서브샘플링은 시간적 세부 정보를 손실합니다. 풀링은 공간적 세부 정보를 손실합니다. Q-former는 두 정보를 모두 약간 손실하지만 토큰 수를 절약합니다.

시간적 위치 인코딩은 또 다른 축입니다: 모델은 프레임 5가 프레임 6보다 먼저 나타났음을 어떻게 알 수 있을까요? 옵션으로는 단순한 1D 시간적 RoPE(Video-LLaMA), 학습된 시간적 임베딩(Video-LLaVA), 그리고 TMRoPE(Qwen2.5-VL, 전체 3D)가 있습니다.

## 개념

### Video-LLaMA: 클립당 Q-former + 오디오 브랜치

Video-LLaMA (2023)는 최초의 오픈 비디오-LLM입니다. 아키텍처:

- 2 FPS의 16프레임 클립 (즉, 8초).
- 프레임별 ViT 특징 → 16프레임 전체에 대해 교차 어텐션하는 Video Q-former → 32개의 학습된 쿼리 → LLM.
- 병렬 오디오 브랜치: 웨이브폼 → ImageBind 오디오 인코더 → Audio Q-former → 32개의 쿼리 → LLM.

강점: 오디오-비주얼 통합 추론. 약점: 고정된 클립 길이, 임의의 시간 그라운딩 불가.

### VideoChat과 Video-LLaVA

VideoChat은 Video-LLaMA 아이디어를 유지하지만 오디오를 제거하고 단순화했습니다. Video-LLaVA (Lin et al., 2023)는 이미지와 비디오 프레임 모두에 대해 단일 시각 인코더를 훈련시켜 ("투영 전 정렬") 통합 표현을 제공했습니다. 둘 다 동결된 CLIP-인코더 + MLP + LLM을 사용합니다.

둘 다 긴 비디오를 처리하지 못합니다. 모두 8-16프레임 시스템입니다.

### Qwen2.5-VL과 TMRoPE

Qwen2.5-VL은 TMRoPE — Temporal-Modality Rotary Position Embedding을 도입했습니다. 각 패치 토큰은 (t, h, w) 위치를 가지며, 여기서 t는 실제 타임스탬프(프레임 인덱스가 아님)입니다.

단순 시간 임베딩과의 주요 차이점:

- 인덱스가 아닌 절대 시간. 모델은 "프레임 15"가 아닌 "4.2초"를 인식합니다.
- 클립 단위가 아닌 토큰별 회전. 각 시각 토큰은 타임스탬프에 따라 독립적으로 회전합니다.
- 동적 FPS와 호환. 여기서 2 FPS, 저기서 4 FPS로 샘플링하면 TMRoPE가 불균일 간격을 네이티브로 처리합니다.

TMRoPE는 "고양이가 몇 초에 점프하나요?"와 같은 쿼리를 가능하게 합니다. 모델은 "4.2초"라고 출력할 수 있습니다. Video-LLaMA는 "클립 초반"만 말할 수 있었습니다.

### 프레임 샘플링 전략

균일: 지속 시간에 걸쳐 N프레임을 균등하게 샘플링. 간단하지만 모션 피크를 놓침.

동적 FPS: 모션 강도에 따라 적응적으로 샘플링. 광학 흐름 또는 프레임 차분으로 고모션 세그먼트를 선택해 밀집 샘플링. Qwen2.5-VL은 이를 기반으로 훈련됩니다.

이벤트 기반: 경량 검출기를 실행하고 액션이 발생하는 곳에 더 많이 샘플링. VideoAgent에서 사용.

키프레임 + 컨텍스트: 샷 경계 + 인접 프레임 몇 개를 샘플링. 시네마틱 콘텐츠에 사용.

### 프레임당 풀링

1 FPS와 프레임당 576토큰에서 5분 클립은 172,800토큰입니다. Qwen2.5-VL-72B의 128k 컨텍스트로는 가능하지만 비용이 큽니다.

3x3 양선형 풀링은 프레임당 64토큰으로 감소 → 5분 클립에 19,200토큰. 대부분의 작업에 적합한 수준.

공간적 디테일이 덜 중요한 에이전트 워크플로우에는 6x6 풀링(프레임당 16토큰)을 적용해 더 공격적으로 풀링할 수 있습니다.

### 4가지 비디오 벤치마크

- VideoMME: 짧은 + 중간 + 긴 비디오에 대한 종합 이해.
- TempCompass: 세밀한 시간 추론, "이전" / "이후" 질문.
- EgoSchema: 3분 이상의 장문 1인칭 비디오.
- Video-MMMU: 다중 모달리티 다학제 비디오 질문.

완전한 비디오-VLM 평가는 이 4가지를 모두 포함합니다. 각각 다른 축을 강조합니다 — TempCompass는 순서, EgoSchema는 3분 이상 추론, VideoMME는 다양한 지속 시간을 다룹니다.

### 출력 형식 그라운딩

시간적 그라운딩을 위한 출력 형식:

- 자유 텍스트: "고양이는 4초 경에 점프합니다." 파싱은 쉽지만 부정확.
- 구조화된 JSON: `{"event": "jump", "start": 4.1, "end": 4.3}`. Qwen2.5-VL은 이를 훈련합니다.
- 토큰 기반: 답변에 `<time>4.1</time>` 특수 토큰을 삽입. Qwen2.5-VL의 내부 형식.

토큰 기반은 다운스트림 사용에 가장 정확합니다. Qwen2.5-VL의 JSON 출력 형식은 직접 파싱 가능합니다.

### 2026년 모범 사례

2026년 비디오 VLMs에 대해:

- 인코더: SigLIP 2 + M-RoPE 또는 TMRoPE (Qwen2.5-VL).
- 프레임 샘플링: 모션에 따라 1-4 FPS의 동적 FPS + 최대 프레임 제한.
- 프레임당 풀링: 3x3 양선형.
- 출력: 시간 + 이벤트 필드가 있는 구조화된 JSON.
- 벤치마크: 일반용 VideoMME + TempCompass; 장문 추론용 EgoSchema.

## 사용 방법

`code/main.py`에는 다음이 포함됩니다:

- 균일 및 동적-FPS 프레임 샘플러.
- 장난감(temporal-grounding) 평가기: 시간 T에 "그라운드 트루스" 이벤트와 모델 출력이 주어졌을 때, 허용 오차 범위 내에서 정확도를 점수화합니다.
- Video-LLaMA(16프레임, Q-former), Video-LLaVA(8프레임, MLP), Qwen2.5-VL(동적 FPS + TMRoPE) 간 비교.

## Ship It

이 레슨은 `outputs/skill-video-vlm-frame-planner.md`를 생성합니다. 비디오 작업(모니터링, 행동 인식, 시간적 근거 설정, 요약)이 주어졌을 때, 프레임 샘플러, 풀링 계수, 출력 형식, 예상 정확도 등급을 선택합니다.

## 연습 문제

1. 3분 분량의 요리 데모에서 균일(uniform) FPS와 동적(dynamic) FPS 중 선택하시오. 토큰 수를 기준으로 정당화하시오.

2. TMRoPE는 단순한 시간 임베딩 테이블이 할 수 없는 어떤 기능을 구체적으로 추가하는가?

3. VLM이 학습할 수 있는 시간 기반 그라운딩(temporal grounding)을 위한 JSON 스키마를 작성하시오. 오류 사례도 포함하시오.

4. Video-LLaVA의 3장 "투영 전 정렬(Alignment Before Projection)"을 읽고, 이 접근법이 별도의 이미지 및 비디오 인코더를 훈련시키는 것보다 왜 더 나은지 설명하시오.

5. VideoMME 리더보드를 기준으로, 2026년 현재 최고 오픈 모델과 최고 프로프라이어터리 모델 간의 성능 격차는 얼마인가? 그 격차 중 시간 인코딩(temporal encoding)과 기본 LLM 규모(scale) 각각이 차지하는 비중은 얼마인가?

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|---------------------|-----------|
| Temporal grounding | "시간 지역화된 답변" | VLM이 이벤트가 발생하는 특정 타임스탬프 범위를 출력 |
| TMRoPE | "시간-멀티모달 RoPE" | 절대 타임스탬프를 가진 3D 회전 위치 인코딩, Qwen2.5-VL에서 사용 |
| Dynamic FPS | "모션 인식 샘플링" | 고모션 구간에서는 더 많은 프레임을, 정적 구간에서는 더 적은 프레임을 샘플링 |
| Frame pooling | "프레임별 공간 압축" | LLM 이전에 쌍선형 보간법으로 프레임당 패치 수 감소 |
| Video Q-former | "클립 압축기" | N개의 프레임을 K개의 학습된 쿼리로 매핑하는 교차-어텐션 병목 구조 |
| VideoMME | "비디오 벤치마크" | 2500개 이상의 샘플로 구성된 종합적인 단/중/장형 비디오 벤치마크 |
