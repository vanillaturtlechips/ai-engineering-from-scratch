# InternVL3: 네이티브 멀티모달 사전 학습

> InternVL3 이전의 모든 오픈 VLM(비전 언어 모델)은 동일한 3단계 레시피를 따랐다: 수조 개의 텍스트 토큰으로 학습된 텍스트 LLM을 가져온 후, 비전 인코더를 추가하고, 접합부를 미세 조정한다. 이 방법은 작동하지만 정렬 부채(alignment debt)가 있다 — 텍스트 LLM은 전체 사전 학습 예산을 순수 텍스트에 사용했으며 시각적 토큰을 네이티브로 이해하지 못한다. 사후에 비전을 추가하면 LLM은 텍스트를 잊지 않으면서 시각적 입력과 텍스트 추론을 연관시키는 방법을 다시 학습해야 한다. InternVL3(Zhu et al., 2025년 4월)는 사후 접근 방식을 거부한다: 하나의 사전 학습 실행에서 텍스트와 멀티모달 데이터를 첫 단계부터 교차시킨다. 결과적으로 78B 파라미터의 오픈 모델에서 MMMU-Pro에서 Gemini 2.5 Pro와 동등한 성능을 달성했다. 이 레슨은 네이티브 사전 학습의 사례와 이를 구현했을 때의 변화를 설명한다.

**유형:** 학습  
**언어:** Python (표준 라이브러리, 학습-코퍼스 믹서)  
**선수 지식:** 12단계 · 05, 12단계 · 07 (레시피)  
**소요 시간:** ~120분

## 학습 목표

- 사후(post-hoc) VLM 훈련이 정렬 부채(alignment debt)를 누적하는 이유를 설명하고, 측정 가능한 세 가지 증상(파괴적 망각(catastrophic forgetting), 답변 드리프트(answer drift), 시각-텍스트 불일치(visual-text inconsistency))을 인용하여 설명하시오.
- InternVL3의 네이티브 사전 훈련 코퍼스 혼합 구성과 텍스트 : 인터리빙 : 캡션 비율이 중요한 이유를 설명하시오.
- V2PE(가변 시각 위치 인코딩)를 Qwen2-VL의 M-RoPE와 비교하시오.
- 시각 해상도 라우터(Visual Resolution Router, ViR)와 분리된 비전-언어(Decoupled Vision-Language, DvD) 배포 최적화 기법을 설명하시오.

## 문제 정의

사후 학습(post-hoc) VLM 훈련이 현재 기본 접근 방식입니다. LLaVA, BLIP-2, Qwen-VL, Idefics 등은 모두 이미 사전 학습된 LLM(Llama, Vicuna, Qwen, Mistral)을 기반으로 시각(vision) 기능을 추가합니다. 일반적인 훈련 단계는 다음과 같습니다:

1. 고정된 LLM + 고정된 비전 인코더 + 학습 가능한 프로젝터(projector)를 사용하여 캡션 쌍으로 임베딩 정렬 훈련.
2. LLM을 해제하고 지시 데이터(LLaVA-Instruct, ShareGPT4V)로 훈련.
3. 선택적 작업별 미세 조정(fine-tuning).

정렬 부채(alignment debt)의 세 가지 증상이 나타납니다:

- **치명적 망각(catastrophic forgetting)**. 사후 학습 VLM은 텍스트 전용 기술을 잊어버립니다. GSM8K 점수가 5-10점 하락합니다. Hellaswag 점수도 감소합니다. 순수 텍스트 에이전트가 퇴보합니다.
- **답변 변동(answer drift)**. 동일한 시각적 질문에 대한 약간의 표현 변화가 다른 답변을 유발합니다. 비전 인코더는 LLM의 자체 토큰보다 약한 결합으로 LLM에 연결됩니다.
- **시각-텍스트 불일치**. VLM은 이미지를 정확히 설명한 후 자신의 설명과 모순되는 질문에 답할 수 있습니다. 시각 토큰은 LLM의 내부 일관성 검증에 텍스트 토큰과 동일한 방식으로 참여하지 않습니다.

이러한 증상은 잘 문서화되어 있습니다. MM1.5 섹션 4에서 이를 정량화했으며, LLaVA-OneVision의 실험 결과도 이를 암시합니다. **네이티브 사전 학습(native pretraining)**이 해결책입니다.

## 개념

### 네이티브 멀티모달 사전 훈련

InternVL3는 처음부터 네이티브 멀티모달 코퍼스에서 처음부터 훈련됩니다. 구성은 다음과 같습니다:

- 40% 텍스트 전용 데이터 (FineWeb, Proof-Pile-2 등)
- 35% 이미지-텍스트 인터리빙 데이터 (OBELICS, MMC4 스타일)
- 20% 이미지-캡션 페어링 데이터
- 5% 비디오-텍스트 데이터

비전 토큰, 텍스트 토큰, 크로스 모달 상호작용은 첫 번째 기울기 단계부터 동일한 손실 함수에 참여합니다. 정렬 사전 훈련, 프로젝터 고정 단계, 복구용 망각 방지 기법이 필요 없습니다.

기본 모델 훈련은 단일 단계로 진행됩니다. 이후 지시 튜닝이 수행되지만, 기본 모델은 이미 시각 토큰을 1급 시민으로 이해합니다.

### V2PE (가변 시각 위치 인코딩)

Qwen2-VL은 고정 축 할당의 M-RoPE를 사용합니다. InternVL3는 V2PE를 도입했습니다: 위치 인코딩은 모달리티 유형(텍스트, 이미지, 비디오)마다 다르며 학습 가능한 스케일링이 적용됩니다. 실제 적용 예:

- 텍스트 토큰은 1D 위치(텍스트 인덱스)를 얻습니다.
- 이미지 패치는 2D 위치(행, 열)를 얻습니다.
- 비디오 프레임은 3D 위치(시간, 행, 열)를 얻습니다.

세 가지 모두 동일한 RoPE 주파수 기반을 공유하지만, 밴드당 은닉 차원 할당은 고정 분할이 아닌 학습 가능한 매개변수입니다. 사전 훈련 중 시간적 vs 공간적 주파수 해상도 간 트레이드오프 자유도가 있습니다.

V2PE의 제거 실험 결과: 동일한 계산 비용으로 M-RoPE 대비 비디오 벤치마크에서 1-2포인트 향상. 혁명은 아니지만 더 깔끔합니다.

### 시각 해상도 라우터 (ViR)

배포 최적화 기법입니다. 모든 이미지가 전체 해상도 인코딩을 필요로 하지는 않습니다. 낮은 디테일의 단일 객체가 있는 사진은 1280px 네이티브로 인코딩 시 토큰을 낭비합니다. ViR은 질문에 답하는 데 필요한 최소 해상도를 예측하는 소형 분류기로, 인코딩 전에 작동합니다.

라우팅은 3단계입니다: 저해상도(256 토큰), 중간(576), 고해상도(2048+). 프로덕션 트래픽의 60% 쿼리에서 저해상도 또는 중간 해상도로 충분합니다. 순 효과: 동일 품질에서 처리량 2-3배 증가.

### 분리된 비전-언어 배포 (DvD)

대규모 VLM을 서비스할 때, 비전 인코더는 이미지당 한 번 실행되지만 LLM은 모든 출력 토큰에 대해 자동회귀적으로 실행됩니다. 두 구성 요소는 서로 다른 병목 현상(비전 = 컨볼루션 + 어텐션용 GPU 메모리 대역폭; LLM = KV 캐시)을 가집니다. DvD는 스트리밍을 통해 별도의 GPU로 분리합니다.

8B + 400M 인코더 모델의 경우, DvD는 공동 배치 대비 노드당 처리량을 대략 2배 증가시킵니다.

### 단일 단계 vs 다단계 품질

InternVL3의 주요 벤치마크 주장: 78B 파라미터에서 Gemini 2.5 Pro의 MMMU-Pro와 동등. 38B에서는 GPT-4o와 동등. 8B에서는 오픈 8B 리더보드에서 선두. 모두 단일 단계 사전 훈련 + 지시 튜닝 레시피 기반입니다.

정렬 부채 가설은 측정 가능합니다: InternVL3-8B는 비전 벤치마크 성능 향상 단위당 Qwen2.5-VL-7B보다 텍스트 벤치마크(MMLU, GSM8K) 점수 감소가 적습니다. 훈련이 두 단계가 아닌 하나로 통합되었기 때문에 모델은 더 일반적인 능력을 가집니다.

### InternVL3.5 및 InternVL-U

InternVL3.5(2025년 8월)는 레시피를 확장합니다. 동일한 네이티브 사전 훈련 접근법, 더 많은 데이터, 더 많은 파라미터. MMMU 개선은 점진적 수준입니다.

InternVL-U(2026)는 통합 생성을 추가합니다 — 동일한 백본 위에 MMDiT 헤드를 통해 이미지 출력을 지원합니다. "U"는 "이해 + 생성"을 의미하며, Transfusion 스타일의 통합 모델(레슨 12.13)을 추구합니다. 동일한 네이티브 사전 훈련 백본이 이해 및 생성 헤드를 모두 지원합니다.

### 네이티브 사전 훈련의 트레이드오프

네이티브 사전 훈련은 무료가 아닙니다:

- 계산 비용. 새로운 VLM을 처음부터 훈련하는 비용은 텍스트 LLM 훈련 비용과 동일 — 수백만 GPU 시간. 사후 적응은 기존 LLM 가중치를 재사용하여 대부분의 비용을 절약합니다.
- 데이터. 대규모 이미지-텍스트 인터리빙 코퍼스는 드뭅니다. OBELICS는 1억 4100만 문서, MMC4는 5억 7100만 문서입니다. 텍스트 단독은 15T 토큰 규모로 제공됩니다. 멀티모달 사전 훈련 데이터 부족은 어려운 제약입니다.
- 기본 LLM 재사용. 네이티브 사전 훈련은 나중에 새로운 LLM을 드롭인할 옵션을 포기합니다. 사후 적응은 어댑터만 재훈련하여 Llama-3.1을 Llama-4로 교체할 수 있습니다.

InternVL3의 베팅: 정렬 부채가 재사용 손실보다 더 나쁩니다. 벤치마크는 이 주장을 뒷받침합니다. 생산 비용은 향후 연구소가 저렴하게 복제하는 것을 막습니다. 사후 VLM은 대부분의 프로젝트에서 여전히 저렴하기 때문에 계속 존재할 것입니다.

## 사용 방법

`code/main.py`는 훈련 코퍼스 믹서 및 ViR 라우터 시뮬레이터 역할을 합니다. 이 스크립트는 다음을 수행합니다:

- 대상 코퍼스 믹스(%텍스트, %인터리빙, %캡션, %비디오)를 입력받아 모달리티별 예상 스텝 수를 계산합니다.
- 50% 저세부, 30% 중간세부, 20% 고세부 분포로 구성된 쿼리 배치에 대해 ViR 라우팅을 시뮬레이션하고 평균 토큰 수를 보고합니다.
- 인코더 대비 LLM FLOPs를 기반으로 DvD 처리량 추정치를 보고합니다.
- 사후 학습(post-hoc) vs 네이티브 사전학습(native pretraining)을 파라미터, 계산량, 데이터, 예상 정렬 부채 증상 측면에서 나란히 비교 출력합니다.

## Ship It

이 레슨은 `outputs/skill-native-vs-posthoc-auditor.md`를 생성합니다. 제안된 VLM(비전-언어 모델) 훈련 계획을 바탕으로 네이티브(native) 방식인지 사후(post-hoc) 방식인지 감사하고, 정렬 부채(alignment-debt) 위험을 표시하며, 코퍼스 혼합을 권장합니다. 새로운 오픈-VLM 프로젝트를 계획하고 훈련 전략을 선택해야 할 때 사용하세요.

## 연습 문제

1. InternVL3-8B(원본 사전 학습)와 LLaVA-OneVision-7B(사후 학습) 간의 계산 리소스 차이(GPU-시간 비율)를 추정하시오. 이 차이를 설명하는 요인은 무엇인가?

2. InternVL3은 40% 텍스트 / 35% 인터리빙 / 20% 캡션 / 5% 비디오 데이터를 보고합니다. 대상 작업이 비디오 중심일 경우 새로운 데이터 비율을 제안하고, 기본 모델이 여전히 상당한 텍스트 및 캡션 데이터를 필요로 하는 이유를 설명하시오.

3. MM1.5 4장 "망각(forgetting)" 부분을 읽고, 사후 학습에서 가장 큰 성능 저하가 발생한 정확한 벤치마크를 제시하시오. 성능 저하 규모는 얼마나 되는가?

4. ViR은 트래픽의 60%를 저해상도 인코딩으로 라우팅합니다. 어떤 유형의 쿼리를 잘못 라우팅(고해상도 필요 시 저해상도로 전송)하는가? 라우터 실패 모드 3가지를 제안하시오.

5. DvD는 비전과 LLM을 별도의 GPU에 분리 배치합니다. DvD가 처리량(throughput)을 향상시키지 못하고 오히려 저해하는 트래픽 패턴은 무엇인가?

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|---------------------|-----------|
| Native multimodal pretraining | "처음부터 함께(from scratch together)" | 텍스트 + 이미지 + 비디오 토큰이 1단계부터 손실 함수에 참여, 나중에 추가되지 않음 |
| Alignment debt | "사후 페널티(post-hoc penalty)" | 고정된 LLM에 비전을 추가(bolt-on)함으로써 발생하는 텍스트 능력 및 답변 일관성 측정 가능한 저하 |
| V2PE | "가변 시각 위치 인코딩(variable visual pos encoding)" | 모달리티별 학습 가능한 위치 인코딩 할당; InternVL3의 M-RoPE 후속 기술 |
| ViR | "해상도 라우터(resolution router)" | 인코딩 전 쿼리당 필요한 최소 해상도를 선택하는 소형 분류기, 추론 토큰 절약 |
| DvD | "분리된 배포(decoupled deployment)" | 비전 인코더는 한 GPU, LLM은 다른 GPU에 배치, 스트리밍 핸드오프; 대형 VLM 처리량 2배 증가 |
| InternVL-U | "통합 이해 + 생성(unified understanding + generation)" | 2026년 후속 모델, 네이티브 사전 훈련 백본에 이미지 생성 헤드 추가 |
| Interleaved corpus | "OBELICS / MMC4" | 텍스트와 이미지가 자연스러운 읽기 순서로 포함된 문서; 네이티브 사전 훈련의 원천 자료 |
