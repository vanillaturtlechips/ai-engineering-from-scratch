# 카멜레온과 초기 결합 토큰 전용 멀티모달 모델

> 지금까지 본 모든 VLM(비전 언어 모델)은 이미지와 텍스트를 분리하여 처리합니다. 시각 토큰은 비전 인코더에서 나와 프로젝터를 거쳐 LLM 내부의 텍스트와 만납니다. 시각과 텍스트 어휘 집합은 절대 겹치지 않습니다. 카멜레온(Meta, 2024년 5월)은 질문했습니다: 만약 겹친다면 어떨까요? 이미지를 공유 어휘 집합의 이산 토큰 시퀀스로 변환하는 VQ-VAE를 훈련시킵니다. 이제 모든 멀티모달 문서는 하나의 시퀀스가 됩니다 — 텍스트 토큰과 이미지 토큰이 교차하며, 단일 자기회귀 손실 함수를 가집니다. 부작용: 모델은 혼합 모달리티 출력을 생성할 수 있습니다 — 단일 추론 호출에서 텍스트와 이미지 토큰을 번갈아 생성. 이 레슨은 초기 결합 논문을 읽고 엔드 투 엔드 장난감 버전을 구축합니다.

**유형:** 구축
**언어:** Python (표준 라이브러리, VQ-VAE 토크나이저 + 교차 디코더)
**선수 과목:** 12단계 · 05, 8단계 (생성형 AI)
**소요 시간:** ~180분

## 학습 목표

- 공유 어휘(shared vocabulary) + 단일 손실(single loss)이 모델의 동작 방식을 어떻게 변화시키는지 설명.
- VQ-VAE가 이미지를 트랜스포머의 다음 토큰 예측 목표(next-token objective)와 호환되는 이산 시퀀스로 토큰화하는 방법 설명.
- Chameleon의 훈련 안정성 기법 이름: QK-Norm, 드롭아웃(dropout) 배치, LayerNorm 순서.
- Chameleon과 BLIP-2의 Q-Former 접근법 비교 및 각각이 적합한 경우 설명.

## 문제 정의

어댑터 기반 VLM(LLaVA, BLIP-2, Qwen-VL)은 텍스트와 이미지를 서로 다른 개체로 취급합니다. 텍스트 토큰은 `embed(text_token)`을 거치고, 이미지는 `visual_encoder(image) → projector → ... pseudo_tokens`를 거칩니다. 모델은 중간에 부분적으로 병합되는 두 개의 입력 경로를 가집니다.

세 가지 결과:

1. LLM은 이미지를 소비할 수는 있지만 생성할 수는 없습니다. 출력은 텍스트만 가능합니다.
2. 다중 모달리티 문서(기사처럼 단락과 이미지가 번갈아 나타나는 경우)는 다루기 어렵습니다. 모델 외부에서 다중 모달 입력을 파싱하거나 생성 과정을 연결해야 합니다.
3. 분포 불일치. 시각 토큰과 텍스트 토큰은 은닉 공간의 서로 다른 영역에 존재하여 미묘한 정렬 문제를 일으킵니다.

카멜레온은 이 전제를 거부합니다: 이미지는 공유 어휘에서 나온 이산 토큰의 시퀀스일 뿐입니다. 교차된 문서에 대해 모델을 훈련시키고, 하나의 손실 함수와 하나의 자기회귀 디코더를 사용하면 다중 모달리티 생성이 자동으로 가능해집니다.

## 개념

### 이미지 토크나이저로서의 VQ-VAE

토크나이저는 벡터 양자화 변분 오토인코더(vector-quantized variational autoencoder)입니다. 아키텍처 구성:

- 인코더(Encoder): 이미지를 32x32 크기의 공간 특징 맵(각각 차원 256)으로 매핑하는 CNN + ViT.
- 코드북(Codebook): 학습된 K개의 벡터(Chameleon은 8192개 사용), 차원 256.
- 양자화(Quantization): 각 공간 특징에 대해 L2 거리로 가장 가까운 코드북 항목을 조회. 연속 특징을 정수 인덱스로 대체.
- 디코더(Decoder): 양자화된 특징을 픽셀로 복원하는 CNN.

훈련: VAE 재구성 손실 + 커밋먼트 손실 + 코드북 손실. 코드북 인덱스는 이미지의 이산 알파벳(discrete alphabet)을 형성합니다.

Chameleon의 경우: 하나의 이미지는 32×32 = 1024개의 토큰으로 변환되며, 이는 8192개의 어휘(vocabulary)에서 추출됩니다. 텍스트 토큰(LLM의 BPE 어휘, 예: 32000개)과 결합. 최종 어휘: 40192개. 트랜스포머는 하나의 시퀀스와 하나의 손실 함수를 사용합니다.

### 공유 어휘(Shared Vocabulary)

Chameleon의 어휘는 텍스트 토큰, 이미지 토큰, 모달리티 구분자를 결합합니다. 각 토큰은 단일 ID를 가집니다. 입력 임베딩 레이어는 모든 ID를 D차원 은닉 벡터로 매핑합니다. 출력 투영 레이어는 은닉 벡터를 어휘 로짓(logits)으로 변환합니다. 소프트맥스는 다음 토큰을 선택하며, 모달리티와 무관합니다.

구분자 역할: `<image>`와 `</image>` 태그는 이미지 토큰 시퀀스를 구분합니다. 생성 시 모델이 `<image>`를 출력하면, 후속 소프트웨어는 다음 1024개 토큰이 디코더로 전송될 VQ 인덱스임을 인지합니다.

### 멀티모달 생성(Mixed-Modality Generation)

추론은 공유 어휘에서 다음 토큰 예측입니다. 예시 프롬프트: "고양이를 그리고 설명해 주세요." Chameleon의 출력:

```
<image> 4821 1029 2891 ... (1024개 이미지 토큰) </image>
고양이는 주황색이며 창가에 앉아 있습니다...
```

모델은 순서를 자율적으로 선택합니다. 이미지 후 텍스트, 텍스트 후 이미지, 또는 교차 생성이 가능합니다. 동일한 디코더와 손실 함수를 사용합니다.

어댑터 기반 VLMs(생성 시 텍스트만 출력)와 비교. Chameleon은 모델 출력 모달리티에 대한 질문을 재개합니다.

### 훈련 안정성 — QK-Norm, 드롭아웃, LayerNorm 순서

초기 융합(early-fusion) 훈련은 대규모에서 불안정합니다. Chameleon 논문에서 제시한 3가지 기법:

- QK-Norm. 어텐션 내 쿼리(query)와 키(key) 투영에 LayerNorm을 적용(도트 곱 전). 심층에서의 로짓 크기 폭주 방지. 2024년 이후 여러 대형 모델에서 사용.
- 드롭아웃 배치. 어텐션 및 MLP 이후뿐만 아니라 모든 잔차 연결(residual-add) 후 드롭아웃 적용. 이미지 토큰의 그래디언트가 지배적일 때 더 많은 정규화 필요.
- LayerNorm 순서. 잔차 분기에 사전 LayerNorm(표준) + 마지막 블록의 스킵 연결에 추가 LayerNorm. 최종 레이어 그래디언트 흐름 안정화.

이 기법들이 없으면 34B 파라미터 Chameleon 훈련은 여러 체크포인트에서 발산합니다. 적용 시 수렴합니다. 훈련 레시피는 아키텍처만큼 중요한 기여입니다.

### 토크나이저의 재구성 한계

VQ-VAE는 손실 압축(lossy)입니다. 8192개 코드북 항목과 512×512 이미지당 1024개 토큰에서 재구성 PSNR은 26–28 dB로 제한됩니다. 이는 인식 가능한 이미지 생성에 충분하지만, 연속 공간 확산(Stable Diffusion 3는 32+ dB)보다 시각적으로 열위입니다.

토크나이저가 병목입니다. 더 나은 토크나이저(MAGVIT-v2, IBQ, SBER-MoVQGAN)는 한계를 극복합니다. Emu3(레슨 12.12)는 토크나이저 개선만으로 SDXL 품질 생성을 달성합니다.

### Chameleon vs BLIP-2 / LLaVA

Chameleon(초기 융합, 공유 어휘):
- 하나의 손실 함수, 하나의 디코더.
- 멀티모달 출력 생성.
- 토크나이저가 품질 한계.
- 비용 높음: 추론 시 생성된 이미지마다 VQ-VAE 디코더 필요.

BLIP-2 / LLaVA(후기 융합, 별도 타워):
- 비전 입력, 텍스트 출력만.
- 사전 훈련된 LLM 재사용.
- 이해를 위한 토크나이저 병목 없음.
- 효율적: 단일 순전파.

작업별 선택. 이미지 생성이 필요하면 Chameleon 계열. 이해만 필요하면 어댑터 기반 VLMs가 더 간단하고 사전 훈련 계산 재사용 가능.

### Fuyu와 AnyGPT

Fuyu(Adept, 2023): 별도의 비전 인코더를 생략하고, 원시 이미지 패치를 토큰처럼 LLM의 입력 투영 레이어에 공급. 토크나이저 없음. Chameleon보다 간단하지만 공유 어휘 생성 기능 상실.

AnyGPT(Zhan et al., 2024): Chameleon을 4개 모달리티(텍스트, 이미지, 음성, 음악)로 확장. 각 모달리티에 VQ-VAE 트릭 적용, 공유 트랜스포머. 모든 조합 생성 가능. 레슨 12.16에서 더 자세히 다룹니다.

## 사용 방법

`code/main.py`는 장난감(end-to-end) 초기 결합(early-fusion) 모델을 구축합니다:

- 8x8 패치를 코드북 인덱스(K=16)로 매핑하는 소형 VQ-VAE 스타일 양자화기(quantizer).
- (텍스트 ID 0..31) + (이미지 ID 32..47) + (구분자 48, 49)로 구성된 공유 어휘(vocabulary).
- 합성 캡션(synthetic captions) 및 이미지 토큰 시퀀스에 대해 학습된 장난감 자기회귀 디코더(bigram 테이블).
- 프롬프트가 주어지면 텍스트 + 이미지 토큰을 번갈아 생성하는 샘플링 루프.

이 코드는 신호 흐름을 처음부터 끝까지 추적할 수 있도록 의도적으로 트랜스포머를 소형(bigrams)으로 유지합니다.

## Ship It

이 레슨은 `outputs/skill-tokenizer-vs-adapter-picker.md`를 생성합니다. 제품 사양(이해만 vs 이해 + 생성, 필요한 이미지 품질, 비용 예산)을 고려하여 Chameleon 계열(early fusion)과 LLaVA 계열(late fusion) 중 하나를 선택하고, 정량적 경험 법칙을 바탕으로 정당화합니다.

## 연습 문제

1. Chameleon은 K=8192 코드북 항목과 512x512 이미지당 1024 토큰을 사용합니다. 24비트 RGB 이미지 대비 압축 비율을 추정하세요. 손실 압축인가요? 손실 정도는 어떻게 되나요?

2. 동일한 VQ-VAE 밀도의 4K 이미지(3840x2160)는 몇 개의 이미지 토큰을 생성하나요? Chameleon 스타일 모델이 한 번의 추론 호출로 4K 이미지를 생성할 수 있나요? 컨텍스트, 토크나이저 품질, KV 캐시 중 무엇이 먼저 한계에 도달할까요?

3. 순수 Python으로 QK-Norm을 구현하세요. 64차원 쿼리와 키에 대해 LayerNorm 적용 전후의 내적 값을 비교하세요. 깊이(depth)에서 크기(magnitude) 제어가 왜 중요한가요?

4. Chameleon 2.3절 훈련 안정성 내용을 읽고, QK-Norm 없이 34B 파라미터에서 관찰된 정확한 실패 모드를 설명하세요. "정규화 폭발(norm explosion)"의 특징은 무엇이었나요?

5. 장난감 디코더를 확장하여 텍스트 전용 프롬프트에 대해 멀티모달 응답을 생성하도록 수정하세요. 훈련 데이터 분포(60% 텍스트 우선 / 40% 이미지 우선)에서 모델이 이미지 우선 vs 텍스트 우선 응답을 선택하는 빈도를 측정하세요.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|---------------------|-----------|
| Early fusion | "통합 토큰" | 이미지를 첫 단계부터 트랜스포머의 어휘(vocabulary)를 공유하는 이산 토큰(discrete tokens)으로 변환 |
| VQ-VAE | "이미지 토크나이저" | CNN + ViT + 코드북(codebook)으로 구성되어 이미지를 트랜스포머가 예측할 수 있는 정수 인덱스로 매핑 |
| Shared vocabulary | "하나의 사전" | 텍스트 + 이미지 + 모달리티 구분자(modality separators)를 포괄하는 단일 토큰 ID 공간 |
| QK-Norm | "어텐션 안정기" | 쿼리(query)와 키(key)의 내적(dot product) 전에 적용되는 LayerNorm, 노름(norm) 폭주 방지 |
| Mixed-modality generation | "텍스트 + 이미지 출력" | 단일 패스에서 텍스트와 이미지 토큰을 자율적으로 교차 생성하는 추론(inference) |
| Codebook size | "K개 항목" | VQ-VAE가 양자화할 수 있는 이산 벡터 수; 압축률과 충실도(fidelity) 간 트레이드오프 |
| Tokenizer ceiling | "재구성 한계" | VQ 토큰 디코딩으로 달성 가능한 최고 PSNR; 모델의 이미지 품질 상한선 |
