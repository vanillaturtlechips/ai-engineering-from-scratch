# LLaVA-OneVision: 단일 이미지, 다중 이미지, 비디오를 하나의 모델로

> LLaVA-OneVision(Li et al, 2024년 8월) 이전에는 오픈 VLM(Visual Language Model) 분야에서 단일 이미지용 LLaVA-1.5, 다중 이미지용 Mantis 및 VILA, 비디오용 Video-LLaVA 및 Video-LLaMA와 같은 별도의 모델 계보가 존재했습니다. 각 모델은 해당 벤치마크에서 우수한 성능을 보였지만 다른 시나리오에서는 실패했습니다. LLaVA-OneVision은 단일 커리큘럼(curriculum)으로 하나의 모델을 훈련시켜 세 가지 시나리오(단일 이미지, 다중 이미지, 비디오) 모두에서 우수한 성능을 달성할 수 있다고 주장했습니다. 또한, 단일 이미지 기술이 비디오로, 다중 이미지 추론 기술이 단일 이미지로 전이되는 현상(emergent task-transfer effects)이 전문가 모델들의 합보다 더 뛰어난 성능을 보인다고 설명했습니다. 이 레시피는 단순해 보이지만, 시나리오에 관계없이 일정한 시각적 토큰 예산(visual-token budget)과 단일 이미지에서 OneVision(다중 이미지)을 거쳐 비디오로 이동하는 명시적인 커리큘럼을 기반으로 합니다. 이 레슨에서는 예산 설정, 커리큘럼, 그리고 나타나는 현상(emergent behaviors)을 다룹니다.

**유형:** 구축(Build)  
**언어:** Python (표준 라이브러리, 토큰 예산 솔버 + 커리큘럼 플래너)  
**사전 요구 사항:** 12단계 · 05(LLaVA), 12단계 · 06(모든 해상도 지원)  
**소요 시간:** ~180분

## 학습 목표

- 단일 이미지, 다중 이미지, 비디오 입력에 걸쳐 일정하게 유지되는 시각적 토큰 예산을 설계한다.
- 단일 이미지에서 비디오로의 기술 전이를 재앙적 망각(catastrophic forgetting) 없이 수행하는 훈련 커리큘럼(curriculum)을 구성한다.
- 커리큘럼을 올바르게 구성했을 때 동일한 파라미터 수에서 단일 모델이 전문가 모델(specialists)을 능가하는 이유를 설명한다.
- LLaVA-OneVision에서 보고된 세 가지 새로운 능력(emergent capabilities)을 명명한다: 다중 카메라 추론(multi-camera reasoning), 마크 집합 프롬프트(set-of-mark prompting), 아이폰 스크린샷 에이전트(iPhone-screenshot agent).

## 문제 정의

이미지, 다중 이미지, 비디오는 각각 모델에 다른 방식으로 부담을 줍니다.

단일 이미지는 OCR 및 세부 사항을 포착하기 위해 고해상도 토큰(AnyRes, ~2880 시각 토큰)을 요구합니다. 샘플당 예산: 이미지 1개, 2880 토큰.

다중 이미지는 이미지 간 추론이 컨텍스트에 적합하도록 중간 해상도(각각 ~576 토큰)의 여러 이미지를 요구합니다. 샘플당 예산: 4-8개 이미지, 각각 576 토큰, 총 2300-4600 토큰.

비디오는 시간적 역학을 포착하기 위해 저해상도(풀링 후 프레임당 ~196 토큰)의 많은 프레임을 요구합니다. 샘플당 예산: 8-32개 프레임, 각각 196 토큰, 총 1600-6200 토큰.

별도의 모델을 훈련하면 하나의 예산만 선택합니다. 단일 모델을 훈련할 경우, 컨텍스트를 초과하지 않으면서 시나리오 간에 예산이 합리적으로 확장되어야 합니다.

OneVision 이전에는 "하나의 시나리오만 훈련하고 나머지는 무시한다"가 기본 접근 방식이었습니다. Video-LLaVA는 추가 훈련 단계를 통해 이미지 모델에 비디오를 개조했습니다. LLaVA-NeXT는 타일링(tiling)을 통해 다중 이미지 지원을 추가했습니다. 그러나 이 중 어느 것도 세 가지 시나리오를 깔끔하게 처리하지 못했습니다.

## 개념

## OneVision 토큰 예산

LLaVA-OneVision은 샘플당 약 3000-4000개의 통합 시각 토큰 예산을 선택하며, 시나리오별로 다르게 할당합니다:

- 단일 이미지: AnyRes-9 (3x3 타일 + 썸네일), 각 타일은 384 해상도, 729 패치, 2x2 공격적 이중 선형 풀링 → 타일당 182 토큰. 총계: 9 * 182 + 182 = 1820 토큰. 또는 AnyRes-4에서 타일당 729 = 2916 + 729.
- 다중 이미지: 각 이미지를 중간 해상도(384, 타일링 없음)로 처리, 풀링 없이 729 토큰. 예산 6개 이미지 → 4374 토큰.
- 비디오: 384 해상도 32프레임에 3x3 공격적 이중 선형 풀링 적용 → 프레임당 81 토큰. 총계: 32 * 81 = 2592 토큰.

할당은 대략 일정한 총 토큰 수를 유지합니다. LLM은 컨텍스트를 초과하는 배치를 절대 보지 않습니다. 인코더는 시나리오별로 다른 기하학을 생성하지만, LLM은 동일한 예산을 소비합니다.

## 3단계 커리큘럼

LLaVA-OneVision은 3단계로 훈련됩니다:

1. 단일 이미지 SFT(단계 SI). 모든 데이터는 단일 이미지+텍스트입니다. 고해상도 AnyRes 입력으로 훈련합니다. 이는 인식, OCR, 세부적 이해를 가르칩니다. LLaVA-NeXT 데이터와 OneVision 전용 단일 이미지 데이터를 사용합니다.
2. OneVision SFT(단계 OV). 단일 이미지 + 다중 이미지 + 비디오(균일 샘플링된 프레임)를 혼합합니다. 통합 토큰 예산으로 훈련합니다. 이는 모델이 이질적인 배치 형태를 처리하도록 가르칩니다. 가중치 재설정 없이 단계 SI에서 계속합니다.
3. 작업 전이(단계 TT). 일반적으로 제품 요구사항에 따라 다중 이미지 또는 비디오에 더 중점을 둔 대상 작업 혼합으로 계속합니다. 배포를 위한 선택적 파인튜닝이 가능합니다.

중요: 커리큘럼 순서가 중요합니다. 비디오-퍼스트 또는 다중 이미지-퍼스트 훈련은 동일한 데이터를 사용하더라도 단일 이미지-퍼스트보다 이미지 성능이 더 나쁩니다. 논문에서는 이를 명시적으로 검증합니다.

## 커리큘럼의 작동 원리

단일 이미지 훈련은 인식 기반을 구축합니다. 패치 토큰은 세부 시각 특징을 운반하며, LLM은 이를 텍스트와 통합하는 방법을 학습합니다. 다중 이미지와 비디오는 구조적 도전(어떤 이미지가 어떤 것인지, 어떤 일이 먼저 발생했는지)을 도입하는데, 이는 강력한 인식 기반 없이는 학습하기 어렵습니다.

모든 시나리오를 처음부터 함께 훈련하면 모델은 인식에 과소적합(배치당 제한된 단일 이미지 데이터)하고 구조에 과적합(다중 이미지/비디오 데이터 과다)합니다. 결과: 교차 이미지 추론 패턴을 따르지만 시각적으로 얕은 모델.

커리큘럼 순서는 단계 SI에서 인식 강도를 제공하고, 단계 OV에서 구성적/시간적 추론을 제공하며, 둘 다 잃지 않습니다.

## 시나리오 간 신생 기술

LLaVA-OneVision 논문에서는 3가지 신생 능력을 보고합니다:

1. 다중 카메라 추론. 다중 이미지 + 비디오를 별도로 훈련; 추론 시 다중 카메라 운전 장면에 대한 추론을 요청받음. 모델은 훈련에서 정확히 그 형식을 본 적 없음에도 뷰를 올바르게 통합합니다.
2. 마크 집합 프롬프트. 사용자가 이미지에 번호 마크를 추가하여 객체를 주석 처리; 모델은 "마크 3이 마크 7에 대해 무엇을 하고 있는가?"에 대해 추론합니다. 마크나 주석에 대해 훈련되지 않음; 공간 기반 + 다중 이미지 참조의 조합에서 학습됨.
3. iPhone-스크린샷 에이전트. 사용자가 iPhone 화면 스크린샷을 제공하고 다음 클릭을 계획하도록 요청. UI 스크린샷, 사용자 워크플로우 비디오, 다중 이미지 전후 쌍으로 훈련. 에이전트 사용 사례로 일반화됩니다.

이들은 훈련된 작업이 아닙니다; 커리큘럼의 구성적 구조에서 발생합니다.

## 시각 토큰 풀링

토큰 예산은 풀링을 필요로 합니다. OneVision은 2D 패치 그리드에 이중 선형 보간을 사용합니다: 24x24 = 576 패치가 12x12 = 144(2x 팩터) 또는 8x8 = 64(3x 팩터)가 됩니다. 풀링은 토큰 공간이 아닌 패치 그리드 공간에서 수행되어 지역성을 보존합니다.

시나리오별 풀링 팩터 선택은 하이퍼파라미터입니다. 풀링이 적을수록 = 토큰이 많을수록 = 표현이 풍부해집니다. 풀링이 많을수록 = 토큰이 적을수록 = 더 많은 프레임/이미지를 수용합니다.

## LLaVA-OneVision-1.5

2025년 후속 연구(LLaVA-OneVision-1.5, arXiv 2509.23661)는 훈련 데이터, 모델 가중치, 코드 모두 "완전 개방"되었습니다. 일부 벤치마크에서 독점 모델과의 격차를 해소하고 레시피를 민주화합니다. 동일한 커리큘럼, 더 많은 데이터, 더 나은 기본 LLM. 아키텍처 변경 없음.

## Qwen2.5-VL과의 대조

Qwen2.5-VL(레슨 12.09)은 다른 선택을 합니다. 고정 풀링 대신 M-RoPE와 동적 FPS를 사용합니다. 입력에 따라 예산이 확장됩니다 — 1분 비디오는 5초 비디오보다 더 많은 토큰을 사용합니다. LLaVA-OneVision은 예산을 고정하고 풀링을 확장합니다. 둘 다 작동하지만, 구성 가능성과 예측 가능성을 교환합니다.

## 사용 방법

`code/main.py`는 OneVision 스타일의 VLM(비전-언어 모델)을 위한 커리큘럼 및 예산 계획 도구입니다. 샘플당 토큰 예산과 목표 시나리오 혼합 비율(예: 단일 이미지 40%, 다중 이미지 30%, 비디오 30%)이 주어지면 다음과 같은 작업을 수행합니다:

- 시나리오별 해상도, 풀링 계수, 프레임 수를 할당합니다.
- 모든 시나리오가 공유 예산 내에 적합한지 확인합니다.
- 예상 토큰 수, LLM FLOPs, 그리고 토큰 부족 시나리오 목록을 보고합니다.
- 단계별 훈련 일정을 출력합니다.

OneVision 파인튜닝 계획 수립이나 VLM 배포 시 요청당 비용 검증에 활용하세요.

| 작업 전이 | "시나리오 간 일반화" | 공유 백본(shared backbone)을 통해 단일 이미지에서 학습한 기술이 비디오에 적용되는 (그 반대도 마찬가지) 현상 |
