# Any-Resolution Vision: Patch-n'-Pack과 NaFlex

> 실제 이미지는 224x224 정사각형이 아닙니다. 영수증은 9:16, 차트는 16:9, 의료 스캔은 4096x4096, 모바일 스크린샷은 9:19.5 비율일 수 있습니다. 2024년 이전의 VLM(Visual Language Model) 접근 방식 — 모든 것을 고정 크기 정사각형으로 리사이즈 — 은 OCR, 문서 이해, 고해상도 장면 파싱에 필요한 신호를 버렸습니다. NaViT(Google, 2023)는 블록-대각선 마스킹을 통해 가변 해상도 패치를 단일 트랜스포머 배치에 패킹할 수 있음을 보였습니다. Qwen2-VL의 M-RoPE(2024)는 절대 위치 테이블을 완전히 제거했습니다. LLaVA-NeXT의 AnyRes는 고해상도 이미지를 베이스 + 서브 이미지로 타일링했습니다. SigLIP 2의 NaFlex 변형(2025)은 현재 모든 종횡비를 지원하는 단일 체크포인트를 원하는 오픈 VLM의 기본 인코더입니다. 이 레슨은 패치-n'-팩을 처음부터 끝까지 구현합니다.

**유형:** 구현
**언어:** Python (표준 라이브러리, 패치 패커 + 블록-대각선 마스크)
**사전 요구 사항:** 12단계 · 01 (ViT 패치), 12단계 · 05 (LLaVA)
**소요 시간:** ~120분

## 학습 목표

- 가변 해상도 이미지 배치에서 패치(patch)를 하나의 시퀀스로 패킹(packing)하고 블록-대각선(block-diagonal) 어텐션 마스크(attention mask)를 구축한다.
- 주어진 작업에 대해 AnyRes 타일링(LLaVA-NeXT), NaFlex(SigLIP 2), M-RoPE(Qwen2-VL) 중 적절한 방식을 선택한다.
- 리사이징 없이 OCR, 차트, 사진 촬영에 대한 토큰 예산(token budget)을 계산한다.
- 정사각형 리사이징(square-resize)의 세 가지 실패 모드인 눌린 텍스트(squished text), 잘린 콘텐츠(cropped content), 패딩에 낭비된 토큰(wasted tokens on padding)을 설명한다.

## 문제 정의

트랜스포머는 시퀀스를 기대합니다. 배치(batch)는 동일한 길이의 시퀀스를 쌓은 것입니다. 이미지가 224x224라면 매번 196개의 패치 토큰이 생성되며, 패딩이 필요 없고 작업이 완료됩니다. 224로 학습하고 224로 추론하며, 해상도는 다시 고려할 필요가 없습니다.

현실 세계는 협조하지 않습니다. 문서는 세로형(8.5x11 인치, 2:3 비율)입니다. 차트 스크린샷은 가로형(16:9)입니다. 영수증은 높고 좁은 형태(1:3)입니다. 의료 영상은 2048x2048 이상의 해상도로 제공됩니다. 모바일 기기 스크린샷은 1170x2532(0.46:1) 비율입니다.

2024년 이전의 세 가지 접근 방식과 각각의 한계:

1. 고정 정사각형(224x224 또는 336x336)으로 리사이즈. 이미지를 찌그러뜨리면 텍스트와 얼굴이 왜곡됩니다. 다운스케일 시 차트 라벨과 OCR 내용이 손실됩니다. LLaVA-1.5 이전까지의 표준 관행입니다.
2. 고정 종횡비로 크롭. 이미지의 대부분을 버리게 되며, 크롭 위치 선정 자체가 별도의 비전 문제입니다.
3. 가장 긴 변 기준으로 패딩. 왜곡은 해결되지만 세로 이미지의 경우 토큰의 50% 이상이 패딩에 낭비됩니다. 모든 패딩 토큰에 대해 이차적(quadratic) 어텐션 비용이 발생합니다.

2024-2025년 해결책: 트랜스포머가 이미지의 원본 해상도에서 패치들을 처리하도록 하고, 이기종 배치를 하나의 시퀀스로 패킹하여 계산 낭비 없이 처리하는 방법을 찾는 것입니다.

## 개념

### NaViT와 패치-앤-팩

NaViT (Dehghani et al., 2023)는 이 방법이 대규모로 작동함을 보여준 논문입니다. 아이디어는 다음과 같이 기계적입니다:

1. 배치 내 각 이미지에 대해 선택한 패치 크기(예: 14)에서 네이티브 패치 그리드를 계산합니다.
2. 각 이미지의 패치를 고유한 가변 길이 시퀀스로 평탄화합니다.
3. 배치 내 모든 이미지의 패치를 하나의 긴 시퀀스로 연결합니다.
4. 블록-대각선 어텐션 마스크를 구성하여 이미지 A의 패치가 이미지 A 내에서만 어텐션하도록 합니다.
5. 패치별 위치 정보(2D RoPE 또는 분수 위치 임베딩)를 전달합니다.

336x336(576 토큰), 224x224(256 토큰), 448x336(768 토큰) 크기의 이미지 3개로 구성된 배치는 1600x1600 블록-대각선 마스크를 가진 1600-토큰 시퀀스가 됩니다. 패딩이 없고, 계산 낭비가 없습니다. 트랜스포머는 임의의 종횡비를 처리할 수 있습니다.

NaViT는 훈련 중 분수 패치 드롭을 도입했습니다. 배치 전체에서 무작위로 50%의 패치를 드롭하여 정규화하고 훈련 속도를 높입니다. SigLIP 2가 이를 계승했습니다.

### AnyRes (LLaVA-NeXT)

LLaVA-NeXT의 AnyRes는 실용적인 대안입니다. 고해상도 이미지와 고정된 인코더(336의 CLIP 또는 SigLIP)가 주어졌을 때 이미지를 타일링합니다:

1. 미리 정의된 세트 — (1x1), (1x2), (2x1), (1x3), (3x1), (2x2) 등 — 에서 이미지 종횡비에 가장 잘 맞는 그리드 레이아웃을 선택합니다.
2. 전체 이미지를 그리드로 타일링합니다. 각 타일은 336x336 크롭이 됩니다.
3. 또한 썸네일을 생성합니다: 전체 이미지를 336x336으로 리사이즈한 글로벌 컨텍스트 토큰입니다.
4. 모든 타일을 고정된 336-인코더로 인코딩합니다. 타일 토큰 + 썸네일 토큰을 연결합니다.

2x2 그리드 + 썸네일로 672x672 이미지를 처리하면 4 * 576 + 576 = 2880개의 시각 토큰이 생성됩니다. 비용이 많이 들지만 효과적입니다. LLM은 로컬 디테일과 글로벌 컨텍스트를 모두 볼 수 있습니다.

AnyRes는 인코더가 고정되어 있고 하나의 해상도만 지원할 때 선택하는 경로입니다. 큰 이미지의 경우 토큰 수가 폭발적으로 증가합니다(4x4 그리드의 1344x1344 이미지는 9216 + 576 ≈ 9800 토큰으로, 8k LLM 컨텍스트의 대부분을 채웁니다).

### M-RoPE (Qwen2-VL)

Qwen2-VL은 멀티모달 로터리 위치 임베딩(M-RoPE)을 도입했습니다. NaViT의 분수 위치나 AnyRes의 타일-앤-썸네일 대신 각 패치는 3D 위치(시간, 높이, 너비)를 가집니다. 쿼리/키 회전은 임의의 H, W, 시간 길이를 처리합니다.

M-RoPE는 재훈련 없이 네이티브 동적 해상도를 지원합니다. 추론 시 임의의 HxW 이미지를 입력하면 패치 임베더가 H/14 x W/14 토큰을 생성하고, 각 토큰은 (t=0, r=행, c=열) 위치를 부여받습니다. RoPE가 적절한 주파수로 어텐션을 회전시킵니다. Qwen2.5-VL과 Qwen3-VL도 이를 계승합니다. InternVL3의 V2PE는 모달리티별 가변 인코딩으로 동일한 아이디어를 구현합니다.

AnyRes와 달리 M-RoPE는 네이티브 해상도에서 O(H x W / P^2) 토큰을 사용합니다. 타일링 오버헤드가 없습니다. NaViT와 달리 단일 이미지를 기대합니다. 해상도 간 배치에는 여전히 패치-앤-팩이 필요합니다.

### NaFlex (SigLIP 2)

NaFlex는 SigLIP 2 체크포인트의 네이티브-플렉스 모드입니다. 단일 모델이 추론 시 여러 시퀀스 길이(256, 729, 1024 토큰)를 지원합니다. 내부적으로는 훈련 시 NaViT 스타일의 패치-앤-팩과 패치별 절대 분수 위치를 사용합니다. 판매 포인트는 다음과 같습니다: 하나의 체크포인트로, 작업에 따라 추론 시 토큰 예산을 선택할 수 있습니다.

시맨틱 작업(분류, 검색)에는 256 토큰을, OCR 또는 차트 이해에는 1024 토큰을 사용합니다. 재훈련이 필요 없습니다.

### 패킹 마스크

블록-대각선 마스크는 대부분의 구현이 실패하는 부분입니다. 길이 `N_total`의 패킹된 시퀀스가 이미지 `i=0..B-1`을 커버하고 각 이미지 길이가 `n_i`일 때, 형태 `(N_total, N_total)`의 마스크 `M`은 두 인덱스가 동일한 이미지 블록에 속하면 1, 아니면 0입니다. 누적 길이 리스트에서 마스크를 구성할 수 있습니다:

```
offsets = [0, n_0, n_0+n_1, ..., N_total]
M[i, j] = 1 iff 존재하는 b에 대해 offsets[b] <= i < offsets[b+1] and offsets[b] <= j < offsets[b+1]
```

이는 PyTorch에서 `torch.block_diag` 또는 명시적 gather로 한 줄로 구현 가능합니다. FlashAttention의 가변 길이 경로(`cu_seqlens`)는 마스크를 완전히 생략하고 누적 길이 텐서를 직접 사용하여 시퀀스 내 어텐션을 수행합니다. 일반적인 배치에서 밀집 마스크보다 ~10배 빠릅니다.

### 토큰 예산

작업별로 전략을 선택하세요:

- OCR / 문서: 1024-4096 토큰. SigLIP 2 NaFlex의 1024 또는 AnyRes 3x3 + 썸네일.
- 차트 및 UI: 384-448 네이티브에서 729-1024 토큰. Qwen2.5-VL의 동적 해상도와 최대 픽셀 제한.
- 자연 사진: 256-576 토큰으로 충분합니다. 다운스트림 LLM이 충분히 볼 수 있습니다. 콘텐츠 밀도가 높은 곳에 토큰을 할당하세요.
- 비디오: 공간 풀링 후 프레임당 64-128 토큰, 2-8 FPS. 12.17장에서 다룹니다.

2026년 프로덕션 규칙: 작업별 최대 픽셀 제한을 선택하고, 해당 제한까지 네이티브 종횡비로 인코딩한 후 배치를 패킹하고 패딩을 건너뜁니다. Qwen2.5-VL은 `min_pixels`와 `max_pixels`를 노출하여 정확히 이 조절을 가능하게 합니다.

## 사용 방법

`code/main.py`는 정수 픽셀 좌표를 가진 이미지들의 이질적 배치에 대해 패치-앤-팩(patch-n'-pack)을 구현합니다. 이 스크립트는 다음을 수행합니다:

- (H, W) 이미지 크기들의 리스트를 입력받습니다.
- 패치 크기 14에서 각 이미지의 패치 시퀀스 길이를 계산합니다.
- 총 길이 `sum(n_i)`를 가진 하나의 시퀀스로 패킹합니다.
- 블록-대각선 어텐션 마스크(명확성을 위해 밀집 형태)를 구성합니다.
- 패킹된 비용 vs 정사각형 리사이즈 및 AnyRes 타일링을 비교합니다.
- 혼합 배치(영수증, 차트, 스크린샷, 사진)에 대한 토큰 예산 표를 출력합니다.

스크립트를 실행해 보세요. 출력되는 숫자들은 2026년 모든 오픈 VLM(Visual Language Model)이 패치-앤-팩을 사용하는 이유가 될 것입니다.

## Ship It

이 레슨은 `outputs/skill-resolution-budget-planner.md`를 생성합니다. 혼합 종횡비 작업 부하(OCR, 차트, 사진, 비디오 프레임)와 총 토큰 예산이 주어졌을 때, 적절한 전략(NaFlex, AnyRes, M-RoPE, 고정 사각형)을 선택하고 요청별 구성을 출력합니다. 제품에 VLM(Vision Language Model) 크기를 조정할 때 이 기술을 사용하세요 — 이는 지연 시간 예산을 초과하는 무음 10x 토큰 폭증을 방지합니다.

## 연습 문제

1. 영수증 크기가 600x1500(1:2.5)입니다. 패치 크기 14에서 원본 해상도 토큰 수는 몇 개인가요? 336으로 정사각형 리사이즈 후 토큰 수는 몇 개인가요? 실제로 어떤 방식이 OCR 정확도를 더 많이 손실시키나요?

2. 길이가 각각 256, 576, 729, 1024인 4개 이미지 배치에 대한 블록-대각선 마스크를 구축하세요. 어텐션 행렬이 2585x2585이고 정확히 `256^2 + 576^2 + 729^2 + 1024^2`개의 비제로(non-zero) 항목을 가지는지 검증하세요.

3. 패치 크기 14에서 1792x896 이미지에 대해 다음을 비교하세요: (a) 336으로 정사각형 리사이즈 후 인코딩, (b) AnyRes 2x1 + 썸네일, (c) 네이티브 해상도 M-RoPE. 어떤 방식이 가장 적은 토큰을 사용하나요? 어떤 방식이 가장 많은 디테일을 보존하나요?

4. 분수형 패치 드롭아웃을 구현하세요: 패킹된 시퀀스가 주어졌을 때, 토큰의 50%를 균일하게 무작위로 드롭하고 블록-대각선 마스크를 업데이트하세요. 마스크의 희소성 변화를 측정하세요.

5. Qwen2-VL 논문(arXiv:2409.12191)의 3.2절을 읽고, `min_pixels`와 `max_pixels`가 무엇을 제어하는지 두 문장으로 설명하고 두 경계값이 왜 중요한지 설명하세요.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|---------------------|-----------|
| Patch-n'-pack | "NaViT-style packing" | 서로 다른 이미지의 가변 길이 패치 시퀀스를 하나의 배치 차원으로 연결 |
| Block-diagonal mask | "Packing mask" | 각 이미지의 패치가 패킹된 이웃이 아닌 자기 자신만 어텐션하도록 제한하는 어텐션 마스크 |
| AnyRes | "LLaVA-NeXT tiling" | 고해상도 이미지를 고정 크기 타일 그리드와 글로벌 썸네일로 분할; 고정 인코더로 모든 타일 인코딩 |
| NaFlex | "SigLIP 2 native-flex" | 재학습 없이 추론 시 256/729/1024 토큰 예산을 지원하는 단일 SigLIP 2 체크포인트 |
| M-RoPE | "Multimodal RoPE" | 위치 테이블 없이 임의의 H, W, T를 처리하는 3D 회전 위치 인코딩(시간, 행, 열) |
