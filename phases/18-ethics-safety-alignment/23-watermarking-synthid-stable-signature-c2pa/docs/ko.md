# 워터마킹 — SynthID, Stable Signature, C2PA

> 2026년 AI 생성 콘텐츠 출처 추적을 구조화하는 세 가지 기술. SynthID(Google DeepMind) — 2023년 8월 출시된 이미지 워터마킹, 2024년 5월 텍스트+비디오(Gemini + Veo), 2024년 10월 Responsible GenAI Toolkit을 통해 텍스트 오픈소스화, 2025년 11월 Gemini 3 Pro와 함께 통합 멀티모달 검출기 출시. 텍스트 워터마킹은 다음 토큰 샘플링 확률을 미묘하게 조정하며, 이미지/비디오 워터마크는 압축, 자르기, 필터, 프레임 레이트 변경에도 살아남음. Stable Signature(Fernandez et al., ICCV 2023, arXiv:2303.15435) — 잠재 확산 디코더를 미세 조정하여 모든 출력에 고정 메시지를 포함시킴; 콘텐츠의 10%가 잘린 생성 이미지도 FPR<1e-6에서 90% 이상 검출. 후속 연구 "Stable Signature is Unstable"(arXiv:2405.07145, 2024년 5월) — 미세 조정으로 워터마크를 제거하면서 품질 유지. C2PA — 암호로 서명되고 변조 방지 메타데이터 표준(C2PA 2.2 설명서 2025). 워터마킹과 C2PA는 상호 보완적: 메타데이터는 제거될 수 있지만 더 풍부한 출처 정보를 담고, 워터마크는 트랜스코딩을 견디지만 적은 정보를 전달.

**유형:** 구축(Build)  
**언어:** Python (표준 라이브러리, token-watermark 임베딩 + 검출)  
**사전 요구 사항:** 10단계 · 04(샘플링), 01단계 · 09(정보 이론)  
**소요 시간:** ~75분

## 학습 목표

- 토큰 수준 워터마킹(SynthID-text 스타일)과 검출 가능한 메커니즘을 설명합니다.
- Stable Signature와 이를 무력화한 2024년 제거 공격을 설명합니다.
- C2PA의 역할과 워터마킹과의 상호 보완적 관계를 명시합니다.
- 주요 한계점(모델별 신호, 의역 시 견고성, 의미 보존 공격(arXiv:2508.20228))을 설명합니다.

## 문제 정의

2023-2024년에는 딥페이크와 AI 생성 콘텐츠가 정치 및 소비자 영역에서 대규모로 확산되었습니다. 워터마킹(watermarking)은 기술적 출처 추적 신호로 제안되었습니다: 생성 시점에 마크를 표시하고, 이후 이를 탐지하는 방식입니다. 2025년 증거 자료에 따르면, 어떤 워터마크도 무조건적으로 견고하지는 않지만, C2PA 메타데이터와 결합했을 때 이 조합은 사용 가능한 출처 추적 스토리를 제공합니다.

## 개념

### 텍스트 워터마킹 (SynthID-text 스타일)

Kirchenbauer et al. 2023 메커니즘, Google에서 상용화:

1. 각 디코딩 단계에서 이전 K 토큰을 해시하여 어휘 사전을 "녹색"과 "적색" 집합으로 의사난수 분할합니다.
2. 녹색 로짓에 δ를 추가하여 녹색 집합으로 샘플링을 편향시킵니다.
3. 생성된 텍스트는 우연보다 더 많은 녹색 토큰을 포함합니다.

탐지: 각 접두사를 재해시하고 생성물의 녹색 토큰 수를 카운트한 후 z-점수를 계산합니다. 워터마킹된 텍스트는 z-점수 >0, 인간 텍스트는 ~0입니다.

특성:
- 독자에게 감지 불가능 (δ가 충분히 작아 품질 손실이 미미).
- 어휘 분할 함수 접근 시 탐지 가능.
- 의역(paraphrase)에 강건하지 않음 — 텍스트 재작성으로 신호 파괴.

SynthID-text는 2024년 10월 Google의 Responsible GenAI 툴킷을 통해 오픈소스화되었습니다.

### 스테이블 시그니처 (이미지)

Fernandez et al. ICCV 2023. 잠재 확산 디코더를 미세조정하여 생성된 모든 이미지가 잠재 표현에 고정된 이진 메시지를 포함하도록 합니다. 탐지는 신경망 디코더로 잠재 공간에서 복호화됩니다. 콘텐츠 10%로 크롭된 이미지도 FPR<1e-6에서 >90% 탐지됩니다.

2024년 5월 "Stable Signature is Unstable" (arXiv:2405.07145): 디코더 미세조정으로 워터마크를 제거하면서 이미지 품질 유지 가능. 적대적 생성 후 미세조정은 저렴하며, 워터마크의 적대적 강건성은 제한적입니다.

### SynthID 통합 탐지기 (2025년 11월)

Gemini 3 Pro와 함께 출시: 텍스트, 이미지, 오디오, 비디오의 SynthID 신호를 하나의 API로 읽는 멀티모달 탐지기. Google 출처 추적 스택을 통합합니다.

### C2PA

콘텐츠 출처 및 진위 연합(Coalition for Content Provenance and Authenticity). 암호로 서명된 변조 방지 메타데이터 표준. C2PA 2.2 설명서(2025). C2PA 매니페스트는 출처 주장(생성자, 생성 시간, 변환 내역)을 생성자 키로 서명하여 기록합니다.

워터마킹과의 상호보완성:
- 메타데이터는 제거 가능; 워터마크는 (쉽게) 제거 불가능.
- 메타데이터는 풍부함(전체 출처 체인); 워터마크는 비트만 전달.
- C2PA는 플랫폼 채택에 의존; 워터마크는 자동 삽입.

Google은 검색, 광고, "이 이미지 정보"에 두 기술을 통합했습니다.

### 한계

- **모델 종속적.** SynthID 워터마크는 SynthID 지원 모델의 생성물만 표시합니다. SynthID가 없는 모델의 생성물은 워터마크되지 않으므로 "SynthID 신호 없음"은 진위성 증명이 아닙니다.
- **의역.** 텍스트 워터마크는 의미 보존 의역에 살아남지 못합니다.
- **변환 공격.** arXiv:2508.20228(2025)은 텍스트 워터마크와 많은 이미지 워터마크를 파괴하는 의미 보존 공격을 제시합니다.
- **미세조정 제거.** "Stable Signature is Unstable"에 따르면 생성 후 미세조정으로 워터마크 제거 가능.

## 사용 방법

`code/main.py`는 장난감 텍스트 워터마크를 구축합니다. 토큰은 정수 0..N-1로 표현되며, 워터마크가 적용된 샘플링은 해시 정의된 녹색 집합(green set)을 선호합니다. 검출기는 녹색 토큰 z-점수를 계산합니다. 1000토큰 생성에서 검출을 관찰할 수 있고, 의역(paraphrase)이 신호를 파괴하는 과정을 지켜보거나, 인간 텍스트에 대한 거짓 긍정률(false-positive rate)을 측정할 수 있습니다.

## Ship It

이 레슨은 `outputs/skill-provenance-audit.md`를 생성합니다. 출처 증명이 포함된 콘텐츠 배포가 주어졌을 때, 다음을 감사합니다: 워터마크 메커니즘(있는 경우), C2PA 서명 체인(있는 경우), 각각의 적대적 강건성, 그리고 모달리티별 커버리지.

## 연습 문제

1. `code/main.py`를 실행하세요. 워터마크가 포함된 1000토큰 생성 텍스트와 인간이 작성한 텍스트의 z-점수(z-score)를 보고하세요. 95% 신뢰 임계값에서 거짓 양성(false-positive) 비율을 식별하세요.

2. 토큰의 30%를 동의어로 대체하는 의역 공격(paraphrase attack)을 구현하세요. z-점수를 다시 측정하세요.

3. Kirchenbauer et al. 2023 섹션 6의 강건성(robustness) 내용을 읽으세요. 텍스트 워터마크는 의역 공격에서 실패하는 반면 이미지 워터마크는 자르기(cropping) 공격에서 살아남는 이유는 무엇인가요?

4. SynthID-text + C2PA 메타데이터를 사용하는 배포 방식을 설계하세요. 소비자가 보는 출처(provenance) 체인을 설명하세요. 각 구성 요소의 실패 모드(failure mode) 하나를 식별하세요.

5. 2024년 "Stable Signature is Unstable" 연구 결과는 파인튜닝(fine-tuning)이 이미지 워터마크를 제거함을 보여줍니다. 이 공격을 제한하는 배포 제어 방식을 설계하세요. 예를 들어, 파인튜닝된 체크포인트(checkpoint)의 서명된 릴리스(signed release)를 요구하도록 하세요.
