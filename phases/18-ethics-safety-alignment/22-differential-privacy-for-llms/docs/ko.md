# 대규모 언어 모델을 위한 차등 프라이버시

> DP-SGD(차등 프라이버시 확률적 경사 하강법)는 여전히 표준으로 남아 있습니다 — 노이즈가 주입된 경사 업데이트는 공식적인 (엡실론, 델타) 보장을 제공합니다. 계산, 메모리, 유용성 측면의 오버헤드는 상당합니다; 파라미터 효율적인 DP 파인튜닝(LoRA + DP-SGD)이 2025년 일반적인 구성입니다(ACM 2025). 두 가지 상반된 증거: 캐니리 기반 멤버십 추론(Duan et al., 2024)은 언어 모델에 대해 제한된 성공만 보고합니다; 훈련 데이터 추출(Carlini et al., 2021; Nasr et al., 2025)은 상당한 그대로의 암기 내용을 복구합니다. 해결 방안(arXiv:2503.06808, 2025년 3월): 차이는 측정 대상에 있습니다 — 삽입된 캐니리 대 "가장 추출 가능한" 데이터. 새로운 캐니리 설계는 섀도우 모델 없이 손실 기반 MIA(멤버십 추론 공격)를 가능하게 하며, 현실적인 DP 보장 하에 실제 데이터로 훈련된 LLM에 대한 최초의 비중요한 DP 감사를 제공합니다. 대안: PMixED(arXiv:2403.15638) — 다음 토큰 분포에 대한 전문가 혼합을 통한 추론 시 프라이빗 예측; DP 합성 데이터 생성(Google Research 2024). 신흥 공격: LLM 피드백을 통한 차등 프라이버시 역전 — 신뢰도 점수 유출.

**유형:** 구축
**언어:** Python (표준 라이브러리, DP-SGD 노이즈 주입 및 엡실론-델타 계정 관리 데모)
**선수 지식:** 01단계 · 09단계(정보 이론), 10단계 · 01단계(대규모 모델 훈련)
**소요 시간:** ~60분

## 학습 목표

- (엡실론, 델타)-차등 프라이버시((epsilon, delta)-differential privacy)를 정의하고 DP-SGD(확률적 경사 하강법) 레시피를 설명하시오.
- 2024-2025년 긴장 관계: 캐너리 MIA(멤버십 추론 공격) vs 학습 데이터 추출이 서로 다른 결과를 제공하는 현상을 설명하시오.
- PMixED(Private Mixture of Experts with Encrypted Decoding)를 설명하고 추론 시 프라이빗 예측이 차등 프라이버시 학습 대안인 이유를 설명하시오.
- LLM 피드백을 통한 차등 프라이버시 역전 공격(Differential Privacy Reversal via LLM Feedback)을 설명하시오.

## 문제 정의

LLM(대형 언어 모델)은 암기합니다. Carlini et al. 2021은 프로덕션 언어 모델이 요청 시 훈련 텍스트를 그대로 재현할 수 있음을 보였습니다. DP(차등 프라이버시)는 공식적인 방어 메커니즘입니다: 출력이 단일 훈련 예제에 대해 증명 가능하게 둔감하도록 훈련합니다. 2024-2025년 증거는 DP-SGD(차등 프라이버시 확률적 경사 하강법)가 필요하지만, 배포된 ε 값이 위협 모델과 일치하지 않을 수 있음을 보여줍니다.

## 개념

### (ε, δ)-차등 프라이버시

랜덤화 알고리즘 M이 (ε, δ)-DP를 만족하려면, 하나의 예시가 다른 두 데이터셋과 모든 사건 S에 대해 다음이 성립해야 합니다:  
P(M(D) ∈ S) ≤ e^ε * P(M(D') ∈ S) + δ.  

해석: 출력 분포가 충분히 가까워서(ε로 파라미터화됨) 어떤 개인의 기여도 δ 확률을 제외하고는 신뢰성 있게 추론할 수 없습니다.

### DP-SGD

Abadi et al. 2016. 표준 레시피:  
1. 미니배치 샘플링.  
2. 예시별 그래디언트 계산.  
3. 각 예시 그래디언트를 임계값 C로 클리핑.  
4. 클리핑된 그래디언트 합산 후 표준편차 σ * C인 가우시안 노이즈 추가.  
5. 노이즈가 추가된 합으로 파라미터 업데이트.  

프라이버시 비용은 계정 도구(Moments Accountant, Rényi DP 계정 도구)로 추적됩니다. LLM 문헌에서 보고된 ε 값은 위협 모델, 데이터 민감도, 유틸리티 목표에 따라 크게 달라집니다. 보편적으로 "안전한" 기본 ε은 없습니다. 일부 LLM 학습 설정에서 ε ≈ 1–10 범위가 보고되었지만, 이는 예시일 뿐 권장 기본값이 아닙니다. 낮은 ε은 일반적으로 더 많은 노이즈를 필요로 하며 유틸리티 손실을 증가시킬 수 있습니다.

### LoRA + DP-SGD

프론티어 모델의 전체 DP-SGD는 비용이 너무 높습니다. LoRA(Hu et al. 2022)는 그래디언트 업데이트를 작은 어댑터로 제한하여 예시별 그래디언트 저장 공간을 줄입니다. LoRA + DP-SGD는 2025년 일반적인 구성입니다. DP 보장은 어댑터에 적용되며, 기본 모델은 고정됩니다.

### 2024-2025년 논쟁

두 가지 증거:  

- **카나리 MIA(Duan et al. 2024).** 훈련 데이터에 고유한 카나리를 삽입하고, 멤버십 추론 공격자가 이를 식별할 수 있는지 측정합니다. 언어 모델에서 제한된 성공만 보고됩니다. MIA가 어렵다는 것을 시사합니다.  
- **훈련 데이터 추출(Carlini 2021, Nasr et al. 2025).** 접두사로 모델을 프롬프트하고, 훈련 데이터의 정확한 텍스트를 복구하는지 측정합니다. 상당한 기억 현상을 보고합니다. 관련 의미에서 MIA가 쉽다는 것을 시사합니다.  

2025년 3월 해결(arXiv:2503.06808): 두 방법은 서로 다른 것을 측정합니다. MIA는 "삽입된 카나리가 D에 있는가?"를 묻고, 추출은 "D에서 무엇을 복구할 수 있는가?"를 묻습니다. 프라이버시에 중요한 것은 "가장 추출 가능한" 예시입니다. 카나리는 추출 가능성을 최적화하지 않아 이를 과소평가합니다.  

새로운 카나리 설계. 섀도우 모델 없는 손실 기반 MIA. 현실적인 DP 보장을 가진 실제 데이터에서의 첫 번째 비중요 DP 감사.

### DP 훈련 대안

- **PMixED(arXiv:2403.15638).** 추론 시 프라이빗 예측. 다음 토큰 분포에 대한 전문가 혼합; 각 전문가는 훈련 데이터의 샤드를 보고, 집계 시 DP를 위해 노이즈 추가. 전체 DP 훈련을 완전히 회피합니다.  
- **DP 합성 데이터 생성(Google Research 2024).** DP-SGD로 LoRA-파인튜닝, 합성 데이터 샘플링, 합성 데이터로 다운스트림 분류기 훈련.  

둘 다 다른 위협 모델을 희생하여 전체 DP 훈련의 유틸리티 비용을 회피합니다.

### LLM 피드백을 통한 차등 프라이버시 역전

2025년 등장한 공격. DP-훈련 모델의 신뢰도 점수를 오라클로 사용하여 개인을 재식별합니다. 출력 자체는 유출되지 않더라도 신뢰도 분포가 유출될 수 있습니다.  

방어: 신뢰도를 노출하지 않거나, 노출 전 절단/양자화합니다. 이는 (ε, δ)-DP 훈련 이상의 추가 요구사항입니다.

레슨 20-21은 편향/공정성. 레슨 22는 프라이버시. 레슨 23은 워터마킹 기반 출처 추적. 레슨 27은 규제 데이터-출처 계층을 다룹니다.

## 사용 방법

`code/main.py`는 장난감 이진 분류 데이터셋에서 DP-SGD(차분 개인 정보 보호 확률적 경사 하강법)를 시뮬레이션합니다. 노이즈 승수 σ와 클리핑 노름 C를 스윕(sweep)하면서 (ε, δ) 예산과 정확도 비용을 추적할 수 있습니다. "캐너리 공격"은 고유한 훈련 예제를 삽입하고 로그-손실 테스트가 DP 적용 전후에 이를 탐지할 수 있는지 측정합니다.

## Ship It

이 레슨은 `outputs/skill-dp-audit.md`를 생성합니다. 언어 모델 배포에 대한 DP(차등 프라이버시) 주장이 주어졌을 때, 다음을 감사합니다: (ε, δ) 값, 사용된 계정(accountant), MIA(멤버십 추론 공격) 평가 프로토콜, 그리고 신뢰-노출 벡터(confidence-exposure vectors) 평가 여부.

5. LLM 피드백을 통한 DP 역전(DP Reversal via LLM Feedback) 공격을 스케치하세요. 신뢰도 점수 누출을 제한하는 대응책을 설계하고 배포 비용을 추정하세요.

## 주요 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|-------------------|-----------|
| DP | "(ε, δ)-차등 프라이버시" | 형식적 프라이버시: 인접 데이터셋 변경 시 출력 분포가 유사함 |
| DP-SGD | "노이즈 주입 SGD" | 그래디언트 클리핑 + 가우시안 노이즈 추가; 표준 차등 프라이버시 학습 |
| LoRA + DP-SGD | "효율적인 개인화 파인튜닝" | 저랭크 어댑터에 대한 DP-SGD 적용; 2025년 표준 구성 방식 |
| MIA | "멤버십 추론" | 특정 예시가 학습 데이터에 포함되었는지 판단하는 공격 |
| Canary | "삽입된 워터마크 예시" | 차등 프라이버시 유출 측정을 위한 고유 학습 예시 |
| PMixED | "개인화 추론 혼합" | 다음 토큰 분포에 대한 전문가 혼합을 통한 추론 시 차등 프라이버시 |
| DP Reversal | "신뢰도 유출 공격" | 모델의 신뢰도를 오라클로 사용하여 재식별하는 공격 |
