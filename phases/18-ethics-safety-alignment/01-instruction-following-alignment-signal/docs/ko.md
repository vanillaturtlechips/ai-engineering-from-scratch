# 명령어 추종이 정렬 신호로 작용하는 방식

> RLHF에 대한 모든 후속 비판은 이 파이프라인을 반대한다. 최적화 압력이 대리 모델을 왜곡하는 방식을 연구하기 전에, 먼저 대리 모델을 확인해야 한다. InstructGPT(Ouyang et al., 2022)는 참조 아키텍처를 정의했다: 명령어-응답 쌍에 대한 지도 미세 조정, 쌍별 선호도 순위로 훈련된 보상 모델, 그리고 SFT 정책에 대한 KL 페널티를 적용한 보상 모델에 대한 PPO. 1.3B InstructGPT가 175B GPT-3보다 선호되었다. 이 단일 결과가 2026년 모든 최전선 연구소가 여전히 RLHF 형태의 사후 훈련 파이프라인을 사용하는 이유이다.

**유형:** 학습
**언어:** Python (표준 라이브러리, 3단계 파이프라인 토이 모델)
**사전 요구 사항:** 10단계 · 06 (지도 미세 조정), 10단계 · 07 (RLHF), 10단계 · 08 (DPO)
**소요 시간:** ~45분

## 학습 목표

- InstructGPT 파이프라인의 세 단계와 각 단계에서 사용되는 손실(loss)을 명명할 수 있다.
- 1.3B 명령어 튜닝(instruction-tuned) 모델이 인간 선호도 평가(human preference evaluation)에서 원시(raw) 175B GPT-3을 능가한 이유를 설명할 수 있다.
- 3단계에서 KL 페널티(KL penalty)가 방지하는 대상과 이를 제거할 경우 모드 탐색(mode-seeking) 행동으로 붕괴되는 이유를 설명할 수 있다.
- 정렬 세금(alignment tax)과 Ouyang 등이 이를 완화하기 위해 사용한 PPO-ptx 기법을 설명할 수 있다.

## 문제 정의

사전 훈련된 언어 모델은 텍스트를 완성합니다. 질문에 답하지 않습니다. GPT-3에 "리스트를 뒤집는 파이썬 함수를 작성해 주세요"라고 질문하면, 종종 다른 프롬프트를 다시 받게 됩니다. 훈련 데이터의 대부분이 웹 텍스트이며, 이는 더 많은 웹 텍스트로 이어지기 때문입니다. 모델은 자신의 역할을 수행하고 있지만 — 그 역할이 잘못된 것입니다.

이 문제를 해결하기 위해 모든 주요 연구실에서 사용하는 대리 지표는 인간의 선호도입니다. 두 개의 완성 결과가 평가자에게 전달되면, 평가자는 더 나은 것을 선택합니다. 이후 보상 모델은 평가자의 선택을 학습합니다. 그런 다음 RL(강화 학습) 루프가 정책을 보상 모델이 높은 점수를 준 출력 쪽으로 이동시킵니다. 이것이 바로 InstructGPT 논문의 핵심 내용을 세 문장으로 요약한 것입니다. 논문의 나머지 부분은 엔지니어링에 관한 내용입니다.

## 개념

### 1단계: 지도형 파인튜닝(SFT)

선한 의도를 가진 인간이 작성할 응답을 포함한 프롬프트-응답 쌍을 수집합니다. Ouyang 등은 라벨러와 OpenAI API에서 13k개의 프롬프트를 사용했습니다. 표준 교차 엔트로피 손실로 이 데이터에 대해 기본 모델을 파인튜닝합니다.

SFT가 제공하는 것: 모델이 질문에 답변을 생성합니다. 제공하지 않는 것: 여러 답변이 가능한 경우 평가자가 선호하는 답변에 대한 신호.

### 2단계: 보상 모델(RM)

각 프롬프트에 대해 SFT 모델에서 K개의 완성본을 샘플링합니다. 라벨러가 이들을 순위 매깁니다. `y_w`가 `y_l`보다 선호되는 쌍에 대해 다음 손실 함수로 모든 프롬프트-응답 쌍의 점수를 매기는 보상 모델을 학습시킵니다:

```
L_RM = -log sigmoid(r(x, y_w) - r(x, y_l))
```

이는 Bradley-Terry 쌍별 선호 손실입니다. RM은 일반적으로 LM 헤드가 스칼라 헤드로 대체된 SFT 모델에서 초기화됩니다.

보상 모델은 작습니다: 175B InstructGPT에 6B로 충분했습니다. 또한 취약합니다 — 논문의 5절은 소규모에서 나타난 보상 해킹 행동에 관한 내용입니다.

### 3단계: KL 페널티가 있는 PPO

목적 함수를 정의합니다:

```
J(pi) = E_{x~D, y~pi(.|x)} [ r(x, y) ] - beta * KL(pi(.|x) || pi_SFT(.|x))
```

PPO로 최대화합니다. KL 항은 `pi`가 SFT 정책에서 너무 멀어지지 않도록 합니다. 이 항이 없으면 최적화기는 보상 모델이 본 적 없는 문자열(인간이 실제로 선호하는 것이 아닌)을 찾습니다.

KL 계수 `beta`는 RLHF에서 가장 중요한 하이퍼파라미터입니다. 너무 낮으면 보상 해킹, 너무 높으면 SFT 대비 개선이 없습니다.

### 정렬 세금

RLHF 후 모델은 인간이 선호하지만 표준 벤치마크(SQuAD, HellaSwag, DROP)에서 성능이 하락합니다. Ouyang 등은 이를 정렬 세금이라 부르며 PPO-ptx로 해결합니다: 사전 학습 그래디언트를 RL 목적 함수에 혼합하여 모델이 보상받지 못한 다운스트림 작업을 잊지 않도록 합니다.

```
J_ptx(pi) = J(pi) + gamma * E_{x~D_pretrain} [ log pi(x) ]
```

PPO-ptx는 표준이 되었습니다. Anthropic, DeepMind, Meta는 모두 변형 버전을 사용합니다.

### 결과

1.3B InstructGPT(SFT + RM + PPO-ptx)는 175B 기본 GPT-3보다 라벨러에게 약 70% 선호됩니다. 프로덕션 트래픽의 숨겨진 테스트 프롬프트에서는 격차가 더 벌어집니다. 이 수치에서 읽을 수 있는 두 가지:

1. 정렬은 능력과 다른 축입니다. 175B 모델은 더 많은 능력을 가졌고, 1.3B 모델은 더 많은 정렬을 가졌으며, 라벨러는 정렬된 모델을 선호했습니다.
2. 능력 하한은 기본 모델에 의해 설정됩니다. 기본 모델이 본 적 없는 사실을 RLHF로 알게 할 수 없습니다.

### 이것이 18단계 기준점인 이유

이후 레슨의 모든 비판 — 보상 해킹(레슨 2), DPO(레슨 3), 아첨(레슨 4), CAI(레슨 5), 슬리퍼 에이전트(레슨 7), 정렬 가짜(레슨 9) — 는 이 파이프라인의 일부를 반박합니다. 보상 해킹은 2단계를 공격합니다. DPO는 2, 3단계를 축소합니다. CAI는 인간 라벨러를 대체합니다. 아첨은 라벨러가 편향된 신호임을 보여줍니다. 정렬 가짜는 정책이 3단계를 완전히 우회할 수 있음을 보여줍니다. 먼저 이 파이프라인을 머릿속에 넣지 않고는 이러한 비판을 따라갈 수 없습니다.

## 사용 방법

`code/main.py`는 장난감 선호도 데이터에서 세 단계를 시뮬레이션합니다. 기본 "정책"은 행동 {A, B, C}에 대한 편향된 동전입니다. 1단계 SFT(Supervised Fine-Tuning)는 200개 프롬프트에 대한 라벨러 행동을 모방합니다. 2단계는 500개의 쌍별 순위로부터 Bradley-Terry 보상 모델을 적합시킵니다. 3단계는 SFT 정책에 대한 KL 페널티(Kullback-Leibler Penalty)를 사용해 단순화된 PPO(Proximal Policy Optimization) 업데이트를 실행합니다. 보상이 증가하는 과정, KL 발산 증가, 정책 변화 과정을 관찰할 수 있으며, KL 항을 비활성화하면 50번의 업데이트 단계 내에 보상 해킹(reward hacking)이 발생하는 것을 확인할 수 있습니다.

확인 사항:

- `beta = 0.1` vs `beta = 0.0`에서의 보상 추이
- 훈련 단계별 KL(pi || pi_SFT) 값
- 최종 행동 분포와 라벨러 선호도 비교

## Ship It

이 레슨은 `outputs/skill-instructgpt-explainer.md`를 생성합니다. RLHF 파이프라인 설명 또는 논문 초록이 주어지면, 세 단계 중 어떤 단계가 수정되고 있는지, 각 단계에서 어떤 손실 함수가 사용되는지, 그리고 KL 페널티(KL penalty) 또는 이와 동등한 정규화기(regularizer)가 존재하는지 식별합니다.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|---------------------|-----------|
| SFT | "지시 튜닝(instruction tuning)" | 단계 1: 프롬프트-응답 쌍에 대한 교차 엔트로피(cross-entropy) 미세 조정 |
| 보상 모델(Reward model) | "the RM" | Bradley-Terry를 사용하여 쌍별 레이블로 훈련된 (프롬프트, 응답)에 대한 스칼라 회귀기(scalar regressor) |
| Bradley-Terry | "쌍별 선호 손실(pairwise preference loss)" | -log sigmoid(r_w - r_l); 쌍별 순위를 이진 분류로 축소 |
| KL 페널티(KL penalty) | "정규화기(the regularizer)" | `beta * KL(pi || pi_SFT)` — RL 정책을 SFT 기준점(anchor) 근처에 유지 |
| PPO-ptx | "사전 훈련 혼합 PPO(PPO with pretraining mix)" | PPO 목적 함수에 사전 훈련 로그 우도(log-likelihood)의 일부를 추가하여 정렬 비용(alignment tax)을 상쇄 |
| 정렬 비용(Alignment tax) | "the RLHF 회귀" | RLHF가 대상으로 하지 않은 표준 벤치마크에서 RLHF 후 발생하는 성능 하락 |
| 라벨러 선호(Labeler preference) | "그라운드 트루스(the ground truth)" | 인간 순위 샘플; RM은 "인간 가치(human values)"가 아닌 이에 대한 통계적 프록시(proxy) |
