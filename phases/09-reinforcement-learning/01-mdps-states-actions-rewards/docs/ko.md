# MDP, 상태, 행동 및 보상

> 마르코프 결정 과정(MDP)은 상태, 행동, 전이, 보상, 할인율이라는 다섯 가지 요소로 구성됩니다. 강화 학습(RL)의 모든 것 — Q-러닝, PPO, DPO, GRPO — 은 이 구조를 최적화합니다. 한 번만 배우면 나머지 강화 학습 내용을 무료로 이해할 수 있습니다.

**유형:** 학습  
**언어:** Python  
**선수 지식:** Phase 1 · 06 (확률 및 분포), Phase 2 · 01 (ML 분류 체계)  
**소요 시간:** ~45분

## 문제 정의

체스 봇을 작성하고 있다. 또는 재고 계획자. 또는 트레이딩 에이전트. 또는 추론 모델을 훈련시키는 PPO 루프. 네 가지 다른 도메인, 하나의 놀라운 사실: 이 네 가지는 모두 동일한 수학적 객체로 귀결된다.

지도 학습은 `(x, y)` 쌍을 제공하고 함수를 피팅하도록 요청한다. 강화 학습은 레이블이 없으며 — 상태(state)의 스트림, 수행한 행동(action), 스칼라 보상(reward)만을 제공한다. 해당 수가 게임에서 승리했는가? 재고 보충 결정이 비용을 절감했는가? 거래가 수익을 냈는가? LLM이 방금 생성한 토큰이 평가자(judge)로부터 더 높은 보상을 이끌어냈는가?

이 스트림으로부터 학습하기 위해서는 이를 형식화(formalize)해야 한다. "내가 본 것", "내가 한 행동", "다음에 발생한 일", "그것이 얼마나 좋았는지" — 각각은 추론할 수 있는 객체가 되어야 한다. 그 형식화가 바로 마르코프 결정 과정(Markov Decision Process, MDP)이다. RLHF 및 GRPO 루프를 포함한 이 단계의 모든 강화 학습 알고리즘은 이 구조를 최적화한다.

## 개념

![마르코프 결정 과정: 상태, 행동, 전이, 보상, 할인](../assets/mdp.svg)

**다섯 가지 객체.**

- **상태** `S`. 에이전트가 결정을 내리는 데 필요한 모든 것. GridWorld에서는 셀. 체스에서는 보드. LLM에서는 컨텍스트 윈도우와 메모리.
- **행동** `A`. 선택 가능한 것들. 위/아래/왼쪽/오른쪽 이동. 체스 수 실행. 토큰 생성.
- **전이** `P(s' | s, a)`. 상태 `s`와 행동 `a`가 주어졌을 때 다음 상태의 분포. 체스에서는 결정적, 재고 관리에서는 확률적, LLM 디코딩에서는 거의 결정적.
- **보상** `R(s, a, s')`. 스칼라 신호. 승리 = +1, 패배 = -1. 수익 - 비용. GRPO의 로그 우도 비율 항.
- **할인** `γ ∈ [0, 1)`. 미래 보상이 현재 대비 얼마나 중요한지. `γ = 0.99`는 약 100단계, `γ = 0.9`는 약 10단계 범위를 구매.

**마르코프 성질** `P(s_{t+1} | s_t, a_t) = P(s_{t+1} | s_0, a_0, …, s_t, a_t)`. 미래는 현재 상태에만 의존. 만약 그렇지 않다면 상태 표현이 불완전한 것 — 방법의 실패가 아닌 상태 표현의 실패.

**정책과 수익.** 정책 `π(a | s)`은 상태를 행동 분포에 매핑. 수익 `G_t = r_t + γ r_{t+1} + γ² r_{t+2} + …`은 할인된 미래 보상의 합. 가치 `V^π(s) = E[G_t | s_t = s]`는 정책 `π` 하에서 상태 `s`에서 시작하는 기대 수익. Q-값 `Q^π(s, a) = E[G_t | s_t = s, a_t = a]`은 특정 행동으로 시작하는 기대 수익. 모든 강화학습 알고리즘은 이 두 값 중 하나를 추정한 후 `π`를 개선.

**벨만 방정식.** 이 단계의 모든 것이 사용하는 고정점 방정식:

`V^π(s) = Σ_a π(a|s) Σ_{s', r} P(s', r | s, a) [r + γ V^π(s')]`
`Q^π(s, a) = Σ_{s', r} P(s', r | s, a) [r + γ Σ_{a'} π(a'|s') Q^π(s', a')]`

이들은 기대 수익을 "이번 단계의 보상"과 "도달한 상태의 할인된 가치"로 분해. 재귀적. 9단계의 모든 알고리즘은 이 방정식을 수렴할 때까지 반복(동적 계획법), 샘플링(몬테카를로), 또는 한 단계 부트스트랩(시간차)합니다.

## 구축 방법

### 1단계: 작은 결정적 MDP

4×4 GridWorld. 에이전트는 왼쪽 상단에서 시작하며, 종료 상태는 오른쪽 하단, 매 스텝마다 보상 -1, 액션 `{위, 아래, 왼쪽, 오른쪽}`. `code/main.py` 참조.

```python
GRID = 4
TERMINAL = (3, 3)
ACTIONS = {"up": (-1, 0), "down": (1, 0), "left": (0, -1), "right": (0, 1)}

def step(state, action):
    if state == TERMINAL:
        return state, 0.0, True
    dr, dc = ACTIONS[action]
    r, c = state
    nr = min(max(r + dr, 0), GRID - 1)
    nc = min(max(c + dc, 0), GRID - 1)
    return (nr, nc), -1.0, (nr, nc) == TERMINAL
```

5줄. 이것이 전체 환경입니다. 결정적 전이, 일정한 스텝 페널티, 흡수 종료 상태.

### 2단계: 정책 롤아웃

정책은 상태에서 액션 분포로의 함수. 가장 간단한 것은 균일 무작위입니다.

```python
def uniform_policy(state):
    return {a: 0.25 for a in ACTIONS}

def rollout(policy, max_steps=200):
    s, total, steps = (0, 0), 0.0, 0
    for _ in range(max_steps):
        a = sample(policy(s))
        s, r, done = step(s, a)
        total += r
        steps += 1
        if done:
            break
    return total, steps
```

무작위 정책을 1000번 실행합니다. 평균 보상은 이 4×4 보드에서 약 -60에서 -80입니다. 최적 보상은 -6(직선 경로 아래-오른쪽)입니다. 이 격차를 줄이는 것이 9단계의 핵심입니다.

### 3단계: 벨만 방정식을 통해 `V^π` 정확히 계산

작은 MDP의 경우 벨만 방정식은 선형 시스템입니다. 상태를 열거하고 기댓값을 적용한 후 값이 변하지 않을 때까지 반복합니다.

```python
def policy_evaluation(policy, gamma=0.99, tol=1e-6):
    V = {s: 0.0 for s in all_states()}
    while True:
        delta = 0.0
        for s in all_states():
            if s == TERMINAL:
                continue
            v = 0.0
            for a, pi_a in policy(s).items():
                s_next, r, _ = step(s, a)
                v += pi_a * (r + gamma * V[s_next])
            delta = max(delta, abs(v - V[s]))
            V[s] = v
        if delta < tol:
            return V
```

이것은 반복적 정책 평가입니다. 서튼 & 바르토의 첫 번째 알고리즘이며 이후 모든 강화학습 방법의 이론적 기반입니다.

### 4단계: `γ`는 물리적 의미를 가진 하이퍼파라미터

유효한 지평선은 대략 `1 / (1 - γ)`입니다. `γ = 0.9` → 10스텝. `γ = 0.99` → 100스텝. `γ = 0.999` → 1000스텝.

너무 낮으면 에이전트가 근시안적으로 행동합니다. 너무 높으면 신용 할당이 노이즈가 많아지는데, 많은 초기 스텝들이 먼 미래 보상에 대한 책임을 공유하기 때문입니다. LLM RLHF는 일반적으로 에피소드가 짧고 유계이기 때문에 `γ = 1`을 사용합니다. 제어 작업은 `0.95–0.99`를 사용합니다. 장기 전략 게임은 `0.999`를 사용합니다.

## 함정

- **비-마르코프 상태(Non-Markovian state).** 마지막 3개의 관측값으로 결정해야 한다면 "상태"는 현재 관측값만이 아님. 해결: 프레임 쌓기(Atari의 DQN은 4개 프레임 쌓기) 또는 순환 상태 사용(관측값에 대한 LSTM/GRU).
- **희소 보상(Sparse rewards).** 승리 시에만 보상을 주면 큰 상태 공간에서 학습이 거의 불가능. 보상 설계(중간 신호) 또는 모방 학습으로 부트스트랩(Phase 9 · 09).
- **보상 해킹(Reward hacking).** 대리 보상(proxy reward)을 최적화하면 병리적 행동이 발생. OpenAI의 보트 레이싱 에이전트는 경주 완주 대신 파워업을 수집하며 영원히 회전. 항상 대리 보상이 아닌 목표 결과로부터 보상을 정의.
- **할인율 오류(Discount mis-spec).** 무한 시간 범위에서 `γ = 1`은 모든 값을 무한대로 만듦. 항상 유한 시간 범위 또는 `γ < 1`로 제한.
- **보상 스케일(Reward scale).** {+100, -100}과 {+1, -1} 보상은 동일한 최적 정책을 생성하지만 기울기 크기가 크게 다름. PPO/DQN에 입력 전 `[-1, 1]` 범위로 정규화.

## 사용 방법

2026 스택은 코드에 손대기 전에 모든 강화학습(RL) 파이프라인을 MDP로 축소합니다:

| 상황 | 상태(State) | 행동(Action) | 보상(Reward) | 감가율(γ) |
|-----------|-------|--------|--------|---|
| 제어(보행, 조작) | 관절 각도 + 속도 | 연속 토크 | 작업별 형성 보상 | 0.99 |
| 게임(체스, 바둑, 포커) | 보드 + 기록 | 합법적 이동 | 승리=+1 / 패배=-1 | 1.0 (유한) |
| 재고/가격 책정 | 재고 + 수요 | 주문량 | 수익 - 비용 | 0.95 |
| LLM을 위한 RLHF | 컨텍스트 토큰 | 다음 토큰 | 종료 시 보상 모델 점수 | 1.0 (에피소드 ~200 토큰) |
| 추론을 위한 GRPO | 프롬프트 + 부분 응답 | 다음 토큰 | 종료 시 검증기 0/1 | 1.0 |

훈련 루프를 작성하기 전에 반드시 5가지 튜플을 정의하세요. "RL이 작동하지 않는다"는 대부분의 버그 보고는 근본적으로 잘못된 MDP 정의에서 비롯됩니다.


## 핵심 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|---------------------|-----------|
| MDP | "강화 학습 설정" | 마르코프 성질을 만족하는 튜플 `(S, A, P, R, γ)`. |
| 상태(State) | "에이전트가 보는 것" | 선택된 정책 클래스 하에서 미래 동역학에 대한 충분 통계량. |
| 정책(Policy) | "에이전트의 행동" | 조건부 분포 `π(a | s)` 또는 결정적 사상 `s → a`. |
| 리턴(Return) | "총 보상" | 현재 단계부터의 할인된 합 `Σ γ^t r_t`. |
| 가치(Value) | "상태의 가치" | `s`에서 시작하여 `π` 하에서 기대되는 리턴. |
| Q-값(Q-value) | "행동의 가치" | `s`에서 시작하여 첫 행동이 `a`일 때 `π` 하에서 기대되는 리턴. |
| 벨만 방정식(Bellman equation) | "동적 프로그래밍 재귀" | 가치/ Q를 1단계 보상과 할인된 후속 가치의 고정점 분해. |
| 할인율 `γ`(Discount `γ`) | "미래 vs 현재" | 먼 미래 보상에 대한 기하학적 가중치; 유효 지평선 `~1/(1-γ)`. |

## 추가 학습 자료

- [Sutton & Barto (2018). 강화 학습: 입문, 2판](http://incompleteideas.net/book/RLbook2020.pdf) — 교과서. 3장은 MDP와 벨만 방정식; 1장은 이후 모든 수업의 기반이 되는 보상 가설(reward hypothesis)을 다룹니다.
- [Bellman (1957). 동적 프로그래밍](https://press.princeton.edu/books/paperback/9780691146683/dynamic-programming) — 벨만 방정식의 기원.
- [OpenAI Spinning Up — 파트 1: 핵심 개념](https://spinningup.openai.com/en/latest/spinningup/rl_intro.html) — 심층 강화 학습(deep-RL) 관점에서의 간결한 MDP 입문서.
- [Puterman (2005). 마르코프 결정 과정](https://onlinelibrary.wiley.com/doi/book/10.1002/9780470316887) — MDP 및 정확한 해법(exact solution methods)에 대한 운영 연구(operations-research) 참고서.
- [Littman (1996). 순차적 의사 결정 알고리즘(박사 학위 논문)](https://www.cs.rutgers.edu/~mlittman/papers/thesis-main.pdf) — 동적 프로그래밍(dynamic-programming) 특화로서의 MDP에 대한 가장 명확한 유도 과정.
