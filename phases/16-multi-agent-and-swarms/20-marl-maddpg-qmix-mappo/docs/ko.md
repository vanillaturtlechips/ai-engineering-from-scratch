# MARL — MADDPG, QMIX, MAPPO

> 2026년 LLM-에이전트 시스템에 여전히 영향을 미치는 다중 에이전트 조정의 강화 학습 유산. **MADDPG** (Lowe et al., NeurIPS 2017, arXiv:1706.02275)는 중앙 집중식 훈련, 분산 실행(CTDE)을 도입했습니다: 각 크리틱은 훈련 중 모든 에이전트의 상태와 행동을 관찰하며, 테스트 시에는 로컬 액터만 실행됩니다. 협력, 경쟁, 혼합 환경에서 작동합니다. **QMIX** (Rashid et al., ICML 2018, arXiv:1803.11485)는 단조 혼합 네트워크를 사용한 가치 분해로, 에이전트별 Q-값이 결합 Q-값으로 통합되어 `argmax`가 깨끗하게 분배됩니다 — 스타크래프트 다중 에이전트 챌린지(SMAC)에서 우세합니다. **MAPPO** (Yu et al., NeurIPS 2022, arXiv:2103.01955)는 중앙 집중식 가치 함수를 가진 PPO로, 입자 환경, SMAC, Google Research Football, Hanabi에서 최소한의 튜닝으로 "놀랍도록 효과적"입니다. 이들은 분산 실행이 필요한 에이전트 팀의 정책 훈련을 뒷받침합니다. MAPPO는 **2026년 협력형 MARL 기본 기준**입니다. 이 레슨은 작은 그리드 월드 장난감 환경에서 각 방법을 구축하고 LLM-에이전트 훈련에 들어가기 전에 세 가지 아이디어를 체화합니다.

**유형:** 학습
**언어:** Python (표준 라이브러리, NumPy 없는 소규모 구현)
**선수 지식:** 09단계(강화 학습), 16·09단계(병렬 스웜 네트워크)
**소요 시간:** ~90분

## 문제

LLM-에이전트 시스템은 점점 더 에이전트 간 협업을 위한 정책(policy)을 훈련시키고 있습니다: 언제 위임할지, 언제 행동할지, 어떤 동료에게 호출할지. 이러한 정책을 훈련시키는 방법을 알려주는 문헌은 다중 에이전트 강화 학습(MARL, Multi-Agent Reinforcement Learning)입니다. 이는 LLM(거대 언어 모델) 붐 이전에 등장했으며 소수의 주요 알고리즘들이 주류를 이루고 있습니다.

패턴 용어 없이 MARL 논문을 읽는 것은 고통스럽습니다. 중앙 집중식 훈련과 분산 실행(CTDE, Centralized Training with Decentralized Execution), 가치 분해(value decomposition), 중앙 집중식 비평가(centralized critics)는 유행어가 아니라 특정 문제에 대한 구체적인 해결책입니다:

- 독립 RL(각 에이전트가 혼자 학습)은 각 에이전트 관점에서 비정상(non-stationary)적입니다. 나쁨.
- 중앙 집중식 RL(하나의 에이전트가 모든 것을 제어)은 확장성이 떨어지며 실행 제약을 위반합니다.
- CTDE는 두 세계의 장점을 결합합니다: 전역 정보로 훈련하고, 지역 정책으로 배포합니다.

## 개념

### 논문에서 사용하는 세 가지 환경

- **Particle World (다중 에이전트 입자 환경).** 협력/경쟁 작업을 위한 간단한 2D 물리 환경. MADDPG의 원래 테스트 환경.
- **StarCraft Multi-Agent Challenge (SMAC).** 협력 미세 관리, 부분 관측. QMIX의 테스트 환경. 이산 행동, 연속 상태.
- **Google Research Football, Hanabi, MPE.** MAPPO 베이스라인.

각 환경은 서로 다른 행동/관측 유형을 가짐. 알고리즘은 이에 맞춰 선택됨.

### MADDPG (2017) — CTDE 패턴

각 에이전트 `i`는 자신의 관측값을 행동으로 매핑하는 액터 `mu_i(o_i)`를 가짐. 각 에이전트는 또한 훈련 중에 모든 관측값과 모든 행동을 보는 크리틱 `Q_i(x, a_1, ..., a_n)`을 가짐. 액터는 크리틱 평가에 대한 정책 기울기로 업데이트됨.

```
액터 업데이트:    grad_theta_i J = E[grad_theta mu_i(o_i) * grad_a_i Q_i(x, a_1..n) at a_i=mu_i(o_i)]
크리틱 업데이트:   다음 상태 결합 추정을 기반으로 Q_i(x, a_1..n)에 대한 TD
```

CTDE 이유: 훈련 시에는 모든 에이전트의 행동을 알 수 있음. 이를 사용해 각 크리틱의 분산을 줄임. 배포 시에는 각 에이전트가 `o_i`만 보고 `mu_i(o_i)`를 호출.

실패 모드: 크리틱은 에이전트 수 N에 따라 증가(모든 행동 포함). 근사 없이 10개 이상의 에이전트로 확장 불가.

### QMIX (2018) — 가치 분해

협력 전용. 글로벌 보상은 에이전트별 Q-값의 단조 함수의 합:

```
Q_tot(tau, a) = f(Q_1(tau_1, a_1), ..., Q_n(tau_n, a_n)),   df/dQ_i >= 0
```

단조성 보장은 `argmax_a Q_tot`가 각 에이전트가 독립적으로 `argmax_{a_i} Q_i`를 선택함으로써 계산 가능함을 의미. 이는 **분산 실행 속성**을 정확히 만족. 훈련 시에는 혼합 네트워크가 에이전트별 Q-값으로부터 `Q_tot`를 생성.

SMAC에서 QMIX가 성공하는 이유: 협력적인 스타크래프트 미세 관리는 동질적 에이전트, 지역 관측, 글로벌 보상을 가짐 — 가치 분해에 완벽히 적합.

실패 모드: 단조성 제약은 제한적. 일부 작업은 단조 분해가 불가능한 보상 구조를 가짐(한 에이전트가 팀을 위해 희생). 확장(QTRAN, QPLEX)은 이를 완화.

### MAPPO (2022) — 간과된 기본값

다중 에이전트 PPO: 중앙 집중식 가치 함수를 가진 PPO. 각 에이전트는 자체 정책을 가짐. 모든 에이전트는 전체 상태를 보는 (또는 에이전트별) 가치 함수를 공유. Yu et al. 2022는 5개 벤치마크에서 MAPPO를 MADDPG, QMIX 및 그 확장과 비교:

- MAPPO는 입자 세계, SMAC, Google Research Football, Hanabi, MPE에서 오프-정책 MARL 방법과 동등하거나 더 나은 성능.
- 최소한의 하이퍼파라미터 튜닝 필요.
- 안정적인 훈련; 시드 간 재현 가능.

이 논문까지 커뮤니티는 온-정책 MARL을 과소평가. 2026년에는 MAPPO가 협력 MARL의 기본 베이스라인; 새로운 방법은 이를 능가해야 함.

### LLM-에이전트 엔지니어가 관심을 가져야 하는 이유

세 가지 직접 활용:

1. **라우터 훈련.** 메타-에이전트가 어떤 하위-에이전트가 작업을 처리할지 선택. 이는 N개의 분산 하위-에이전트와 하나의 중앙 집중식 라우터를 가진 MARL 문제. MAPPO가 적합.
2. **역할 발생.** 생성 에이전트 시뮬레이션에서 시간이 지남에 따라 상호 보완적 역할을 채택하도록 에이전트를 훈련시키는 것은 MARL 문제. QMIX 스타일의 가치 분해는 구조적으로 상호 보완성을 강제.
3. **다중 에이전트 도구 사용.** 에이전트가 도구를 공유하고 예산을 경쟁할 때, CTDE를 통해 훈련하면 리소스 제약을 존중하는 배포 가능한 로컬 정책이 생성.

실용적 주의 사항: 2026년 대부분의 프로덕션 LLM-에이전트 시스템은 정책을 훈련시키기보다 프롬프트를 사용. MARL은 (a) 많은 상호작용 데이터, (b) 명확한 보상 신호, (c) 훈련 인프라 투자 의지가 있을 때 사용.

### RL을 넘어선 설계 패턴으로서의 CTDE

훈련 없이도 CTDE는 유용한 아키텍처 패턴:

- *설계* 중에는 전체 팀 가시성을 가정.
- *런타임*에는 분산 실행을 강제: 각 에이전트는 `o_i`만 봄.

이 패턴은 에이전트별 상태를 명시적으로 유지하고 부분 관측성을 사전에 고려하도록 강제. 많은 프로덕션 다중 에이전트 시스템은 모든 곳에서 공유 상태를 암묵적으로 가정 — CTDE 규율은 이를 방지.

### 비정상성 문제

여러 에이전트가 동시에 학습할 때, 각 에이전트의 환경(다른 에이전트의 정책 포함)은 비정상성. 고전적 단일 에이전트 RL 증명은 실패. 이 레슨의 MARL 알고리즘은 모두 이를 해결:

- MADDPG: 글로벌 크리틱이 모든 행동을 보므로 가치 추정은 정상성.
- QMIX: 가치 분해는 최적성이 잘 정의된 결합-Q 공간으로 학습을 이동.
- MAPPO: 중앙 집중식 가치 함수는 다른 정책 변경으로 인한 분산을 감소.

LLM-에이전트 시스템에서 비정상성은 "지난달에는 내 에이전트가 작동했는데, 이제 상류의 다른 에이전트가 변경되어 내 에이전트가 오작동"으로 나타남. CTDE로 MARL을 훈련하는 것이 원칙적 해결책; 프롬프트 수준의 수정은 더 빠르지만 덜 지속적.

### 이 레슨에서 다루지 않는 내용

실제 네트워크 훈련은 9단계 주제. 이 레슨은 그래디언트 업데이트 없이 CTDE, 가치 분해, 중앙 집중식 가치 패턴을 보여주는 스크립트 정책 버전을 구축. 목표는 전체 MARL 라이브러리(PyMARL, MARLlib, RLlib 다중 에이전트)를 사용하기 전에 패턴을 내재화하는 것.

## 빌드 및 실행

`code/main.py`는 2개의 에이전트가 협력하는 4x4 그리드 월드 환경에서 3가지 패턴 데모를 구현합니다:

- **환경**: 4x4 그리드에 2개의 에이전트와 1개의 보상 펠릿. 펠릿에 도달한 에이전트가 있으면 보상 = 1, 작업 종료.
- **`IndependentAgents`** — 각 에이전트는 다른 에이전트를 환경의 일부로 처리. 베이스라인.
- **`MADDPGStyle`** — 중앙 집중형 크리틱이 공동 가치 계산; 액터 정책은 이를 기반으로 업데이트. 스크립트 기반 정책 개선.
- **`QMIXStyle`** — 단조 믹서를 사용한 가치 분해.
- **`MAPPOStyle`** — 중앙 집중형 가치 함수; 정책은 공유 베이스라인에 대항해 업데이트.

4가지 방법 모두 동일한 에피소드를 실행하며 평균 목표 도달 단계 수를 보고합니다. CTDE(중앙 집중식 훈련-분산 실행) 변형들은 독립 베이스라인보다 더 짧은 경로로 수렴합니다.

실행:

```
python3 code/main.py
```

예상 출력: 독립 에이전트는 평균 ~6단계 소요; CTDE 변형들은 ~3.5단계로 수렴(4x4 그리드에서 최적 경로는 3단계). 스크립트 기반 정책에도 불구하고 패턴 차이가 나타납니다.

## 사용 방법

`outputs/skill-marl-picker.md`는 주어진 다중 에이전트 작업에 대해 MARL(Multi-Agent Reinforcement Learning) 알고리즘을 선택하는 스킬입니다. 다음 요소를 고려합니다:  
- 협력(cooperative) vs 경쟁(competitive)  
- 동종(homogeneous) vs 이종(heterogeneous)  
- 행동 공간(action-space) 유형  
- 규모(scale)  
- 보상 신호(reward signal)

## Ship It

실제 프로덕션 환경에서 MARL(Multi-Agent Reinforcement Learning) 적용은 드뭅니다. MARL을 사용할 때는 다음을 따르세요:

- **MAPPO로 시작하세요.** 2022년 논문에서 이 방법을 기준으로 확립했으며, 먼저 재현하면 더 복잡한 방법을 탐색하는 데 걸리는 시간을 몇 주 절약할 수 있습니다.
- **모든 에이전트의 관측(observation)과 행동(action) 스트림을 로깅하세요.** 에이전트별 추적 없이 MARL을 디버깅하는 것은 불가능합니다.
- **훈련 코드와 실행 코드를 분리하세요.** CTDE(Centralized Training with Decentralized Execution)는 규율이 필요합니다. 실행 경로에서는 실제로 `o_i`(에이전트의 관측값)만 볼 수 있도록 구성하세요.
- **보상 형성(Reward Shaping) 경고.** MARL은 보상 설계에 극도로 민감합니다. 보상 형성에 조율이 하나라도 잘못되면 에이전트가 이를 악용하는 방법을 학습합니다. 적대적 테스트를 실행하세요.
- **LLM(Large Language Model) 에이전트**의 경우, 먼저 프롬프트 수준의 정책을 고려하세요. 상호작용 데이터, 보상 신호, 인프라가 모두 갖춰졌을 때만 MARL 훈련에 투자하세요.

## 연습 문제

1. `code/main.py`를 실행하세요. 6x6 그리드에서 독립형(Independent) 에이전트와 MAPPO 스타일 에이전트 간의 목표 달성 단계 차이를 측정하세요. 이 차이는 커지나요, 아니면 줄어드나요?
2. 경쟁 변형(competitive variant)을 구현하세요: 두 에이전트, 하나의 펠릿(pellet), 먼저 도달한 에이전트만 보상을 받습니다. 어떤 패턴이 경쟁을 깔끔하게 처리하나요? 역사적으로 MADDPG가 이 역할을 했습니다.
3. MADDPG(arXiv:1706.02275) 3절을 읽으세요. 의사코드(pseudocode)로 정확한 크리틱(critic) 업데이트 규칙을 기호적으로 구현하세요. (자신의 표현으로)
4. MAPPO(arXiv:2103.01955)를 읽으세요. 저자들이 왜 중앙집중식 가치(centralized value) + PPO가 오프라인 정책(MARL) 벤치마크에서 더 우수하다고 주장하는지 설명하세요. 가장 강력한 세 가지 주장을 나열하세요.
5. CTDE(중앙집중식 훈련 + 분산 실행)를 가상의 LLM-에이전트 시스템(예: 연구 에이전트 + 요약기 + 코더)에 디자인 패턴으로 적용하세요. 설계 시점에 사용 가능하지만 런타임에는 사용할 수 없는 공동 정보(joint information)는 무엇인가요?

| MADDPG | "Multi-Agent DDPG" | 모든 관측값 + 행동을 보는 에이전트별 크리틱을 사용하는 CTDE. |
| QMIX | "Value decomposition" | 에이전트별 Q값의 단조 혼합. 협력형. |
| MAPPO | "Multi-Agent PPO" | 중앙 집중식 가치 함수를 사용하는 PPO. 2026년 기본 베이스라인. |
| Value decomposition | "개별 Q값의 합" | 에이전트별 Q값의 단조 함수로 표현되는 공동 Q값. |
| Non-stationarity | "움직이는 목표" | 다른 에이전트가 학습함에 따라 각 에이전트의 환경이 변화. MARL의 핵심 문제. |
| On-policy / off-policy | "현재 데이터 학습 / 재생 데이터 학습" | PPO는 온-폴리시(MAPPO); DDPG와 Q-러닝은 오프-폴리시. |
| SMAC | "StarCraft Multi-Agent Challenge" | 협력형 미세 관리 벤치마크; QMIX의 자체 개발 환경.
