# 투표, 자기 일관성, 그리고 토론 토폴로지

> 가장 저렴한 집계 방법: N개의 독립적인 에이전트를 샘플링하고 다수결 투표. Wang et al. 2022의 자기 일관성은 하나의 모델을 N번 샘플링하여 이를 수행했습니다. 다중 에이전트 확장은 **이질적인** 에이전트를 사용하여 단일 문화(monoculture)를 탈출합니다 — 서로 다른 모델, 다른 프롬프트, 다른 온도, 다른 컨텍스트. 다수결 투표를 넘어서 토론 토폴로지가 중요합니다: MultiAgentBench (arXiv:2503.01935, ACL 2025)는 스타/체인/트리/그래프 조정을 평가하고 **그래프가 연구에 가장 적합**하며, 약 4개 이상의 에이전트에서는 "조정 비용(coordination tax)"이 발생한다고 밝혔습니다. AgentVerse (ICLR 2024)는 두 가지 발생 패턴 — 자원 행동(volunteer behaviors)과 순응 행동(conformity behaviors) — 을 문서화하며, 순응은 특징(합의 도출)이자 위험(집단 사고, 레슨 24)입니다. 이 레슨은 토폴로지 공간을 매핑하고 각 변형을 구축하며 조정 비용을 측정합니다.

**유형:** 학습 + 구현  
**언어:** Python (표준 라이브러리)  
**선수 조건:** 16단계 · 07 (마음의 사회와 토론), 16단계 · 14 (합의와 BFT)  
**소요 시간:** ~75분

## 문제

토론은 정확도를 향상시킬 수 있습니다(Du et al., arXiv:2305.14325). 하지만 정확도를 저하시킬 수도 있습니다. 토론이 도움이 되는지 여부는 다음 네 가지 구조적 선택에 따라 달라집니다:

1. 누가 누구와 대화하는지(구조).
2. 몇 라운드인지(Du 2023: 라운드 수와 에이전트 수가 독립적으로 중요).
3. 에이전트가 이질적인지(서로 다른 기본 모델이 단일 문화(monoculture)를 깨뜨림).
4. 적대적 목소리가 있는지(강하게 반박(steel-manning) vs. 허수아비 공격(straw-manning)).

"5개 에이전트를 실행하고 투표하는" 방식을 작업에 무리하게 적용하는 팀은 종종 단일 에이전트 대비 성능이 저하됩니다. 이러한 실패는 무작위가 아닙니다. 구조(구조)와 이질성(heterogeneity)에 따라 발생합니다. 이 교훈이 바로 구조 지도(topology map)입니다.

## 개념

### 자기 일관성, 단일 모델 기준선

Wang et al. 2022("Self-Consistency Improves Chain of Thought Reasoning")는 동일한 모델을 온도 > 0에서 N회 샘플링하고 추론 경로 답변에 대해 다수결 투표를 수행했습니다. GSM8K에서의 결과: 단일 탐욕적 디코딩 대비 N=40 샘플로 상당한 성능 향상. 자기 일관성은 다중 에이전트 투표의 단일 에이전트 전구체입니다.

한계: 자기 일관성은 하나의 기본 모델을 사용합니다. 오류는 구조적으로 상관관계가 있습니다. 모델에 체계적 편향이 있다면 모든 N 샘플이 이를 공유합니다.

### 다중 에이전트 투표, 이질적 확장

N개의 샘플을 N개의 *서로 다른* 에이전트로 대체합니다. 다른 기본 모델(Claude, GPT, Llama), 다른 프롬프트, 다른 도구 접근. 이점: 비상관 오류. 비용: 다른 에이전트는 비용이 다르며, 조정 시 오버헤드가 추가됩니다.

이질적 토론의 2026년 표준 이름은 **A-HMAD**(Adversarial Heterogeneous Multi-Agent Debate)입니다. 보편적으로 채택되지는 않았지만, 논문들은 "서로 다른 모델들이 토론하여 단일 문화 붕괴의 상관 오류를 줄이는" 경우에 이 용어를 사용합니다.

### 네 가지 토폴로지

```
star                chain               tree                graph

    ┌─A─┐           A─B─C─D         ┌──A──┐              A───B
    │   │                           │     │              │ × │
    B   C                           B     C              D───C
    │   │                          / \   / \
    D   E                         D   E F   G           (완전 연결)
```

별(Star): 하나의 허브, 나머지는 허브와만 통신. 역방향 채널 없는 감독자-작업자 구조와 동등합니다.  
체인(Chain): 선형, 각 에이전트는 이전 에이전트의 출력을 확인. 파이프라인 유사.  
트리(Tree): 계층적, 계층적 에이전트 시스템(레슨 06)에서 사용.  
그래프(Graph): 임의 연결. 완전 연결 클리크 및 임의 DAG 포함.

### 조정 비용 (MultiAgentBench)

MultiAgentBench(MARBLE, ACL 2025, arXiv:2503.01935)는 연구, 코딩, 계획 작업을 포함한 작업 세트에서 별, 체인, 트리, 그래프 토폴로지를 벤치마킹했습니다. 주요 측정 결과:

- **그래프** 토폴로지는 연구 작업에서 우승. 정보가 양방향으로 흐르며 에이전트들이 서로 비평할 수 있습니다.  
- **별** 토폴로지는 빠른 답변 사실 작업에서 우승. 허브가 필터링 및 통합합니다.  
- **체인** 토폴로지는 단계적 파이프라인(단계별 개선)에서 우승.  
- **조정 비용**은 그래프 토폴로지에서 에이전트 수 ~4개 이후부터 나타납니다. 벽시계 시간과 토큰 비용이 품질보다 빠르게 증가합니다.

4개 에이전트 한계는 경험적이며 근본적이지 않습니다. 2026년 LLM 컨텍스트 용량을 반영합니다: 각 에이전트의 컨텍스트는 동료들의 출력으로 채워지며, 에이전트 N+1 추가의 한계 효용은 모든 에이전트가 서로를 볼 수 있게 된 후 감소합니다.

### 다중 에이전트 토론 전략("Should we be going MAD?")

arXiv:2311.17371은 2023년 MAD 전략 조사 논문입니다. 다른 연구들에서 재현된 주요 결과: 자기 일관성과 *구조적으로 유사한* MAD 변형(독립 샘플링 + 집계)은 동일한 예산 사용 시 자기 일관성보다 성능이 낮은 경우가 많습니다. MAD는 에이전트들이 진정으로 이질적이고 토론에 적대적 구조(한 에이전트가 반대 주장)가 있을 때 가장 도움이 됩니다.

### 에이전트버스(AgentVerse)의 발생적 패턴

에이전트버스(ICLR 2024, https://proceedings.iclr.cc/paper_files/paper/2024/file/578e65cdee35d00c708d4c64bce32971-Paper-Conference.pdf)는 명시적 설계 없이도 다중 에이전트 토론에서 발생하는 두 가지 행동을 문서화했습니다:

- **자원자(Volunteer).** 에이전트가 요청 없이 도움을 제공합니다("다음 단계를 수행할 수 있습니다"). 유용: 하위 작업에 가장 능력 있는 에이전트에게 작업을 할당합니다.  
- **순응(Conformity).** 에이전트가 비평자와 입장을 맞추려 조정합니다. 비평자가 틀렸을 때도 발생합니다. 이는 토론에서의 아첨(레슨 14)에 해당합니다.

순응은 합의까지 토론 시 괴롭히는 에이전트에게 보상을 주는 이유입니다. 별도의 판사와 함께 라운드 수를 제한하면 완화됩니다.

### 이질성: 정확도를 움직이는 실제 조절 장치

2024-2026년 실무 문헌의 패턴: N개 에이전트 중 하나를 다른 기본 모델로 교체하는 것이 N을 1 증가시키는 것보다 더 큰 정확도 향상을 제공합니다. 직관: 단일 문화 — 각 새로운 독립 오류 소스는 추가 상관 샘플보다 가치가 있습니다.

극단적으로 이질성이 수적 우위를 이깁니다. 세 가지 다른 모델은 대부분의 깨끗한 정답 작업(clean ground truth)에서 하나의 모델 5개 복사본보다 성능이 좋습니다.

### 배심원 방법

시빌(Sibyl) 프레임워크(밍스키-LLM 문헌에서 인용)는 "배심원"을 공식화합니다. 각 단계에서 투표로 답변을 개선하는 소규모 전문 에이전트 집합입니다. 일반 다수결 투표와 달리 배심원은 역할을 가집니다: 한 에이전트는 반대 심문, 한 에이전트는 맥락 제공, 한 에이전트는 타당성 평가. 배심원 방법은 일반 투표(저렴하지만 단일 문화 경향)와 완전 MAD(비싸지만 순응 경향)의 중간 지점입니다.

### 토론-투표가 지배적인 경우

- 질문에 정답(사실, 수학, 코드 동작)이 있습니다. 투표 수렴이 의미 있습니다.  
- 에이전트들이 서로 다른 소스나 도구에 접근할 수 있습니다(이질성 가능).  
- 라운드가 제한됩니다(2-3회 일반적) 그리고 별도의 판사 또는 검증자가 있습니다.  
- 예산이 3-5개 에이전트를 허용합니다. 그래프 토폴로지에서 5-7개 이상 시 조정 비용이 지배적입니다.

### 토론-투표가 해로운 경우

- 질문이 의견 형태입니다. 에이전트들은 가장 확신하는 답변으로 수렴하며, 가장 정확한 답변으로 수렴하지 않습니다.  
- 모든 에이전트가 기본 모델을 공유합니다. 단일 문화로 인해 합의가 무의미합니다.  
- 라운드가 무제한입니다. 순응이 항상 승리합니다.  
- 작업이 단순합니다. N=5의 자기 일관성을 가진 단일 에이전트가 더 저렴하고 정확합니다.

## 구축 방법

`code/main.py`는 다음을 구현합니다:

- `run_star(agents, hub, question)` — 허브가 각 작업자를 폴링하고 집계합니다.
- `run_chain(agents, question)` — 순차적 정제(sequential refinement).
- `run_tree(root, children, question)` — 깊이 2의 계층적 집계(hierarchical with depth-2 aggregation).
- `run_graph(agents, question, rounds)` — 모든 에이전트 간 토론(all-to-all debate), 제한된 라운드.
- 각 에이전트의 체계적 오류(error_bias)를 나타내는 사전 구성된 이질성 다이얼(scripted heterogeneity dial).
- N=3, 5, 7에서 각 토폴로지를 실행하고 (정확도, 총 토큰 수, 시뮬레이션된 벽시계 시간)을 보고하는 측정 하니스(measurement harness).

실행 방법:

```
python3 code/main.py
```

예상 출력: 토폴로지 × N → (정확도, 토큰 수, 지연 시간) 표. 그래프 방식은 연구 스타일 작업에서 N=3-5일 때 승리하며, 스타 방식은 빠른 사실 확인 작업에서 승리합니다. N=7에서 그래프 방식은 조정 비용(coordination tax)을 보여줍니다(정확도보다 지연 시간이 더 빠르게 증가).

## 사용 방법

`outputs/skill-topology-picker.md`는 작업 설명을 읽고 토폴로지(스타/체인/트리/그래프), N(에이전트 수), 이질성 프로파일(사용할 기본 모델), 라운드 상한을 추천하는 스킬입니다.

## Ship It

모든 앙상블에 대해:

- **N=5에서 자기 일관성(self-consistency)**을 강력한 기본 모델 하나로 시작하세요. 이는 저렴한 기준선입니다.
- 정확도가 중요하다면 **N=3에서 이질적 투표(heterogeneous voting)**로 업그레이드하세요. 변화량(delta)을 측정하세요.
- **토론 토폴로지(debate topology)**는 작업에 구조(연구, 다단계)가 있고 제한된 라운드가 가능한 경우에만 업그레이드하세요.
- 항상 소수 클러스터(minority cluster)를 기록하세요. 소수 집단이 지속적으로 옳다면 다양성 신호(diversity signal)가 있는 것입니다.
- 정확도뿐만 아니라 **벽시계 시간(wall-clock)**과 **토큰 수(tokens)**도 벤치마크하세요. "10배 비용에서 더 나은 정확도"는 비즈니스 결정 사항입니다.

## 연습 문제

1. `code/main.py`를 실행하세요. 그래프 토폴로지에 대한 조정-세금 곡선을 플롯하세요: 정확도 vs N, 토큰 vs N. 곡선이 변하는 지점은 어떤 N에서 발생하나요?
2. A-HMAD를 구현하세요: 세 에이전트가 의도적으로 서로 다른 편향(bias)을 갖도록 구성하세요. 모든 편향이 동일한 기준선(baseline)과 비교하여, 레슨 14의 단일문화 공격(monoculture attack)에서 A-HMAD는 어떻게 다른가요?
3. 그래프 토폴로지에 "심사자(judge)" 역할을 추가하세요. 이 역할은 투표하지 않고 최종 합의(consensus)에만 점수를 매깁니다. 이렇게 하면 발생하는 순응 행동(emergent conformity behavior)이 달라지나요?
4. AgentVerse 논문(ICLR 2024)을 읽으세요. 구현에서 가장 강하게 나타나는 발생적 행동(emergent behavior)은 무엇인가요? 프롬프트 변경으로 반대 행동을 유도할 수 있나요?
5. MultiAgentBench 논문(arXiv:2503.01935) 4절(토폴로지 실험)을 읽으세요. 하니스(harness)를 사용하여 논문 내 한 작업에서 "그래프-승리-연구(graph-wins-research)" 결과를 재현하세요.

## 주요 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|----------------|------------------------|
| 자기 일관성(Self-consistency) | "N번 샘플링, 투표" | Wang 2022. 단일 모델에서 N개의 temperature>0 샘플 생성, 추론 경로에 대한 다수결 투표. |
| 이질성(Heterogeneity) | "다른 모델들" | 서로 다른 기본 모델 또는 프롬프트 패밀리의 앙상블. 단일 문화(monoculture)를 깨뜨림. |
| MAD | "다중 에이전트 토론" | 에이전트들이 여러 라운드에 걸쳐 비평을 교환하는 일반적인 용어. Du 2023 참조. |
| A-HMAD | "적대적 이질성 MAD" | 서로 다른 모델 + 적대적 구조를 강조하는 MAD 변형. |
| 토폴로지(Topology) | "누가 누구와 대화하는가" | 스타, 체인, 트리, 그래프. 정보 흐름 결정. |
| 조정 비용(Coordination tax) | "감소하는 수익" | 그래프 상에서 ~4개 이상의 에이전트부터 비용이 품질보다 빠르게 증가. |
| 자원봉사 행동(Volunteer behavior) | "자발적 도움" | AgentVerse에서 나타나는 패턴: 에이전트가 스스로 단계를 수행하겠다고 제안. |
| 순응 행동(Conformity behavior) | "압박 하의 동의" | AgentVerse에서 나타나는 패턴: 에이전트가 비평자와 의견을 맞춤. |
| 배심원(Jury) | "소규모 전문 패널" | 역할(검사자, 컨텍스트, 평가자)이 있는 Sibyl 스타일 앙상블. |
