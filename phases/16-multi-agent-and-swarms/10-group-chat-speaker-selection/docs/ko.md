# 그룹 채팅과 화자 선택

> AutoGen GroupChat과 AG2 GroupChat은 N명의 에이전트 간에 하나의 대화를 공유합니다. 선택기 함수(LLM, 라운드 로빈, 또는 커스텀)가 다음에 말할 사람을 선택합니다. 이는 에머전트 다중 에이전트 대화의 원형입니다. 에이전트들은 정적 그래프에서의 역할을 알지 못하며, 공유된 풀에 반응만 합니다. AutoGen v0.2의 GroupChat 의미론은 AG2 포크에서 보존되었습니다. AutoGen v0.4는 이벤트 기반 액터 모델로 재작성했습니다. Microsoft는 2026년 2월 AutoGen을 유지보수 모드로 전환하고 Semantic Kernel과 병합하여 Microsoft 에이전트 프레임워크(RC 2026년 2월)를 출시했습니다. GroupChat 프리미티브는 AG2와 Microsoft 에이전트 프레임워크 모두에서 살아남았습니다. 한 번 배우면 어디서나 사용할 수 있습니다.

**유형:** 학습 + 구축  
**언어:** Python (stdlib)  
**사전 요구 사항:** 16단계 · 04 (프리미티브 모델)  
**소요 시간:** ~60분

## 문제

정적 그래프(LangGraph)는 워크플로우가 알려져 있을 때 훌륭합니다. 하지만 실제 대화는 정적이 아닙니다: 때로는 코더가 리뷰어에게, 때로는 연구자가, 때로는 작가가 다른 사람에게 말을 걸기도 합니다. 모든 가능한 핸드오프를 하드코딩하면 엣지 폭발(edge explosion)이 발생합니다. 공유 풀에 반응하는 에이전트들이 있고, 다음 발화자를 결정하는 어떤 함수가 있는 방식을 원합니다.

바로 AutoGen GroupChat이 이를 정확히 구현합니다.

## 개념

### 형태

```
              ┌─── 공유 풀 ────┐
              │   m1  m2  m3  ...  │
              └─────────┬──────────┘
                        │ (모두가 모든 메시지 읽음)
      ┌───────┬─────────┼─────────┬───────┐
      ▼       ▼         ▼         ▼       ▼
    에이전트 A  에이전트 B  에이전트 C  에이전트 D  선택기
                                           │
                                           ▼
                                  "다음 발화자 = C"
```

모든 에이전트는 모든 메시지를 볼 수 있습니다. 각 턴마다 선택기 함수가 호출되어 다음 발화자를 선택합니다.

### 세 가지 선택기 유형

**라운드 로빈.** 고정된 순환 방식. 결정론적. N에 따라 선형적으로 확장되지만 컨텍스트를 무시함 — 주제가 법률 검토일 때도 코더에게 턴이 주어질 수 있음.

**LLM 선택.** 최근 풀을 읽고 최적의 다음 발화자를 반환하는 LLM 호출. 컨텍스트 인식이 가능하지만 느림: 매 턴마다 LLM 호출이 추가됨. AutoGen의 기본값.

**커스텀.** 원하는 로직을 가진 Python 함수. 일반적: LLM 선택에 폴백 규칙 추가 (예: "코더 다음에는 항상 검증자에게 턴 부여").

### ConversableAgent API

```
agent = ConversableAgent(
    name="코더",
    system_message="당신은 Python을 작성합니다.",
    llm_config={...},
)
chat = GroupChat(agents=[코더, 검토자, 테스터], messages=[])
manager = GroupChatManager(groupchat=chat, llm_config={...})
```

`GroupChatManager`는 선택기를 보유합니다. 에이전트가 턴을 완료하면 매니저가 선택기를 호출하여 다음 에이전트를 반환합니다. 종료 조건이 충족될 때까지 루프가 계속됩니다.

### 종료

세 가지 일반적인 패턴:

- **최대 라운드.** 총 턴 수에 대한 하드 캡.
- **"TERMINATE" 토큰.** 에이전트가 센티넬 메시지를 방출할 수 있음; 매니저는 이 메시지가 나타나면 중지.
- **목표 달성 확인.** 가벼운 검증기가 매 턴 실행되며 완료되면 채팅을 중지.

### AutoGen → AG2 분할과 Microsoft Agent Framework 통합

2025년 초, Microsoft는 이벤트 기반 액터 모델을 중심으로 AutoGen(v0.4)을 대대적으로 재작성했습니다. 커뮤니티는 AutoGen v0.2의 GroupChat 의미론을 AG2로 포크하여 초기 채택자들이 통합한 API를 보존했습니다.

2026년 2월, Microsoft는 AutoGen이 유지보수 모드로 전환되고 이벤트 기반 액터 모델이 **Microsoft Agent Framework**(2026년 2월 RC, 현재 Semantic Kernel과 통합)로 통합될 것이라고 발표했습니다. GroupChat 개념은 두 트랙에서 모두 유지되지만 구현 세부사항은 다릅니다. AG2는 v0.2 호환 코드의 선호되는 업스트림입니다.

### GroupChat이 적합한 경우

- **발생적 대화.** 모든 가능한 다음 발화자를 미리 연결하지 않으려 함.
- **역할 혼합 작업.** 코더가 연구자에게, 연구자가 기록 보관자에게, 기록 보관자가 다시 코더에게 질문. 흐름이 DAG가 아님.
- **탐색적 문제 해결.** "조립 라인"이 아닌 "브레인스토밍 회의"를 생각.

### 실패하는 경우

- **엄격한 결정론.** LLM 선택기는 일관성이 없을 수 있음. 동일한 프롬프트라도 실행마다 다른 다음 발화자가 선택될 수 있음.
- **아첨 캐스케이드.** 에이전트가 가장 자신 있게 발언한 사람에게 양보. 명시적으로 반-프롬프트 사용.
- **컨텍스트 팽창.** 모든 에이전트가 모든 메시지를 읽음; 10턴 후에는 컨텍스트가 매우 커짐. 프로젝션(레슨 15)을 사용하여 뷰를 범위 지정.
- **핫 스피커.** 선택기가 특정 에이전트의 전문 분야를 선호하여 한 에이전트가 대화를 지배. 선택기 기능으로 발화자 균형 도입.

### 그룹 채팅 vs 감독자

동일한 기본 요소, 다른 기본값:

- 감독자: 한 에이전트가 계획하고 다른 에이전트가 실행. 선택기는 "계획자에게 무엇을 할지 묻기".
- 그룹 채팅: 모든 에이전트가 피어; 선택기는 공유 풀에 대한 함수.

둘 다 레슨 04의 네 가지 기본 요소를 사용합니다. 그룹 채팅은 LLM 선택 오케스트레이션과 전체 풀 공유 상태를 기본으로 합니다.

## 빌드하기

`code/main.py`는 표준 라이브러리(stdlib)를 사용하여 GroupChat을 처음부터 구현합니다. 세 명의 에이전트(coder, reviewer, manager), 라운드 로빈(round-robin) 및 LLM 선택(LLM-selected) 변형, 그리고 `TERMINATE` 토큰에 의한 종료 기능이 포함되어 있습니다.

데모는 대화 기록과 두 변형에 대한 선택기(selector)의 결정 추적을 출력합니다.

실행 방법:

```
python3 code/main.py
```

## 사용 방법

`outputs/skill-groupchat-selector.md`는 주어진 작업(라운드 로빈 vs LLM 선택 vs 사용자 정의)에 대한 그룹 채팅 선택기를 구성하며, 어떤 선택기 입력(최근 메시지, 에이전트 전문 분야, 턴 횟수)을 사용할지 정의합니다.

## Ship It

체크리스트:

- **최대 라운드 제한.** 항상 적용. 일반적인 작업의 경우 10-20회.
- **화자 균형 지표.** 에이전트별 발화 횟수 추적; 불균형이 임계값을 초과할 경우 경고.
- **종료 토큰.** `TERMINATE` 또는 전용 검증 에이전트.
- **투영 또는 범위 기반 메모리.** 약 10개 메시지 이후, 컨텍스트 과부하 방지를 위해 각 에이전트에게 범위 제한된 뷰 제공 고려.
- **선택기 로깅.** LLM 선택 변형체의 경우, 선택기의 입력과 선택 사항 모두 기록. 그렇지 않으면 디버깅이 불가능함.

## 연습 문제

1. `code/main.py`를 실행해 보세요. 라운드 로빈(round-robin) 방식과 LLM 선택(LLM-selected) 방식에서의 대화를 비교해 보세요. 각 방식에서 어떤 에이전트가 주도권을 가지나요?
2. 선택기(selector)에 "에이전트당 최대 발화 횟수(max-speaks-per-agent)" 규칙을 추가해 보세요. 이것이 대화 기록(transcript)에 어떤 영향을 미치나요?
3. 목표 달성 종료 조건(goal-reached termination)을 구현해 보세요: 검토자(reviewer)가 "approved"를 반환하면 종료되도록 합니다. 라운드 제한(round cap) 전에 이 조건이 얼마나 자주 발동하나요?
4. AutoGen 안정 버전 문서(https://microsoft.github.io/autogen/stable/user-guide/core-user-guide/design-patterns/group-chat.html)의 그룹 채팅(GroupChat) 섹션을 읽어보세요. `GroupChatManager`에서 사용하는 기본 선택기(default selector)는 무엇인가요?
5. AG2 저장소(https://github.com/ag2ai/ag2)를 읽고 v0.2 그룹 채팅과 v0.4 이벤트 기반 버전을 비교해 보세요. v0.4가 추가한 구체적인 특성(처리량(throughput), 장애 허용성(fault-tolerance), 조합 가능성(composability))은 무엇인가요?

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|----------------|------------------------|
| GroupChat | "한 채팅방에 있는 에이전트들" | 공유 메시지 풀 + 선택자 함수. AutoGen / AG2 기본 구성 요소. |
| Speaker selection | "누가 다음에 말할지" | 다음 에이전트를 선택하는 함수. 라운드 로빈, LLM 선택, 또는 커스텀 방식. |
| GroupChatManager | "회의 진행자" | 선택자를 소유하고 턴을 반복하는 AutoGen 구성 요소. |
| ConversableAgent | "기본 에이전트" | AutoGen 기본 클래스; 메시지를 송수신할 수 있는 에이전트. |
| Termination token | "'중지' 단어" | 채팅을 종료하는 센티넬 문자열(보통 `TERMINATE`). |
| Hot speaker | "한 에이전트가 독점" | 선택자가 계속 같은 에이전트를 선택하는 실패 모드. |
| Context bloat | "풀의 무제한 증가" | 각 에이전트가 모든 이전 메시지를 읽음; 턴이 증가할수록 컨텍스트 확장. |
| Projection | "범위 지정 뷰" | 컨텍스트 폭증을 방지하기 위한 역할별 공유 풀 뷰.
