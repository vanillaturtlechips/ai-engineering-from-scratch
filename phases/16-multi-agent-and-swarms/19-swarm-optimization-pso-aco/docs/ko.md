# LLM을 위한 군집 최적화 (PSO, ACO)

> 생체 영감 최적화가 LLM 분야에서 다시 주목받고 있습니다. **LMPSO** (arXiv:2504.09247)는 각 입자의 속도를 프롬프트로 사용하고 LLM이 다음 후보를 생성하는 PSO를 적용하며, 구조화된 시퀀스 출력(수학 표현식, 프로그램 등)에서 우수한 성능을 보입니다. **Model Swarms** (arXiv:2410.11163)는 각 LLM 전문가를 모델-가중치 매니폴드 상의 PSO 입자로 취급하며, 9개 데이터셋에서 12개 베이스라인 대비 **평균 13.3% 성능 향상**을 200개 인스턴스만으로 달성했습니다. **SwarmPrompt** (ICAART 2025)는 프롬프트 최적화를 위해 PSO와 Grey Wolf를 혼합합니다. **AMRO-S** (arXiv:2603.12933)는 다중 에이전트 LLM 라우팅을 위한 ACO 영감 페로몬 전문가 시스템으로, **4.7배 속도 향상**, 해석 가능한 라우팅 증거, 추론과 학습을 분리하는 품질-게이트 비동기 업데이트를 제공합니다. 이 강의에서는 프롬프트 매개변수 공간에 PSO를 적용하고 에이전트 라우팅에 ACO를 구현하며, 이러한 고전적 알고리즘이 LLM 시대에 적합한 이유와 한계를 분석합니다.

**유형:** 학습 + 구현  
**언어:** Python (표준 라이브러리)  
**선수 지식:** 16단계 · 09 (병렬 군집 네트워크), 16단계 · 14 (합의 및 BFT)  
**소요 시간:** ~75분

## 문제

프롬프트가 작업 평가에서 62%의 점수를 기록했습니다. 이를 개선하고 싶습니다. 순진한 접근 방식은 기울기(gradient) 없는 수동 조정이지만, 이는 확장성이 매우 떨어집니다. 강화 학습(Reinforcement Learning)은 보상 신호(reward signal)와 충분한 롤아웃(rollout)이 필요합니다. 프롬프트를 통한 역전파(backpropagation)는 실제로 불가능합니다. 프롬프트는 이산적인 문자열(discrete string)이며, 미분 가능한 매개변수(differentiable parameter)가 아니기 때문입니다.

고전적인 생체 영감 최적화 기법(PSO: 연속 탐색 공간, ACO: 경로 선택)은 정확히 이러한 환경을 위해 설계되었습니다. 기울기(gradient) 없이, 개체군 기반(population-based)으로, 평가 비용이 저렴한 방식입니다. 이를 LLM과 결합하여 기울기 없는 탐색 단계를 수행하면, 놀랍도록 실용적인 최적화기를 얻을 수 있습니다.

동일한 패턴이 다중 에이전트 시스템의 에이전트 *라우팅*에도 적용됩니다. ACO 스타일의 페로몬 트레일(pheromone trail)은 어떤 에이전트가 어떤 작업 유형에서 가장 잘 수행했는지 기록하고, 라우터가 이 트레일을 활용하도록 하며, 페로몬이 감쇠되도록 해 경로가 재발견될 수 있게 합니다.

## 개념

### PSO 복습 (Kennedy & Eberhart 1995)

입자 군집 최적화(Particle Swarm Optimization): 연속 검색 공간에 있는 입자들의 집단. 각 입자는 위치 `x_i`와 속도 `v_i`를 가짐. 매 반복:

```
v_i <- w * v_i + c1 * r1 * (p_best_i - x_i) + c2 * r2 * (g_best - x_i)
x_i <- x_i + v_i
evaluate fitness(x_i)
update p_best_i if improved
update g_best if global best
```

여기서 `p_best`는 입자 자신의 최적값, `g_best`는 군집의 최적값, `w, c1, c2`는 관성 + 인지 + 사회적 가중치, `r1, r2`는 무작위 요소.

### LLM 출력에 대한 PSO — LMPSO

arXiv:2504.09247은 LLM 생성 구조 출력(수학 표현식, 프로그램)에 PSO를 적용. 각 입자는 후보 출력. 속도는 현재 출력을 개인/군집 최적값으로 수정하는 방법을 설명하는 *프롬프트*. LLM은 속도 프롬프트로부터 새 출력을 생성. 속도의 "관성"은 "작은 증분 변경을 수행하라"와 같은 프롬프트.

다음 조건에서 잘 작동:
- 출력이 구조적(파싱 가능, 평가 가능).
- 적합도 평가 자동화(테스트 실행, 산술 평가).
- 인구 규모 작음(~10-30 입자) → 총 LLM 호출 관리 가능.

적합도 평가에 인간 검토가 필요할 때는 잘 작동하지 않음 — 반복당 비용 과도.

### 모델 군집

arXiv:2410.11163은 PSO를 출력 계층에서 *모델* 계층으로 이동. 각 "입자"는 전문가 LLM(파라미터). 군집은 경사 없는 업데이트를 통해 파라미터를 집단 최적값으로 이동. 보고: 9개 데이터셋에서 12개 베이스라인 대비 평균 13.3% 향상, 반복당 200개 인스턴스만 사용.

핵심 통찰: LLM 전문가 모델은 이미 공유 파라미터 매니폴드(어댑터 가중치, LoRA 델타) 근처에 있음. 이 저차원 부분공간에서의 PSO는 저렴하고 효과적.

### ACO 복습 (Dorigo 1992)

개미 군집 최적화(Ant Colony Optimization): 개미가 그래프 탐색; 각 경로에는 페로몬 흔적. 개미 이동 확률은 페로몬 강도에 가중치. 작업 완료 개미는 솔루션 품질에 비례해 페로몬을 침적. 페로몬은 시간에 따라 감소.

### AMRO-S — 에이전트 라우팅을 위한 ACO

arXiv:2603.12933은 다중 에이전트 라우팅에 ACO를 사용. 각 작업 유형은 "목적지"; 각 에이전트는 가능한 경로. 좋은 출력을 생성하는 경로는 페로몬 강화. 주요 기여:

- **해석 가능한 라우팅 증거.** 페로몬 강도는 인간이 읽을 수 있는 신호.
- **품질 게이트 비동기 업데이트.** 품질 검사 통과 후 페로몬 업데이트, 추론과 학습 분리.
- **4.7배 속도 향상** 다중 에이전트 라우팅 벤치마크.

품질 게이트 중요: 없으면 빠르지만 부정확한 에이전트가 페로몬을 축적, 시스템이 나쁜 경로에 고정.

### LLM에 PSO/ACO 사용 시기

**PSO 사용 시기:**
- 검색 공간이 연속적이거나 연속 파라미터에 매핑(프롬프트 임베딩, LoRA 가중치, 수치 생성 파라미터).
- 적합도 평가가 저렴하고 자동화.
- 인구 규모 작음(10-30).

**ACO 사용 시기:**
- 라우팅 또는 경로 선택 문제.
- 결정이 시간에 따라 강화(동일 작업 유형 반복).
- 라우팅 결정에 대한 해석 가능한 증거 필요.

**둘 다 사용하지 않을 시기:**
- 적합도 평가에 인간 검토 필요(반복당 비용 과도).
- 검색 공간이 이산적이고 조합적이며 PSO가 커버하지 않는 경우(유전 알고리즘 사용).
- 실시간 결정에 엄격한 지연 시간 필요(PSO/ACO는 단일 패스 휴리스틱 대비 수렴 느림).

### 생체 영감 방식이 여전히 승리하는 이유

경사 기반 방법은 미분 가능한 신호 필요. LLM 출력과 라우팅 결정은 자명하게 미분 불가능. 의사 경사 방법(강화 학습 라우터, DPO 스타일 프롬프트 튜너)은 작동하지만 비싼 훈련 필요.

PSO와 ACO는 *평가자* 함수만 필요. 후보 출력이나 라우팅 결정을 점수화할 수 있다면 공간 최적화 가능. 이는 적용 가능성 기준을 훨씬 낮춤.

### 실용적 한계

- **인구 예산.** N 입자 × T 반복 × 평가당 비용. LLM 평가당 ~$0.02일 때, 20입자 PSO 50반복은 ~$20. 계획 수립.
- **탐색 vs 활용.** 페로몬 감소율과 PSO 관성은 트레이드오프; 너무 빠른 감소 → 솔루션 잊음; 너무 느림 → 초기 지역 최적에 갇힘.
- **파국적 표류.** 두 알고리즘 모두 적합도 지형 변화(새 데이터 분포) 시 수렴 후 발산 가능. 최적 적합도 안정성 모니터링.

## 구축 방법

`code/main.py`는 다음을 구현합니다:

- `LMPSO` — 수치적 프롬프트 파라미터(온도, top_k 가중치)에 대한 PSO(입자 군집 최적화). 각 입자의 "LLM 생성"은 스크립트된 적합도 함수로 시뮬레이션됩니다. 30회 반복 동안 알고리즘을 실행하고 g_best 수렴을 보여줍니다.
- `AMRO_S` — ACO(개미 집단 최적화) 스타일 라우팅. 3개의 에이전트, 4개의 작업 유형, 페로몬 행렬, 100개의 라우팅된 작업. 시간에 따른 (작업 유형 → 에이전트 선택) 분포를 출력하여 트레일 형성을 보여줍니다.
- 비교: 동일한 작업 스트림에서 무작위 라우팅 대 ACO 라우팅. 품질과 지연 시간을 측정합니다.

실행 방법:

```
python3 code/main.py
```

예상 출력:
- LMPSO: g_best 적합도가 30회 반복 동안 무작위에서 최적에 가까운 값으로 개선됩니다.
- AMRO-S: 페로몬 테이블이 작업 유형별 올바른 에이전트에 안정화되며, ACO 라우팅이 무작위 대비 품질에서 ~30-40% 우수하고 지연 시간(재시도 감소)도 단축됩니다.

## 사용 방법

`outputs/skill-swarm-optimizer.md`는 LLM/에이전트 최적화 문제에 대해 PSO(입자 군집 최적화), ACO(개미 군집 최적화), 유전 알고리즘, 경사 기반 최적화 방법 중 적절한 선택 방법을 제시합니다.

## Ship It

- **작은 규모로 시작.** 10-20개의 입자, 20-50회 반복. 수렴 곡선이 명확한 이득을 보일 때만 규모를 확장.
- **반복별 페로몬 또는 g_best 기록.** 추적 기록 없이 군집 최적화 알고리즘을 디버깅하는 것은 고통스러움.
- **품질 게이트 업데이트.** 특히 ACO 라우팅의 경우: 빠르고 부정확한 에이전트는 페로몬을 축적하지 않아야 함.
- **분포 변화 시 감쇠 재설정.** 평가 분포가 변경되면 오래된 페로몬은 더 이상 유효하지 않음; 일시적으로 감쇠율을 재설정하거나 2배로 증가.
- **반복당 비용 제한.** 반복당 비용 메트릭을 생성. 반복당 $500의 비용이 들고 0.5%의 이득만 얻는 PSO는 배포 불가능.

## 연습 문제

1. `code/main.py`를 실행하세요. LMPSO 수렴을 관찰하세요. 개체군 크기를 5, 10, 20, 50으로 변화시키세요. 어떤 크기에서 수렴 시간이 포화 상태에 도달하나요?
2. "파국적 드리프트" 실험을 구현하세요: 30번째 반복 이후 적합도 함수를 변경하세요. PSO는 얼마나 빠르게 적응하나요? `p_best`를 재설정하는 것이 도움이 되나요?
3. AMRO-S에 품질 게이트를 추가하세요: 평가 점수가 0.7을 초과하는 실행에서만 페로몬을 축적하세요. 이것이 게이트가 없는 버전과 비교하여 수렴에 어떤 변화를 주나요?
4. LMPSO 논문(arXiv:2504.09247)을 읽으세요. 논문에서 "프롬프트로서의 속도"를 수치적 속도로 매핑하세요. 시뮬레이션에서 무엇이 손실되고 무엇이 보존되나요?
5. AMRO-S 논문(arXiv:2603.12933)을 읽으세요. 비동기 페로몬 업데이트와 함께 분리된 "추론 고속 경로"를 구현하세요. 이것이 지속적인 부하 하에서 시스템 지연 시간에 어떤 변화를 주나요?

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|----------------|------------------------|
| PSO | "입자 군집 최적화(Particle Swarm Optimization)" | Kennedy-Eberhart 1995. 개체군 기반 경사 하강법 없는 최적화 알고리즘. |
| ACO | "개미 군집 최적화(Ant Colony Optimization)" | Dorigo 1992. 페로몬 흔적을 통한 경로/루트 최적화. |
| LMPSO | "LLM 생성 기반 PSO(PSO with LLM generation)" | arXiv:2504.09247. 속도가 프롬프트로 사용됨; LLM이 후보 생성. |
| 모델 스웜(Model Swarms) | "전문가 가중치 기반 PSO(PSO on expert weights)" | arXiv:2410.11163. 모델 파라미터 부분공간에 대한 경사 하강법 없는 업데이트. |
| AMRO-S | "에이전트 라우팅을 위한 ACO(ACO for agent routing)" | arXiv:2603.12933. 작업 유형 × 에이전트 간 페로몬 행렬. |
| p_best / g_best | "개인/전역 최적(personal / global best)" | 입자별 및 군집 전체 최적해(지금까지 발견된). |
| 페로몬(Pheromone) | "라우팅 메모리(Routing memory)" | 엣지 상의 강도; 시간에 따라 감소; 품질에 따라 축적. |
| 품질 기반 업데이트(Quality-gated update) | "좋은 실행 결과만 학습" | 품질 검증에 조건부로 페로몬 축적. |
| 치명적 드리프트(Catastrophic drift) | "분포 변화(Distribution shift)" | 적합도 지형 변화; 이전 p_best 및 페로몬이 구식이 됨. |
