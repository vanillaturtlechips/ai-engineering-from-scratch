# 에이전트 경제, 토큰 인센티브, 평판

> 장기 자율 에이전트(METR의 1시간~8시간 작업 곡선)는 경제적 행위 능력이 필요합니다. 새롭게 부상하는 **5계층 스택**은 다음과 같습니다: **DePIN**(물리적 컴퓨팅) → **신원**(W3C DIDs + 평판 자본) → **인지**(RAG + MCP) → **결제**(계정 추상화) → **거버넌스**(에이전트 DAO). 생산 에이전트 인센티브 네트워크에는 **Bittensor**(TAO 서브넷이 작업별 모델에 보상), **Fetch.ai / ASI Alliance**(ASI-1 미니 LLM + FET 토큰), **Gonka**(트랜스포머 기반 PoW로 컴퓨팅 자원을 생산적 AI 작업에 재할당)가 포함됩니다. 학술 연구: AAMAS 2025의 분산형 LaMAS는 **샤플리 값 신용 할당**을 사용해 기여 에이전트에게 공정하게 보상하며, Google Research의 "대규모 언어 모델을 위한 메커니즘 설계"는 **단조 집계 하에서 2차 가격 지불**을 적용한 토큰 경매를 제안합니다. 이 레슨에서는 최소 에이전트 마켓플레이스를 구축하고, 다중 에이전트 파이프라인에 샤플리 값 신용 할당을 적용하며, 2차 가격 토큰 경매를 실행하여 게임 이론 메커니즘을 구체적으로 구현합니다.

**유형:** 학습  
**언어:** Python (표준 라이브러리)  
**선수 지식:** 16단계 · 16 (협상 및 교섭), 16단계 · 09 (병렬 스웜 네트워크)  
**소요 시간:** ~75분

## 문제

에이전트들이 공동으로 가치를 생성하지만 개별적으로 보상을 받아야 할 때 다중 에이전트 시스템은 복잡해집니다. 기존의 메커니즘인 균등 분배(equal split)나 마지막 기여자 독점(last-contributor-takes-all)은 불공평하거나 게임화(gameable)될 수 있습니다. 샤플리 값(Shapley value) 기반의 연합(coalition) 보상 방식은 구조적으로 공정하지만 계산에 비용이 많이 듭니다. 2025-2026년 문헌에서는 샤플리 샘플링(Shapley sampling), 단조 집계 경매(monotone aggregation auctions), 확인된 기여로부터 축적되는 온체인 평판(on-chain reputation)과 같은 유용한 근사 방법들을 제시하고 있습니다.

크레딧 할당(credit attribution)을 넘어서, 이 분야는 실제 경제 주체(agent)로 눈을 돌렸습니다: Bittensor TAO는 서브넷(subnet)별 모델 미세 조정(fine-tuning)을 위한 컴퓨팅 자원 마이닝에 보상을 제공하고, Fetch.ai/ASI는 FET 토큰으로 ASI-1 Mini LLM 사용을 보상하며, Gonka는 생산적인 AI 작업을 위해 트랜스포머 작업 증명(transformer proof-of-work)을 재할당합니다. 자율적으로 거래하는 에이전트들은 이미 현재 존재하지만, 문제는 어떻게 인센티브를 정렬할 것인가입니다.

이 레슨은 에이전트 경제를 특정 문제 군(credit attribution, 메커니즘 설계, 평판)으로 취급하며, 각 개념을 최소한의 수학적 도구로 구축하여 아이디어가 잘 기억되도록 합니다.

## 개념

### 5계층 에이전트-경제 스택

1. **DePIN(물리적 컴퓨팅).** GPU, 스토리지, 대역폭을 임대하는 분산 인프라. Bittensor 서브넷, Render Network, Akash 등이 해당. 에이전트 전용이 아니며, 에이전트가 이를 활용.
2. **신원(Identity).** W3C 분산 식별자(DID)는 각 에이전트에게 플랫폼과 독립적인 지속 ID를 제공. 평판은 DID에 누적. 에이전트 네트워크 프로토콜(ANP)은 DID를 발견 계층으로 사용.
3. **인지(Cognition).** 에이전트의 추론 루프: LLM + RAG + MCP. 다른 계층이 구축되는 핵심.
4. **결제(Settlement).** 계정 추상화(ERC-4337)를 통해 에이전트는 ETH 보유 없이 자체 잔액에서 가스비를 지불. 에이전트는 서비스, 서로, 또는 컴퓨팅 비용을 지불 가능.
5. **거버넌스(Governance).** 에이전트 DAO: 인간과 에이전트가 프로토콜 변경에 투표하는 거버넌스 구조. 투표권은 평판에 연동.

모든 프로덕션 시스템이 5계층을 모두 사용하는 것은 아님. Bittensor는 1, 2, 부분적 3, 4를 사용하나 5는 미사용. OpenAI 에이전트는 3만 사용. 이 스택은 참고 지도일 뿐 필수 조건이 아님.

### Bittensor, Fetch.ai, Gonka — 실행 중인 시스템

**Bittensor(TAO).** 서브넷은 특수 작업(언어 모델링, 이미지 생성, 예측)을 수행. 마이너는 모델 출력을 제출. 검증자는 순위를 매기고, 스테이크 가중 점수 기반으로 TAO 보상을 분배. 각 서브넷은 자체 평가 방식을 가짐. 경제적 교훈: 사용된 컴퓨팅이 아닌 작업별 출력 품질에 대한 보상.

**Fetch.ai / ASI Alliance.** ASI-1 Mini LLM은 Fetch.ai 네트워크에서 실행. 사용자는 추론에 FET 토큰을 지불. 에이전트 간 피어 간 협력 내러티브가 강함: Fetch의 에이전트는 다른 에이전트를 호출해 작업을 수행하고 FET로 지불.

**Gonka.** 트랜스포머 작업 증명(PoW): "작업"은 트랜스포머의 순전파. 마이너는 훈련 데이터에서 알려진 정답 출력을 가진 추론 작업을 실행해 보상 획득. 해시 기반 PoW 대신 자원 생산적 PoW.

세 시스템 모두 2026년 4월 기준 프로덕션 등급. 보상 분배 방식 차이: Bittensor는 서브넷 검증자 대비 품질 보상, Fetch는 유료 사용자 측정 유틸리티 보상, Gonka는 검증 가능한 추론 작업 보상.

### 샤플리 값 기여도 할당

3개 에이전트가 협업해 작업 수행. 출력 점수는 0.8. 각 기여도는?

샤플리 값: 효율성, 대칭성, 선형성, 무효성 4가지 공리를 만족하는 유일한 기여도 할당. 에이전트 `i`에 대해:

```
shapley(i) = (1/N!) * sum over all orderings O of (v(S_i_O ∪ {i}) - v(S_i_O))
```

여기서 `S_i_O`는 순서 `O`에서 `i` 이전의 에이전트 집합. 실제 적용: 모든 순열 열거, 각 순열에서 에이전트의 한계 기여도 기록, 평균 계산.

N=3 에이전트일 때 6개 순열. N=10일 때 360만 개 — 실제로는 순열 샘플링.

### 집계를 위한 2차 가격 경매

Google Research("대규모 언어 모델을 위한 메커니즘 설계")는 LLM 출력 집계를 위한 2차 가격 토큰 경매 제안. 설정: N개 에이전트가 각각 완성본을 제안. 각각은 선택될 때 사적 가치를 가짐. 경매인은 최고 가치 제안 선택 후 *두 번째 최고* 가치를 지불. 단조 집계(선택된 제안에 따라 가치 결정, 입찰 수 무관) 하에서 진실적 — 에이전트는 실제 가치를 입찰.

LLM 시스템에 중요한 이유: 다양한 가격 정책을 가진 여러 에이전트에게 완성 작업을 아웃소싱 가능. 경매는 최적 선택 + 공정한 지불, 에이전트는 허위 보고 유인 없음.

### 평판 자본

DID에 바인딩된 평판 점수는 확인된 기여로부터 누적. 단순 업데이트 규칙:

```
rep(i, t+1) = alpha * rep(i, t) + (1 - alpha) * contribution_quality(i, t)
```

감쇠 계수 `alpha`는 1에 가까움. 평판 특징:
- 라우팅 결정("고평판 에이전트에게 어려운 작업 전송")에 읽기 비용 저렴.
- 위조 비용 높음(시간 누적, DID 바인딩).
- 슬래싱 가능: 검증 실패 기여는 평판 감소.

### AAMAS 2025 분산 LaMAS

LaMAS 제안(AAMAS 2025)은 DID 신원, 샤플리 값 기여도 할당, 단순 경매 메커니즘을 결합. 핵심 주장: 기여도 할당 분산화는 시스템 감사 가능성 및 단일 지점 조작 방지.

### 경제학이 실패하는 지점

- **가격 오라클 조작.** 기여도 함수를 악용할 수 있다면 에이전트는 이를 악용. 모든 메커니즘은 적대적 테스트 필요.
- **시빌 공격.** 한 운영자가 N개의 가짜 에이전트를 생성해 기여도 부풀리기. DID는 속도를 늦추나 방지하지 못함; 평판 위조 비용이 완화책.
- **검증 비용.** 기여도 할당은 검증자만큼 공정. 검증이 저렴(소형 LLM)하면 조작 가능; 고가(인간 패널)면 시스템 확장 불가.
- **규제 부담.** 에이전트 경제는 금융 규제와 충돌. Bittensor, Fetch, Gonka는 2026년 기준 일부 관할권에서 법적 회색지대에서 운영.

### 에이전트 경제가 의미 있는 경우

- **이기종 운영자가 있는 개방형 네트워크.** 단일 팀이 모든 에이전트를 통제하지 않음.
- **검증 가능한 출력.** 검증 없이는 기여도 할당이 추측.
- **장기 워크플로우.** 일회성 작업은 평판 누적 혜택 없음.
- **토큰화 결제가 관할권에서 법적 가능.**

폐쇄적 기업 시스템에서는 경제학이 단순 할당(관리자가 작업 지정, 메트릭은 내부)으로 대체. 경제학 문헌은 주로 개방형 네트워크에 적용.

## 구축 방법

`code/main.py`는 다음을 구현합니다:

- `shapley(value_fn, agents)` — 소규모 N에 대한 열거 기반 정확한 샤플리(Shapley) 값 계산.
- `second_price_auction(bids)` — 진실적(truthful) 메커니즘; 낙찰자는 두 번째로 높은 금액을 지불.
- `Reputation` — 지수 감쇠 및 슬래싱(slashing)이 적용된 DID 기반 평판 시스템.
- 데모 1: 세 에이전트가 협력, 정확한 샤플리 값이 기여도를 할당.
- 데모 2: 다섯 에이전트가 작업 슬롯에 입찰; 두 번째 가격 경매가 낙찰자 + 지불금 결정.
- 데모 3: 100라운드 동안 이질적 평판을 가진 에이전트에게 작업 할당; 평판 가중치 기반 라우팅이 무작위 대비 10-20% 품질 향상.

실행 방법:

```
python3 code/main.py
```

예상 출력: 각 에이전트의 샤플리 값; 진실적 입찰 균형을 보여주는 경매 결과; 워밍업 후 무작위 대비 10-20% 품질 향상을 보이는 평판 가중치 기반 라우팅.

## Ship It

2026년 에이전트 경제 운영:

- **토큰이 아닌 평판부터 시작하라.** 평판은 구현이 간단하면서도 그 자체로 가치가 있으며, 토큰은 법적·경제적 복잡성을 추가한다.
- **보상하기 전에 검증하라.** 독립적인 검증 단계 없이 신용을 분배하지 마라. 자체 보고된 품질은 시빌 공격(sybil attack)을 유발한다.
- **정확한 샤플리(Shapley-exact)가 아닌 샤플리 샘플링(Shapley-sample)을 사용하라.** 100-1000개의 순서를 샘플링하라. 정확한 열거는 확장성이 없다.
- **감쇠 계수(decay factor)를 제한하고 평판의 최소값을 설정하라.** 무제한 감쇠는 정당한 기여자를 삭제하며, 너무 느린 감쇠는 오래된 고평판 에이전트를 보상한다.
- **적대적으로 감사 메커니즘을 테스트하라.** 네트워크를 공개하기 전에 레드 팀 시나리오를 실행하라. 모든 메커니즘에는 게임 이론이 존재한다. 공격자가 아닌 당신이 취약점을 찾아야 한다.

## 연습 문제

1. `code/main.py`를 실행하세요. 샤플리 값(Shapley value)이 총 가치(효율성 공리, efficiency axiom)에 합산되는지 확인하세요. 가치 함수(value function)를 변경해 보세요. 샤플리 할당(Shapley allocation)이 예상된 방향으로 변화하나요?

2. 샤플리 *샘플링*(Monte Carlo를 통한 K개의 순서 샘플링)을 구현하세요. K가 근사 정확도에 어떤 영향을 미치나요? N=4일 때 정확한 계산과 비교해 보세요.

3. 경매 전에 연합 형성(coalition-forming) 단계를 구현하세요: 에이전트(agent)가 팀으로 병합(merge)하여 단일 입찰자로 참여할 수 있습니다. 어떤 연합이 형성되나요? 이 결과가 개별 입찰보다 파레토 우월(Pareto-better)한가요?

4. Google Research의 메커니즘 설계(mechanism-design) 포스트를 읽으세요. 만약 위반된다면 진실성(truthfulness)이 깨지는 하나의 가정을 식별하세요. LLM 설정에서 이 실패 모드(failure mode)는 어떻게 나타나나요?

5. AAMAS 2025의 분산형 LaMAS 논문을 읽으세요. 합성 작업(synthetic task)에서 10개 에이전트(agent)에 대한 그들의 샤플리 단계(Shapley step)를 구현하세요. 정확한 계산은 얼마나 걸리나요? 100회 샘플링으로 얼마나 근접하게 계산되나요?
