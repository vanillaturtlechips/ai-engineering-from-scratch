# 평가 및 협업 벤치마크

> 2025-2026년 5개 벤치마크는 다중 에이전트 평가 영역을 포괄합니다. **MultiAgentBench / MARBLE** (ACL 2025, arXiv:2503.01935)은 스타/체인/트리/그래프 토폴로지를 마일스톤 KPI로 평가하며, **그래프가 연구에 가장 적합**하고 인지 계획 시 마일스톤 달성률이 ~3% 증가합니다. **COMMA**는 다중 모달 비대칭 정보 협업을 평가하며, GPT-4o를 포함한 최신 모델들은 무작위 기준선을 넘지 못합니다. **MedAgentBoard** (arXiv:2505.12371)는 4개 의료 작업 범주를 다루며, 다중 에이전트가 단일 LLM을 항상 능가하지는 않음을 보여줍니다. **AgentArch** (arXiv:2509.10769)는 도구 사용 + 메모리 + 오케스트레이션을 결합한 기업용 에이전트 아키텍처를 벤치마크합니다. **SWE-bench Pro** ([arXiv:2509.16941](https://arxiv.org/abs/2509.16941))는 비즈니스 앱, B2B 서비스, 개발자 도구 분야 41개 리포지토리에 걸친 1865개 문제를 포함하며, 프론티어 모델들은 Pro에서 ~23% 점수를 기록하지만 Verified에서는 70%+를 달성해 오염(contamination) 현실 점검을 제공합니다. Claude Opus 4.7(2026년 4월)은 명시적 에이전트 팀 협업으로 Pro에서 **64.3%**를 기록했으나(아직 Anthropic 공식 출처 미발표 — 예비 결과로 간주), Verdent(에이전트 스캐폴드)는 Verified에서 **76.1% pass@1**을 달성했습니다([Verdent 기술 보고서](https://www.verdent.ai/blog/swe-bench-verified-technical-report)). **AAAI 2026 Bridge Program WMAC** (https://multiagents.org/2026/)는 2026년 커뮤니티 중심 프로그램입니다. 이 레슨은 MARBLE의 메트릭을 기반으로 토폴로지-메트릭 스윕을 실행하고, "SWE-bench Verified 통과만으로는 일반화 증거 불충분" 규칙을 강조합니다.

**유형:** 학습  
**언어:** Python (표준 라이브러리)  
**선수 조건:** 16단계 · 15 (투표 및 토론 토폴로지), 16단계 · 23 (실패 모드)  
**소요 시간:** ~75분

## 문제

논문이 "우리의 다중 에이전트 시스템이 더 낫다"고 주장할 때, 질문은 다음과 같다: 무엇과 비교해, 어떤 측면에서, 어떻게 측정한 결과인가? 2023-2024년 다중 에이전트 평가 시대는 혼란스러웠다 — 모두가 자신만의 지표, 자신만의 베이스라인, 자신만의 작업 세트를 선택했다. 2025-2026년 벤치마크는 구조를 도입했다.

공유된 벤치마크가 없으면 두 다중 에이전트 시스템을 의미 있게 비교할 수 없다. 더 나쁜 것은, 홀드아웃 벤치마크가 없으면 최첨단 모델이 오염될 수 있다는 점이다. SWE-bench Verified는 2025년 중반까지 훈련 코퍼스에 부분적으로 오염되었고, 최첨단 점수가 부풀려졌다. Pro는 오염되지 않은 현실 검증 도구로 설계되었다.

이 레슨은 2026년의 5가지 표준 벤치마크를 열거하고, 각각이 무엇을 측정하는지 설명하며, 벤치마크 주장을 회의적으로 읽는 방법을 알려준다.

## 개념

### MultiAgentBench (MARBLE) — ACL 2025

arXiv:2503.01935. 연구, 코딩, 계획 작업 시 네 가지 조정 토폴로지(별, 체인, 트리, 그래프)를 평가합니다. 마일스톤 기반 KPI는 최종 성공뿐만 아니라 부분 진행 상황을 추적합니다.

측정 결과:

- **그래프** 토폴로지는 연구 시나리오에 가장 적합하며, 모든 에이전트 간 비평을 지원합니다.
- **체인**은 단계적 개선 코딩에 가장 적합합니다.
- **별**은 빠른 사실 통합에 가장 적합합니다.
- 그래프 토폴로지에서 ~4개 이상의 에이전트부터는 **조정 비용**이 발생합니다.
- **인지 계획**은 모든 토폴로지에서 마일스톤 달성률을 약 3% 향상시킵니다.

사용 시기: 조정 토폴로지를 공정하게 비교하고자 할 때. MARBLE 저장소(https://github.com/ulab-uiuc/MARBLE)에서 평가기를 제공합니다.

### COMMA — 다중 모달 비대칭 정보

에이전트들이 서로 다른 관측 모달리티를 가지고 완전한 정보 공유 없이 협력해야 하는 작업을 다룹니다. 보고된 결과는 불편합니다: GPT-4o를 포함한 프론티어 모델들은 COMMA에서 에이전트 간 협력 시 **무작위 기준선**을 이기지 못합니다. 이는 다중 에이전트 모달리티가 충분히 훈련되지 않고 평가되지 않았음을 시사합니다. LLM은 단일 모달리티 협력은 합리적으로 처리하지만, 다중 모달리티 조정은 실패합니다.

사용 시기: 시스템에 다중 모달리티 또는 비대칭 정보 조정이 필요한 경우. COMMA의 무효 결과는 주장 전에 측정할 것을 경고합니다.

### MedAgentBoard — 도메인 스트레스 테스트

arXiv:2505.12371. 진단, 치료 계획, 보고서 생성, 환자 커뮤니케이션 등 네 가지 의료 작업 범주를 비교합니다. 다중 에이전트 vs 단일 LLM vs 기존 규칙 기반 시스템을 평가합니다.

발견: 다중 에이전트는 대부분의 범주에서 단일 LLM을 압도하지 않습니다. 다중 에이전트 장점은 좁습니다. 작업 분해가 명확한 경우(진단 + 치료)에는 도움이 되지만, 조정 오버헤드가 전문화 이득을 초과하는 경우(보고서 생성)에는 해롭습니다.

사용 시기: 도메인에 명확한 단일 LLM 기준선이 있는 경우. MedAgentBoard의 교훈이 일반화된다면, 많은 제안된 다중 에이전트 시스템은 과도하게 설계된 것입니다.

### AgentArch — 엔터프라이즈 아키텍처

arXiv:2509.10769. 도구 사용, 메모리, 오케스트레이션이 계층화된 엔터프라이즈 환경을 평가합니다. 벤치마크는 각 계층의 기여도를 분리합니다: 도구 추가가 얼마나 도움이 되는가? 메모리 추가는? 다중 에이전트 오케스트레이션 추가는?

사용 시기: 엔터프라이즈 에이전트 스택을 설계하고 각 계층의 가치를 정당화해야 할 때. AgentArch는 가치를 측정할 수 없는 기능을 구매하는 것을 방지합니다.

### SWE-bench Pro — 현실 검증

arXiv:2509.16941. 비즈니스 앱, B2B 서비스, 개발자 도구 등 41개 저장소에 걸친 1865개 문제. 이후 훈련 컷오프와 **오염되지 않도록** 설계되었습니다. 프론티어 모델들은 Pro에서 ~23%, Verified에서 70%+를 기록합니다. 이 격차는 오염 신호입니다.

2026년 4월 점수:
- Claude Opus 4.7 on Pro: **64.3%** (명시적 에이전트 팀 조정으로 보고됨; 아직 Anthropic의 공식 출처 없음 — 예비로 간주).
- Verdent(에이전트 스캐폴딩) on Verified: **76.1% pass@1** ([기술 보고서](https://www.verdent.ai/blog/swe-bench-verified-technical-report)).
- 에이전트 스캐폴딩 없이 Pro에서 프론티어 모델의 원시 점수: ~23-35% ([SWE-bench Pro 논문](https://arxiv.org/abs/2509.16941)).

교훈: "SWE-bench Verified를 이겼다"는 더 이상 능력 증거가 아닙니다. Pro가 현재 관문 테스트입니다. 에이전트 팀 스캐폴딩은 Pro에서 측정 가능한 향상(~30-40점 차이)을 생성하며, 이는 2026년 다중 에이전트 조정에 대한 가장 강력한 실증적 논거 중 하나입니다.

### AAAI 2026 WMAC

AAAI 2026 Bridge Program — 다중 에이전트 조정 워크숍(https://multiagents.org/2026/). 2026년 다중 에이전트 AI 연구의 커뮤니티 초점. 승인된 논문과 워크숍 진행 자료는 새로운 방법 평가를 위한 정설 장소입니다. 프로덕션 결정 시 arXiv 프리프린트보다 WMAC 승인 주장을 우선시하세요.

### 벤치마크 주장을 회의적으로 읽기 — 2026 체크리스트

누군가가 다중 에이전트 결과를 주장할 때:

1. **어떤 벤치마크, 어떤 분할?** SWE-bench Verified vs Pro는 매우 중요합니다. 잘못된 분할에서 보고된 수치는 무의미합니다.
2. **오염 확인.** 벤치마크가 모델 훈련 컷오프 이후에 공개되었는가? 그렇지 않다면 주의하여 다루세요.
3. **기준선 비교.** 단일 LLM 기준선, 무작위, 이전 다중 에이전트 작업과 비교. "동일한 시스템의 조정되지 않은 버전"과 비교는 제외.
4. **통계적 유의성.** N 시행, p-값, 신뢰 구간. 프론티어 모델은 분산이 큽니다. 단일 실행은 오해를 일으킵니다.
5. **작업 다양성.** 하나의 작업 또는 여러 작업? 일반화는 프로덕션에 중요합니다.
6. **비용 공개.** 작업당 토큰, 벽시계 시간. 20배 비용으로 90% 솔루션은 비즈니스 결정이지 능력 주장이 아닙니다.

### 벤치마크가 잘 측정하지 못하는 것

- **장기 조정.** 벽시계 기준 며칠 간의 상호작용. 모든 현재 벤치마크는 짧게 실행됩니다.
- **적대적 회복력.** 한 에이전트가 악의적이거나 손상되었을 때 발생하는 현상.
- **배포 시 드리프트.** 벤치마크는 정적이지만, 프로덕션 분포는 변화합니다.
- **비용 정규화 성능.** 대부분의 벤치마크는 원시 정확도만 보고하며, 달러당 정확도는 보고하지 않습니다.

실제로 관심 있는 축에 대해 내부 벤치마크를 구축하는 것이 종종 올바른 선택입니다.

## 빌드하기

`code/main.py`는 비대화형 워크스루입니다:

- 장난감 작업에서 3개의 다중 에이전트 시스템을 시뮬레이션합니다.
- 각각에 대해 MARBLE 스타일의 마일스톤 메트릭을 계산합니다.
- "훈련" 세트에서 작업을 보류하여 오염 검사를 실행합니다.
- 명시적 랜덤 베이스라인과 비교합니다.
- 벤치마크-클레임 스코어카드를 출력합니다.

실행:

```bash
python3 code/main.py
```

예상 출력: 원시 정확도, 마일스톤 달성도, 작업당 비용, 랜덤 베이스라인 대비 델타, 오염 검사 메모가 포함된 시스템 스코어카드.

## 사용 방법

`outputs/skill-benchmark-reader.md`는 모든 다중 에이전트 벤치마크 주장을 읽고 검토 체크리스트를 적용합니다. 출력: 등급과 주의 사항.

## Ship It

프로덕션 평가 원칙:

- 실제 프로덕션 분포를 반영하는 **내부 벤치마크**를 구축하세요. 공개 벤치마크는 참고용이지만 대체할 수 없습니다.
- 모든 비교에 **무작위 기준선**을 포함하세요. 협업 작업에서 무작위 기준선을 큰 마진으로 이기지 못한다면, 작업 정의가 잘못되었을 수 있습니다.
- **정확도와 함께 비용**을 보고하세요. 토큰 비용과 실제 소요 시간(벽시계 시간) 모두 필요합니다. 운영 팀은 두 가지 모두 필요합니다.
- **분기별로 벤치마크를 재구축**하세요. 프로덕션 분포는 변화합니다. 오래된 벤치마크는 오해를 일으킵니다.
- **공개 벤치마크 과적합을 피하세요.** 팀이 SWE-bench Pro 수치에만 최적화한다면, 프로덕션에서 성능이 저하될 것입니다.

## 연습 문제

1. `code/main.py`를 실행하세요. 세 개의 시뮬레이션 시스템 중 어떤 것이 마일스톤당 비용(cost-per-milestone)이 가장 좋은지 확인하세요. 이 시스템이 가장 높은 원시 정확도(raw-accuracy)를 가진 시스템과 일치하나요?

2. MultiAgentBench 논문(arXiv:2503.01935)을 읽으세요. 자신의 작업 도메인에 대해 MARBLE이 추천할 네 가지 토폴로지 중 어떤 것을 선택할지 결정하고, 논문 결과를 바탕으로 근거를 제시하세요.

3. SWE-bench Pro 논문을 읽으세요. 구체적으로 어떤 요소가 오염 저항성(contamination-resistant)을 만드는지 설명하세요. 동일한 기법을 관심 있는 다른 벤치마크에도 적용할 수 있을까요?

4. COMMA의 다중 모달 협력(multimodal coordination) 연구 결과를 읽고, 내부 벤치마크에 추가할 수 있는 간단한 다중 모달 협력 과제를 설계하세요. 유용한 신호(useful signal)로 간주되는 것은 무엇인가요?

5. 최근 다중 에이전트 논문의 주요 결과(headline result)에 벤치마크-주장 체크리스트(benchmark-claims checklist)를 적용하세요. 이 주장에 어떤 등급을 줄 것인가요?

| 마일스톤 달성 | "부분 점수" | 최종 성공뿐 아니라 진행 상황을 보상하는 벤치마크. |
| 오염 | "벤치마크가 훈련에 유출" | 출시 후 벤치마크가 훈련 코퍼스로 유입; 점수 인플레이션 발생. |
| WMAC | "AAAI 2026 Bridge Program" | 다중 에이전트 협업 워크숍; 커뮤니티 초점 포인트. |
