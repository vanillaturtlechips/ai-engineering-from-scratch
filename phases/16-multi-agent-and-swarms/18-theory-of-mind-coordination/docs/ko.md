# 마음 이론(Theory of Mind)과 창발적 조정(Emergent Coordination)

> Li et al. (arXiv:2310.10701)은 협력적인 텍스트 게임에서 LLM 에이전트들이 **창발적 고차 마음 이론(ToM)** — 제3의 에이전트의 믿음에 대해 다른 에이전트가 어떻게 추론하는지 — 을 보이지만, 컨텍스트 관리와 환각(hallucination) 문제로 인해 장기 계획(long-horizon planning)에는 실패한다는 것을 보였다. Riedl (arXiv:2510.05174)은 집단 전반에 걸쳐 고차 시너지(higher-order synergy)를 측정하고, **오직** ToM-프롬프트 조건에서만 정체성 연결 분화(identity-linked differentiation)와 목표 지향적 상호보완성(goal-directed complementarity)이 생성된다는 것을 발견했다. 낮은 용량의 LLM은 오직 우연한 창발(spurious emergence)만 보인다. 즉, 조정 창발은 프롬프트에 조건적이며 모델에 의존적이지, 자유롭지 않다. 이 레슨은 최소한의 ToM-인식 에이전트를 구현하고, ToM 프롬프트 유무에 따른 협력 작업을 실행한 후, Riedl 2025 프로토콜에 따른 조정 델타(delta)를 측정한다.

**유형:** 학습 + 구현  
**언어:** Python (표준 라이브러리)  
**선수 지식:** 16단계 · 07 (마음의 사회와 토론), 16단계 · 17 (생성 에이전트)  
**소요 시간:** ~75분

## 문제

다중 에이전트 조정은 종종 마법처럼 보입니다: 에이전트들이 업무를 분담하고, 서로를 예측하며, 중복을 피합니다. 일반적으로 이러한 "창발(emergence)"은 프롬프트 엔지니어링의 부산물입니다 — 누군가가 에이전트들에게 "협력하라"고 지시한 것입니다. 프롬프트를 제거하면 조정도 사라집니다.

Riedl의 2025년 연구 결과는 더 엄격합니다: 통제된 조건에서 조정은 에이전트들이 **다른 에이전트의 마음(ToM, Theory of Mind)**에 대해 추론하도록 프롬프트를 받았을 때만 나타납니다. ToM 프롬프트가 없으면 강력한 모델조차도 통계적 검증을 견디지 못하는 조정 패턴을 보입니다. 이는 실제 적용에서 중요합니다: 팀들은 프롬프트에 의존적이고 취약한 "다중 에이전트 조정" 기능을 출시하고 있습니다.

이 레슨은 ToM을 특정 능력(믿음에 대한 믿음에 대한 추론)으로 취급하고, 최소한의 ToM 인식 에이전트를 구축하며, 실제 조정과 프롬프트 장식이 어떻게 다른지 측정합니다.

## 개념

### ToM이 의미하는 것

발달 심리학: 3세 아이는 모든 사람의 내면 세계가 자신과 일치한다고 생각한다. 5세 아이는 다른 사람이 다른 믿음을 가질 수 있음을 이해한다. 7세 아이는 "그녀는 내가 공이 컵 아래에 있다고 생각한다고 생각한다"와 같은 믿음-믿음 관계를 추론한다. 이는 각각 0차, 1차, 2차 ToM이다.

LLM 에이전트의 경우 ToM 차수는 다음과 같이 매핑된다:

- **0차:** 다른 에이전트에 대한 모델이 없음. 에이전트는 자신의 관측만으로 행동한다.
- **1차:** 각 다른 에이전트의 믿음에 대한 모델을 가짐. "앨리스는 X를 믿는다."
- **2차:** 재귀적 믿음을 모델링. "앨리스는 밥이 X를 믿는다고 생각한다."

Li et al. 2023은 협력 게임에서 1차 및 2차 ToM이 LLM 에이전트에서 나타나지만, 긴 시간 범위와 신뢰할 수 없는 통신으로 인해 성능이 저하됨을 발견했다.

### 샐리-앤 테스트, 간략히

1985년 거짓 믿음 테스트: 샐리가 구슬을 바구니 A에 넣고 떠난다. 앤이 구슬을 바구니 B로 옮긴다. 샐리가 돌아왔을 때 어디서 구슬을 찾을까? 1차 ToM을 가진 아이는 바구니 A(샐리의 믿음이 현실과 다름)라고 답한다. 그렇지 않은 아이는 바구니 B라고 답한다.

GPT-4 시대 LLM은 샐리-앤 스타일 테스트를 평이한 질문으로 제시하면 통과한다. 그러나 서사가 길거나 장면이 여러 번 변경되거나 질문이 간접적으로 표현되면 실패한다. 이는 2026년 현재 프로덕션 LLM의 ToM 현황이다.

### 리드(Reidl)의 조정 측정

리드(arXiv:2510.05174)는 인구 규모 테스트를 구축했다: N명의 에이전트, 협력 목표, 가변 프롬프트 조건. 측정 항목:

1. **정체성 연결 분화.** 에이전트가 시간에 따라 안정적인 역할 구분을 발전시키는지?
2. **목표 지향적 상호보완성.** 에이전트 행동이 서로 중복되지 않고 다른 하위 작업을 보완하는지?
3. **고차 시너지.** 그룹이 어떤 부분집합도 달성할 수 없는 것을 달성하는지 통계적 측정.

결과: ToM 프롬프트 조건에서만 세 가지 지표가 기준치 이상의 신호를 생성한다. ToM 프롬프트가 없으면 중간 규모 모델의 지표는 무작위 수준 근처에 머문다. 대형 모델은 명시적 ToM 프롬프트 없이도 일부 조정을 보이지만 효과는 명시적 프롬프트 사용 시보다 작다.

### 조정 환상

통계적 통제가 없을 때 데모의 "발현적 조정"은 종종 다음을 반영한다:

- 조정을 내재화한 프롬프트 엔지니어링("함께 일하라"는 시스템 프롬프트).
- 관찰자 편향(기대하는 패턴을 본다).
- 성공적인 실행의 사후 선택.

측정 가능한 신호 없이 "발현적 조정"을 마케팅하는 프로덕션 시스템은 마케팅으로 간주해야 한다. 주장하기 전에 측정하라.

### 최소한의 ToM 인식 에이전트

구조:

```
에이전트 상태:
  own_beliefs:    {에이전트가 믿는 사실}
  other_models:   {다른_에이전트_ID -> {에이전트가 그들에게 귀속시키는 믿음}}
  actions_last_N: [다른 에이전트의 행동 기록]

관측 업데이트:
  - 직접 관측으로부터 own_beliefs 업데이트
  - 다른 에이전트의 행동 + 이전 믿음을 기반으로 other_models[에이전트_ID] 업데이트

행동 선택:
  - 후보 행동 열거
  - 각각에 대해 모델링된 믿음을 바탕으로 다른 에이전트가 다음에 할 행동 예측
  - 예측 하에서 공동 결과를 최대화하는 행동 선택
```

`other_models` 속성은 ToM 상태이다. 1차 ToM은 한 수준만 유지한다. 2차 ToM은 `other_models[i][other_models_of_j]`를 추가한다 — 내가 에이전트 i가 에이전트 j의 믿음을 어떻게 생각하는지.

### 장기 시간 범위가 성능을 저하시키는 이유

Li et al.은 다음과 같이 기록했다: 컨텍스트 제한으로 에이전트가 어떤 믿음이 누구에게 속하는지 잊는다. 환각은 다른 에이전트 모델에 거짓 믿음을 추가한다. 둘 다 시간이 지남에 따라 "그가 X를 생각한다고 생각했는데" 오류를 누적시킨다.

논문 및 2024-2026 후속 연구에서 문서화된 완화 방법:

- **프롬프트 내 명시적 ToM 상태.** 구조화된 형식: `{에이전트_ID: 믿음_목록}`. 신원-믿음 바인딩 보존을 위한 검색 강제.
- **더 짧은 추론 체인.** 턴당 ToM 업데이트 횟수 감소로 환각 누적 완화.
- **외부 ToM 저장소.** LLM 컨텍스트 외부에 모델 유지; 턴당 관련 부분만 주입.

### 프로덕션에서 ToM이 실패하는 경우

- **적대적 환경.** ToM이 좋은 에이전트는 조작하기 쉬움(그들이 당신을 어떻게 모델링하는지 예측한 후 악용 가능).
- **이질적 팀.** 모델이 다를 때 한 상대방에게 작동하는 ToM 모델이 일반화되지 않음.
- **사실 기반 작업.** ToM은 믿음에 관한 것; 정확성이 사실에 의존할 때 ToM은 주의 분산 요인이 될 수 있음.

### 실제로 측정 가능한 조정

팀의 조정이 프롬프트에 의한 것이 아닌 실제임을 나타내는 세 가지 실용적 신호:

1. **시간에 따른 상호보완성.** 다단계 작업에서 에이전트 행동이 서로 다른 하위 작업을 다루는지?
2. **예측.** 에이전트 A의 T+1 행동이 B T+2 행동에 대한 예측에 의존하며 그 예측이 정확했는지?
3. **수정.** A가 T턴에서 B의 믿음을 잘못 읽었을 때, T+2턴까지 수정하는지?

이들은 로깅된 다중 에이전트 시스템에서 측정 가능하다. "조정" 내러티브의 실질적 버전이다.

## 구축 방법

`code/main.py`는 다음을 구현합니다:

- `ToMAgent` — 자신의 믿음(belief)과 다른 에이전트별 믿음 모델을 추적합니다.
- 협력 과제: 세 에이전트가 세 개의 상자에서 세 개의 토큰을 수집해야 합니다. 각 상자는 하나의 토큰만 보유할 수 있습니다. 에이전트들은 의사소통할 수 없으며, 서로의 행동으로부터 의도를 추론합니다.
- 두 가지 구성: `zeroth_order` (ToM 없음)와 `first_order` (1단계 믿음 모델을 가진 ToM).
- 200회의 무작위 시행에서 측정: 완료율, 중복률(두 에이전트가 동일한 상자를 대상으로 하는 경우), 완료까지 평균 턴 수.

실행 방법:

```
python3 code/main.py
```

예상 결과: 제로차(zeroth-order) 에이전트는 약 35%의 중복률을 보이며 10턴 내 약 60%의 시행을 완료합니다. 1차 ToM 에이전트는 약 5%의 중복률을 보이고 약 95%를 완료합니다. 이 차이가 측정 가능한 협력 효과입니다.

## 사용 방법

`outputs/skill-tom-auditor.md`는 다중 에이전트 시스템의 "발현적 조정(emergent coordination)" 주장을 감사하는 스킬입니다. 프롬프트 드레싱(prompt dressing), 대조군 대비 통계적 유의성, 측정된 상호보완성(complementarity)을 검증합니다.

## Ship It

협업 주장 체크리스트:

- **통제 조건(Control condition).** 협업 프롬프트가 없는 시스템 버전. 두 버전 모두 측정.
- **통계적 검정(Statistical test).** 시스템과 통제 조건 간 차이가 메트릭에서 `p < 0.05` 수준에서 유의미한가?
- **상호보완성 측정(Complementarity measure).** 최종 성공뿐만 아니라 시간에 따른 행동-불분리성(action-disjointness).
- **실패 사례 로그(Failure-case log).** 에이전트가 협업에 실패할 때, ToM 상태(theory of mind state)는 어떻게 나타나는가?
- **모델 용량 공개(Model-capacity disclosure).** 더 작은 모델에서 효과가 사라진다면, 이를 명시하라.

## 연습 문제

1. `code/main.py`를 실행하세요. 1차 ToM(이론 of 마음)이 중복률을 ~7배 감소시키는지 확인하세요. 5명의 에이전트와 5개의 상자로 확장했을 때 이 격차가 유지되나요?
2. 2차 ToM(에이전트 A가 B가 C에 대해 어떻게 생각하는지 모델링)을 구현하세요. 1차 ToM보다 개선되나요? 어떤 작업에서 개선되나요?
3. ToM 상태에 **환각(hallucination)**을 주입하세요: 매 턴마다 무작위로 하나의 믿음을 뒤집을 때, 1차 ToM 성능이 얼마나 저하되나요?
4. Li et al. (arXiv:2310.10701)을 읽고 "장기적 성능 저하" 결과를 재현하세요: 턴 수가 10에서 30으로 증가할 때, 1차 ToM 성능은 어떻게 변하나요?
5. Riedl 2025 (arXiv:2510.05174)을 읽고 시뮬레이션 로그에 고차 시너지 통계량을 구현하세요. ToM 프롬프트 조건 없이도 이 효과가 나타나나요?

## 주요 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|----------------|------------------------|
| 마음 이론(Theory of Mind) | "다른 사람의 마음 이해" | 다른 에이전트의 믿음을 모델링하는 능력. 차수(0, 1, 2+)로 등급화됨. |
| 샐리-앤 테스트(Sally-Anne test) | "거짓 믿음 테스트" | 1985년 발달 심리학; LLM은 일반 버전은 통과하지만 복잡한 버전은 실패함. |
| 1차 마음 이론(First-order ToM) | "A는 X를 믿는다" | 사실에 대한 다른 에이전트의 믿음을 모델링함. |
| 2차 마음 이론(Second-order ToM) | "A는 B가 X를 믿는다고 믿는다" | 한 단계 더 깊은 재귀적 모델링. |
| 정체성 연결 분화(Identity-linked differentiation) | "시간에 걸쳐 안정적인 역할" | Riedl의 지표: 역할이 무작위로 변하지 않고 지속됨. |
| 목표 지향적 상호보완성(Goal-directed complementarity) | "분리된 행동" | 에이전트들이 동일한 하위 작업이 아닌 서로 다른 하위 작업을 목표로 함. |
| 고차 시너지(Higher-order synergy) | "그룹이 어떤 부분집합보다 우수함" | Riedl의 실제 협력을 위한 통계적 측정 지표. |
| 협력 착각(Coordination illusion) | "협력처럼 보인다" | 측정 가능한 신호 없이 프롬프트로 꾸며진 협력 모습. |
