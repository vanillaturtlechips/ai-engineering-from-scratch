# 에이전트를 위한 합의(Consensus)와 비잔틴 장애 허용(Byzantine Fault Tolerance, BFT)

> 고전적인 분산 시스템 BFT와 확률적 LLM의 만남. 2025-2026년에 세 가지 연구 방향이 등장했습니다: **CP-WBFT**(arXiv:2511.10400)는 각 투표에 신뢰도 프로브를 가중치로 적용; **DecentLLMs**(arXiv:2507.14928)는 리더 없이 병렬 작업자 제안과 기하 중앙값(geometric-median) 집계를 사용; **WBFT**(arXiv:2505.05103)는 가중치 투표와 계층적 구조 클러스터링(Hierarchical Structure Clustering)을 결합해 코어(Core)와 엣지(Edge) 노드를 분리합니다. "Can AI Agents Agree?"(arXiv:2603.01213)의 정직한 실험 결과는 스칼라 합의(scalar agreement)조차 현재 취약하다는 것입니다 — 단일 기만적 에이전트가 에이전트 혼합체(Mixture-of-Agents)를 훼손할 수 있습니다. BFT는 필수적이지만 충분하지는 않습니다. 이 레슨은 최소한의 BFT 프로토콜을 구축하고, 세 가지 에이전트 특화 공격(비잔틴 거짓말, 아첨적 순응, 상관 오류 단일문화)을 주입하며, 각 합의 변형이 어떻게 대응하는지 측정합니다.

**유형:** 학습 + 구현  
**언어:** Python (표준 라이브러리)  
**선수 지식:** 16단계 · 07 (마음의 사회와 토론), 16단계 · 13 (공유 메모리)  
**소요 시간:** ~75분

## 문제

N개의 LLM 에이전트가 각각 답변을 생성하지만 서로 의견이 일치하지 않습니다. 다수결 투표는 잘못된 답변을 선택할 수 있는데, 이는 두 에이전트가 상관관계를 가지기 때문입니다(동일한 기본 모델, 동일한 학습 데이터, 동일한 실패 모드). 세 번째 에이전트는 우연히 새로운 방식으로 오답을 생성하므로, 다수결은 거짓 다수결이 됩니다.

이제 기만적인 에이전트를 추가해 보겠습니다. 이 에이전트는 고의로 거짓말을 합니다. 또는 아첨하는 에이전트를 추가할 수도 있습니다. 이 에이전트는 마지막으로 발언한 의견에 동의합니다. 고전적인 BFT(Byzantine Fault Tolerance)에서는 비잔틴 노드가 `f < n/3` 비율이며 임의로 행동한다는 가정을 합니다. 그러나 2026년의 현실은 LLM 노드들이 정직할 때도 확률적이며, 모델 간 상관관계가 있고, 서로의 출력에 영향을 받는다는 것입니다. 따라서 이들을 독립적인 베르누이 투표자로 취급할 수 없습니다.

고전적인 BFT(예: PBFT, 1999년)가 틀린 것은 아닙니다. 다만 불완전할 뿐입니다. 고전적인 BFT는 임의의 비트 플리핑을 처리하지만, "세 개의 정직한 에이전트가 학습 데이터를 공유하기 때문에 환각을 공유하는" 경우는 처리하지 못합니다. 이 강의에서는 PBFT의 기반을 출발점으로 삼아 2025-2026년에 필요한 세 가지 적응 방식을 추가합니다.

## 개념

### 고전적 BFT가 제공하는 것

실용적 비잔틴 장애 허용(Practical Byzantine Fault Tolerance, Castro & Liskov, OSDI 1999)은 `f < n/3` 비잔틴 노드를 허용합니다. 이 프로토콜은 3단계(사전 준비(pre-prepare), 준비(prepare), 커밋(commit))와 2가지 기본 요소(서명된 메시지, 쿼럼 인증서)를 가지며, `n >= 3f + 1` 정직한 노드 또는 악의적 노드 간 단일 값에 대한 합의를 보장합니다.

보장은 강력하지만 다음을 가정합니다:

1. **독립적 장애.** 비잔틴 노드가 협력하지 않음.
2. **정직한 노드는 완전히 정직함.** 정직한 출력의 정확성은 문제되지 않음; 프로토콜은 의견 불일치만 조정.
3. **질문에 대한 객관적 정답이 존재.** 잘못된 사실에 대한 합의도 여전히 합의임.

LLM 에이전트는 이 세 가지를 모두 위반합니다. 동일한 기본 모델을 실행하는 두 에이전트는 장애를 공유합니다. "정직한" LLM도 환각(hallucination)을 일으킵니다. 모호한 질문에 대해 "진실"은 에이전트가 결정하는 것이며, 외부 오라클이 없습니다.

### LLM 특화 공격 3가지

**비잔틴 거짓말(Byzantine lie).** 한 에이전트가 고의로 잘못된 답을 출력. 고전적 BFT는 `f < n/3`일 때 이를 처리합니다.

**아첨적 순응(Sycophantic conformity).** 한 에이전트가 투표 전 다른 에이전트의 답을 읽고 마지막에 발언한 쪽에 맞춰줌. 악의적이진 않지만 가장 큰 목소리를 내는 쪽과 상관관계가 있음. 고전적 BFT는 서명 검사를 통과하므로 이를 방지하지 못함.

**상관 오류 단일 문화(Correlated-error monoculture).** 세 에이전트가 기본 모델을 공유. 동일한 잘못된 답을 환각함. 다수가 틀림. 고전적 BFT는 세 에이전트가 "정직하게" 동의하므로 도움이 되지 않음.

### 2025-2026년 대응 방안

**CP-WBFT** (arXiv:2511.10400) — 신뢰도 탐색 가중치 BFT. 각 투표자는 답변에 신뢰도 탐색(self-reported probability 또는 별도 보정 모델의 예측)을 첨부. 투표 가중치는 신뢰도에 비례. 완전 그래프에서 +85.71% BFT 개선 보고. 완화 대상: 아첨적 순응(순응하는 에이전트는 자발적 입장에 대한 신뢰도가 낮은 경향).

**DecentLLMs** (arXiv:2507.14928) — 리더리스. 작업자 에이전트가 병렬로 제안, 평가자 에이전트가 제안을 점수화, 최종 답은 점수화된 위치의 기하 중앙값. `f < n/2`일 때 강건. 완화 대상: 비잔틴 거짓말 및 상관 오류(기하 중앙값은 이상치에 강건하고 모델 편향 평균이 아닌 밀집 클러스터로 끌어당김).

**WBFT** (arXiv:2505.05103) — 계층 구조 클러스터링을 통한 가중치 BFT. 투표 가중치는 응답 품질 및 이력에서 학습된 신뢰 점수로 할당. 에이전트를 코어(Core)와 에지(Edge)로 클러스터링; 코어 에이전트가 먼저 합의 달성, 에지 에이전트는 따름. 완화 대상: 확장성(코어 합의는 작고 빠름) 및 부분적으로 단일 문화(코어는 다양성 기준으로 선택 가능).

### 실증 연구: "AI 에이전트는 합의할 수 있는가?" (arXiv:2603.01213)

이 논문은 여러 프론티어 모델에서 스칼라 합의(LLM 에이전트가 단일 수치 값에 합의)를 측정합니다. 결과는 불편합니다:

- 적대적 에이전트가 없어도 LLM 에이전트는 많은 벤치마크에서 30% 이상 스칼라 질문에 대해 불일치.
- 기만적 페르소나를 채택한 단일 에이전트는 혼합 에이전트 합의를 정직한 기준선에서 40% 이상 벗어나게 할 수 있음.
- 불일치율은 모델 다양성과 상관관계가 있음 — 이질적 앙상블은 동질적 앙상블보다 더 많이 불일치(좋음: 비상관 오류)하지만 더 느리게 표류함(나쁨: 합의 도달 시간 증가).

교훈: BFT는 출력을 정렬하는 도구를 제공하지만, 정렬된 출력이 옳은지 알려주지는 않습니다. 검증(16단계 · 08 역할 전문화), 다양성(16단계 · 15 토론 변형), 평가자 에이전트(16단계 · 24 벤치마크)와 결합해야 합니다.

### 핵심 프로토콜, 간소화

LLM 에이전트를 위한 최소 BFT 라운드:

```
1. 작업 도착; 각 에이전트 i는 답변 a_i 생성
2. 각 에이전트는 [0, 1] 범위의 신뢰도 탐색 c_i 첨부
3. 집계기는 모든 n 에이전트의 (a_i, c_i) 수집
4. 집계기는 의미적 클러스터(동등한 답변)로 그룹화
5. 집계기는 각 클러스터 C에 대한 가중치 계산:
     w(C) = C에 속한 i의 c_i 합계
6. 승자 = 최대 가중치 클러스터, 단 최대값이 임계값 * sum(c_i)보다 클 때
   아니면: 재시도 또는 에스컬레이션
7. 소수 클러스터는 사후 감사를 위해 출처와 함께 기록
```

의미적 클러스터링 단계는 LLM 특화 변형입니다. "연구는 4.2%를 보고한다"와 "4.2% 개선"은 동일 클러스터입니다. 순진한 문자열 일치 검사는 이를 놓칩니다. 프로덕션에서는 저렴한 임베딩 모델이나 명시적 정규화(canonicalization)를 사용합니다.

### 임계값 조정

`threshold` 매개변수는 수락 여부와 재시도 여부를 결정합니다. 너무 낮으면 약한 다수를 수락합니다. 너무 높으면 아무것도 수락하지 않습니다. 경험적 범위: `n=5-7` 에이전트의 경우 0.5-0.67, 더 작은 `n`의 경우 더 높음. 임계값 미만일 때는 인간 또는 다른 에이전트 앙상블로 에스컬레이션합니다.

### 합의가 도움이 되지 않는 경우

- **모호한 질문.** 질문에 객관적 진실이 없으면 합의는 의견일 뿐입니다. 그렇게 부르세요.
- **복합 질문.** "코드를 작성하고 설명하라" — 두 답변. 각각 독립적으로 투표.
- **적대적 다단계.** 에이전트가 이전 라운드를 관찰하고 모방할 수 있으면(Du 2023 토론), 진실과 무관하게 서로 동의하기 시작. 라운드 제한(일반적으로 2-3회).

## 구축 방법

`code/main.py`는 다음을 구현합니다:

- `AgentVoter` — (답변, 신뢰도)를 가진 스크립트 정책.
- `MajorityVote` — 고전적 다수결(plurality).
- `CPWBFT` — 의미론적 클러스터링을 통한 신뢰도 가중 투표.
- `DecentLLMs` — 점수 부여 제안에 대한 기하 중앙값(geometric-median) 집계.
- `Scenario` — 세 가지 공격 패턴 하에서 각 집계기 실행.

구현된 공격 패턴:

1. `byzantine`: 한 에이전트가 높은 신뢰도로 거짓 답변.
2. `sycophancy`: 한 에이전트가 첫 번째로 본 답변을 복사하고 신뢰도를 맞춤.
3. `monoculture`: 세 에이전트가 중간 신뢰도로 잘못된 답변을 공유(상관 오류).

실행 방법:

```
python3 code/main.py
```

예상 출력: (공격, 집계기) → 최종 답변 표. 올바른 답변은 강조 표시됨. 다수결은 단일 문화(monoculture) 사례에서 실패. CPWBFT의 신뢰도 가중치는 아첨(sycophancy) 완화. DecentLLMs의 기하 중앙값은 단일 문화가 전체 인구의 절반 미만일 때 정직한 클러스터로 수렴.

## 사용 방법

`outputs/skill-consensus-designer.md`는 다중 에이전트 앙상블을 위한 합의 프로토콜을 설계합니다: 클러스터링 방법, 가중치, 임계값, 그리고 임계값 미달 라운드에 대한 에스컬레이션 정책.

## Ship It

어떤 합의 메커니즘을 출시하기 전에:

- **위의 최소 세 가지 패턴**으로 공격 테스트를 수행하세요. 프로토콜은 예측 가능하게 실패해야 하며, 침묵적으로 실패해서는 안 됩니다.
- **모든 소수 클러스터**를 출처(provenance)와 함께 기록하세요. 소수 클러스터는 상관 오류(correlated errors)에 대한 조기 경고 시스템입니다.
- **유계 라운드(bounded rounds)를 강제하세요.** "합의에 도달할 때까지 계속 토론"은 아첨(sycophancy)을 조장합니다.
- **합의(agreement)와 정확성(correctness)을 분리하세요.** 합의 출력은 검증자(verifier)로 전달되며, 검증자는 앙상블(ensemble)과 독립적이어야 합니다.
- **합의율(agreement rate)을 모니터링하세요.** 급격한 상승은 동조 편향(conformity bias)을 의미하며, 급격한 하락은 모델 드리프트(model drift)를 의미합니다.

## 연습 문제

1. `code/main.py`를 실행하세요. 복수형(plurality)이 단일문화 공격(monoculture attack)에 실패하는 반면, CPWBFT는 단일문화 신뢰도가 0.7 미만일 때 부분적으로 완화하는지 확인하세요.
2. 네 번째 공격 패턴을 추가하세요: **침묵 기권(silent abstention)** — 한 에이전트가 답변을 거부합니다("I don't know"). 각 집계기(aggregator)는 기권을 어떻게 처리해야 할까요? 선택한 방식을 구현하세요.
3. 의미 클러스터링(semantic clustering)을 문자열 정규화(string canonicalization)에서 임베딩 유사도(embedding-similarity)로 변경하세요(오픈소스 임베딩 모델 사용). 아첨 공격(sycophancy attack)에 어떤 영향이 발생하나요?
4. CP-WBFT(arXiv:2511.10400)를 읽으세요. 신뢰도 탐색 보정(confidence-probe calibration) 단계를 구현하세요(별도의 보정 모델이 각 에이전트의 자체 보고 신뢰도를 검증). 단일문화 시나리오에서 정확도 향상을 측정하세요.
5. "Can AI Agents Agree?"(arXiv:2603.01213)를 읽으세요. 단순화된 스칼라 합의(scalar-agreement) 실험을 재현하세요: 세 에이전트, 하나의 스칼라 질문, 기만적 페르소나(deceptive-persona) 프롬프트. CPWBFT 또는 DecentLLMs가 이를 탐지할 수 있나요?
