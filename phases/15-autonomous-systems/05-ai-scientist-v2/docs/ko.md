# AI Scientist v2 — 워크숍 수준의 자율 연구

> Sakana의 AI Scientist v2 (Yamada et al., arXiv:2504.08066)는 가설 설정, 코드 작성, 실험, 시각화, 논문 작성, 제출에 이르는 전체 연구 루프를 실행합니다. 이 시스템은 생성된 논문이 ICLR 2025 워크숍의 동료 검토를 통과한 최초의 시스템입니다. 독립 평가(Beel et al.)에 따르면 42%의 실험이 코딩 오류로 실패했으며 문헌 검토 과정에서 기존 개념을 새로운 것으로 잘못 표기하는 경우가 빈번했습니다. Sakana의 공식 문서는 코드베이스가 LLM이 작성한 코드를 실행한다는 점을 경고하며 Docker 격리를 권장합니다. 이 두 가지 측면 모두 핵심 포인트입니다.

**유형:** 학습
**언어:** Python (표준 라이브러리, 연구 루프 상태 머신 토이)
**선수 지식:** 15단계 · 03 (AlphaEvolve), 15단계 · 04 (DGM)
**소요 시간:** ~60분

## 문제 정의

연구는 열린 과제(open-ended task)입니다. AlphaEvolve의 알고리즘 검색이나 DGM의 벤치마크 범위 내 자기 수정과 달리, 연구 결과에는 기계가 검증 가능한 정확성 기준이 없습니다. 논문은 단위 테스트(unit test)가 아닌 리뷰어(reviewer)에 의해 평가됩니다. 이는 피드백 루프를 닫는 것을 더 어렵게 만들지만, 만약 닫힌다면 더 큰 가치를 지닙니다. 왜냐하면 연구는 복리적 진보(compounding progress)가 발생하는 영역이기 때문입니다.

AI Scientist v1(Sakana, 2024)은 인간이 작성한 템플릿에서 시작하여 루프를 닫았습니다. LLM은 고정된 구조(scaffolding) 내에서 실험을 채웠습니다. AI Scientist v2(Yamada et al., 2025)는 비전-언어 모델(vision-language model) 기반 크리티크 루프(critique loop)와 에이전트 기반 트리 검색(agentic tree search)을 사용하여 템플릿 요구 사항을 제거했습니다. 이 시스템은 아이디어를 생성하고, 실험을 구현하며, 그림을 제작하고, 논문을 작성한 후 리뷰어 피드백에 따라 반복적으로 개선합니다.

동료 평가(peer review) 결과: v2로 생성된 논문 한 편이 ICLR 2025 워크숍에 승인되었습니다(공개 포함). 독립 평가 결과: 이 시스템은 아직 신뢰할 수 있는 수준에 미치지 못합니다. 두 평가 모두 사실입니다.

## 개념

### 아키텍처

1. **아이디어 생성.** LLM은 주제와 선행 문헌을 조건으로 연구 아이디어를 제안합니다. v1은 템플릿을 사용했고, v2는 가설 공간을 대상으로 에이전트 기반 검색을 사용합니다.
2. **참신성 검증.** 문헌 검색 단계에서 아이디어가 출판되었는지 확인합니다. 이 단계에서 Beel 등은 오분류 문제를 발견했습니다. 기존 방법이 자주 참신하다고 분류되었습니다.
3. **실험 계획.** 에이전트가 실험 프로토콜을 작성하고 코드를 생성합니다.
4. **실행.** 코드는 샌드박스에서 실행됩니다. 실패 사례는 재시도 루프로 피드백됩니다. Beel 등의 측정에서 이 단계에서 42%의 실험이 코딩 오류로 실패했습니다.
5. **그림 생성.** 비전-언어 모델이 생성된 그림을 읽고 명확성을 위해 재작성합니다. 이는 v2의 주요 기술적 추가 사항입니다.
6. **작성.** LLM이 논문을 초안으로 작성하고 내부 검토자와 반복적으로 수정합니다.
7. **선택 사항: 제출.** 논문을 학회에 제출합니다.

### 워크숍 논문 채택 결과의 의미

v2로 생성된 논문 1편이 ICLR 2025 워크숍의 동료 검토를 통과했습니다. 저자들은 프로그램 위원회에 논문의 출처를 공개했습니다. 이 채택은 데이터 포인트일 뿐, "시스템이 연구를 수행한다"는 주장을 허용하는 라이선스가 아닙니다.

중요 맥락: 워크숍 논문은 메인 학회 논문보다 기준이 낮습니다. 동료 검토는 잡음이 많으며, 특정 날짜에 제출된 논문 중 소수만 채택됩니다. 한 번의 성공은 개념 증명일 뿐, 신뢰성 주장이 아닙니다. Nature 2026 논문은 종단 간 루프를 문서화했으며, 인간 연구자들이 공동 저자로 참여했습니다. "시스템이 Nature 논문을 작성했다"는 의미가 아닙니다.

### 독립 평가 결과

Beel 등(arXiv:2502.14297)이 외부 평가를 수행했습니다. 주요 결과:

- **실험 실패.** 42%의 실험이 코딩 오류(잘못된 임포트, 형태 불일치, 정의되지 않은 변수)로 실패했습니다. 재시도 루프가 일부는 잡았지만 전부는 아닙니다.
- **참신성 오분류.** 문헌 검색 단계가 기존 개념을 자주 참신하다고 표시했습니다. 이는 연구 분야의 환각(hallucination)과 동일합니다.
- **발표 품질 격차.** 비전-언어 모델의 그림 검토는 출판 수준의 시각 자료를 생성했지만, 근본적인 실험 약점을 가렸습니다.

마지막 결과가 이 단계에서 가장 중요합니다. 설득력 있는 연구를 수행하지 않으면서 설득력 있는 출력을 생성하는 시스템은 명백히 실패하는 시스템보다 더 위험합니다. 평가는 그림 수준에서 멈추지 않고 근본적인 주장에 도달해야 합니다.

### 샌드박스 탈출 우려

Sakana의 공식 저장소 README는 다음과 같이 경고합니다:

> LLM이 생성한 코드를 실행하는 소프트웨어의 특성상 안전을 보장할 수 없습니다. 위험한 패키지, 제어되지 않은 웹 접근, 의도하지 않은 프로세스 생성 등의 위험이 있습니다. 사용자 책임 하에 사용하며, Docker 격리를 고려하세요.

이는 검증되지 않은 영역에서 자율성의 운영적 형태입니다. LLM이 코드를 작성하고, 코드가 실행되며, 프로세스가 허용된 모든 작업을 수행할 수 있습니다. 파일 시스템, 네트워크, 프로세스 작업을 엄격히 제한하는 샌드박스가 없으면, 어떤 자기 주도 연구 에이전트도 데이터를 유출하거나, 컴퓨팅 자원을 소모하거나, 자기 자신을 재작성할 수 있습니다.

AlphaEvolve의 샌드박스 사례는 평가기가 엄격하기 때문에 더 쉽습니다. AI Scientist v2의 루프는 열린 목표와 함께 열린 코드를 실행합니다. 따라서 더 강력한 격리(Docker 최소 요건; seccomp / gVisor 권장)와 시스템 이탈 전 모든 제출물에 대한 수동 검토가 필요합니다.

### v2의 프론티어 스택 위치

| 시스템 | 대상 | 출력 유형 | 평가기 | 알려진 실패 사례 |
|---|---|---|---|---|
| AlphaEvolve | 알고리즘 | 코드 | 단위 + 벤치마크 | 평가기 엄격성에 의해 제한됨 |
| DGM | 에이전트 스캐폴딩 | 코드 | SWE-bench | 보상 해킹 |
| AI Scientist v2 | 연구 논문 | 텍스트 + 코드 + 그림 | 동료 검토(약한) | 실험 실패, 오분류, 약점이 가려진 폴리시 |

v2는 세 시스템 중 가장 약한 자동 평가기를 가지며, 가장 넓은 출력 범위와 공개 아티팩트까지의 최단 경로를 가집니다. 운영 제어(샌드박스, 검토, 공개)가 대부분의 안전 작업을 수행하고 있습니다.

## 사용 방법

`code/main.py`는 상태 머신으로서 v2 루프를 시뮬레이션합니다: 아이디어 → 참신성 검토 → 실험 → 그림 → 작성 → 검토 → 승인-반복. 각 상태는 Beel et al.의 연구 결과에서 추출한 구성 가능한 실패 확률을 가집니다. N번의 루프에 대해 시뮬레이터를 실행하고 다음을 계산합니다:

- 제출에 도달한 아이디어의 수.
- 완성된 논문이 숨기는 치명적인 실험 결함을 가진 제출물의 수.
- 재시도 예산이 품질 대비 산출량과 어떻게 상충되는지.

## Ship It

`outputs/skill-ai-scientist-sandbox-review.md`는 샌드박스를 떠나기 전 연구-루프 에이전트가 생성한 모든 산출물에 대한 2단계 검토 체크리스트입니다.

## 연습 문제

1. 기본 파라미터로 `code/main.py`를 실행해 보세요. 전체 루프 실행 중 몇 %가 "깨끗한" 논문을 생성하나요? 또 몇 %가 실험 실패 결함을 포함하지만 그림 검토로 다듬어진 논문을 생성하나요?

2. 기본값은 이미 Beel et al.의 42% / 25%를 사용합니다. `--experiment-failure 0.20 --novelty-mislabel 0.10`로 재실행한 후, `--experiment-failure 0.60 --novelty-mislabel 0.40`으로도 실행해 보세요. 두 실행 간 다듬어진 결함 논문의 비율이 어떻게 변하나요?

3. Sakana의 AI Scientist v2 레포지토리 README에서 샌드박스 요구사항을 읽고, Docker 외에 다일간 자율 실행을 위해 추가할 두 가지 제한 사항을 제안하세요.

4. Beel et al. 4장의 발표 품질 격차 내용을 읽고, 외관은 완벽하지만 실험적 결함이 있는 논문을 추가로 탐지할 수 있는 평가자(evaluator) 하나를 설계해 보세요.

5. "박사 과정 학생이 모든 논문을 직접 읽는" 방식보다 확장성 있는 연구 에이전트 출력물 검토 프로토콜을 제안하세요. 병목 현상을 식별하고 이를 해결할 방안을 설계해 보세요.

## 주요 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|---|---|---|
| AI Scientist v1 | "Sakana의 템플릿 연구 에이전트" | 고정된 실험 틀에 실험 내용을 채움 |
| AI Scientist v2 | "템플릿 없는 연구 에이전트" | VLM 그림 비평을 통한 에이전트 기반 트리 탐색 |
