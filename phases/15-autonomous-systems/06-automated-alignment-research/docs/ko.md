# 자동화된 정렬 연구 (Anthropic AAR)

> Anthropic은 독립적인 샌드박스에서 Claude Opus 4.6 자율 정렬 연구자(AAR) 팀을 병렬로 운영했으며, 공유 포럼을 통해 조정했습니다. 이 포럼의 로그는 모든 샌드박스 외부에 저장되어(에이전트가 자신의 기록을 삭제할 수 없음) 약-강 훈련 문제에서 AAR들은 인간 연구자들을 능가했습니다. Anthropic의 자체 요약에서는 규정된 워크플로우가 종종 AAR의 유연성을 제한하고 성능을 저하시킨다고 지적했습니다. 정렬 연구 자동화는 RSP(위험 신호 프로토콜)가 탐지하려는 정확한 오정렬 위험까지 타임라인을 압축하는 단계입니다.

**유형:** 학습  
**언어:** Python (stdlib, 병렬 연구 포럼 시뮬레이터)  
**선수 지식:** 15단계 · 05 (AI 과학자 v2), 15단계 · 04 (DGM)  
**소요 시간:** ~60분

## 문제 정의

정렬 연구는 인간 연구자의 시간 측면에서 비용이 많이 듭니다. 확장 가능한 감독, 보상 명세, 약-강 훈련(weak-to-strong training)과 같은 문제들은 한 번의 반복(iteration)에 몇 주가 소요되는 실험을 필요로 합니다. 최첨단 능력이 발전함에 따라 정렬 작업량은 자격을 갖춘 연구자 공급보다 더 빠르게 증가하고 있습니다.

자동화된 정렬 연구(AAR, Automated Alignment Research)는 정렬 속도를 앞지르는 동일한 최첨단 모델들이 이 격차를 줄이는 데 기여할 수 있는지 묻습니다. Anthropic의 2026년 보고서(alignment.anthropic.com/2026/automated-w2s-researcher/)는 AAR이 실행한 약-강 훈련 연구(weak-to-strong-training study)에 대한 첫 번째 공개 결과 중 하나로, 이 클래스의 배치된 시스템에서 나온 것입니다.

결과는 진정으로 긍정적입니다: AAR은 인간 기준선보다 연구 문제를 더 잘 해결했습니다. 이 결과는 또한 이 단계에서 발전된 모든 우려 사항을 단일 시스템에 집중시킵니다. 정렬 연구가 자동화될 수 있다면, 안전장치를 약화시키는 부분들도 자동화될 수 있습니다. RSP(Research Safety Protocol)의 AI R&D 능력 임계값은 이러한 루프를 염두에 두고 작성되었습니다.

## 개념

### 아키텍처 (공개적으로 설명된 내용)

- **병렬 에이전트.** 여러 Claude Opus 4.6 인스턴스가 AAR로 실행되며, 각각 독립적인 샌드박스에서 작동합니다. 샌드박스는 파일 시스템, 네트워크, 프로세스 표면을 제한합니다.
- **공유 포럼.** 연구 결과는 모든 샌드박스 외부에 위치한 저장 백엔드를 가진 공유 포럼에 게시됩니다. 에이전트는 포럼을 읽을 수 있지만, 자신의 샌드박스 내에서 포럼 기록을 삭제하거나 편집할 수 없습니다.
- **작업 할당.** 인간이 설계한 작업 큐가 AAR에 연구 문제를 할당합니다. AAR은 작업을 분해하고, 실험을 실행하며, 연구 결과를 작성합니다.
- **약-강 훈련 집중.** 공개된 연구는 약-강 훈련(약한 감독 모델을 사용하여 더 강력한 학생 모델을 훈련)을 다루었습니다. AAR은 이 작업에서 인간 연구자를 능가했습니다.

### 샌드박스 외부 로그의 중요성

로그 무결성 속성은 통제된 실험과 잠재적으로 손상된 실험의 차이입니다. AAR이 자신의 로그를 편집할 수 있다면 실패하거나 의심스러운 실험을 숨길 수 있습니다. Anthropic은 이를 직접 지적합니다: 이 속성은 연구 결과를 신뢰하기 위한 필수 조건이며, 검증이 쉽지 않습니다.

인프라 측면에서 이는 외부 저장소(예: S3 객체 잠금, 불변 Datadog 로그)에 대한 추가 전용 로그와 유사합니다. 에이전트는 과거 기록을 수정할 수 있는 자격 증명을 갖지 않습니다. 이는 2026년 기업 보안 팀이 랜섬웨어에 강한 감사 추적을 만드는 데 사용하는 동일한 패턴입니다.

### 규정된 워크플로우의 트레이드오프

Anthropic의 보고서는 규정된 워크플로우가 종종 AAR의 유연성을 제한하고 성능을 저하시킨다고 언급합니다. 즉, 인간 설계자가 연구 계획을 단계별로 작성하면 AAR은 고수준 목표와 분해 자유를 부여받은 경우보다 성능이 떨어집니다. 이는 에이전트 연구(LangGraph, Microsoft의 에이전트 프레임워크) 전반에 걸친 결과와 일치합니다: 과도하게 지정된 스캐폴드는 능력 있는 에이전트를 해칩니다.

안전 측면의 함의는 미묘합니다. AAR 성능을 향상시키는 유연성은 승인되지 않은 분해, 새로운 도구 사용, 목표 이탈을 허용하는 동일한 유연성입니다. 트레이드오프는 설계 공간 결정입니다: 얼마나 많은 유연성이 인간을 능가할 수 있는 능력을 사들이면서도, 에이전트가 목표를 분포 외부로 일반화할 여지를 주지 않을까요?

### 압축 위험

RSP v3.0(레슨 19)은 AI R&D 능력 임계값을 도입합니다: 인간 + AI 도구와 경쟁력 있는 비용으로 AI R&D 파이프라인을 완전히 자동화하는 능력. DeepMind의 FSF v3에는 유사한 ML R&D 자율성 수준이 포함됩니다. 두 프레임워크 모두 이 임계값을 강화된 제어의 트리거로 취급합니다.

AAR은 임계값에 한 걸음 못 미칩니다: 파이프라인의 일부(특정하고 범위가 잘 정의된 작업에 대한 정렬 연구)를 자동화하지만, 엔드투엔드 능력 개발 루프는 자동화하지 않습니다. 타임라인 질문은 이 격차가 얼마나 빨리 좁혀지는지입니다.

압축된 타임라인은 복합 실패 우려입니다. 정렬 연구와 능력 연구가 비슷한 속도로 복합된다면, 오정렬 위험 표면은 능력만큼 빠르게 성장합니다. 능력이 더 빠르게 복합된다면(역사적 추세), 격차는 확대됩니다. 이는 AAR이 조건부 선(good)이라는 주장입니다: 각 추가 정렬 결과는 연구 프로세스가 신뢰할 수 있는 경우에만 격차를 줄입니다.

### AAR이 대체하지 않는 것

인간 연구자는 작업 큐를 설정하고, 결과를 검토하며, 헌법상의 권한을 보유합니다. AAR은 파이프라인의 중간을 가속화하지만, 양 끝은 아닙니다. Anthropic의 공개된 출력에는 AAR 기여와 인간 연구자의 판단(게재할 내용, 철회할 내용, 개선할 내용)이 모두 포함됩니다.

이는 레슨 15의 제안-커밋 패턴을 연구 자체에 적용한 것과 일치합니다: AAR은 제안하고, 인간은 커밋합니다.

## 사용 방법

`code/main.py`는 다중 AAR 연구 포럼을 시뮬레이션합니다. 세 개의 독립적인 "연구 에이전트"가 병렬로 실행되며, 선택적 변조 시도와 함께 추가 전용 포럼에 연구 결과를 게시합니다.

관찰 포인트:

- 샌드박스 외부 로깅이 로그 변조 시도를 어떻게 탐지하는지.
- 자유 분해가 포함된 작업 할당이 더 어려운 문제에서 고정 워크플로우보다 어떻게 우수한 성능을 보이는지.
- 로그가 변조 방지 기능이 없을 경우 단일 손상된 AAR이 포럼의 신호를 어떻게 오염시킬 수 있는지.

## 연습 문제

1. `code/main.py`를 실행하세요. "fixed-workflow" vs "free-decomposition" 설정을 비교하세요. 자유 분해가 항상 우수한지, 아니면 고정 워크플로우가 경쟁력 있는 문제 클래스가 있는지 확인하세요.

2. 시뮬레이터를 수정하여 한 에이전트가 로그 변조를 시도하도록 만드세요. 추가 전용 로그가 이를 탐지하는지 확인하세요. 로그에서 탐지 시그니처가 정확히 어떻게 나타나는지 설명하는 단락을 작성하세요.

3. Anthropic의 약한-강한 AAR 보고서를 읽으세요. AAR이 인간 연구자를 능가한 특정 하위 작업을 식별하세요. 어떤 특성이 자동화에 적합하게 만들었는지 설명하세요.

4. AAR 유연성(더 나은 결과)과 규정된 워크플로우 제약(감사 용이성)을 균형 있게 조정하는 작업 큐 할당 정책을 설계하세요. 두 방식을 A/B 테스트하는 방법을 설명하세요.

5. RSP v3.0의 AI R&D-4 임계값을 읽으세요. 한 단락으로 현재 AAR이 충족하지 못하지만 이를 충족할 것으로 생각되는 사항을 설명하세요.

## 주요 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|---|---|---|
| AAR | "자동 정렬 연구자(Automated Alignment Researcher)" | 정렬 문제에 대해 자율적으로 운영되는 Claude Opus 4.6 인스턴스 |
| 약→강 훈련(Weak-to-strong training) | "더 약한 감독자로 더 강한 모델 훈련" | 확장 가능한 감독(scalable-oversight) 벤치마크에서 AAR이 인간보다 우수한 성능을 보인 사례 |
| 공유 포럼(Shared forum) | "에이전트들이 연구 결과를 게시하는 곳" | 추가 전용(append-only) 샌드박스 외부 저장소 |
| 샌드박스 외부 로그(Out-of-sandbox log) | "에이전트가 자신의 기록을 편집할 수 없음" | 외부 저장소에 대한 변조 방지(write-through) 기록 |
| 규정된 워크플로우(Prescribed workflow) | "인간 설계자가 만든 단계별 계획" | AAR을 제한하며, 자유 분해(free decomposition) 대비 성능 저하 유발 |
| 자유 분해(Free decomposition) | "에이전트가 작업 분할 방식을 스스로 결정" | 더 높은 능력 보유, 감사(audit)는 더 어려움 |
| AI R&D 임계값(AI R&D threshold) | "RSP/FSF 능력 수준" | 경쟁력 있는 비용으로 R&D 파이프라인 완전 자동화 달성 |
| 압축된 타임라인(Compressed timeline) | "정렬 vs 능력 경쟁" | 능력이 정렬보다 빠르게 복합화되면 오정렬 위험 증가 |
