# STaR, V-STaR, Quiet-STaR — 자기 주도 추론(Self-Taught Reasoning)

> 가장 작은 자기 개선 루프는 근거(rationale) 내부에 위치합니다. 모델은 사고 연쇄(chain of thought)를 생성하고, 정답에 도달한 것들을 유지한 후, 그것들로 파인튜닝(fine-tuning)합니다. 이것이 STaR입니다. V-STaR은 검증기(verifier)를 추가하여 추론 시간 선택(inference-time selection)을 개선합니다. Quiet-STaR은 근거(rationale)를 모든 토큰(token)으로 확장합니다. 이 세 방법 모두 효과가 있습니다. 하지만 마법은 없습니다 — 이 루프는 우연히 정답에 도달한 어떤 지름길(shortcut)도 보존합니다.

**유형:** 학습(Learn)  
**언어:** Python (stdlib, 부트스트랩 루프 시뮬레이터)  
**선수 지식:** 13단계 · 01-03 (추론 및 사고 연쇄), 15단계 · 01 (장기적 프레이밍)  
**소요 시간:** ~60분

## 문제 정의

모델에 추론 능력을 가르치는 직관적인 방법은 인간이 작성한 추론 과정을 수집하는 것입니다. 그러나 이 방법은 비용이 많이 들고 느리며, 인간이 기꺼이 작성할 수 있는 고품질 사고 체인의 양에 의해 제한됩니다.

STaR(Self-Taught Reasoner, Zelikman et al., 2022)은 다음과 같은 질문을 던집니다: 모델이 스스로 근거를 작성하고 알려진 답변과 비교하여 평가한다면 어떨까요? 이 루프는 다음과 같습니다:

1. 추론 과정과 답변을 샘플링합니다.
2. 최종 답변이 정확하면 해당 추론 과정을 유지합니다.
3. 유지된 추론 과정으로 파인튜닝(fine-tuning)을 수행합니다.
4. 반복합니다.

이 방법은 효과가 있습니다. GSM8K와 CommonsenseQA 모두에서 새로운 인간 주석 없이도 성능이 향상되었습니다. 그러나 이 루프에는 내재된 편향이 있습니다: 추론 과정 자체의 타당성과 관계없이 정답을 생성한 모든 근거가 유지됩니다. V-STaR(Hosseini et al., 2024)은 학습된 검증기(verifier)를 통해 이 문제를 해결하며, Quiet-STaR(Zelikman et al., 2024)은 이 아이디어를 토큰별 내부 근거(per-token internal rationales)로 일반화합니다.

## 개념

### STaR: 성공한 것에 대한 부트스트랩

약한 추론 능력을 가진 기본 모델에서 시작합니다. 각 훈련 문제에서 근거(rationale)와 답변을 샘플링합니다. 답변이 라벨과 일치하면 (문제, 근거, 답변) 트리플을 유지합니다. 유지된 집합으로 모델을 파인튜닝합니다. 이 과정을 반복합니다.

한 가지 중요한 변형이 있습니다. 모델이 문제를 절대 맞출 수 없다면 루프는 해당 문제에 대해 학습할 수 없습니다. STaR은 **합리화(rationalization)**를 추가합니다. 모델이 실패한 문제에 대해 정답을 힌트로 주입하고, 모델이 이를 유도하는 근거를 생성하도록 다시 프롬프팅합니다. 합리화된 근거는 훈련 세트에 추가됩니다.

원본 논문(Zelikman et al., 2022) 결과: GPT-J 기본 모델이 합리화를 포함한 반복적인 STaR 라운드를 통해 GSM8K에서 5.8%에서 10.7%로 개선되었습니다. 절대 5% 포인트 향상입니다. CommonsenseQA에서는 STaR로 훈련된 GPT-J 6B가 72.5%를 달성했으며, 이는 핸드 어노테이션된 근거로 훈련된 GPT-3 175B(~73%)와 비슷한 성능입니다. 이는 약 30배 더 큰 모델입니다.

### V-STaR: DPO로 검증자 훈련

STaR은 잘못된 근거를 버립니다. Hosseini et al. (2024)는 이러한 근거들도 데이터임을 관찰했습니다. (근거, "이것이 맞는가?") 쌍은 검증자를 훈련할 수 있습니다. 그들은 올바른 해와 잘못된 해 모두에 대해 직접 선호 최적화(Direct Preference Optimization)를 사용하여 랭커를 구축합니다. 추론 시에는 N개의 근거를 샘플링하고 검증자의 최상위 선택을 고릅니다.

보고된 성능 향상: GSM8K와 MATH에서 기존 자기 개선 기준선 대비 +4~+17% 포인트 향상. 이 중 대부분은 추가 생성기 파인튜닝이 아닌 추론 시 검증자 선택에서 비롯됩니다.

### Quiet-STaR: 토큰별 내부 근거

Zelikman et al. (2024)는 다음과 같은 질문을 던졌습니다. 모델이 문제와 답변 사이뿐만 아니라 모든 토큰 위치에서 짧은 내부 근거를 생성하도록 학습한다면 어떨까요? Quiet-STaR은 모델이 각 예측 토큰 전에 숨겨진 "생각"을 생성하도록 훈련한 후, 학습된 가중치를 통해 생각이 포함된 예측과 기본 예측을 혼합합니다.

결과: Mistral 7B는 작업별 파인튜닝 없이도 GSM8K에서 5.9%에서 10.9%, CommonsenseQA에서 36.3%에서 47.2%로 절대 제로샷 개선되었습니다. 모델은 "언제 생각해야 하는지"를 학습했습니다. 어려운 토큰은 더 긴 내부 근거를 생성하고, 쉬운 토큰은 거의 생성하지 않습니다.

### 세 방법 모두 공유하는 안전 문제

세 방법 모두 최종 답변을 그래디언트 신호로 사용합니다. 올바른 답변에 도달하지만 결함이 있는 추론(숏컷 활용, 추측, 일반화되지 않는 패턴 사용)을 하는 근거는 긍정적으로 강화됩니다. 숏컷이 작동하는 분포 내 문제에서는 잘 동작하지만, 분포 외 문제에서는 소리 없이 실패합니다.

V-STaR의 검증자는 근거 순위를 학습함으로써 완화하지만, 검증자는 동일한 라벨 세트로 훈련됩니다. 이는 정직한 불확실성보다 잘 포맷된 잘못된 추론을 선호하도록 학습할 수 있습니다. 더 안전한 설계는 (a) 중간 단계를 보상하는 프로세스 감독 보상 모델과 (b) 간단한 숏컷을 깨뜨리는 외부 OOD 평가를 STaR 스타일 데이터와 결합하는 것입니다.

### 비교

| 방법 | 훈련 신호 | 추론 비용 | 데이터 낭비 | 알려진 실패 모드 |
|---|---|---|---|---|
| STaR | 정답일 경우 (근거, 답변) 유지 | 1x | 모든 잘못된 근거 폐기 | 숏컷 근거 |
| STaR + 합리화 | 위 + 정답 힌트 재시도 | 1x | 적음 | 합리화된 근거가 비현실적일 수 있음 |
| V-STaR | STaR + 두 클래스의 DPO 검증자 | Nx (N개 중 최고) | 최소 | 검증자가 확신하는 오류를 강화할 수 있음 |
| Quiet-STaR | 토큰별 근거 + 혼합 가중치 | 1.5-3x | 최소 | 여전히 답변 조건부 그래디언트 |

### 2026년 스택에서의 위치

STaR은 오래되었습니다. 그러나 이 패턴은 2025-2026년 도처에서 재등장합니다. 검증 가능한 수학 문제에 대한 RL(DeepSeek-R1, Kimi-k1.5, o1)은 STaR의 답변 조건부 그래디언트 신호를 확장한 것입니다. 프로세스 보상 모델(Lightman et al., 2023; OpenAI의 "단계별로 검증해 보자")은 프로세스 감독 대안입니다. AlphaEvolve(레슨 3)는 라벨 대신 프로그램 평가자를 사용하는 코드용 STaR입니다. Darwin Godel Machine(레슨 4)은 에이전트 스캐폴딩 자체를 위한 STaR입니다.

STaR을 이해하면 이 모든 것이 이해됩니다. 이는 최소 실행 가능 자기 개선 루프입니다.

## 사용 방법

`code/main.py`는 장난감 산술 작업에서 시뮬레이션된 STaR 루프를 실행합니다. 다음을 확인할 수 있습니다:

- 부트스트랩 라운드에 따라 정확도가 어떻게 상승하는지.
- "게으른" 추론 클래스가 40% 확률로 정답을 얻지만 일반화가 잘 되지 않는 상황에서, STaR이 이러한 **지름길(shortcut)**을 유지하는지 관찰.
- 검증기(V-STaR 스타일)가 추론 시 도움이 되지만, 훈련 중 도입된 **지름길(shortcut)**을 완전히 제거하지는 못하는지 확인.

## Ship It

`outputs/skill-star-loop-reviewer.md`는 훈련 전에 제안된 자기-교수-추론 파이프라인을 감사하는 데 도움을 줍니다.

## 연습 문제

1. 시뮬레이터를 실행합니다. 단축키 빈도를 0으로 설정한 후 0.4로 변경합니다. 두 실행 모두 훈련 분포에서 >90% 정확도를 달성했음에도 최종 정확도가 얼마나 차이 나는지 확인합니다.

2. 시뮬레이터에 별도의 OOD(분포 외) 테스트 세트를 추가합니다. 다른 분포에서 문제를 샘플링하고 부트스트랩된 모델을 분포 내 및 OOD 세트에서 평가합니다. 성능 차이를 정량화합니다.

3. Quiet-STaR 논문(arXiv:2403.09629) 3절을 읽고 다음 내용을 각각 3문장으로 설명합니다.  
   - "사고 종료(end-of-thought)" 토큰의 역할  
   - 혼합 가중치(mixing-weight) 헤드의 기능  

4. STaR의 "정답일 때만 유지(keep-if-correct)" 필터와 각 추론 단계를 독립적으로 보상하는 프로세스 감독 방식을 비교합니다.  
   - 라벨링 비용 차이  
   - 추론 품질 차이  

5. 배포된 모델에서 단축키 추론을 탐지할 수 있는 평가 방법을 설계합니다. 완벽할 필요는 없으며, STaR 루프가 강화할 수 있는 가장 단순한 단축키 추론을 포착할 수 있어야 합니다.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|---|---|---|
| STaR | "Self-Taught Reasoner" | 정답에 도달한 모델 생성 근거(rationales)로 미세 조정(fine-tuning) 후 반복 |
| Rationalization | "힌트 재시도" | 기본 모델이 실패한 문제에 정답을 주입하고 재프롬프트하여 근거 생성 |
| V-STaR | "검증자 STaR" | 정답과 오답 근거 모두로 DPO 훈련한 검증자, 추론 시 선택용으로 사용 |
| Quiet-STaR | "토큰별 근거" | 모든 토큰 위치에서 숨겨진 생각 생성, 기본 예측과 혼합 |
| 답변 조건 기울기 | "결과 기반 신호" | 훈련 루프는 추론 단계가 아닌 최종 답변에 보상 부여 |
| 과정 보상 모델 | "단계별 검증자" | 결과가 아닌 단계별 정확도로 훈련된 보상 모델 — STaR과 대조적 |
| 숏컷 근거 | "정답은 맞았지만 추론은 틀림" | 일반화되지 않는 패턴으로 라벨에 도달한 근거; STaR은 이를 유지 |
