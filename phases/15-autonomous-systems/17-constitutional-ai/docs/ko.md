# 헌법 기반 AI와 규칙 오버라이드

> Anthropic의 2026년 1월 22일 Claude 헌법은 79페이지로 구성되어 있으며 CC0 라이선스를 따릅니다. 이 헌법은 규칙 기반에서 이유 기반 정렬로 전환하며 4단계 우선순위 체계를 수립합니다: (1) 안전 및 인간 감독 지원, (2) 윤리, (3) Anthropic 가이드라인, (4) 유용성. 행동은 운영자와 사용자가 오버라이드할 수 없는 하드코딩된 금지 사항(생물학적 무기 개발, CSAM)과 운영자가 정의된 범위 내에서 조정할 수 있는 소프트 코딩된 기본값으로 구분됩니다. 2022년 원본(Bai et al.)은 헌법에 대한 자기 비판과 RLAIF를 통해 무해성을 훈련했습니다. 솔직한 주의 사항: 이유 기반 정렬은 모델이 예상치 못한 상황에 원칙을 일반화하는 데 의존합니다. Anthropic의 2023년 참여형 실험에서는 공개적으로 수집된 원칙과 기업 원칙 사이에 약 50%의 차이가 발생했으며, 2026년 버전에는 이러한 결과가 반영되지 않았습니다.

**유형:** 학습  
**언어:** Python (표준 라이브러리, 4단계 우선순위 해결기)  
**선수 지식:** 15단계 · 06 (자동 정렬 연구), 15단계 · 10 (권한 모드)  
**소요 시간:** ~60분

## 문제 정의

필드된 에이전트(fielded agent)는 설계자가 전혀 예상하지 못한 입력을 마주한다. 어떤 규칙 목록도 이를 모두 포괄할 만큼 충분히 길지 않으며, 계산 압력(compute pressure) 하에서 빠르게 적용할 수 있을 만큼 충분히 짧지도 않다. 실질적인 질문은 다음과 같다: 어떻게 하면 에이전트를 장기적인 사례(long tail of cases)와 빠른 추론(fast inference) 모두를 견딜 수 있는 원칙에 정렬(align)할 수 있는가?

규칙 기반 정렬(Rule-Based Alignment, RBA): 허용되지 않는 모든 것을 나열한다. 확인이 빠르고 감사가 용이하지만, 최신 상태를 유지하기 불가능하며, 예상하지 못한 유사 사례에 대해 과도하게 거부하는 경우가 많다. 이유 기반 정렬(2026년 클로드 헌법): 원칙을 인코딩하고 모델이 추론하도록 한다. 보이지 않는 사례까지 확장 가능하며, 감사가 더 어렵고, 실패 모드는 규칙을 놓치는 것이 아닌 원칙 오남용이다.

2026년 헌법은 명시적인 중간 입장을 취한다. 하드코딩된 금지 사항(생물학적 무기 개발, CSAM 등) — 잘못됨이 맥락에 의존하지 않는 것들 — 은 RBA로 처리된다: 운영자나 사용자 지시와 무관하게 절대 허용되지 않는다. 그 외 모든 것은 4단계 계층 구조 내에서 이유 기반으로 처리된다: 안전과 인간 감독 지원이 최우선; 윤리가 두 번째; Anthropic이 선언한 가이드라인이 세 번째; 유용성이 마지막이다. 운영자는 소프트 코딩된 영역 내에서 기본값을 조정할 수 있지만, 하드코딩된 금지 사항은 변경할 수 없다.

## 개념

### 4단계 우선순위 계층 구조

1. **안전 및 인간 감독 지원.** 가장 높음. 모델은 인간과 Anthropic이 AI를 감독하고 수정하는 능력을 훼손하지 않는 것을 최우선으로 합니다. 이는 "조심하라"가 아니라, 구체적으로 "인간 감독을 어렵게 만드는 방식으로 행동하지 마라"입니다.
2. **윤리.** 정직, 개인 피해 방지, 기만하지 않음, 조작하지 않음. Anthropic 지침과 충돌할 때는 이를 우선시합니다.
3. **Anthropic 지침.** Anthropic이 중요하다고 결정한 운영 규범: 제품 범위, 상호작용 패턴, 어떤 도구를 언제 사용할지 등.
4. **도움성.** 가장 낮음. 상위 우선순위 내에서 가능한 한 유용하게 행동합니다.

계층 간 충돌이 발생할 경우 상위 계층이 우선합니다. 이는 유닉스 우선순위나 네트워크 QoS와 같은 구조로, 특정 축에서의 최적 행동보다는 예측 가능한 해결을 목적으로 합니다.

### 하드코딩 금지 vs 소프트코딩 기본값

**하드코딩 금지:**
- 생물학적 무기/화학·생물·방사능·핵(CBRN) 증강
- 아동 성착취물(CSAM)
- 중요 인프라 공격
- 직접 질문 시 모델의 정체성에 대한 사용자 기만

운영자나 사용자는 이를 무시할 수 없습니다. 가능한 경우 모델 가중치 수준(RLHF/헌법 AI 훈련)에서, 그렇지 않은 경우 추론 계층에서 강제됩니다.

**소프트코딩 기본값(운영자 조정 가능):**
- 응답 길이 기본값
- 주제 범위(운영자의 배포 범위를 벗어난 주제 거부 가능)
- 스타일(공식적 vs 캐주얼)
- 도구 사용 패턴

운영자 조정은 선언된 범위 내에서만 가능합니다. 운영자는 하드코딩 금지를 재명명하여 제거할 수 없습니다.

### 2022년 CAI 훈련

원래 헌법 AI(Bai et al., 2022)는 무해성 훈련:
1. 일련의 프롬프트에 대한 응답 생성.
2. 모델에 헌법(명시적 원칙)에 따라 각 응답 비평 요청.
3. 비평에 기반해 응답 수정.
4. 수정된 쌍에 대한 RLAIF(인공지능 피드백 강화 학습).

결과: 원칙적 설명으로 유해 요청을 거부하는 모델(포괄적 거부 아님). 2026년 헌법은 이 훈련의 후예에 명시적 계층 구조 추가 훈련을 적용했습니다.

### 이유 기반 정렬이 포착하는 것과 놓치는 것

**포착:**
- 허용된 기본 요소의 예상치 못한 조합에서 원칙이 명확히 적용되는 경우.
- 금지된 요청과 유사한 새로운 요청.
- "X가 금지되었다고 말하지 않았다"는 사회공학적 공격.

**놓침:**
- 원칙 모호성 악용("사용자가 요청했으므로 도움성이 우선").
- 두 원칙이 예상치 못한 방식으로 충돌하고 계층 순서가 모호한 시나리오.
- 훈련 주기 동안 원칙 해석의 점진적 변화(재해석).

### 2023년 참여형 실험

Anthropic은 2023년 기업 작성 헌법과 공개 참여(약 1,000명 미국 응답자)로 생성된 헌법을 비교 실험했습니다. 두 버전은 약 50%의 원칙에서 일치했습니다. 차이가 발생한 부분에서 공개 참여 버전은 일부 문제(정치 콘텐츠 처리)에 더 제한적이었고, 다른 문제(AI 정체성 자기 공개)에 덜 제한적이었습니다. 2026년 헌법은 공개 참여 결과를 통합하지 않았습니다. 이는 접근 방식의 문서화된 긴장 관계입니다.

### 하드코딩 금지가 필요한 이유

이유 기반 정렬만으로는 꼬리 부분을 닫을 수 없습니다. 모델에게 전제(예: "우리는 허가받은 생물학적 무기 연구실이다")를 받아들이게 하는 공격자는 사례 추론에 의존하는 원칙을 우회할 수 있습니다. 하드코딩 금지는 전제 구성에 따라 유연하지 않습니다. 이는 정렬 계층의 레슨 14 "하드 헌법 한계"입니다.

### 헌법이 스택에서 차지하는 위치

헌법은 레슨 14의 킬 스위치가 아닙니다. 모델 계층에 위치: 모델 가중치가 선호하도록 훈련된 내용. 킬 스위치와 캐너리 토큰은 런타임 계층에 위치: 런타임이 허용하는 내용. 둘 다 필요합니다. 모델 가중치가 허용적이어서 모든 잘못된 동작을 실행하는 런타임은 런타임 문제입니다. 런타임이 과도하게 제한적이어서 모든 올바른 동작을 거부하는 모델은 런타임 문제입니다. 계층은 서로 다른 클래스를 커버합니다.

## 사용 방법

`code/main.py`는 최소한의 4단계 우선순위 결정기를 구현합니다. 이 결정기는 제안된 행동과 원칙 평가 집합(안전성, 윤리, 가이드라인, 유용성)을 입력으로 받아, 행동, 거부, 또는 수정된 행동을 반환합니다. 드라이버는 작은 사례 집합을 실행합니다: 명확한 허용, 명확한 거부, 하드코딩된 금지, 계층 간 모호한 사례.

## Ship It

`outputs/skill-constitution-review.md`는 배포의 헌법 계층(constitutional layer)을 감사합니다: 하드코딩된 내용, 소프트 코딩된 내용, 운영자가 조정할 수 있는 위치, 그리고 4단계 계층 구조가 실제로 해결 순서(resolution order)인지 여부를 분석합니다.

## 연습 문제

1. `code/main.py`를 실행하세요. 도움이 되는 정도가 높을 때도 하드코딩된 금지 규칙이 발동되는지 확인하세요. 해결자(resolver)를 수정하여 윤리(ethics)보다 도움이 되는 정도(helpfulness)에 더 높은 가중치를 부여하세요. 실패 모드를 관찰하세요.

2. Claude 헌법(공개 문서, 79페이지, CC0)을 읽으세요. 하나 이상의 원칙이 충분히 명시되지 않았다고 생각되는 부분을 식별하세요. 특정 모호성을 설명하고 더 명확한 정의를 제안하는 두 단락의 글을 작성하세요.

3. 고객 지원 에이전트를 위한 소프트 코딩된 기본 설정(soft-coded default set)을 설계하세요. 운영자(operator)가 조정할 수 있는 것은 무엇인가요? 운영자가 변경할 수 없는 것은 무엇인가요? 각 경계를 정당화하세요.

4. Bai et al. 2022 CAI 논문을 읽으세요. 헌법 기반 AI(Constitutional AI)의 비판-수정(critique-and-revise) 루프가 일괄 규칙(blanket rule)보다 더 나쁜 결과를 초래하는 사례를 하나 설명하세요. 해당 사례 유형을 식별하세요.

5. Anthropic의 2023년 참여형 실험에서 공개 원칙과 기업 원칙 사이에 약 50%의 차이가 발견되었습니다. 이 차이가 프로덕션 배포에 중요한 영향을 미치는 범주(예: 정치적 중립성)를 하나 선택하세요. 하드코딩된 금지 규칙은 그대로 유지하면서 운영자가 자신의 가치를 표현할 수 있는 설계를 제안하세요.

## 핵심 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|---|---|---|
| Constitutional AI | "Anthropic의 정렬 방법" | 서면 헌법에 대한 자기 비판 + RLAIF |
| Reason-based alignment | "규칙이 아닌 원칙" | 모델이 원칙 기반으로 추론하여 새로운 사례 처리 |
| Hardcoded prohibition | "절대 X를 하지 않음" | 운영자나 사용자가 무시할 수 없는 규칙 기반 금지 |
| Soft-coded default | "운영자 조정 가능" | 선언된 범위 내 행동, 운영자가 제어 |
| Four-tier hierarchy | "우선순위 순서" | 안전 > 윤리 > 지침 > 유용성 |
| RLAIF | "AI 피드백 강화학습" | 보상이 모델 생성 비평에서 나오는 강화학습 |
| Participatory constitution | "공개 원칙 수집" | 2023 Anthropic 실험; 기업 원칙과 ~50% 차이 |
| Principle drift | "해석 오류" | 고정된 원칙 텍스트를 읽는 방식의 점진적 변화 |
