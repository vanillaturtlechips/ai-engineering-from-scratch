# 브라우저 에이전트와 장기 웹 작업

> ChatGPT 에이전트(2025년 7월)는 Operator와 심층 연구를 하나의 브라우저/터미널 에이전트로 통합했으며 BrowseComp SOTA를 68.9%로 설정했습니다. OpenAI는 2025년 8월 31일 Operator를 종료했습니다 — 제품 계층에서의 통합. Anthropic의 Vercept 인수는 OSWorld에서 Claude Sonnet을 15% 미만에서 72.5%로 끌어올렸습니다. WebArena-Verified(ServiceNow, ICLR 2026)는 원본 WebArena의 11.3% 포인트 위음성률을 수정하고 258개 작업 하드 서브셋을 출시했습니다. 이 숫자들은 실제입니다. 공격 표면도 마찬가지입니다: OpenAI의 준비 담당 책임자는 브라우저 에이전트로의 간접 프롬프트 주입이 "완전히 패치할 수 있는 버그가 아니다"라고 공개적으로 언급했습니다. 2025–2026년 문서화된 공격: Tainted Memories(Atlas CSRF), HashJack(Cato Networks), Perplexity Comet의 원클릭 하이재킹.

**유형:** 학습  
**언어:** Python(표준 라이브러리, 간접 프롬프트 주입 공격 표면 모델)  
**선수 지식:** 15단계 · 10(권한 모드), 15단계 · 01(장기 에이전트)  
**소요 시간:** ~45분

## 문제 정의

브라우저 에이전트(browser agent)는 신뢰할 수 없는 콘텐츠를 읽고 중대한 행동을 취하는 장기 호라이즌(long-horizon) 에이전트입니다. 에이전트가 방문하는 모든 페이지는 사용자가 작성하지 않은 입력입니다. 모든 페이지의 폼(form)은 잠재적인 명령 채널입니다. 2025–2026 공격 코퍼스(attack corpus)는 이것이 가설이 아님을 보여줍니다: "Tainted Memories"는 공격자가 조작된 페이지를 통해 악성 명령을 에이전트의 메모리에 바인딩할 수 있게 하고, "HashJack"은 에이전트가 방문하는 URL 조각에 명령을 숨기며, "Perplexity Comet"은 단일 클릭으로 히트를 탈취합니다.

방어 상황은 불편합니다. OpenAI의 준비 담당 책임자는 공개적으로 언급했습니다: 간접 프롬프트 주입(indirect prompt injection)은 "완전히 패치할 수 있는 버그가 아니다." 이는 공격이 에이전트의 읽기-행동 경계에서 발생하기 때문입니다. 이 경계는 구조적으로 모호하며 — 모델이 읽는 모든 토큰이 원칙적으로 명령으로 읽힐 수 있습니다.

이 레슨은 공격 표면을 명명하고, 벤치마크 환경(BrowseComp, OSWorld, WebArena-Verified)을 소개하며, 최소한의 간접 프롬프트 주입 시나리오를 모델링하여 레슨 14와 18에서 실제 방어 전략을 추론할 수 있도록 합니다.

## 개념

### 2026년 시스템별 전망, 시스템당 한 단락으로 요약

**ChatGPT 에이전트(OpenAI).** 2025년 7월 출시. 오퍼레이터(브라우저 검색)와 딥 리서치(다중 시간 연구)를 통합. 독립형 오퍼레이터는 2025년 8월 31일 종료. BrowseComp에서 SOTA(68.9%), OSWorld 및 WebArena-Verified에서 강력한 성능.

**Claude Sonnet + Vercept(Anthropic).** Vercept 인수는 컴퓨터 사용 능력에 집중. OSWorld에서 Claude Sonnet 성능을 <15%에서 72.5%로 향상. Claude Computer Use는 도구 API로 제공.

**Gemini 3 Pro with Browser Use(DeepMind).** Browser Use 통합은 컴퓨터 사용 제어 기능 제공; FSF v3(2026년 4월, 레슨 20)은 ML R&D 도메인에서의 자율성 추적.

**WebArena-Verified(ServiceNow, ICLR 2026).** 잘 알려진 문제 해결: 원본 WebArena은 약 11.3%의 거짓 음성률(실제 해결된 작업이 실패로 표시됨)을 가짐. Verified 릴리스는 인간이 선별한 성공 기준으로 재평가하고 258개 작업의 Hard 서브셋 추가(ICLR 2026 논문, openreview.net/forum?id=94tlGxmqkN).

### BrowseComp vs OSWorld vs WebArena

| 벤치마크 | 측정 항목 | 시간 범위 |
|---|---|---|
| BrowseComp | 시간 압박 하에서 오픈 웹에서 특정 사실 찾기 | 수분 |
| OSWorld | 전체 데스크톱(마우스, 키보드, 셸) 운영 에이전트 | 수십 분 |
| WebArena-Verified | 시뮬레이션 사이트에서 트랜잭션 웹 작업 | 수분 |
| Hard 서브셋 | 다중 페이지 상태 전환이 필요한 WebArena-Verified 작업 | 수십 분 |

서로 다른 축. 높은 BrowseComp 점수는 에이전트가 사실을 찾는다는 의미이지만, 항공권 예약 능력을 의미하지는 않음. OSWorld 점수는 "내 데스크톱에서 작동하는가"에 가까움. WebArena-Verified는 "플로우를 완료할 수 있는가"에 가까움. 프로덕션 결정은 작업 분포와 일치하는 벤치마크가 필요.

### 명명된 공격 표면

1. **간접 프롬프트 주입.** 신뢰할 수 없는 페이지 내용에 지시사항이 포함. 에이전트가 이를 읽고 실행. 공개 사례: 2024년 Kai Greshake 등, 2025년 Tainted Memories 논문, 2026년 HashJack(Cato Networks).
2. **URL 조각/쿼리 주입.** 크롤링된 URL의 `#조각` 또는 쿼리 문자열에 명령어 포함. 시각적으로 렌더링되지 않지만 에이전트 컨텍스트 내부에 존재.
3. **메모리 바인딩 공격.** 페이지가 에이전트에게 지속적 메모리 기록을 지시(레슨 12는 지속적 상태 다룸). 다음 세션에서 메모리가 시각적 트리거 없이 페이로드 실행.
4. **인증 세션에 대한 CSRF 형태 공격.** Tainted Memories 클래스: 에이전트가 어딘가에 로그인 상태; 공격자 페이지가 사용자의 쿠키로 상태 변경 요청 실행.
5. **원클릭 탈취.** 시각적으로 무해한 버튼이 에이전트가 따라가는 페이로드 포함. Comet 클래스.
6. **에이전트 호스트 표면의 Content-Security-Policy 취약점.** 렌더링 및 도구 계층 자체가 공격 벡터가 될 수 있음; 브라우저 내 브라우저 에이전트 스택은 광범위.

### "완전히 패치 불가능"한 이유

공격은 에이전트의 능력과 동형(isomorphic). 에이전트는 작업을 수행하기 위해 신뢰할 수 없는 콘텐츠를 읽어야 함. 에이전트가 읽는 모든 콘텐츠에 지시사항이 포함될 수 있음. 에이전트가 따르는 모든 지시사항은 사용자의 실제 요청과 불일치할 수 있음. 방어(신뢰 경계, 분류기, 도구 허용 목록, 중대한 작업에 대한 HITL)는 공격 비용을 높이고 영향 범위를 줄임. 그러나 공격 클래스 자체를 닫지는 못함.

이는 Lob의 정리(레슨 8)와 동일한 추론 패턴: 에이전트는 다음 토큰이 안전하다는 것을 증명할 수 없음; 단지 안전하지 않은 토큰이 더 탐지 가능하도록 시스템을 구성할 수 있음.

### 실제로 배포되는 방어 태세

- **읽기/쓰기 경계.** 읽기는 절대 중대한 결과를 초래하지 않음. 쓰기(폼 제출, 콘텐츠 게시, 부작용이 있는 도구 호출)는 신뢰 경계 외부에서 시작된 콘텐츠인 경우 신선한 인간 승인 필요.
- **작업별 도구 허용 목록.** 에이전트는 브라우징 가능; 해당 도구가 작업에 명시적으로 활성화되지 않은 경우 송금 시작 불가. 레슨 13은 예산 다룸.
- **세션 격리.** 브라우저 에이전트 세션은 범위 지정된 자격 증명으로만 실행. 프로덕션 인증, 개인 이메일 없음. 감사용 모든 HTTP 요청 로그 보관.
- **콘텐츠 정화기.** 모델 컨텍스트에 결합되기 전, 가져온 HTML에서 알려진 악성 패턴 제거. (쉬운 공격 감소; 정교한 페이로드는 차단 불가.)
- **중대한 작업에 대한 HITL.** 제안-커밋 패턴(레슨 15).
- **메모리에 대한 캐나리 토큰.** 메모리 항목이 실행되면 사용자가 확인(레슨 14).

## 사용 방법

`code/main.py`는 세 개의 합성 페이지를 대상으로 실행되는 소규모 브라우저-에이전트 모델을 구현합니다. 한 페이지는 정상(benign) 페이지이고, 한 페이지는 보이는 텍스트에 직접 프롬프트 인젝션(prompt-injection) 블롭이 포함되어 있으며, 한 페이지는 URL-프래그먼트 인젝션(가시적이진 않지만 에이전트의 컨텍스트 내부에 존재)이 있습니다. 이 스크립트는 (a) 순진한(naïve) 에이전트가 어떻게 행동하는지, (b) 읽기/쓰기 경계(read/write boundary)가 무엇을 포착하는지, (c) 살균기(sanitizer)가 무엇을 포착하는지, (d) 둘 다 포착하지 못하는 것은 무엇인지를 보여줍니다.

## Ship It

`outputs/skill-browser-agent-trust-boundary.md`는 제안된 브라우저-에이전트 배포 범위를 정의합니다: 어떤 신뢰 영역(trust zone)에 영향을 미치는지, 어떤 데이터를 쓸 수 있는 권한이 있는지, 그리고 첫 실행 전에 반드시 구현되어야 하는 방어(security defenses) 메커니즘을 명시합니다.

## 연습 문제

1. `code/main.py`를 실행하세요. 샌더라이저가 탐지하지만 읽기/쓰기 경계 검사가 탐지하지 못하는 공격과, 읽기/쓰기 경계 검사만 탐지하는 공격을 식별하세요.

2. 샌더라이저를 확장하여 HashJack 스타일의 URL-프래그먼트 주입 공격 한 종류를 탐지하도록 개선하세요. 합법적인 프래그먼트가 포함된 정상 URL에 대한 오탐률을 측정하세요.

3. 알고 있는 실제 브라우저-에이전트 워크플로우(예: "항공편 예약")를 선택하세요. 모든 읽기 작업과 쓰기 작업을 나열하세요. 어떤 쓰기 작업에 HITL(인간 개입)이 필요한지 그 이유를 설명하세요.

4. WebArena-Verified ICLR 2026 논문을 읽으세요. 원본 WebArena의 채점 방식이 불안정했던 작업 범주 하나를 식별하고, 검증된 서브셋이 이를 어떻게 해결하는지 설명하세요.

5. 브라우저-에이전트 환경을 위한 메모리 캐니리를 설계하세요. 어떤 정보를 어디에 저장할지, 어떤 조건에서 경보를 트리거할지 설명하세요.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|---|---|---|
| 간접 프롬프트 인젝션(Indirect prompt injection) | "나쁜 페이지 텍스트" | 에이전트가 읽는 페이지의 신뢰할 수 없는 콘텐츠에 에이전트가 실행하는 지시사항이 포함됨 |
| 오염된 메모리(Tainted Memories) | "메모리 공격" | 에이전트가 공격자가 제공한 지시사항을 영구 메모리에 기록; 다음 세션에서 트리거됨 |
| 해시잭(HashJack) | "URL 조각 공격" | URL 조각/쿼리 문자열에 숨겨진 페이로드가 에이전트의 컨텍스트에 포함되지만 시각적으로 렌더링되지 않음 |
| 원클릭 탈취(One-click hijack) | "나쁜 버튼" | 가시적인 UI 요소가 후속 페이로드를 에이전트가 실행하도록 유도 |
| 브라우즈컴프(BrowseComp) | "웹 검색 벤치마크" | 오픈 웹에서 특정 사실 찾기; 분 단위 시간 범위 |
| OS월드(OSWorld) | "데스크톱 벤치마크" | 전체 OS 제어; 다단계 GUI 작업 |
| 웹아레나-검증(WebArena-Verified) | "고정된 웹 작업 벤치마크" | ServiceNow가 재평정한 WebArena의 Hard 하위 집합 |
| 읽기/쓰기 경계(Read/write boundary) | "부작용 게이트" | 읽기는 절대 결과를 초래하지 않음; 쓰기는 콘텐츠가 신뢰 영역 밖일 경우 새로운 승인 필요 |
