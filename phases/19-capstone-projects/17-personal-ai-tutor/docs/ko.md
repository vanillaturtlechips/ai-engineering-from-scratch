# 캡스톤 17 — 개인 AI 튜터 (적응형, 멀티모달, 메모리 기능)

> Khanmigo(Khan Academy), Duolingo Max, Google LearnLM / Gemini for Education, Quizlet Q-Chat, Synthesis Tutor는 모두 2026년에 대규모 적응형 멀티모달 튜터링 시스템을 출시했습니다. 공통 구조는 소크라테스식 정책(절대 답을 바로 제공하지 않음), 모든 상호작용 후 업데이트되는 학습자 모델(베이지안 지식 추적 방식), 음성 + 텍스트 + 사진-수학 입력, 커리큘럼 그래프 검색, 간격 반복 스케줄링, 연령에 적합한 콘텐츠를 위한 강력한 안전 필터입니다. 캡스톤 과제는 특정 과목 튜터(K-12 대수학 또는 입문 Python)를 출시하고, 10명의 학습자와 함께 2주간의 효능 연구를 실행하며, 콘텐츠 안전 감사를 통과하는 것입니다.

**유형:** 캡스톤  
**언어:** Python(백엔드, 학습자 모델), TypeScript(웹 앱), SQL(Postgres + Neo4j를 통한 커리큘럼 그래프)  
**선수 과목:** 5단계(NLP), 6단계(음성), 11단계(LLM 엔지니어링), 12단계(멀티모달), 14단계(에이전트), 17단계(인프라), 18단계(안전)  
**활용 단계:** P5 · P6 · P11 · P12 · P14 · P17 · P18  
**소요 시간:** 30시간

## 문제

적응형 튜터링은 과거 교육 기술 연구 분야의 틈새 시장이었습니다. 그러나 2026년에는 소비자 제품으로 자리 잡았습니다. Khanmigo는 미국 대부분의 학군에 배치되었고, Duolingo Max는 수천만 명의 월간 활성 사용자(MAU)를 기록했습니다. Google의 LearnLM / Gemini for Education은 Google Classroom에서 튜터링 기능을 구동하며, Quizlet Q-Chat은 플래시카드와 함께 사용됩니다. Synthesis Tutor는 "호기심 많은 아이들을 위한 튜터"로 바이럴 성공을 거두었습니다. 공통 요소: 다중 모달 입력(텍스트, 음성, 방정식 사진 촬영), 소크라테스식 교수법(먼저 질문, 나중에 설명), 각 상호작용 후 업데이트되는 학습자 모델, 그리고 엄격한 연령 적합성 안전 기준.

특정 집단을 대상으로 이러한 시스템 중 하나를 구축하게 됩니다. 평가 기준은 실제 효과 연구입니다: 10명의 학습자를 대상으로 2주간의 사전-사후 테스트 점수 비교. 음성 루프는 자연스럽게 느껴져야 합니다(캡스톤 03 서브스택). 메모리는 개인 정보를 존중해야 하며, 안전 필터는 K-12를 고려한 COPPA 준수 레드 팀 테스트를 통과해야 합니다.

## 개념

네 가지 구성 요소. **튜터 정책**은 소크라테스식 루프입니다: 학습자가 답을 요청하면 정책이 유도 질문을 하고, 정답을 맞히면 다음 개념으로 이동하며, 막히면 단계적 힌트를 제공합니다. **학습자 모델**은 베이지안 지식 추적(또는 단순 변형)으로, 각 상호작용 후 커리큘럼 노드별 숙달 확률을 업데이트합니다. **커리큘럼 그래프**는 선행 관계 엣지를 가진 개념들의 Neo4j 그래프이며, 정책은 그래프를 탐색하여 다음 개념을 선택합니다. **메모리**는 과거 상호작용, 오류, 선호도를 저장하는 에피소드+의미 저장소(agentmemory 스타일)입니다.

UX는 다중 모달입니다. 텍스트 입력은 타이핑된 답변을 위해 사용됩니다. 음성 입력은 LiveKit + Whisper(캡스톤 03 재사용)를 통해 처리됩니다. 수학 문제용 사진 입력은 dots.ocr 또는 PaliGemma 2로 지원됩니다. 음성 출력은 Cartesia Sonic-2를 통해 제공됩니다. 안전 기능은 Llama Guard 4와 연령 적합 필터(성인 콘텐츠, 폭력, 자해 차단) 및 COPPA 인식 메모리 보존 정책을 사용합니다.

성과 연구가 최종 결과물입니다. 10명의 학습자, 사전/사후 테스트, 2주간 진행. 학습 성과 변화량(delta)과 신뢰 구간을 보고합니다. 비적응형 베이스라인(튜터 정책 없이 선형적으로 제공되는 동일 콘텐츠)과 비교합니다.

## 아키텍처

```
학습자 장치
  |
  +-- 텍스트         -> 웹 앱
  +-- 음성         -> LiveKit Agents (ASR + TTS)
  +-- 사진 수학   -> dots.ocr / PaliGemma 2
       |
       v
  튜터 정책 (LangGraph)
       - 소크라테스식 결정 헤드
       - 다음 개념 선택기 (커리큘럼 그래프 탐색)
       - 힌트 스캐폴더
       - 숙달도 업데이트
       |
       v
  학습자 모델 (BKT / 문항 반응 이론)
       - 개념별 숙달도 확률
       - 간격 반복 스케줄러 (SM-2 또는 FSRS)
       |
       v
  메모리 (에이전트 메모리 스타일)
       - 에피소드: 모든 상호작용
       - 의미적: 학습된 실수, 선호도
       - 보존 정책: COPPA / GDPR 준수
       |
       v
  커리큘럼 그래프 (Neo4j)
       - 선수 조건 엣지
       - OER 콘텐츠 첨부
       |
       v
  안전성:
    Llama Guard 4 + 연령 적합 필터
    메모리 접근: 학습자 ID 범위 제한
```

## 스택

- 주제 선택: K-12 대수학 또는 입문 Python (심화를 위해 하나 선택)
- 튜터 정책: LangGraph 대 Claude Sonnet 4.7 (프롬프트 캐싱 포함)
- 학습자 모델: 베이지안 지식 추적(classic) 또는 FSRS(간격 반복)
- 커리큘럼 그래프: 개념 + 선수과목 엣지 + OER 콘텐츠를 위한 Neo4j
- 메모리: agentmemory 스타일의 지속적 벡터 + 에피소드 + 의미론적 저장소
- 음성: LiveKit Agents 1.0 + Cartesia Sonic-2 (캡스톤 03 하위 스택 재사용)
- 사진 수학: dots.ocr 또는 PaliGemma 2를 이용한 방정식 인식
- 안전성: Llama Guard 4 + 맞춤형 연령 적합 필터
- 평가: 블룸 수준 질문 생성, 사전/사후 테스트 하니스, 효과 연구 툴링

## 구축 방법

1. **커리큘럼 그래프.** 50-150개의 개념 노드(예: K-12 대수학 "수직선"에서 "이차 방정식 공식"까지)와 선수과목 엣지로 구성된 Neo4j를 구축합니다. 각 노드에 OER(Open Educational Resource) 콘텐츠(Open Textbook, OpenStax)를 첨부합니다.

2. **학습자 모델.** 베이지안 지식 추적(Bayesian Knowledge Tracing)을 초기화합니다. 사전 확률(guess, slip, learn-rate)을 설정하고, 각 상호작용 후 개념별 숙달도(mastery)를 업데이트합니다. 학습자별 데이터를 지속합니다.

3. **튜터 정책.** LangGraph로 다음 노드를 구현합니다:  
   - `read_signal`(학습자의 답변이 정답/부분 정답/막힌 상태인지 확인)  
   - `select_concept`(커리큘럼 그래프를 탐색하여 가장 우선순위가 높은 개념 선택)  
   - `scaffold`(소크라테스식 질문 제공)  
   - `update_mastery`(숙달도 업데이트)

4. **메모리.** 모든 상호작용은 에피소드 저장소(episodic store)에 기록됩니다. 실수와 선호도는 의미 기억(semantic memory)으로 승격됩니다. COPPA 준수 보존 정책: 1년 후 자동 삭제, 부모 접근 가능.

5. **음성 경로.** 튜터 정책에 LiveKit Agents 워커를 연결합니다. Whisper-v3-turbo를 통한 ASR(Automatic Speech Recognition), Cartesia Sonic-2를 통한 TTS(Text-to-Speech) 지원. 캡스톤 03 메커니즘을 재사용하여 바지인(barge-in) 기능 지원.

6. **사진-수학 경로.** 이미지 업로드 또는 캡처 후 dots.ocr 또는 PaliGemma 2로 방정식 인식. 구조화된 입력으로 튜터에 전달.

7. **안전성.** 모든 모델 출력은 Llama Guard 4와 연령 적합성 필터(자살 유발, 성인 콘텐츠, 폭력 차단)를 통과합니다. 메모리 접근은 학습자 ID로 범위 제한. 부모용 삭제 접근 표면 제공.

8. **효과성 연구.** 10명의 학습자 대상, 사전 테스트(표준화된 30문항 기준 평가), 2주간 튜터 상호작용(주 3회), 사후 테스트. 동일 콘텐츠에서 비적응형(non-adaptive) 기준 집단 10명과 비교.

9. **주간 진행 보고서.** 학습자별로 PDF 형식의 요약 보고서를 자동 생성합니다. 탐색한 주제, 숙달도 변화 추이, 권장 다음 단계 포함.

## 사용 방법

```
learner: "3x + 6 = 12가 x = 2를 의미하는 이유를 이해하지 못하겠어요"
[signal]   막힘
[concept]  '변수 분리하기' (선수 개념: 덧셈-뺄셈-등식)
[scaffold] "양변에서 어떤 수를 빼야 할까요?"
learner: "6"
[signal]   정답
[mastery]  덧셈-뺄셈-등식: 0.62 → 0.77
[concept]  '변수 분리하기' 계속
[scaffold] "좋아요. 이제 3x / 3은 얼마일까요?"
```

## 연습 문제

1. 적응형 학습자 모델(랜덤 개념 순서) 적용 여부에 따른 효능 연구를 실행하세요. 델타(차이)를 보고하세요. 적응형 모델이 우세할 것으로 예상되지만, 그 차이가 흥미로운 수치입니다.

2. 다중 모달 프로브(probe)를 추가하세요: 동일한 개념 질문을 텍스트, 음성, 사진으로 제공하세요. 학습자가 선호하는 모달리티(방식)로 더 빠르게 수렴하는지 측정하세요.

3. 부모 대시보드를 구축하세요: 연습한 주제, 숙달 궤적, 예정된 개념, 안전 이벤트(가드레일 위반 사항)를 포함하세요. COPPA(아동 온라인 개인정보 보호법) 준수.

4. 언어 전환 모드를 추가하세요: 튜터가 스페인어 입력을 받아들이고 스페인어로 가르치도록 구현하세요. X-Guard 적용 범위를 측정하세요.

5. 메모리 프라이버시를 강화하세요: 음성 클립 재입력 공격을 통해서도 학습자 A가 학습자 B의 데이터를 볼 수 없음을 검증하세요. 시도된 접근 기록을 남기고 경고를 발령하세요.

## 주요 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|-------------------|-----------|
| 소크라틱 정책(Socratic policy) | "묻지 말고 답하지 마라" | 튜터가 답을 주는 대신 유도 질문을 함 |
| 베이지안 지식 추적(Bayesian knowledge tracing) | "BKT" | 개념별 숙달 확률을 위한 클래식 학습자 모델 방정식 |
| FSRS | "자유 간격 반복 스케줄러" | 2024년 간격 반복 스케줄러, SM-2보다 우수 |
| 커리큘럼 그래프(Curriculum graph) | "개념 DAG" | 선행 조건 엣지를 가진 개념들의 Neo4j |
| 에피소드 기억(Episodic memory) | "상호작용별 로그" | 모든 상호작용이 이후 검색을 위해 저장됨 |
| 의미 기억(Semantic memory) | "학습된 패턴 저장소" | 에피소드 기억에서 승격된 압축된 실수와 선호도 |
| COPPA | "아동 개인정보 보호법" | 13세 미만 아동의 데이터 수집을 제한하는 미국 법률 |
