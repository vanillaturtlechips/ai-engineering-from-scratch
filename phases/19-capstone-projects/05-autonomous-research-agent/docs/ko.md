# 캡스톤 05 — 자율 연구 에이전트(AI-과학자 클래스)

> 사카나의 AI-Scientist-v2가 전체 논문을 출판했습니다. 에이전트 랩에서 실험을 실행했고, 앨런 AI가 트레이스를 공유했습니다. 2026년 형태는 실험, 예산 비용, 샌드박싱된 코드 실행, 비전 피드백 LaTeX 라이터, 자동화된 NeurIPS 스타일 리뷰어 앙상블에 대한 계획-실행-검증 트리 탐색입니다. 캡스톤은 이를 구축하고, 논문당 $30 예산 내에서 엔드 투 엔드로 실행하며, 사카나가 문서화한 샌드박스 탈출 레드 팀을 통과하는 것입니다.

**유형:** 캡스톤  
**언어:** Python (에이전트 + 샌드박스), LaTeX (출력)  
**선수 과목:** 2단계(ML), 3단계(딥러닝), 7단계(트랜스포머), 10단계(LLM 처음부터), 14단계(에이전트), 15단계(자율성), 16단계(멀티 에이전트), 18단계(안전성)  
**적용 단계:** P0 · P2 · P3 · P7 · P10 · P14 · P15 · P16 · P18  
**소요 시간:** 40시간

## 문제

자율 연구 에이전트는 2026년에 중요한 전환점을 넘었습니다. Sakana AI의 AI-Scientist-v2는 워크숍 동료 평가를 통과한 생성 논문으로 Nature에 게재되었습니다. ShinkaEvolve(ICLR 2026)는 가설 진화 분야로 이 연구 라인을 확장했습니다. AMD의 Agent Laboratory는 재현 가능한 실험 기록을 출시했습니다. 이 에이전트들은 마법이 아닙니다 — 비용 제한, 시드 기반 샌드박스, 자동화된 검토를 갖춘 후보 실험 트리 위에서 실행되는 계획-실행-검증 루프입니다. 핵심은 루프 구조, 예산 관리, 안전성 설계에 있습니다.

좁은 도메인(예: 100M 파라미터 트랜스포머의 어텐션-희소성(attention-sparsity) 분석)에서 시드 아이디어를 바탕으로 루프를 구현하며 학습합니다. 첫 실행에서 새로운 발견을 하는 것이 목적이 아닙니다. 인프라는 가치가 있습니다: 트리 탐색(tree-search), 실험 샌드박스, 작성자-검토자 루프, 레드 팀 보고서 등이 핵심입니다. Sakana 팀은 샌드박스 탈출 실패 사례를 문서화했으며, 여러분의 에이전트도 동일한 레드 팀 테스트를 통과해야 합니다.

## 개념

에이전트는 최선-우선 트리 탐색(best-first tree search)입니다. 노드는 실험 사양(experiment specifications)으로 구성됩니다: (가설(hypothesis), 설정(config), 코드(code), 예상 결과(expected outcome)). 확장(expand) 단계는 작은 수정(small edits)을 통해 자식 노드를 제안합니다(예: 옵티마이저(optimizer) 교체, 배치 크기(batch size) 조정, 구성 요소 제거(ablate a component)). 각 자식 노드는 엄격한 리소스 제한(hard resource cap)이 있는 새로운 샌드박스(sandbox)에서 실행됩니다. 결과는 노드를 (새로움(novelty) × 품질(quality) × 남은 예산(remaining budget)) 기준으로 순위 매기는 점수 함수(scoring function)에 피드백됩니다. 트리는 예산이 소진될 때까지 성장한 후, 최적의 브랜치가 작성됩니다.

작성기(writer)는 멀티모달(multimodal)입니다. LaTeX 초안을 생성하고, 컴파일한 후, 그림을 렌더링하며, 렌더링된 PDF를 Claude Opus 4.7의 비전 모드(vision mode)에 피드백하여 레이아웃, 그림 가독성, 주장-증거 일치도 등을 비평합니다. 5개의 LLM 심사자(reviewer ensemble)로 구성된 검토 그룹은 NeurIPS 스타일의 점수(새로움(novelty), 엄격성(rigor), 명확성(clarity), 재현성(reproducibility), 영향력(impact))를 제공합니다. 평균이 임계값 미만으로 떨어지면, 논문은 비평과 함께 작성기로 반환됩니다.

안전성은 핵심 요소입니다. 모든 실험은 네트워크 접근 차단(no network egress), 최대 실행 시간 제한(bounded wall-clock), 고정된 리소스 한도(pinned resource limits)가 있는 E2B 또는 Daytona 샌드박스에서 실행됩니다. 에이전트의 코드 생성 단계는 샌드박스를 탈출하는 시스템 콜(syscalls)을 차단하는 정책 계층(policy layer)을 통과합니다. 레드 팀 보고서(red-team report)는 Sakana에 문서화된 공격 표면(attack surface)(포크 폭탄(fork bombs), 파일 시스템 탈출, LLM이 작성한 네트워크 호출)을 재현합니다.

## 아키텍처

```
시드 아이디어 + 도메인
      |
      v
  문헌 조사 (Semantic Scholar + OpenAlex + FAISS 캐시)
      |
      v
  LangGraph 계획-실행-검증 트리
      |
      v
  +--- 노드 확장 ----+      노드별 샌드박스
  |                    |      (E2B / 데이토나)
  v                    v      리소스 제한
  child_1           child_k   네트워크 차단
  |                    |      결정적 시드
  v                    v
  실험 실행           실험 실행
  |                    |
  v                    v
  (새로움, 품질, 예산) 기준 노드 점수화
      |
      v
  최적 분기 -> LaTeX 작성기
      |
      v
  컴파일 + 시각 평가 (Opus 4.7 비전)
      |
      v
  검토자 앙상블 (5개 LLM 심사자, NeurIPS 평가 기준)
      |
      v
  paper.pdf + review.md + trace.json
```

## 스택

- 오케스트레이션: 체크포인팅 및 인간 승인 게이트가 있는 LangGraph
- 트리 탐색: 실험 노드(AB-MCTS 스타일, Sakana v2 기반) 위의 커스텀 최선 우선 탐색
- 샌드박스: 실험별 E2B, Docker-in-Docker 폴백; cgroups를 통한 리소스 제한
- 문헌: Semantic Scholar Graph API + OpenAlex + 초록의 로컬 FAISS 캐시
- 작성자: LaTeX 템플릿 + 그림 비평 및 레이아웃을 위한 Claude Opus 4.7(비전 모드)
- 검토자: 가중 집계를 사용하는 5명의 심사자(Opus 4.7, GPT-5.4, Gemini 3 Pro, DeepSeek R1, Qwen3-Max) 앙상블
- 실험 프레임워크: 물리 실험을 위한 PyTorch 2.5, 로깅을 위한 W&B
- 관측 가능성: 에이전트 추적을 위한 Langfuse, 논문당 $30 하드 예산

## 구축 방법

1. **시드 및 도메인 범위 설정.** 시드 아이디어(예: "10억 미만 파라미터 트랜스포머의 어텐션 맵 희소성 패턴 조사")를 선정합니다. 검색 공간(모델, 데이터셋, 컴퓨팅 예산)을 정의합니다.

2. **문헌 조사.** Semantic Scholar + OpenAlex에서 가장 많이 인용된 관련 논문 50편을 조회합니다. 초록을 로컬에 캐시하고 1페이지 분량의 도메인 요약문을 생성합니다.

3. **트리 구조 설계.** 시드 가설로 루트 노드를 초기화합니다. `expand(node) -> children`을 구현하여 소규모 수정 제안(자식 노드당 하나의 설정 변경)을 생성합니다. `score(node)`를 가중치 적용 신규성 × 품질 × 예산 항으로 구현합니다.

4. **샌드박스 래핑.** 모든 실험은 `docker run --network=none --memory=8g --cpus=2 --pids-limit=256 --read-only`(또는 E2B 정책 동등물)로 실행됩니다. 시드는 샌드박스에 기록되며, 출력은 읽기 전용으로 마운트됩니다.

5. **계획-실행-검증 루프.** `plan`은 자식 노드를 제안합니다. `execute`는 샌드박스를 실행하고 로그 및 메트릭을 수집합니다. `verify`는 메트릭에 대한 단위 검사를 수행합니다(손실 감소 여부? 제거 실험이 효과를 분리했는지?). 실패한 노드는 실패 사유가 트리에 저장됩니다.

6. **작성기.** 예산 소진 후 최적 분기를 선택합니다. matplotlib로 시각화를 렌더링합니다. Claude Opus 4.7을 사용해 분기 추적 내용을 컨텍스트로 포함한 LaTeX 초안을 생성합니다. 컴파일 후 생성된 PDF를 Opus 4.7 비전 모델에 피드백으로 제공합니다. 반복합니다.

7. **심사자 앙상블.** 5명의 심사자가 NeurIPS 스타일 평가 기준(신규성, 엄격성, 명확성, 재현성, 영향력)으로 초안을 평가합니다. 평균이 4.0/5 미만일 경우 비평과 함께 작성 단계로 반환합니다. 3회 재작성 후 강제 종료합니다.

8. **레드 팀.** 샌드박스를 대상으로 한 적대적 작업 세트(포크 폭탄, 네트워크 유출 시도, 파일 시스템 탈출, LLM 생성 셸 메타문자)를 구축하거나 통합합니다. 모든 공격이 차단되었는지 확인합니다. 결과를 문서화합니다.

9. **재현성.** 모든 논문은 트리 검색 추적 JSON, 시드, W&B 실행 링크, 샌드박스 설정, 그리고 전체 과정을 재현하는 README와 함께 배포됩니다.

## 사용 방법

```
$ ai-scientist run --seed "서브-1B 트랜스포머에서의 어텐션 희소성" --budget 30
[lit]    50편의 논문, 12초 내 요약 완료
[tree]   8개 노드 확장, 예산 12/30 사용
[exec]   노드 #3 희소성=top-8, 손실=2.83 (현재까지 최고)
[exec]   노드 #6 희소성=top-4, 손실=3.12 (더 나쁨)
[exec]   ...
[tree]   노드 #3을 루트로 하는 분기 선택 (참신성 0.62, 품질 0.81)
[write]  LaTeX 초안 v1 완성
[vision] 비평: 그림 2 범례 너무 작음, 주장-근거 적절
[write]  3회 수정 후 초안 v2
[review] 평균 4.2/5 (참신성 3.9, 엄밀성 4.3, 명확성 4.1, 재현성 4.5, 영향력 4.2)
[done]   paper.pdf + review.md + trace.json     $28.40 사용
```

## Ship It

`outputs/skill-ai-scientist.md`가 최종 결과물입니다. 시드 아이디어 + 도메인 + $30 예산을 기반으로 전체 파이프라인을 실행하고 검토 가능한 논문 및 재현성 번들을 출력합니다.

| 20 | 실험 엄격성 | 베이스라인, 시드, 애블레이션; 결과 표의 모든 셀에 의해 주장 입증 |

| 15 | 재현성 | 동일한 시드로 한 번의 명령으로 논문 재현 |

## 연습 문제

1. 동일한 도메인에서 세 가지 다른 시드 아이디어에 대해 파이프라인을 실행해 보세요. 트리-서치에서 중복되는 부분을 비교하고, 낭비된 계산 리소스를 식별하세요.

2. $5 이상으로 추정된 노드에 대해 실험 실행 전 인간 검토 게이트를 추가하세요. 총 비용이 얼마나 감소하는지 측정하세요.

3. 리뷰어 앙상블을 단일 심사자로 교체하세요. 알려진 불량 논문 홀드아웃 세트에서 거짓 수용률을 측정하세요.

4. 네트워크 유출 레드 팀 테스트를 도입하세요: 에이전트가 외부 주소에 `curl`을 시도하는 코드를 작성하도록 합니다. `--network=none` 정책이 이를 차단하는지 확인하고, 시도 기록을 로그에 남기세요.

5. 트리-서치를 평탄한 무작위 베이스라인(동일 예산, 확장 전략 없음)과 비교하세요. 참신성 × 품질 향상도를 보고하세요.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|---------------------|----------|
| 트리 탐색 | "AB-MCTS-style 확장" | 실험 노드에서 novelty×quality×budget 점수를 기반으로 한 최적 우선 탐색 |
| 샌드박스 | "실험 격리" | 네트워크 차단, CPU/메모리 제한, 고정된 시드, 읽기 전용 입력을 가진 컨테이너 |
| 비전 크리티크 | "렌더-후-읽기" | 논문을 PDF로 컴파일하고, PDF를 VLM에 다시 입력하여 레이아웃 및 주장-근거 분석 수행 |
| 리뷰어 앙상블 | "자동화된 동료 검토" | NeurIPS 평가 기준으로 논문을 점수화하는 여러 LLM 심사관; 가중치 집계 결과가 파이프라인 통과 여부 결정 |
| 신규성 점수 | "이것은 새로운가?" | 50편의 논문 문헌 캐시와 유사성을 페널티로 부과하는 휴리스틱 |
| 비용 상한 | "$ 예산" | 논문당 총 지출에 대한 하드 캡; Langfuse 카운터 + 사전 실행 추정치 |
| 레드 팀 | "샌드박스 탈출 감사" | 정책이 잘못되었을 경우 샌드박스를 탈출할 수 있는 적대적 작업 |
