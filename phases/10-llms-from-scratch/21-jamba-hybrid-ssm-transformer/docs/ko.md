# Jamba — 하이브리드 SSM-트랜스포머

> 상태 공간 모델(SSM)과 트랜스포머는 서로 다른 목표를 추구합니다. 트랜스포머는 이차 비용(quadratic cost)으로 품질(quality)을 얻는 반면, SSM은 재귀(recurrence)를 통해 선형 시간 추론(linear-time inference)과 상수 메모리(constant memory)를 얻지만 품질 측면에서 뒤처집니다. AI21의 Jamba(2024년 3월)와 Jamba 1.5(2024년 8월)는 두 방식을 동일 모델에 통합했습니다: 7개의 Mamba 레이어당 1개의 트랜스포머 레이어, 매 다른 블록마다 MoE(Mixture of Experts) 적용, 그리고 단일 80GB GPU에 적합한 256k 컨텍스트 윈도우. Mamba-3(ICLR 2026)은 복소수 상태 공간(complex-valued state spaces)과 MIMO(Multi-Input Multi-Output) 투영을 통해 SSM 측면을 강화했습니다. 이 레슨은 두 아키텍처를 처음부터 끝까지 읽고, 순수 SSM 및 순수 트랜스포머 기반 장문 컨텍스트 시도가 실패한 반면 하이브리드 레시피가 3년간 확장성을 유지한 이유를 설명합니다.

**유형:** 학습
**언어:** Python (표준 라이브러리, 레이어 혼합 계산기)
**선수 지식:** Phase 10 · 14 (오픈 모델 아키텍처), Phase 10 · 17 (네이티브 희소 어텐션)
**소요 시간:** ~60분

## 학습 목표

- Jamba 블록의 세 가지 기본 구성 요소(트랜스포머 레이어(Transformer layers), 맘바 레이어(Mamba layers), MoE(Mixture of Experts))와 1:7:even 인터리빙(interleaving) 레시피를 설명.
- SSM(상태 공간 모델)의 재귀(recurrence)가 높은 수준에서 어떻게 구성되는지, 그리고 왜 상수 메모리 추론(constant-memory inference)을 가능하게 하는지 설명.
- 256k 컨텍스트에서 Jamba 모델의 KV 캐시 풋프린트(KV cache footprint)를 계산하고, 순수 트랜스포머 모델(pure-Transformer model)이 필요로 하는 것과 비교.
- 맘바-3(Mamba-3)의 세 가지 혁신(지수-사다리꼴 이산화(exponential-trapezoidal discretization), 복소수 상태 업데이트(complex-valued state update), MIMO(Multi-Input Multi-Output))과 각각이 해결하는 문제를 명명.

## 문제 정의

어텐션(attention)은 시퀀스 길이에 대해 2차(quadratic) 복잡도를 가지지만, 상태 공간 모델(SSM)은 선형(linear) 복잡도를 가집니다. 이 차이는 시퀀스 길이가 길어질수록 더욱 두드러집니다. 예를 들어, 256k 토큰에서 트랜스포머 어텐션 맵은 헤드당 650억(65B) 항목을 가지지만, SSM의 순환 상태(recurrent state)는 시퀀스 길이와 무관하게 고정 크기를 유지합니다.

순수 SSM 모델(Mamba, Mamba-2)은 소규모 작업에서 트랜스포머와 유사한 퍼플렉서티(perplexity)를 보이지만, 상태 추적(state-tracking) 작업에서는 성능이 떨어지고 일부 범주의 인-컨텍스트 검색(in-context retrieval) 작업에서는 실패합니다. 그 이유는 SSM이 역사를 고정 크기의 상태로 압축하기 때문에 시퀀스가 길어질 경우 정보 누출(information leak)이 발생하기 때문입니다. 반면 어텐션은 모든 정보를 정확히 기억하지만 2차 계산 비용이 듭니다.

명백한 해결책: 두 방식을 함께 사용하는 것입니다. 정확한 기억(exact recall)이 중요한 위치에는 트랜스포머 레이어를 배치하고, 다른 위치에는 SSM 레이어를 사용합니다. 그리고 이 비율을 조정합니다. Jamba는 이 하이브리드 방식을 대규모로 구현한 최초의 프로덕션급 모델입니다(전체 52B 파라미터, 활성 12B, 256k 컨텍스트, 단일 80GB GPU). Jamba 1.5는 이 패밀리를 확장하여 전체 398B/활성 94B로 성장했습니다. Mamba-3(ICLR 2026)은 하이브리드가 재구성될 수 있는 현재 최고의 순수 SSM 베이스라인입니다.

이 문서는 세 논문을 모두 읽고 "적절한 비율 선택"을 위한 정신 모델(mental model)을 제공합니다.

## 개념

### 한 페이지로 보는 SSM

상태 공간 모델(SSM)은 고정 크기의 상태 `h`를 통해 시퀀스 `x_1, ..., x_N`을 처리합니다:

```
h_t = A h_{t-1} + B x_t
y_t = C h_t
```

각 단계에서 상태는 선형 동역학 `A`에 따라 진화하며, 입력 `B x_t`를 받고 출력 `C h_t`를 생성합니다. `A, B, C`는 학습 가능합니다. 중요한 특성: `y_t` 계산에는 `h_{t-1}`과 `x_t`만 필요하며, 이전 `x`는 필요하지 않습니다. 메모리는 상수입니다. 추론은 토큰당 O(1)입니다.

모델 품질을 위한 핵심은 `A`의 구조입니다. S4(Gu 2021)는 훈련 중 긴 컨볼루션으로 효율적으로 평가할 수 있는 고구조 행렬을 사용했습니다. Mamba(Gu, Dao 2023)는 고정된 `A, B, C`를 데이터 의존적 구조로 대체했습니다("선택적" 부분). Mamba-2(2024)는 구조를 더 단순화했습니다. Mamba-3(2026)는 특정 위치에서 복잡성을 다시 추가했습니다.

핵심 특성: 디코더 LLM에서 SSM 레이어는 어텐션 레이어를 대체하며, 증가하는 KV 캐시 대신 레이어당 고정 크기 상태를 사용합니다.

### Jamba 블록

Jamba 블록은 두 숫자에 따라 레이어를 교차 배치합니다:

- `l`: 어텐션 대 Mamba 비율. Jamba는 `l = 8`을 사용하며, 7개의 Mamba 레이어당 1개의 Transformer 레이어를 의미합니다(7 Mamba + 1 어텐션 = 그룹당 8 레이어).
- `e`: MoE 빈도. Jamba는 `e = 2`를 사용하며, 매 다른 레이어에서 MoE를 적용합니다.

블록 내 레이어 순서:

```
M  M  M  M  M  M  M  A    (7 Mamba + 1 어텐션)
|  M  |  M  |  M  |  M    (|는 MoE 적용 위치)
```

각 Jamba 블록은 8개의 레이어로 구성됩니다. 4개의 블록(32개 레이어 총계)에서는 28개의 Mamba 레이어와 4개의 어텐션 레이어를 얻으며, 이 중 16개는 MoE를 사용합니다.

### 1:7 비율의 이유

AI21은 다양한 비율을 실험했습니다: 어텐션 대 Mamba의 어떤 비율이 파라미터당 최고의 퍼플렉서티와 장문 컨텍스트 평가에서의 인컨텍스트 리콜을 제공하는가?

- 어텐션이 너무 많음(1:1): 품질은 올라가지만 메모리와 속도가 저하됩니다.
- 어텐션이 너무 적음(1:15): 메모리는 훌륭하지만 인컨텍스트 검색이 실패합니다.
- 최적점: 1:7 또는 1:8.

직관: Transformer 레이어는 정확한 리콜과 상태 추적을 처리합니다. Mamba 레이어는 저렴한 대량 처리를 담당합니다.

### 위치 인코딩

Mamba 레이어는 자체적으로 위치를 인식합니다(재귀적 구조 덕분). 원래 Mamba 기반 하이브리드의 어텐션 레이어는 RoPE를 사용하지 않았습니다. SSM 레이어가 위치 정보를 제공했습니다. Jamba 1.5는 경험적 장문 컨텍스트 평가를 기반으로 어텐션 레이어에 RoPE를 추가하여 장문 일반화를 개선했습니다.

### 메모리 예산

Jamba-1 구조(32 레이어: 28 Mamba + 4 어텐션, 히든 4096, 32 어텐션 헤드)의 경우:

- KV 캐시(어텐션 레이어만): 256k BF16에서 `2 * 4 * 32 * 128 * 256k * 2 = 8.4 GB`. 4개의 어텐션 레이어만 기여합니다.
- SSM 상태: 토큰 접두사당 `28 * hidden * state_size`이지만, 이는 시퀀스 길이에 비례하지 않는 레이어당 고정 크기입니다. 일반적인 Mamba 상태는 피처당 16, 히든 4096: `28 * 4096 * 16 * 2 = 3.7 MB` 총계.

동일한 히든 크기, 32개 레이어, 32개 헤드의 순수 Transformer와 비교: 256k BF16에서 `2 * 32 * 32 * 128 * 256k * 2 = 128 GB`. KV 캐시가 8배 감소합니다. 2024년 모델 대부분이 사용하는 GQA(8) 기준선(`2 * 32 * 8 * 128 * 256k * 2 = 32 GB`)에 비해서도 Jamba의 16GB는 2배 더 작습니다.

이것이 AI21이 "80GB GPU 1개에서 256k 컨텍스트"라고 의미하는 바입니다. 순수 Transformer의 KV 캐시는 맞지 않으며, GQA 기준선조차 가중치와 활성화 공간을 남기지 않습니다. Jamba는 가능합니다.

### Mamba-3: 2026년 순수 SSM 기준선

Mamba-3(ICLR 2026, arXiv:2603.15569)은 순수 SSM 측면에서 세 가지 혁신을 도입합니다:

1. **지수-사다리꼴 이산화.** Mamba-2의 오일러 방법 이산화를 더 표현력 있는 재귀 구조로 대체합니다. `x_t`에 대한 외부 컨볼루션 대신 상태-입력 내에서 컨볼루션 유사 연산을 적용합니다.

2. **복소수 상태 업데이트.** 이전 Mamba는 상태 행렬을 복소수(S4)에서 실수 대각(Mamba), 스케일링된 항등(Mamba-2)으로 축소했습니다. Mamba-3는 복소수 값을 다시 도입합니다. 이는 상태에 데이터 의존적 로터리 임베딩을 적용하는 것과 동일하며, 이전 실수 단순화가 희생한 상태 추적 능력을 복원합니다.

3. **다중 입력 다중 출력(MIMO) 투영.** 피처당 스칼라 투영 대신 행렬 값 투영을 사용합니다. 디코딩 지연 시간 증가 없이 모델링 능력과 추론 시 하드웨어 활용도를 향상시킵니다.

1.5B 파라미터에서 Mamba-3는 Gated DeltaNet 대비 평균 다운스트림 정확도를 0.6포인트 향상시켰습니다. MIMO 변형은 1.2포인트를 추가로 향상시켜 총 1.8포인트 이득을 제공합니다. 동일한 상태 크기에서 Mamba-3는 Mamba-2의 절반 상태 크기로 동일한 성능을 달성합니다.

Mamba-3는 아직 대규모 하이브리드 모델로 출시되지 않았지만, 차세대 Jamba급 모델의 SSM 측 후보로 명확합니다.

### 하이브리드를 선택할 때

하이브리드가 승리하는 경우:

- 컨텍스트가 충분히 길어 순수 Transformer의 KV 캐시가 부담되는 경우(64k+).
- 작업이 단거리 구조(SSM에 적합)와 장거리 리콜(Transformer 필요)을 혼합하는 경우.
- Transformer의 KV 캐시만으로도 맞지 않는 단일 GPU 메모리 예산에 배포하려는 경우.

하이브리드가 패배하는 경우:

- 컨텍스트가 짧은 경우(16k 미만). SSM 오버헤드가 낭비되며, 순수 Transformer로 충분합니다.
- 작업이 모든 위치 간 어텐션(심층 추론, 다중 문서 상호 참조)을 필요로 하는 경우. 하이브리드의 어텐션 레이어 희소성이 해가 됩니다.
- 조 단위 파라미터 프론티어 모델로 확장하는 경우. 순수 Transformer + MLA + MoE(DeepSeek-V3 스타일)가 현재 능력 경쟁에서 승리하고 있습니다.

### 경쟁 환경

| 모델 | 패밀리 | 규모 | 고유 강점 |
|-------|--------|------|-------------|
| Mamba-2 | 순수 SSM | 3B | 선형 시간, 상수 메모리 |
| Jamba | 하이브리드 | 52B/12B | 80GB에서 256k |
| Jamba 1.5 Large | 하이브리드 | 398B/94B | 엔터프라이즈급 장문 컨텍스트 |
| Mamba-3 | 순수 SSM | 1.5B(논문) | 상태 추적 복원 |
| DeepSeek-V3 | 순수 Transformer + MoE | 671B/37B | 프론티어 능력 |

2026년 환경: 순수 Transformer MoE가 프론티어를 지배하지만, 하이브리드는 256k+ 컨텍스트 니치를 점유합니다. Mamba-3의 상태 추적 강점은 차세대에서 하이브리드 비율을 낮출 수 있습니다(더 많은 SSM, 더 적은 어텐션).

## 사용 방법

`code/main.py`는 하이브리드 아키텍처용 메모리 계산기입니다. SSM-Transformer 비율과 은닉 크기/레이어 수 구성이 주어지면 다음을 계산합니다:

- 목표 컨텍스트에서의 KV 캐시.
- SSM 상태 메모리.
- 다양한 모델 형태에 대한 컨텍스트 N에서의 총 메모리.

이 계산기는 다음을 지원합니다:

- 순수-트랜스포머 기준선(KV 캐시는 N에 따라 증가).
- Jamba 스타일 1:7 하이브리드.
- 순수-SSM(KV 캐시 없음).

이 수치는 Jamba-1 및 Jamba-1.5 논문에서 발표된 형태에 대한 직접적인 값이며, 가상의 변형에 대해서는 외삽된 값입니다.

실제 배포를 위한 통합 고려사항:

- 대부분의 프로덕션 추론 서버(vLLM, SGLang)는 Jamba와 Mamba를 지원합니다. 특정 버전을 확인하세요.
- 256k 컨텍스트에서 Jamba의 메모리 이점은 동시 요청 처리량에서 나타납니다. 동일한 VRAM에서 트랜스포머 시퀀스보다 더 많은 Jamba 시퀀스를 처리할 수 있습니다.
- Mamba-3 독립형 모델은 아직 프로덕션에 배포되지 않았습니다 — 1.5B 규모의 연구 프리뷰 단계입니다.

## 연습 문제

## 주요 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|----------------|------------------------|
| 상태 공간 모델(State space model, SSM) | "고정된 상태를 가진 순환" | 학습된 순환 `h_t = A h_{t-1} + B x_t`을 가진 레이어; 토큰당 상수 메모리 |
| 선택적 SSM(Selective SSM) | "Mamba의 트릭" | 데이터 의존적 A, B, C 파라미터로 선형 시간에 게이팅(gating) 같은 선택성을 제공 |
| 어텐션-투-맘바 비율(Attention-to-Mamba ratio) | "어텐션 레이어 수" | Jamba에서 `l = 8`은 7개의 Mamba 레이어당 1개의 어텐션 레이어를 의미 |
| Jamba 블록(Jamba block) | "8-레이어 그룹" | 하나의 어텐션 + 7개의 Mamba + 번갈아 위치한 MoE |
| SSM 상태(SSM state) | "숨겨진 버퍼" | Mamba 레이어의 KV 캐시를 대체하는 레이어당 고정 크기 상태 |
| 256k 컨텍스트(256k context) | "Jamba의 대표 수치" | Jamba-1이 단일 80GB GPU에 적합한 시퀀스 길이; 순수 Transformer는 해당 크기에서 불가능 |
| Mamba-3 | "2026 순수 SSM" | 복잡한 상태 + MIMO를 가진 현재 최고 순수-SSM 아키텍처; 하이브리드 모델들이 이를 기반으로 재구축 |
| MIMO | "다중 입력 다중 출력" | 스칼라 특성 대신 행렬 값 투영을 사용하는 Mamba-3의 혁신 |
| 지수-사다리꼴 이산화(Exponential-trapezoidal discretization) | "Mamba-3의 순환" | Mamba-2의 오일러 방법 이산화를 포괄하는 더 표현력이 높은 순환 |
| 하이브리드 아키텍처(Hybrid architecture) | "어텐션과 SSM 혼합" | Transformer와 SSM 레이어를 교차시키는 모든 모델; Jamba는 프로덕션 기준 아키텍처 |
