# Speculative Decoding과 EAGLE-3

> Phase 7 · Lesson 16은 레비아탄(Leviathan) 거부 규칙이 검증자(verifier)의 분포를 정확히 보존한다는 수학적 증명을 다뤘습니다. 이 레슨은 2026년 프로덕션 환경에서 사용되는 추측 디코딩(speculative decoding)의 훈련 스택 관점을 설명합니다. EAGLE-3는 초안 모델(draft model)을 단순한 근사 모델에서 검증자의 은닉 상태(hidden states)에 특화된 소형 네트워크로 전환했으며, 훈련 시 테스트 루프를 추가하여 훈련 및 추론 분포를 정렬했습니다. 결과: 엔드-투-엔드 속도 3×~6.5× 향상, 채팅에서 0.9 이상의 토큰별 수용률, 분포적 트레이드오프 없음. 2026년 모든 프로덕션 추론 스택은 이를 기본값으로 탑재합니다.

**유형:** 구축(Build)  
**언어:** Python (표준 라이브러리)  
**선수 지식:** Phase 7 · 16 (추측 디코딩 수학), Phase 10 · 12 (추론 최적화)  
**소요 시간:** ~75분

## 학습 목표

- 레비아탄 정리(Leviathan theorem)를 한 문장으로 서술하고, 추측 루프(speculative loop)가 검증자(verifier)와 동일한 분포를 갖는 샘플을 생성함을 증명하라.
- 바닐라 스펙-디코딩(vanilla spec-decoding, Leviathan 2023)부터 EAGLE, EAGLE-2, EAGLE-3까지의 2년 간의 발전 과정을 설명하고, 각 단계에서 제거한 정확한 한계점(limitation)을 명시하라.
- 수용률(acceptance rate) `α`와 초안-검증자 비용 비율(draft-to-verifier cost ratio) `c`로부터 기대되는 속도 향상(speedup)을 계산하고, 각 영역(regime)에 대해 최적의 초안 길이(draft length) `N`을 선택하라.
- 처음부터 전체 추측 루프(speculative loop)를 구현하라: 초안(draft) 생성, 검증(verify), 잔차(residual)로부터의 거부-샘플링(reject-sampling), 거부 시 KV 캐시 롤백(roll back), 완전 수용 시 보너스 토큰(bonus token) 출력.

## 문제

70B 모델에서의 자기회귀적 디코딩은 H100에서 초당 약 35토큰 속도로 실행됩니다. GPU는 포화 상태에 전혀 가깝지 않습니다. 메모리 대역폭이 병목입니다: 매 토큰마다 HBM에서 70B의 가중치를 로드하고, 한 단계의 연산을 수행한 후 하나의 부동소수점 값을 생성합니다. 연산 유닛은 대부분 유휴 상태입니다.

추측 디코딩은 이를 실제로 해결할 수 있는 처리량 문제로 전환합니다. 저렴한 드래프트가 `N`개의 작은 순전파 단계에서 `N`개의 토큰을 제안합니다. 검증기는 접두사와 모든 `N`개의 드래프트에 대해 한 번 실행됩니다. 검증기의 분포가 위치 `i`에서 드래프트와 (통계적 의미에서) 일치하면 수락하고, 그렇지 않으면 거부하고 잔여 분포에서 수정 토큰을 샘플링합니다. 단일 대형 모델 순전파는 1개 대신 최대 `N+1`개의 수락된 토큰을 생성합니다.

중요한 정리는 Leviathan, Kalman, Matias (ICML 2023)입니다: 출력 분포는 검증기에서 직접 샘플링한 결과와 동일합니다. 근사치가 아닌 정확히 동일합니다. 이것이 추측 디코딩이 프로덕션에서 허용되는 유일한 이유입니다—품질 저하 없이 순수한 지연 시간 최적화입니다.

Phase 7 · Lesson 16에서 제공한 것은 수학 이론입니다. 이 레슨에서 제공하는 것은 훈련 스택입니다. 좋은 드래프트는 저렴한 드래프트보다 2배 더 많은 속도 향상을 제공합니다. EAGLE, EAGLE-2, EAGLE-3 (Li et al., 2024–2025)은 "드래프트 = 동일 모델의 소형 버전"을 정밀한 엔지니어링 분야로 발전시켰습니다. 2026년 프로덕션 추론 서버는 기본적으로 EAGLE-3을 사용합니다.

## 개념

### 불변량: 리바이어던 거부 샘플링

`p(t)`를 특정 접두사에 주어진 다음 토큰에 대한 드래프트 분포를, `q(t)`를 검증기의 분포를 나타낸다고 하자. 드래프트 토큰 `d ~ p`를 샘플링하고, 확률 `min(1, q(d) / p(d))`로 수락한다. 거부 시, 잔여 분포 `(q - p)_+ / ||(q - p)_+||_1`에서 샘플링한다. 결과적으로 샘플은 `q`에 따라 분포된다. 이는 `p`가 얼마나 나쁘든 상관없이 참이다 — `p`가 나쁠수록 거부 횟수가 증가하지만 출력은 정확하게 유지된다.

`prefix + d_1 + ... + d_N`에 대한 검증기 순전파 1회를 사용하여 이러한 호출 `N`개를 연속적으로 스택한다. 검증기는 `q_1, q_2, ..., q_{N+1}`을 동시에 반환한다. 왼쪽에서 오른쪽으로 이동한다. 위치 `j`에서 첫 번째 거부가 발생하면 `residual(q_j, p_j)`에서 샘플링하고 중단한다. 전체 수락 시 `q_{N+1}`에서 보너스 토큰 1개를 샘플링한다.

### 속도 향상 결정 요소

`α`를 드래프트된 토큰당 기대 수락률이라고 하자. `c = cost(draft) / cost(verifier)`를 비용 비율이라고 하자. 검증기 순전파당 기대 수락 토큰 수는 다음과 같다:

```
E[accepted] = (1 - α^(N+1)) / (1 - α)
```

수락된 토큰당 기대 총 벽시계는 `(N * c + 1) / E[accepted]`이다. `N`에 대해 이를 최소화하면 최적점을 얻는다. `α = 0.8, c = 0.05`의 경우 최적 `N`은 약 5–7, 속도 향상은 3.2×이다. `α = 0.95, c = 0.02`의 경우 최적 `N`은 약 8–10, 속도 향상은 5×까지 증가한다.

가장 큰 레버는 `α`이다. `N = 5`로 고정했을 때 `α = 0.6`(바닐라 드래프트)에서 `α = 0.9`(EAGLE-3)로 이동하면 검증기 순전파당 기대 수락 토큰이 2.2에서 4.1로 증가한다. 동일한 검증기로부터 거의 2배 더 많은 처리량을 얻는다.

### 2년간의 발전

**바닐라 추측(리바이어던, 2023).** 드래프트 모델은 동일한 패밀리의 독립적으로 훈련된 더 작은 LLM이다. 쉽게 연결할 수 있으며, `α ≈ 0.6`, 최대 속도 향상은 약 2×이다.

**EAGLE-1 (Li et al., 2024).** 드래프트는 검증기의 마지막 레이어 은닉 상태를 입력으로 받아 다음 토큰을 직접 예측하는 1~2개 레이어의 작은 트랜스포머다. 드래프트가 검증기의 특징 표현을 보기 때문에 분포가 검증기에 훨씬 가깝다. `α`는 0.7–0.8로 상승한다.

**EAGLE-2 (Li et al., 2024).** 동적 드래프트 트리를 추가한다: `N`개 토큰의 단일 시퀀스를 제안하는 대신, 후보들의 작은 트리를 제안하고 검증기로 한 번의 순전파로 각 후보를 점수화한 후(트리 어텐션), 가장 높은 확률 경로를 따라간다. 드래프트 길이는 단계마다 적응적으로 변한다. 수락된 경로 토큰당 `α`는 0.85 이상으로 상승한다.

**EAGLE-3 (Li et al., 2025, NeurIPS).** 두 가지 변경 사항. 첫째, 특징 예측 손실을 완전히 제거한다 — EAGLE-1/2는 드래프트를 검증기의 은닉 상태와 일치하도록 훈련시켰는데, 이는 데이터 활용도를 제한한다. EAGLE-3는 토큰 예측에 직접 훈련한다. 둘째, 훈련-시간 테스트(TTT): 드래프트 훈련 중 드래프트의 이전 예측들을 여러 단계에 걸쳐 입력으로 다시 공급하며, 이는 추론 시 동작 방식과 동일하다. 이는 훈련 및 테스트 분포를 정렬하고 오류 누적을 방지한다. 측정된 속도 향상: 채팅에서 최대 6.5×, H100의 SGLang에서 배치 64 시 38% 처리량 향상.

### KV 캐시 롤백

검증은 한 번의 순전파로 검증기의 KV 캐시를 `N`개 항목만큼 확장한다. 위치 `j`에서 거부가 발생하면 `j-1` 위치 이후의 캐시 내용은 이제 잘못된다. 두 가지 일반적인 구현: 수락 시 스크래치 버퍼에 쓰고 커밋(vLLM, TensorRT-LLM), 또는 물리적 KV 캐시와 논리적 길이를 유지하고 거부 시 잘라내기. 어느 쪽이든 롤백 비용은 레이어당 헤드당 바이트 단위이며, 순전파 비용에 비해 무시할 수 있다.

EAGLE-2 트리 검색의 경우, 검증기는 트리 토폴로지를 존중하는 비인과적 마스크로 어텐션을 실행한다. 엔지니어링은 까다롭지만 계산은 사용자 정의 마스크를 가진 표준 플래시-어텐션 호출이다.

### 2026년 드래프트 아키텍처

| 전략 | 드래프트 유형 | `α` | 속도 향상 | 훈련 비용 |
|----------|-----------|-----|---------|---------------|
| 바닐라 | 별도 소형 LLM | 0.55-0.70 | 1.8-2.3× | 없음 (기존 소형 모델 재사용) |
| 메두사 | 검증기에 추가 LM 헤드 | 0.65-0.75 | 2-3× | ~1B SFT 토큰 |
| EAGLE-1 | 은닉 상태 기반 1-레이어 트랜스포머 | 0.70-0.80 | 2.5-3× | ~60B 토큰 |
| EAGLE-2 | EAGLE-1 + 동적 드래프트 트리 | 0.80-0.88 | 3-4× | ~60B 토큰 |
| EAGLE-3 | 다중 레이어 특징 융합 + TTT | 0.88-0.92 | 3.5-6.5× | ~60-200B 토큰 |
| Lookahead | 드래프트 없음 (Jacobi 반복) | N/A | 1.3-1.6× | 없음 |

2026년 프로덕션: vLLM과 SGLang은 사용 가능한 경우 EAGLE-3, 그렇지 않으면 EAGLE-2를 기본으로 한다. TensorRT-LLM은 Meta 및 NVIDIA 공개 모델에 대해 가장 빠른 메두사 경로를 제공한다. llama.cpp는 CPU 배포를 위해 바닐라 드래프트를 제공한다.

## 빌드하기

`code/main.py`를 참조하세요. 여기에는 모든 구성 요소가 포함된 완전한 리바이어던 추측 루프(draft-of-N, 검증자 병렬 패스, 위치별 거부, 잔여 샘플링, 보너스 토큰, KV 롤백, 그리고 출력 분포가 `q`의 직접 샘플링과 일치함을 경험적으로 검증하는 부분)가 구현되어 있습니다.

### 1단계: 거부 규칙

```python
def accept(q_prob, p_prob, u):
    if p_prob <= 0:
        return True
    return u < min(1.0, q_prob / p_prob)
```

### 2단계: 잔여 분포

```python
def residual(q, p):
    raw = [max(0.0, qi - pi) for qi, pi in zip(q, p)]
    s = sum(raw)
    if s == 0:
        return list(q)
    return [r / s for r in raw]
```

### 3단계: 전체 추측 단계

`spec_step` 함수는 `p`에서 `N`개의 토큰을 초안으로 작성한 후, 하나의 병렬 `q` 평가에서 모든 토큰을 검증합니다. 각 초안 토큰에 대해 거부 규칙을 적용하고, 첫 번째 거부 시 잔여 분포에서 수정 토큰을 샘플링합니다. 모든 토큰이 수락되면 `q_{N+1}`에서 보너스 토큰을 출력합니다.

### 4단계: KV 롤백 장부 관리

시뮬레이터는 작업자별로 논리적 `kv_length`를 추적합니다. `k`개의 초안이 수락되면 `kv_length += k`가 됩니다. 위치 `j`에서 거부가 발생하면 캐시는 이미 `j`를 넘어 쓰여졌지만, 논리적 길이는 `prefix_length + j + 1`로 설정됩니다. 이후 읽기는 논리적 길이로 잘립니다.

### 5단계: 리바이어던 검증

50,000번의 추측 단계를 실행합니다. 수락된 토큰의 경험적 분포를 세고, `q`에서 직접 샘플링한 50,000개의 샘플과 비교합니다. 카이제곱 통계량은 임계값보다 훨씬 낮아야 합니다. 이 정리는 실제로 통과됩니다.

### 6단계: α 대비 속도 향상

`p`를 `q`에서 다양한 진폭으로 변형하여 초안 품질을 스위핑합니다. `α`를 측정한 후, `α`와 `N`에 따른 검증자 호출당 예상 토큰 수를 그래프로 그립니다. 코드는 EAGLE-3급 초안 품질(`α ≈ 0.9`)이 검증자 호출당 4–5개의 토큰을 생성하는 방법을 보여주는 표를 출력합니다.

## 사용 방법

프로덕션 수준의 `vllm serve`와 EAGLE-3 사용:

```bash
vllm serve meta-llama/Llama-3.3-70B-Instruct \
  --speculative-config '{
    "model": "yuhuili/EAGLE3-LLaMA3.3-Instruct-70B",
    "num_speculative_tokens": 5,
    "method": "eagle3"
  }'
```

H100에서 배치 64로 SGLang과 EAGLE-3 사용 시: EAGLE-3 논문에 따르면 배치-64 기본 디코딩 대비 약 1.38배 더 높은 처리량.

추측 디코딩을 사용해야 하는 경우:

- p50 레이턴시가 최대 처리량보다 중요한 모든 대화형 채팅 작업.
- 코드 생성 및 구조화된 출력(JSON, SQL). 대상 분포가 매우 예측 가능하므로 `α`가 0.9 이상.
- 장문 생성(수천 개의 토큰). 상각된 속도 향상이 지속적으로 유효.

사용하지 말아야 할 경우:

- 매우 작은 모델(< 3B). 초안 모델이 검증 모델보다 크게 저렴하지 않음.
- 작은 배치-1 CPU 배포. 초안 모델의 메모리 오버헤드가 가치가 없을 수 있음.
- `α`가 붕괴되는 매우 높은 온도(temperature)의 창의적 샘플링.

## 연습 문제

1. `code/main.py`를 실행하세요. 50,000개 샘플에서 Leviathan 분포 검정의 카이제곱 통계량(chi-square statistic)이 95% 임계값(critical value) 이하로 유지되는지 확인하세요.

2. `α`를 0.9로 고정하고 `c`를 0.04로 고정한 상태에서 `N`을 1부터 10까지 스위핑(sweep)하세요. 검증자 호출(verifier call)당 예상 토큰 수와 실제 벽시계 시간(wall time)당 토큰 수를 플롯하세요. 벽시계 시간을 최소화하는 `N`을 찾고 곡선의 형태를 설명하세요.

3. EAGLE-2 트리 검색(tree search)을 시뮬레이션하도록 코드를 수정하세요: 각 단계에서 드래프트(draft)는 `[2, 2, 2]` 형태의 트리(여덟 개의 후보 경로)를 제안합니다. 검증자(verifier)는 한 번 실행되며, 가장 높은 확률의 수용된 경로(accepted path)가 승리합니다. 리프(leaf)당 `α`와 검증자 호출당 총 토큰 수를 계산하세요. 동등한 계산량에서 선형 체인(linear-chain) 스펙 디코딩(spec-decoding)과 비교하세요.

4. 두 개의 동시 시퀀스(sequence)에 대한 배치 KV 롤백(batch KV rollback) 시뮬레이터를 구현하세요. 시퀀스 A는 모든 드래프트가 수용되고, 시퀀스 B는 위치 2에서 거부됩니다. 시퀀스별로 올바른 `kv_length`가 업데이트되고 작업이 낭비되지 않음을 보여주세요.

5. EAGLE-3 논문의 4절(Training-Time Test)을 읽으세요. TTT 없이 순진한 드래프트 훈련(naive draft training)이 노출 편향(exposure bias)에 시달리는 이유와 훈련 중 드래프트에 자신의 예측을 제공하는 것이 이를 해결하는 이유를 두 문장으로 설명하세요. 이를 seq2seq의 예약 샘플링(scheduled-sampling) 문헌과 연결지어 설명하세요.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|----------------|------------------------|
| 리바이어던 규칙(Leviathan rule) | "min(1, q over p)" | 베르누이(Bernoulli) 수용/거부 규칙으로 확률 `min(1, q(d)/p(d))`를 사용, 거부 시 잔여 분포에서 샘플링할 때 검증자 분포를 정확히 보존 |
| 잔여 분포(Residual distribution) | "(q minus p) plus, normalized" | `(q - p)_+`를 0으로 클램핑하고 재정규화한 분포 — 거부 시 샘플링해야 할 올바른 분포 |
| 수용률 α(Acceptance rate α) | "초안이 얼마나 자주 맞는지" | 거부 규칙 하에서 예상되는 토큰별 베르누이 성공 확률, 모든 속도 향상 계산을 지배 |
| EAGLE-1 | "은닉 상태 초안" | 검증자의 마지막 레이어 은닉 상태에 조건화된 소형 트랜스포머 초안 (Li et al., 2024) |
| EAGLE-2 | "동적 초안 트리" | EAGLE-1에 트리 어텐션으로 점수화한 후보 연속체 트리를 추가, 단일 검증자 패스로 처리 |
| EAGLE-3 | "훈련 시간 테스트" | 특징 예측 손실 제거, 훈련 시 초안 출력을 직접 토큰 예측에 사용 |
| 훈련 시간 테스트(TTT) | "노출 편향 수정" | 훈련 시 초안을 자기회귀적으로 실행하여 훈련 및 테스트 입력 분포를 일치시킴 — 예약 샘플링의 직접적 유사체 |
| KV 롤백(KV rollback) | "거부된 초안 취소" | 거부 후 검증자의 KV 캐시를 수용된 접두사 길이로 재설정하는 부기 작업 |
| 보너스 토큰(Bonus token) | "무료 토큰" | 모든 `N` 초안이 수용될 때, 추가 검증자 비용 없이 `q_{N+1}`에서 하나 더 샘플링 |
| 트리 어텐션(Tree attention) | "한 번에 여러 후보 검증" | 초안 트리 위상을 존중하는 비인과적 마스크를 사용한 어텐션, 단일 순방향 패스로 트리 내 모든 노드의 `q_i` 계산 |
