# 네이티브 스파스 어텐션 (DeepSeek NSA)

> 64k 토큰에서 어텐션은 디코딩 지연 시간의 70-80%를 차지합니다. 모든 오픈소스 모델 연구실은 이를 해결할 계획을 가지고 있습니다. DeepSeek의 NSA(ACL 2025 최우수 논문)는 실제로 효과를 발휘한 방법입니다: 압축된 coarse-grained 토큰, 선택적으로 유지된 fine-grained 토큰, 로컬 컨텍스트를 위한 슬라이딩 윈도우라는 세 가지 병렬 어텐션 브랜치를 학습된 게이트를 통해 결합합니다. 이는 하드웨어에 최적화되어 있으며(커널 친화적), 네이티브 훈련이 가능하고(추론 시 추가되는 것이 아닌 사전 훈련에서 작동), 64k 디코딩에서 FlashAttention보다 빠르게 실행되면서 전체 어텐션 품질과 동등하거나 더 나은 성능을 보입니다. 이 강의에서는 세 가지 브랜치를 종단간으로 구축하고 희소성이 종단간으로 미분 가능한 이유를 설명합니다.

**유형:** 구축
**언어:** Python (표준 라이브러리)
**사전 요구 사항:** 7단계 · 12 (KV 캐시, 플래시 어텐션), 7단계 · 15 (어텐션 변형), 10단계 · 16 (미분 가능 어텐션)
**소요 시간:** ~60분

## 학습 목표

- 세 가지 NSA 어텐션 브랜치(attention branch)와 각 브랜치가 포착하는 내용을 설명하시오.
- 기존 희소 어텐션(sparse-attention) 방법들이 추론(inference-only)에만 국한되었던 반면, NSA가 "네이티브 훈련 가능(natively trainable)"인 이유를 설명하시오.
- 압축 블록 크기(compression block size)와 선택 상위-k(selection top-k)에 따른 64k 컨텍스트에서 NSA와 전체 어텐션(full attention) 간의 어텐션 계산 비용 절감량을 계산하시오.
- 짧은 합성 시퀀스(synthetic sequence)에 대해 stdlib Python으로 세 가지 브랜치 조합을 구현하고 게이팅 가중치(gating weights)의 동작을 검증하시오.

## 문제

시퀀스 길이 N에 대한 전체 어텐션(full attention)은 레이어당 `O(N^2)` 시간과 `O(N)` KV 캐시 메모리를 소모합니다. 64k 토큰에서는 계산 및 메모리 대역폭 수치가 재앙적입니다. NSA 논문의 이론적 측정 추정치에 따르면, 64k에서 어텐션은 전체 디코딩 지연 시간의 70-80%를 차지합니다. TTFT(첫 토큰 생성 시간), 초당 토큰 수, 백만 토큰당 비용 등 모든 후속 지표는 어텐션 비용에 의해 지배됩니다.

희소 어텐션(sparse attention)이 명백한 해결책입니다. 이전 시도들은 두 가지 범주로 나뉩니다. 고정 패턴 희소화(슬라이딩 윈도우, 스트라이드, 블록-로컬)는 정보를 폐기하며 장거리 기억 작업에서 실패합니다. 추론 시간 희소화(KV 캐시 프루닝, H2O, StreamingLLM)는 밀집 어텐션으로 사전 학습된 모델에 적용되며, 모델이 희소 패턴을 통해 정보를 라우팅하도록 학습되지 않았기 때문에 잠재적 속도 향상의 일부만 회복합니다.

네이티브 희소 어텐션(Native Sparse Attention, Yuan et al., DeepSeek + PKU + UW, ACL 2025 최우수 논문, arXiv:2502.11089)은 두 가지를 모두 수행합니다: 사전 학습 중 모델이 학습하는 희소화 패턴과, 추론 시 실제 계산 절감 효과를 제공하는 커널 정렬 알고리즘으로 구현됩니다. 2년 후, NSA 또는 그 직계 후손은 모든 최첨단 장문 컨텍스트 모델의 기본 어텐션 방식이 될 것입니다.

## 개념

### 세 개의 병렬 분기

각 쿼리에 대해 NSA는 KV 캐시의 세 가지 다른 뷰에 대해 세 번 어텐션을 실행합니다:

1. **압축 분기(Compressed branch).** 토큰은 크기 `l`(일반적으로 32 또는 64)의 블록으로 그룹화됩니다. 각 블록은 작은 학습된 MLP를 통해 단일 요약 토큰으로 압축됩니다. 쿼리는 이러한 압축된 토큰에 대해 어텐션을 수행하여 전체 시퀀스에 대한 거친(coarse-grained) 뷰를 얻습니다.

2. **선택 분기(Selected branch).** 압축 분기의 어텐션 점수를 사용하여 현재 쿼리와 가장 관련성이 높은 상위-k 블록을 식별합니다. 해당 블록의 세밀한(fine-grained) 압축되지 않은 토큰을 읽고 쿼리는 이들 모두에 대해 어텐션을 수행합니다. 압축 분기의 어텐션을 선택 라우팅 신호로 생각할 수 있습니다.

3. **슬라이딩 윈도우 분기(Sliding-window branch).** 쿼리는 최근 `W` 토큰(일반적으로 512)에 대해 어텐션을 수행하여 로컬 컨텍스트를 포착합니다. 이 분기는 다른 두 분기가 놓칠 수 있는 구조 중심의 단거리 패턴(구문, 로컬 코레퍼런스)을 포착합니다.

세 분기 출력은 학습된 위치별 게이트를 통해 결합됩니다:

```
out = g_cmp * out_cmp + g_sel * out_sel + g_win * out_win
```

`g_cmp, g_sel, g_win`은 쿼리에 대한 작은 MLP에서 나온 게이트 가중치입니다. 이들은 합이 1이 될 필요가 없으며 독립적으로 분기에 가중치를 부여할 수 있습니다.

### 이것이 "네이티브 훈련 가능"인 이유

선택 단계(상위-k 블록)는 이산적입니다. 이산 연산은 기울기 흐름을 끊습니다. 이전의 희소 어텐션 작업은 선택을 통한 역전파를 생략하거나(훈련 제한) 추론 시 실제 희소성을 제공하지 않는 연속 완화를 사용했습니다.

NSA는 이를 우회합니다: 압축 분기 어텐션은 전체 시퀀스에 대한 미분 가능한 거친 어텐션입니다. 상위-k 연산은 압축 분기의 최고 어텐션 점수를 재사용하여 어떤 세밀한 블록을 로드할지 선택합니다. 기울기는 압축 분기 점수(압축 출력 및 선택 로직 모두에 영향을 미침)를 통해 흐르며, 선택된 블록의 최종 출력 기여도 미분 가능합니다. 비미분 가능한 `top_k` 연산은 순방향 계산 그래프에서 무의미(no-op)하며, 메모리에서 로드할 블록만 제어합니다.

이것이 NSA가 사전 훈련에 종단간 사용될 수 있는 이유입니다. 모델은 세 분기를 통해 정보를 공동으로 라우팅하는 방법을 학습하며, 추론 시 실제로 약속된 속도 향상을 제공하는 희소 패턴을 생성합니다.

### 하드웨어 정렬 커널

NSA의 커널은 현대 GPU 메모리 계층 구조에 맞게 설계되었습니다. 커널은 GQA 그룹 단위로 쿼리(외부 루프)를 로드하고, 그룹별 해당 희소 KV 블록을 페치(내부 루프)한 후 SRAM에서 어텐션을 실행합니다. 각 쿼리 그룹이 동일한 선택된 블록을 보기 때문에(선택은 쿼리 그룹별, 쿼리 헤드별이 아님), KV 로드는 그룹 전체에 걸쳐 분산됩니다. 산술 강도는 높게 유지됩니다.

논문은 64k 디코딩에서 FlashAttention보다 9배 빠른 Triton 커널을 보고하며, 시퀀스 길이가 증가함에 따라 속도 향상 비율도 증가합니다. 순방향 및 역방향 커널이 모두 제공됩니다.

### 계산 예산

`N`을 시퀀스 길이, `l`을 압축 블록 크기, `k`를 상위-k 선택 수, `w`를 슬라이딩 윈도우, `b`를 선택된 블록 크기(일반적으로 `l`과 동일)로 가정합니다.

- 압축 분기: 쿼리당 `O(N/l)` 키, 총 `O(N * N / l)`.
- 선택 분기: 쿼리당 `O(k * b)` 키, 총 `O(N * k * b)`.
- 슬라이딩 분기: 쿼리당 `O(w)` 키, 총 `O(N * w)`.

총계: `O(N * (N/l + k*b + w))`.

`N = 64k, l = 64, k = 16, b = 64, w = 512`일 때: 쿼리당 비용은 `1000 + 1024 + 512 = 2536` 키입니다. 전체 어텐션은 `64000` 키입니다. 25배 계산 감소.

`N = 128k, l = 64, k = 16, b = 64, w = 512`일 때: 쿼리당 비용은 `2000 + 1024 + 512 = 3536` 키입니다. 전체 어텐션은 `128000` 키입니다. 36배 감소. 시퀀스 길이가 증가함에 따라 이점이 커지며, 이것이 전체 목적입니다.

### 비교 방법

| 방법 | 미분 가능 | 실제 추론 속도 향상 | 장거리 재현율 |
|------|-----------|---------------------|---------------|
| 슬라이딩 윈도우만 | 예 | 예 | 실패 |
| 스트라이드/블록 희소 | 예 | 예 | 부분적 |
| KV 프루닝(H2O, StreamingLLM) | N/A(추론 시) | 예 | 부분적 |
| MoBA(Moonshot) | 부분적 | 예 | 우수 |
| NSA | 예(네이티브) | 예(64k에서 9배) | 전체 어텐션과 동일 |

MoBA(Moonshot, arXiv:2502.13189)는 동시에 발표되었으며, 어텐션 블록에 MoE 원리를 적용하는 유사한 "세 가지가 하나보다 낫다" 접근법을 취합니다. NSA와 MoBA는 2026년 장문 컨텍스트 사전 훈련을 위한 두 가지 주요 아키텍처입니다.

## 빌드하기

`code/main.py`는 짧은 합성 시퀀스에서 세 가지 브랜치를 구현하며 다음을 보여줍니다:

- 압축 MLP (교육적 명확성을 위해 간단한 평균 풀링 기준선을 사용함; 실제 NSA는 학습된 MLP를 사용)
- 압축 브랜치 점수에 의해 구동되는 상위-k 블록 선택
- 마지막 `w` 토큰에 대한 슬라이딩 윈도우 어텐션
- 게이팅 결합
- 전체 어텐션과 비교한 계산 횟수 출력

### 단계 1: 토큰을 블록으로 압축

```python
def compress(K, l):
    n = len(K)
    n_blocks = (n + l - 1) // l
    out = []
    for b in range(n_blocks):
        start, end = b * l, min((b + 1) * l, n)
        block = K[start:end]
        summary = [sum(row[d] for row in block) / len(block) for d in range(len(K[0]))]
        out.append(summary)
    return out
```

### 단계 2: 압축 브랜치 어텐션

쿼리에 대해 압축된 키에 대한 소프트맥스 어텐션을 실행합니다. 압축 브랜치 점수는 상위-k 선택을 위한 신호로 이중 사용됩니다.

### 단계 3: 상위-k 블록 선택

가장 높은 점수를 받은 `k`개의 압축 블록 인덱스를 선택합니다. 해당 블록에서 원본 압축되지 않은 토큰을 로드하고 이들에 대해 어텐션을 실행합니다.

### 단계 4: 슬라이딩 윈도우 어텐션

마지막 `w` 토큰을 가져와 이들에 대해 표준 어텐션을 실행합니다.

### 단계 5: 게이트 + 결합

쿼리에 대한 소형 MLP가 세 개의 게이트 가중치를 생성합니다. 최종 출력은 세 브랜치 출력의 가중 합입니다.

### 단계 6: 계산 횟수 계산

각 브랜치 및 전체 쿼리당 어텐션된 키 수를 출력합니다. 전체 어텐션(`N`)과 비교합니다. `l = 32, k = 4, w = 128`인 1024 토큰 합성 데이터에서 NSA는 쿼리당 `32 + 128 + 128 = 288`개의 키를 처리하는 반면 전체 어텐션은 1024개를 처리합니다 — 3.5배 더 적음.

## 사용 방법

NSA는 DeepSeek의 자체 장문 컨텍스트 사전 학습 파이프라인에 탑재되어 있습니다. 2026년 4월 기준 공개 추론 스택 통합 현황:

- **DeepSeek 내부**: 네이티브 지원, 공개된 가중치들은 NSA 또는 후속 기술인 DSA(Deepseek Sparse Attention)를 사용.
- **vLLM**: DeepSeek-V3.x 가중치용 실험적 NSA 지원 개발 중.
- **SGLang**: NSA 벤치마크 공개됨; 프로덕션 경로는 vLLM을 따름.
- **llama.cpp / CPU**: 미지원; 커널 분해 오버헤드가 CPU 처리량 대비 비효율적.

NSA를 사용해야 하는 경우:

- 64k 이상의 컨텍스트를 목표로 하는 사전 학습 또는 추가 학습 실행 시, 충분한 컴퓨팅 예산 확보 시.
- DeepSeek의 자체 장문 컨텍스트 체크포인트 추론 시. 가중치는 NSA 네이티브.

사용하지 말아야 할 경우:

- 기존 밀집 어텐션 사전 학습 모델 서빙 시. 추가 학습 없이는 NSA를 적용할 수 없음.
- 16k 미만 컨텍스트. 3개 분기 오버헤드가 절감 효과를 압도.
- 배치-1 대화형 채팅. 지연 시간 민감 디코딩에는 이점이 있으나, 장문 컨텍스트에서만 유효.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|----------------|------------------------|
| 압축 분기(Compressed branch) | "Coarse view" | 블록 평균화된 키(block-averaged keys)에 대한 어텐션(attention)으로, 쿼리당 O(N/l) 키로 전역 컨텍스트(global context)를 제공 |
| 선택 분기(Selected branch) | "Top-k blocks" | 압축 분기 점수가 가장 높은 `k`개 블록에 대한 세밀한 어텐션(fine-grained attention) |
| 슬라이딩 윈도우(Sliding window) | "Local context" | 단거리 패턴(short-range patterns)을 위한 마지막 `W`개 토큰에 대한 어텐션 |
| 네이티브 훈련 가능성(Native trainability) | "Pre-train with the sparsity on" | 추론 시 추가되는 것이 아닌, 사전 훈련 중에 희소성 패턴(sparsity pattern)이 학습됨 |
| 압축 블록 크기 l(Compression block size l) | "Group size for coarse view" | 하나의 요약으로 병합되는 토큰 수; 일반적으로 32-64 |
| Top-k(Top-k) | "Blocks to keep" | 압축 해제된 토큰이 읽히는 압축 블록 수; 일반적으로 16 |
| 슬라이딩 윈도우 W(Sliding window W) | "Local attention radius" | 일반적으로 512; 짧으면 지역 일관성(local coherence)이 손상되고, 길면 계산 자원이 낭비됨 |
| 분기 게이트(Branch gate) | "How to mix the three" | 세 분기의 기여도를 가중치로 부여하는 위치별 MLP 출력 |
| 하드웨어 정렬(Hardware alignment) | "Kernel-friendly sparsity" | 실제 GPU 커널이 이론적 속도 향상을 달성할 수 있도록 선택된 희소 패턴 |
| DSA(DSA) | "NSA's successor" | DeepSeek의 계보에서 NSA를 계승한 아키텍처, Deepseek Sparse Attention |
