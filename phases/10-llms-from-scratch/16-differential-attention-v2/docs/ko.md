# 차등 어텐션(Differential Attention, V2)

> 소프트맥스 어텐션은 모든 비매칭 토큰에 작은 확률을 분산시킵니다. 10만 개 이상의 토큰에서는 이 노이즈가 누적되어 신호를 압도합니다. 차등 트랜스포머(Differential Transformer, Ye et al., ICLR 2025)는 두 소프트맥스의 차이로 어텐션을 계산하여 공유되는 노이즈 바닥을 제거함으로써 이 문제를 해결합니다. DIFF V2(Microsoft, 2026년 1월)는 프로덕션 스택에 최적화된 재구현 버전으로, 기존 트랜스포머와 동일한 디코딩 지연 시간을 가지며 커스텀 커널이 필요 없고 FlashAttention과 호환됩니다. 이 레슨은 V1에서 V2까지의 전체 과정을 다루며, 표준 Python 라이브러리에서 실행할 수 있는 차등 연산 토이 구현체를 포함합니다.

**유형:** 구현(Build)
**언어:** Python (표준 라이브러리)
**선수 지식:** Phase 7 · 02 (셀프 어텐션), Phase 7 · 15 (어텐션 변형), Phase 10 · 14 (아키텍처 워크스루)
**소요 시간:** ~60분

## 학습 목표

- 소프트맥스 어텐션(softmax attention)이 노이즈 플로어(noise floor)를 가지는 정확한 이유와 컨텍스트 길이(context length)가 증가함에 따라 노이즈 플로어가 성장하는 이유를 설명할 수 있다.
- 미분 어텐션(differential attention) 공식을 유도하고, 뺄셈 연산이 공유 노이즈 성분(shared noise component)을 상쇄시키면서 신호(signal)는 보존하는 이유를 설명할 수 있다.
- V1에서 V2로의 변경 사항(diff)을 설명할 수 있다: 무엇이 더 빨라졌는지, 무엇이 더 단순해졌는지, 무엇이 더 안정화되었는지, 그리고 각 변경 사항이 프로덕션 사전 학습(production pre-training)에 왜 필요했는지 설명할 수 있다.
- 순수 Python으로 미분 어텐션을 처음부터 구현하고, 합성 신호-노이즈(synthetic signal-plus-noise) 쿼리에서 노이즈 상쇄(noise-cancellation) 특성을 실험적으로 검증할 수 있다.

## 문제 정의

표준 소프트맥스 어텐션은 수학적으로 확장 시 운영상의 골칫거리로 변하는 특성을 가집니다. 쿼리 `q`에 대해 어텐션 가중치는 `softmax(qK^T / sqrt(d))`로 계산됩니다. 소프트맥스는 절대 정확한 0을 생성할 수 없습니다 — 모든 불일치 토큰은 약간의 양의 확률을 할당받습니다. 이 잔여 확률은 노이즈이며, 컨텍스트 길이에 비례하여 증가합니다. 128k 토큰에서 각 불일치 토큰이 0.001%의 확률만 할당받더라도, 127,999개 토큰이 합쳐지면 전체 확률의 약 12%를 차지하게 됩니다. 모델은 컨텍스트 길이에 따라 증가하는 노이즈 바닥을 우회하는 방법을 학습해야 합니다.

경험적으로 이는 어텐션 헤드 간섭 현상으로 나타납니다: 긴 컨텍스트 RAG에서의 환각 인용, 100k 토큰 검색 작업에서의 중간 토큰 손실, 32k를 넘는 니들-인-헤이스택 벤치마크에서의 미묘한 정확도 저하 등입니다. Differential Transformer 논문(arXiv:2410.05258, ICLR 2025)은 이 격차를 측정했습니다: DIFF Transformer는 동일 규모 베이스라인 대비 낮은 퍼플렉서티, 높은 긴 컨텍스트 정확도, 적은 환각 현상을 보였습니다.

DIFF V1은 프론티어 사전 학습 파이프라인에 적용되지 못하게 한 세 가지 문제점이 있었습니다. 디코딩 단계마다 값 캐시를 두 번 로드해야 했고, FlashAttention 호환성을 깨는 커스텀 CUDA 커널이 필요했으며, 헤드별 RMSNorm이 70B+ 규모에서 장기 학습을 불안정하게 만들었습니다. DIFF V2(Microsoft unilm 블로그, 2026년 1월 20일)는 이 세 가지 문제를 모두 해결했습니다. 이 레슨에서는 두 버전을 모두 살펴보고, 차분 연산자를 구축하며, 장난감 쿼리에서 노이즈 제거 성능을 벤치마크합니다.

## 개념

### 소프트맥스의 노이즈 바닥

쿼리 `q`와 키 `K = [k_1, ..., k_N]`에 대해 어텐션 가중치는 다음과 같습니다:

```
w_i = exp(q . k_i / sqrt(d)) / sum_j exp(q . k_j / sqrt(d))
```

어떤 `w_i`도 0이 되지 않습니다. `k_i`가 `q`와 완전히 무관하면 점수 `q . k_i`는 0이 아닌 — 분산 `||q||^2 / d`로 0 주변에서 변동합니다. 소프트맥스 정규화 후, 무관한 토큰은 여전히 가중합에 `O(1/N)`을 기여합니다. 무관한 토큰의 총 기여는 `O((N-1)/N) = O(1)`로, 작은 양이 아닙니다.

모델이 원하는 것은 하드 탑-k와 유사한 것입니다: 일치하는 토큰에 높은 가중치, 나머지에는 거의 0에 가까운 가중치. 소프트맥스는 너무 부드러워서 이를 직접 수행할 수 없습니다.

### 미분 기반 아이디어

각 헤드의 Q와 K 투영을 두 부분으로 분할합니다: Q = (Q_1, Q_2) 및 K = (K_1, K_2). 두 어텐션 맵을 계산합니다:

```
A_1 = softmax(Q_1 K_1^T / sqrt(d))
A_2 = softmax(Q_2 K_2^T / sqrt(d))
```

출력:

```
DiffAttn = (A_1 - lambda * A_2) V
```

뺄셈은 두 맵이 공유하는 노이즈 분포를 상쇄합니다. 두 맵이 127k개의 무관한 토큰에 대해 대략 균일한 가중치를 가지면(무작위 초기화 시 발생) 이들은 상쇄됩니다. 신호 — 실제로 관련된 소수의 토큰에 집중된 가중치 — 는 두 맵에 동일한 크기로 나타나지 않으므로 상쇄되지 않습니다. 모델이 훈련되면 이 현상이 발생합니다.

`lambda`는 헤드당 학습 가능한 스칼라로, `lambda = exp(lambda_q1 dot lambda_k1) - exp(lambda_q2 dot lambda_k2) + lambda_init`로 파라미터화됩니다. 음수일 수 있습니다. `lambda_init`은 기본적으로 0.8과 같은 작은 양수로 설정됩니다.

### 헤드별 노이즈 상쇄와의 일치

두 개의 노이즈가 있는 마이크가 동일한 음성을 녹음하는 상황을 생각해 보세요. 두 마이크에는 화자 음성과 상관된 배경 노이즈가 모두 포함됩니다. 하나를 다른 하나에서 빼면 공유된 노이즈가 제거됩니다. 음성은 두 신호가 위상이나 진폭에서 충분히 달라 완전한 상쇄가 방지되므로 살아남습니다. 헤드별 `lambda`는 정확히 이 균형을 학습합니다.

### V1 vs V2: 차이점

V1은 파라미터 수를 기본 트랜스포머와 동일하게 유지했습니다. 헤드당 두 개의 쿼리를 얻기 위해 헤드 차원을 절반으로 줄였습니다. 이는 헤드의 표현력을 감소시켰고, 더 고통스럽게도 헤드당 값 캐시를 절반으로 줄였습니다. 디코딩 시 값 캐시를 단계당 두 번 로드해야 했습니다(한 소프트맥스 분기당 한 번). 결과: 파라미터 수는 일치하지만 디코딩 속도가 기본 모델보다 느렸습니다.

V2는 쿼리 헤드 수를 두 배로 늘리고 KV 헤드는 동일하게 유지합니다(업-프로젝션에서 파라미터를 차용). 헤드 차원은 기본 모델과 동일하게 유지됩니다. 뺄셈 후, 추가 차원은 기본 트랜스포머의 O_W 투영과 일치하도록 다시 축소됩니다. 세 가지 일이 동시에 발생합니다:

1. 디코딩 속도가 기본 모델과 일치합니다(KV 캐시는 한 번만 로드됨).
2. FlashAttention이 변경 없이 실행됩니다(커스텀 커널 불필요).
3. 디코딩 시 산술 강도가 증가합니다(HBM에서 로드된 바이트당 더 많은 계산).

V2는 V1에서 뺄셈을 안정화하기 위해 사용했던 헤드별 RMSNorm을 제거합니다. 70B급 사전 훈련 규모에서 이 RMSNorm은 훈련 후반에 불안정성을 유발했습니다. V2는 추가 모듈 없이도 훈련을 안정화하는 더 간단한 초기화 방식으로 대체합니다.

## 구축 방법

`code/main.py`는 순수 Python으로 미분 어텐션(differential attention)을 구현합니다. 알려진 신호-잡음 구조를 가진 장난감 쿼리(toy query)를 통해 잡음-제거 비율(noise-cancellation ratio)을 직접 측정할 수 있습니다.

### 1단계: 표준 소프트맥스 어텐션

표준 라이브러리 행렬 연산: 리스트 안의 리스트, 수동 행렬 곱셈(matmul), 수치적 안정성을 위한 최댓값(max) 감산 후 소프트맥스.

```python
def softmax(row):
    m = max(row)
    exps = [math.exp(x - m) for x in row]
    s = sum(exps)
    return [e / s for e in exps]
```

### 2단계: Q, K를 두 부분으로 분할

V1 스타일: 헤드 차원(head dimension)을 절반으로 분할. V2 스타일: 헤드 차원은 유지하고 헤드 수를 두 배로 증가. 장난감 구현체는 교육적 명확성을 위해 V1을 사용 — 수학적 계산은 동일하며, 관리 방식(bookkeeping)만 다름.

### 3단계: 두 개의 소프트맥스 분기 + 감산

```python
A1 = [softmax([dot(q1, k) / scale for k in K1]) for q1 in Q1]
A2 = [softmax([dot(q2, k) / scale for k in K2]) for q2 in Q2]
diff_weights = [[a1 - lam * a2 for a1, a2 in zip(r1, r2)] for r1, r2 in zip(A1, A2)]
out = [[sum(w * v[j] for w, v in zip(row, V)) for j in range(d_v)] for row in diff_weights]
```

참고: 출력 가중치는 음수일 수 있습니다. 이는 문제되지 않습니다 — 값 캐시(value cache)는 여전히 부호 있는 기여를 처리합니다. 이후 V 투영(projection)이 부호를 흡수합니다.

### 4단계: 잡음 제거 측정

길이 1024의 합성 시퀀스(synthetic sequence)를 구축합니다. 신호 토큰(signal token)을 알려진 위치에 배치하고 나머지는 잡음으로 채웁니다. (a) 표준 소프트맥스 어텐션의 신호 위치 가중치와 (b) 미분 어텐션 가중치를 계산합니다. 각각의 신호-대-잡음 비율을 측정합니다. DIFF 어텐션은 두 분기가 얼마나 다르게 훈련되었는지에 따라 3x-10x 더 높은 신호-대-잡음 비율을 안정적으로 생성합니다.

### 5단계: V1 vs V2 파라미터 계산

구성(config: hidden=4096, heads=32, d_head=128)이 주어졌을 때, 다음을 출력합니다:

- 베이스라인 트랜스포머: Q, K, V 각각 크기 `hidden * hidden`, MLP는 4 * hidden.
- DIFF V1: Q, K 각각 크기 `hidden * hidden`, V 크기 `hidden * hidden` (변경 없음), 내부적으로 헤드 차원(head dim) 절반. 헤드별 `lambda` 파라미터 추가 (O(heads * d_head)).
- DIFF V2: Q 크기 `2 * hidden * hidden`, K 크기 `hidden * hidden`, V 크기 `hidden * hidden`. 추가 차원은 O_W 이전에 다시 축소. 동일한 `lambda` 파라미터 추가.

장난감 구현체는 V2의 추가 파라미터 비용(대략 어텐션 블록당 `hidden * hidden` 추가)을 측정하고 출력합니다.

## 사용 방법

DIFF V2는 2026년 4월 기준으로 모든 프로덕션 추론 서버에서 아직 제공되지는 않지만, vLLM과 SGLang에서 통합 작업이 진행 중입니다. 한편 이 패턴은 다음에서 나타납니다:

- Microsoft 내부 장문 컨텍스트 프로덕션 모델.
- 256k 이상의 컨텍스트를 목표로 하는 여러 오픈 모델 훈련 실행에서의 연구 재현.
- DIFF 어텐션과 슬라이딩 윈도우 어텐션을 번갈아 적용하는 하이브리드 아키텍처.

2026년에 이 기술을 사용해야 하는 경우:

- 64k 이상의 효과적인 컨텍스트를 목표로 새 모델을 처음부터 훈련할 때. 처음부터 차등 어텐션을 추가하세요. 나중에 재훈련하는 것은 비용이 많이 듭니다.
- 중간 부분 정보 손실(lost-in-the-middle) 실패가 평가를 지배하는 장문 컨텍스트 모델을 파인튜닝할 때. Q 프로젝션에 LoRA를 적용하면 DIFF 구조를 근사할 수 있습니다.

사용하지 말아야 하는 경우:

- 안정적인 장문 컨텍스트 성능을 가진 사전 훈련된 밀집 모델을 서비스할 때. 기존 가중치에 대한 재훈련 비용은 거의 회수되지 않습니다.
- 컨텍스트가 항상 16k 미만일 때. 노이즈 플로어(noise floor)는 무시할 수 있습니다.

## 주요 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|----------------|------------------------|
| 차분 어텐션(Differential attention) | "두 소프트맥스(softmax)를 서로 뺀 것" | Q(Query), K(Key)를 두 부분으로 분할, 두 개의 소프트맥스 맵 계산, 첫 번째에서 두 번째(람다(lambda)로 스케일링)를 뺀 후 V(Value)와 곱함 |
| 노이즈 플로어(Noise floor) | "소프트맥스의 비영(non-zero) 꼬리" | 소프트맥스가 모든 관련 없는 토큰에 부여하는 O(1/N) 가중치, 긴 컨텍스트에서 O(1)로 합산됨 |
| 람다(lambda) | "뺄셈 스케일" | 헤드별 학습 가능한 스칼라 파라미터, `exp(lq1.lk1) - exp(lq2.lk2) + lambda_init`로 파라미터화됨; 음수 가능 |
| DIFF V1 | "ICLR 2025 버전" | 원본 차분 트랜스포머(Differential Transformer); 파라미터 수 보존을 위해 헤드 차원(head dim)을 절반으로 분할, 커스텀 커널 필요, 디코딩 속도 느림 |
| DIFF V2 | "2026년 1월 수정판" | KV 헤드는 유지한 채 Q(Query) 헤드를 두 배로 증가; 베이스라인 디코딩 속도와 일치, FlashAttention과 호환 |
| 헤드별 RMSNorm(Per-head RMSNorm) | "V1 안정화 장치" | 차분 후 V1에 적용된 추가 정규화(norm); V2에서는 학습 후기 불안정성 방지를 위해 제거됨 |
| 신호 대 잡음비(Signal-to-noise ratio) | "얼마나 많은 어텐션이 낭비되는가" | 실제 신호 위치에 대한 가중치와 관련 없는 위치의 평균 가중치 간 비율 |
| 중간에서 길을 잃음(Lost in the middle) | "긴 컨텍스트 실패 모드" | 긴 컨텍스트 중간에 위치한 문서의 검색 정확도가 떨어지는 경험적 현상 — 차분 어텐션(DIFF attention)이 이를 감소시킴 |
| 산술 강도(Arithmetic intensity) | "로드된 바이트당 FLOPs" | KV 로드당 쿼리(Q) 두 배로 인해 V2에서 증가한 디코딩 시 비율; 메모리 바운드 디코딩에 중요 |
