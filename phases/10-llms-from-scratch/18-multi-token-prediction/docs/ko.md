# 멀티토큰 예측(Multi-Token Prediction, MTP)

> GPT-2부터 Llama 3까지 모든 자기회귀적 LLM은 한 위치당 하나의 손실(다음 토큰 예측)로 훈련됩니다. DeepSeek-V3는 한 위치당 두 번째 손실(다음다음 토큰 예측)을 추가했습니다. 추가 140억 개의 파라미터(6710억 개 모델 기준)는 그래디언트 흐름을 통해 메인 모델로 증류되었고, 훈련된 MTP 헤드는 추론 시 80% 이상의 수용률을 보이는 추측 디코딩 드래프터로 재활용되었습니다. 1.8배 생성 처리량이 무료로 제공되었습니다. 이 레슨은 DeepSeek 기술 보고서에서 순차적 MTP 모듈을 구축하고, 손실 및 공유 헤드 파라미터 구성을 계산하며, MTP가 인과적 체인을 유지하는 반면 Gloeckle 등의 원래 병렬 MTP는 이를 깨뜨린 이유를 설명합니다.

**유형:** 구축(Build)
**언어:** Python (stdlib)
**선수 지식:** Phase 10 · 04 (미니 GPT 사전 훈련), Phase 10 · 15 (추측 디코딩)
**소요 시간:** ~60분

## 학습 목표

- MTP 훈련 목표를 명시하고 예측 깊이(prediction depths)에 걸친 결합 손실(joint loss)을 유도해 내라.
- Gloeckle et al.의 병렬 MTP 헤드(2024)와 DeepSeek-V3의 순차적 MTP 모듈 간의 차이점을 설명하고, 순차적 설계가 인과적 연쇄(causal chain)를 보존하는 이유를 설명하라.
- 사전 훈련(pre-training) 실행에 MTP 모듈을 추가할 때의 파라미터 및 메모리 오버헤드를 계산하라.
- 하나의 MTP 모듈을 처음부터 구현하라: 공유 임베딩(shared embedding), 깊이별 트랜스포머 블록(per-depth transformer block), 프로젝션(projection), 공유 출력 헤드(shared output head).

## 문제 정의

다음 토큰 예측은 표준 LLM 학습 목표입니다. 모든 은닉 상태는 단 하나의 것, 즉 바로 다음 토큰을 예측하도록 감독됩니다. 이는 놀랍도록 약한 신호입니다. 시퀀스의 대부분의 정보(구조, 일관성, 사실성, 산술적 흐름)는 한 토큰을 넘어 확장됩니다. 모델은 수조 개의 토큰에 걸쳐 많은 단일 토큰 신호를 축적함으로써 이를 학습해야 합니다.

MTP는 다음과 같은 질문을 던집니다: 만약 모든 은닉 상태가 한 번에 여러 미래 토큰을 예측하도록 감독된다면 어떨까요? Gloeckle 등(Meta, 2024)은 이 접근법이 도움이 된다는 것을 보여주었습니다. 그들의 구현은 백본 위에 여러 독립적인 출력 헤드를 배치하여 각각 다른 오프셋을 예측하도록 했습니다. 이는 병렬적이고 단순하지만, 헤드들은 계층적 정제 없이 동일한 은닉 상태를 보았으며, 예측들은 인과적으로 연쇄되지 않아 추측 디코딩에 사용될 수 없었습니다.

DeepSeek-V3(2024년 12월)는 각 예측 깊이에서 인과적 연쇄를 유지하는 순차적 모듈로 MTP를 재설계했습니다. 모델은 `h_i^(0)`에서 `t+1`을 예측한 후, `h_i^(0)`과 `E(t+1)` 임베딩을 결합한 새로운 은닉 상태 `h_i^(1)`에서 `t+2`를 예측하는 식으로 진행합니다. 각 깊이는 자체적인 소형 트랜스포머 블록입니다. 공유 임베딩과 공유 출력 헤드는 매개변수 오버헤드를 적당히 유지합니다. DeepSeek-V3 규모에서 671B 메인 모델 가중치 위에 MTP 모듈에 14B 추가 매개변수가 사용되었습니다. 이 2%의 오버헤드는 더 밀도 있는 학습 신호와 추론 시 준비된 추측 디코딩 초안을 제공했습니다.

이 레슨은 단일 MTP 모듈과 D-깊이 손실을 처음부터 구축합니다. 수학은 깔끔합니다. 구현은 150줄입니다.

## 개념

### 순차적 MTP 레시피

DeepSeek-V3는 메인 모델 위에 `D`개의 MTP 모듈을 추가합니다. 각 모듈 `k`(여기서 `k = 1..D`)는 깊이 `k`에서의 토큰 — 즉, 위치 `i`까지의 접두사가 주어졌을 때 `t_{i+k}`를 예측합니다.

모듈 `k`는 다음을 포함합니다:

- 자체 어텐션과 MLP를 가진 트랜스포머 블록 `T_k`.
- 이전 깊이의 은닉 상태와 다음 깊이의 실제 토큰 임베딩을 결합하는 투영 행렬 `M_k`.
- 공유 임베딩 `E`(메인 모델과 동일).
- 공유 출력 헤드 `Out`(메인 모델과 동일).

훈련 시, 위치 `i`까지의 접두사에 대해 깊이별 은닉 상태는 다음과 같습니다:

```
h_i^(0) = 위치 i에서의 메인 모델 백본
h_i^(k) = T_k( M_k * concat(RMSNorm(h_i^(k-1)), RMSNorm(E(t_{i+k}))) )   for k >= 1
```

깊이별 예측은 다음과 같습니다:

```
logits_{i+k} = Out(h_i^(k-1))   for k = 1..D
```

깊이별 손실은 실제 토큰 `t_{i+k}`에 대한 교차 엔트로피입니다:

```
L_k = CE(logits_{i+k}, t_{i+k})
```

전체 깊이 손실은 다음과 같습니다:

```
L_MTP = (lambda / D) * sum_{k=1..D} L_k
```

`lambda`는 작은 가중치 인자입니다 — DeepSeek-V3는 훈련 초기 10% 동안 0.3을 사용하고 이후에는 0.1을 사용합니다. 총 훈련 손실은 `L_main + L_MTP`입니다.

### 병렬이 아닌 순차적 설계 이유

Gloeckle의 원래 병렬 MTP는 `D`개의 출력 헤드를 가지며, 각각은 `h_i^(0)`에 직접 적용됩니다. 각 헤드는 동일한 백본 은닉 상태에서 `t_{i+k}`를 예측합니다. 이는 잘 훈련되지만 예측들이 서로 조건화되지 않습니다. `head_1`의 출력을 `head_2`에 활용할 수 없습니다 — 헤드들이 병렬로 실행되기 때문입니다.

DeepSeek-V3의 순차적 설계는 `h_i^(k-1)`과 실제 다음 토큰 임베딩 `E(t_{i+k})`으로부터 `h_i^(k)`를 구축합니다. 이는 인과적 체인을 보존합니다: `t_{i+k+1}`을 예측하기 위해 깊이 `k+1`의 모듈은 `t_{i+k}`에서 무엇이 있었는지 확인합니다. 이는 자기회귀 디코더가 자신의 출력을 소비하는 방식과 구조적으로 동일합니다 — 따라서 MTP 모듈들을 추측 디코딩 드래프터로 직접 사용할 수 있습니다.

추론 시: `h_i^(k-1)`과 드래프트된 `t_{i+k}`를 모듈 `k+1`에 입력하여 `t_{i+k+1}` 예측을 얻습니다. 이를 반복합니다. 이는 훈련된 MTP 모듈을 드래프트 네트워크로 사용하는 EAGLE 스타일의 드래프트와 정확히 일치합니다. DeepSeek-V3는 첫 번째 MTP 모듈에서 80% 이상의 수용률을 보고하며 약 1.8배의 속도 향상을 달성합니다.

### 파라미터 계산

은닉 상태 크기 `h`와 어휘 크기 `V`를 가진 모델에 대해:

- 메인 모델: 수십억 개의 파라미터, 출력 헤드 크기 `V * h`.
- 공유 출력 헤드: 메인 모델의 헤드 재사용. 추가 파라미터 없음.
- 공유 임베딩: 메인 모델의 임베딩 재사용. 추가 파라미터 없음.
- MTP 모듈당:
  - 투영 `M_k`: `(2h) * h = 2h^2`.
  - 트랜스포머 블록 `T_k`: 어텐션(MHA에 `4h^2`) + MLP(일반적으로 SwiGLU에 `8h^2`, 비율 8/3). 블록당 약 `12h^2`.

모듈당 추가 파라미터: `~14h^2`. DeepSeek-V3의 `h = 7168`, D = 1 모듈일 때: `~14 * 7168^2 = ~720M` 파라미터(이론상). DeepSeek-V3는 14B를 보고합니다 — 차이는 주로 MTP 모듈의 MoE 전문가 레이어에서 발생합니다.

### 추측 디코딩의 이점

사전 훈련 중 MTP 모듈은 훈련 속도를 약 10% 저하시킵니다(추가 순전파 계산, 추가 손실). 그 이점은 두 가지입니다:

1. 더 밀도 있는 훈련 신호. 각 은닉 상태는 D+1개의 감독 대상을 봅니다. MMLU, GSM8K, MATH, HumanEval에서 측정된 효과: DeepSeek-V3의 실험에서 일관된 몇 퍼센트 포인트 향상.

2. 추론 시 무료 추측 디코딩 드래프트. MTP 모듈은 이미 다음 몇 토큰을 예측하도록 훈련되었습니다. 드래프트 네트워크로 재활용 시 80% 이상의 수용률을 제공합니다. 이 수준에서 N=3 또는 N=5 추측 디코딩은 1.8배의 처리량을 제공합니다. 10%의 훈련 시간 비용은 추론을 처음 실행할 때 회수됩니다.

### EAGLE과의 관계

EAGLE은 사전 훈련 후 별도의 작은 드래프트 모델을 훈련합니다. MTP는 드래프트를 사전 훈련에 통합합니다. 두 접근법은 유사한 수용률로 수렴하지만 다른 파이프라인을 사용합니다:

| 차원 | EAGLE-3 | MTP (DeepSeek-V3) |
|-----------|---------|------------------|
| 훈련 시기 | 사전 훈련 후 | 사전 훈련 중 |
| 기존 가중치와의 호환성 | 예 | 아니오(재훈련 필요) |
| 드래프트 파라미터 | 1-2 트랜스포머 레이어 | 1 트랜스포머 블록 + 투영 |
| 수용률 | 0.88-0.92 | 깊이 1에서 0.80+ |
| 속도 향상 외 이점 | 추측 디코딩 전용 | 밀도 있는 훈련 신호 + 속도 향상 |

## 구축 방법

`code/main.py`는 공유 임베딩, 프로젝션, 트랜스포머 블록, 공유 출력 헤드로 구성된 단일 MTP 모듈을 종단간(end-to-end)으로 구축합니다. 이후 짧은 합성 시퀀스에 대해 깊이별 교차 엔트로피 손실을 계산하고 구성 요소별 파라미터 수를 출력합니다. 32개 토큰의 장난감 어휘 집합은 숫자 가독성을 유지합니다.

### 1단계: 공유 임베딩 테이블

`vocab_size x hidden` 크기의 단일 테이블은 메인 모델 **및** 모든 깊이의 MTP 모듈에서 사용됩니다. 두 번째 복사본이 아닌 **동일한 텐서**입니다.

### 2단계: 깊이별 결합

```python
def combine(prev_hidden, next_token_embed, M_k):
    # 특성 차원 축 기준 연결 후 hidden 차원으로 프로젝션
    concat = rms_norm(prev_hidden) + rms_norm(next_token_embed)  # 벡터 덧셈 대체
    projected = matvec(M_k, concat)
    return projected
```

실제 DeepSeek-V3는 두 RMSNorm 벡터를 `[2h]`로 연결한 후 `h x 2h` 행렬로 프로젝션합니다. 장난감 모델은 표준 라이브러리 간결성을 위해 벡터 덧셈을 사용합니다.

### 3단계: 깊이 k의 트랜스포머 블록

셀프 어텐션 + MLP. 장난감 모델에서는 1층 선형 어텐션 블록과 SwiGLU MLP가 넘파이 없이 구조를 가시화합니다.

### 4단계: 공유 출력 헤드

메인 모델의 출력 프로젝션을 재사용합니다. 어휘 집합에 대한 로짓(logits)을 생성합니다.

### 5단계: 깊이별 손실

`offset k` 위치의 실제 토큰에 대한 `softmax(logits)`의 교차 엔트로피를 계산합니다. `lambda / D` 스케일링 인자로 깊이들을 집계합니다.

### 6단계: 파라미터 계산

총 파라미터 수, 공유(embedding, head) 파라미터 수, 모듈별 추가 파라미터 수를 출력합니다. MTP 추가 파라미터와 메인 모델 크기의 비율을 표시합니다.

## 사용 방법

MTP는 DeepSeek-V3(2024년 12월)와 DeepSeek-R1 시리즈에 통합되었습니다. 추론 시:

- DeepSeek의 자체 서빙 스택은 MTP 모듈을 즉시 사용 가능한(speculative) 디코더로 소비합니다.
- vLLM과 SGLang은 2026년 4월 기준으로 DeepSeek-V3 MTP 통합 경로를 제공합니다.
- AMD의 ROCm SGLang 튜토리얼은 V3 체크포인트에서 측정된 1.8× 속도 향상을 보이는 특정 MTP 추측 디코딩 구성을 보여줍니다.

새로운 사전 학습 실행에서 MTP를 사용해야 하는 경우:

- 전체 사전 학습 파이프라인을 제어하고 더 밀도 있는 학습 신호를 확보하려는 경우.
- 모델을 대규모로 서비스할 예정이며 추측 디코딩을 무료로 활용하려는 경우.
- 은닉층 크기(hidden size)가 최소 4096인 경우. 1B 규모에서는 오버헤드가 이득보다 더 큽니다.

사용하지 말아야 할 경우:

- 기존 사전 학습된 밀집 모델을 파인튜닝하는 경우. MTP 모듈은 학습되지 않습니다.
- 비교 대상 클린 베이스라인이 필요한 연구용 모델. MTP는 아키텍처를 변경합니다.

## Ship It

이 레슨은 `outputs/skill-mtp-planner.md`를 생성합니다. 사전 학습 실행 사양(모델 크기, 데이터, 컴퓨팅)이 주어졌을 때, MTP 통합을 위한 계획을 반환합니다: 깊이 수 D, `lambda` 스케줄, 메모리 오버헤드, 그리고 추론 시 추측 디코딩 배선 방식입니다.

## 연습 문제

1. `code/main.py`를 실행하세요. 합성 신호(synthetic signal)가 강화됨에 따라 깊이별 손실(per-depth loss)이 단조 감소하는 것을 확인하세요. 합성 신호를 고정 패턴(fixed pattern)으로 수정하고 깊이-1과 깊이-2 손실이 모두 수렴하는지 검증하세요.

2. D=1 MTP 모듈을 사용하는 밀집(dense) 70B 모델(은닉 크기 8192, 80층)의 파라미터 오버헤드를 계산하세요. DeepSeek-V3에서 보고된 14B 오버헤드와 비교하세요. DeepSeek의 수치가 더 높은 이유를 설명하세요: MTP 트랜스포머 블록은 동일한 MoE 구조를 상속받아 모듈당 파라미터 수가 증가하기 때문입니다.

3. 토이 모델에 D=2를 구현하세요: 두 번째 MTP 모듈을 추가하여 h^(1)을 입력으로 받아 `t_{i+2}`를 예측하게 하세요. 결합 손실(joint loss)과 파라미터 계산이 DeepSeek 논문의 식 (19-21)과 일치하는지 확인하세요.

4. 토이 모델을 병렬 MTP(Gloeckle 스타일)로 전환하세요: 주 은닉 상태(main hidden state) 위에 D개의 출력 헤드(output heads)를 추가하고, 각각이 다른 오프셋(offset)을 예측하도록 구성하세요. 동일한 합성 신호에서 깊이별 손실이 순차적 버전(sequential version)과 어떻게 비교되는지 측정하세요. 순차적 버전은 중간 예측(intermediate predictions)에 조건화(conditioning)하기 때문에 k > 1일 때 깊이-k 손실이 더 낮게 나타나야 합니다.

5. 훈련된 MTP 모듈을 EAGLE 스타일의 초안(draft)으로 사용하세요: 추론 시 모듈 k를 호출하여 `t_{i+k}`를 제안하게 하세요. 보유된 시퀀스(held-out sequence)에서 주 모델의 예측과 비교하여 이 초안 토큰의 수용률(acceptance rate)을 측정하세요. 토이 모델에서 50% 이상을 달성하면 경험적 MTP-as-draft 속성을 재현한 것입니다.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|----------------|------------------------|
| MTP 모듈 | "추가 손실 블록" | 메인 모델보다 `k` 토큰 앞을 예측하는 작은 트랜스포머 블록과 프로젝션 |
| 예측 깊이 | "어떤 오프셋" | 모듈 `k`가 위치 `i`까지의 접두사(prefix)로부터 `t_{i+k}`를 예측하는 정수 `k` |
| 병렬 MTP | "Gloeckle 스타일" | 동일한 백본 은닉 상태에 대한 D개의 독립적인 헤드, 조건부 체인 없음 |
| 순차적 MTP | "DeepSeek-V3 스타일" | 각 모듈이 이전 깊이의 은닉 상태와 다음 토큰 임베딩을 조건으로 함; 인과적 체인 유지 |
| 공유 출력 헤드 | "메인 헤드 재사용" | MTP 모듈이 별도의 출력 프로젝션이 아닌 메인 모델의 LM 헤드를 호출 |
| 공유 임베딩 | "메인 테이블 재사용" | 모든 곳에서 동일한 어휘 임베딩 테이블 사용; 중복 파라미터 없음 |
| 프로젝션 행렬 M_k | "은닉 상태 + 다음 토큰 결합" | 이전 은닉 상태와 대상 토큰 임베딩을 다음 깊이 입력으로 접는 `h x 2h` 선형 계층 |
| 결합 손실 L_MTP | "평균 추가 손실" | `lambda`로 스케일링된 각 깊이별 교차 엔트로피 손실의 산술 평균 |
| 깊이 1에서의 수용률 | "MTP 초안이 얼마나 자주 맞는가" | D=1 MTP 모듈의 top-1 예측이 메인 모델의 top-1 예측과 일치하는 비율; DeepSeek-V3에서 80% 이상 |
| 람다 가중치 | "추가 손실 중요도" | 깊이별 스케일링 계수; DeepSeek-V3에서 학습 시작 시 0.3, 이후 0.1로 조정 |
