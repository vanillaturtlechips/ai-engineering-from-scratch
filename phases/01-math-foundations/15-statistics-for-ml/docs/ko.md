# 머신러닝을 위한 통계학

> 통계학은 모델이 실제로 작동하는지, 아니면 단지 운이 좋았는지 확인하는 방법입니다.

**유형:** 구축(Build)
**언어:** Python
**선수 지식:** 1단계, 06강(확률과 분포), 07강(베이즈 정리)
**소요 시간:** ~120분

## 학습 목표

- 기술 통계량, 피어슨/스피어만 상관계수, 공분산 행렬을 처음부터 계산
- 가설 검정(t-검정, 카이제곱 검정) 수행 및 p-값과 신뢰 구간 올바르게 해석
- 분포 가정 없이 부트스트랩 리샘플링을 활용해 모든 지표에 대한 신뢰 구간 구성
- 효과 크기 측정을 통해 통계적 유의성과 실용적 유의성 구분

## 문제

두 모델을 훈련시켰다. 모델 A는 테스트 세트에서 0.87점을 기록했고, 모델 B는 0.89점을 기록했다. 모델 B를 배포했다. 3주 후, 프로덕션 메트릭이 이전보다 나빠졌다. 무슨 일이 일어난 걸까?

모델 B는 실제로 모델 A를 능가하지 못했다. 0.02 차이는 노이즈였다. 테스트 세트가 너무 작았거나, 분산이 너무 높았거나, 둘 다였다. 개선처럼 보이지만 실제로는 무작위성을 배포한 것이다.

이런 일은 끊임없이 발생한다. 캐글 리더보드의 순위 변동, 재현에 실패한 논문, 수백 개의 샘플을 기반으로 승자를 선언하는 A/B 테스트. 근본 원인은 항상 같다: 누군가가 통계 분석을 생략했다.

통계는 신호와 노이즈를 구분하는 도구를 제공한다. 차이가 실제인지, 얼마나 확신해야 하는지, 결과를 신뢰하기 전에 얼마나 많은 데이터가 필요한지 알려준다. 모든 ML 파이프라인, 모델 비교, 실험에는 통계가 필요하다. 통계 없이는 추측에 불과하다.

## 개념

### 기술 통계: 데이터 요약

모델링을 시작하기 전에 데이터의 형태를 알아야 합니다. 기술 통계는 데이터셋을 그 형태를 포착하는 몇 개의 숫자로 압축합니다.

**중심 경향성 측정**은 "중간값은 어디에 있는가?"에 답합니다.

```
평균:   모든 값의 합 / 개수
        mu = (1/n) * sum(x_i)

중앙값: 정렬했을 때 가운데 값
        이상치에 강건함. [1, 2, 3, 4, 1000]의 경우 평균은 202
        이지만 중앙값은 3입니다.

최빈값:   가장 빈번한 값
        범주형 데이터에 유용. 연속형 데이터에서는 거의 정보를 제공하지 않음.
```

평균은 균형점입니다. 중앙값은 중간 지점입니다. 이 둘이 벌어지면 분포가 왜곡되었습니다. 소득 분포는 평균 >> 중앙값(억만장자로 인한 오른쪽 왜곡)입니다. 학습 중 손실 분포는 평균 << 중앙값(쉬운 샘플로 인한 왼쪽 왜곡)인 경우가 많습니다.

**산포 측정**은 "데이터는 얼마나 퍼져 있는가?"에 답합니다.

```
분산:   평균으로부터의 제곱 편차의 평균
            sigma^2 = (1/n) * sum((x_i - mu)^2)

표준 편차:  분산의 제곱근
                     sigma = sqrt(sigma^2)
                     데이터와 동일한 단위이므로 해석이 용이.

범위:      최대값 - 최소값
            이상치에 민감. 거의 단독으로 유용하지 않음.

IQR:        Q3 - Q1 (사분위 범위)
            데이터의 중간 50% 범위.
            이상치에 강건. 상자 그림과 이상치 탐지에 사용.
```

**백분위수**는 정렬된 데이터를 100개의 동일한 부분으로 나눕니다. 25번째 백분위수(Q1)는 25%의 값이 이 지점 아래에 있음을 의미합니다. 50번째 백분위수는 중앙값입니다. 75번째 백분위수는 Q3입니다.

```
지연 시간 모니터링의 경우:
  P50 = 중앙값 지연 시간        (일반적인 사용자 경험)
  P95 = 95번째 백분위수       (나쁘지만 최악은 아님)
  P99 = 99번째 백분위수       (꼬리 지연, 종종 중앙값의 10배)
```

ML에서는 추론 지연, 예측 신뢰도 분포, 오차 분포 이해를 위해 백분위수를 고려합니다. 평균 오차는 낮지만 P99 오차가 심각한 모델은 안전-중요 애플리케이션에 쓸모없을 수 있습니다.

**표본 vs 모집단 통계량.** 표본에서 분산을 계산할 때 n 대신 (n-1)로 나눕니다. 이는 베셀 보정입니다. 표본 평균이 모집단 평균이 아니라는 사실을 보정합니다. n으로 나누면 분산을 체계적으로 과소평가합니다. (n-1)로 나누면 추정치가 불편해집니다.

```
모집단 분산: sigma^2 = (1/N) * sum((x_i - mu)^2)
표본 분산:     s^2     = (1/(n-1)) * sum((x_i - x_bar)^2)
```

실제 적용: n이 크면(수천 개 표본) 차이는 무시할 수 있습니다. n이 작으면(수십 개 표본) 중요합니다.

### 상관관계: 변수들의 움직임

상관관계는 두 변수 간 선형 관계의 강도와 방향을 측정합니다.

**피어슨 상관계수**는 선형 연관성을 측정합니다.

```
r = sum((x_i - x_bar)(y_i - y_bar)) / (n * s_x * s_y)

r = +1:  완벽한 양의 선형 관계
r = -1:  완벽한 음의 선형 관계
r =  0:  선형 관계 없음 (비선형 관계는 있을 수 있음!)

범위: [-1, 1]
```

피어슨은 관계가 선형이고 두 변수가 대략 정규 분포를 따른다고 가정합니다. 이상치에 민감합니다. 하나의 극단값이 r을 0.1에서 0.9로 끌어올릴 수 있습니다.

**스피어만 순위 상관**은 단조 관계를 측정합니다.

```
1. 각 값을 순위(1, 2, 3, ...)로 대체
2. 순위에 대해 피어슨 상관 계산

스피어만은 선형이 아닌 단조 관계도 포착합니다.
y = x^3인 경우 피어슨은 r < 1이지만 스피어만은 rho = 1입니다.
```

**사용 시기:**

```
피어슨:    두 변수 모두 연속형이고 대략 정규 분포를 따름.
            구체적으로 선형 관계에 관심 있음.
            극단 이상치 없음.

스피어만:   순서형 데이터(순위, 평점).
            데이터가 정규 분포를 따르지 않음.
            단조하지만 선형이 아닌 관계 의심.
            이상치 존재.
```

**황금률:** 상관관계는 인과관계를 의미하지 않습니다. 아이스크림 판매량과 익사 사망은 여름에 모두 증가하기 때문에 상관관계가 있습니다. 모델의 정확도와 파라미터 수는 상관관계가 있지만 파라미터 추가가 자동으로 정확도를 향상시키지는 않습니다(과적합 참조).

### 공분산 행렬

두 변수 간 공분산은 함께 어떻게 변하는지를 측정합니다.

```
Cov(X, Y) = (1/n) * sum((x_i - x_bar)(y_i - y_bar))

Cov(X, Y) > 0:  X와 Y는 함께 증가하는 경향
Cov(X, Y) < 0:  X가 증가하면 Y는 감소하는 경향
Cov(X, Y) = 0:  선형적 공동 변화 없음
```

d개의 특성에 대해 공분산 행렬 C는 d x d 행렬이며 C[i][j] = Cov(특성_i, 특성_j)입니다. 대각 성분 C[i][i]는 각 특성의 분산입니다.

```
C = | Var(x1)      Cov(x1,x2)  Cov(x1,x3) |
    | Cov(x2,x1)  Var(x2)      Cov(x2,x3) |
    | Cov(x3,x1)  Cov(x3,x2)  Var(x3)     |

특성:
  - 대칭: C[i][j] = C[j][i]
  - 준정부호: 모든 고유값 >= 0
  - 대각 = 분산
  - 비대각 = 공분산
```

**PCA와의 연결.** PCA는 공분산 행렬을 고유분해합니다. 고유벡터는 주성분(최대 분산 방향)입니다. 고유값은 각 주성분이 포착하는 분산량을 나타냅니다. 이는 레슨 10에서 다룬 내용으로, 공분산 행렬이 데이터의 모든 쌍별 선형 관계를 인코딩하기 때문에 분해에 적합합니다.

**상관관계와의 연결.** 상관 행렬은 표준화된 변수(각 변수를 표준편차로 나눈)의 공분산 행렬입니다. 상관관계는 공분산을 정규화하여 모든 값이 [-1, 1]에 오도록 합니다.

### 가설 검정

가설 검정은 불확실성 하에서 결정을 내리는 프레임워크입니다. 주장으로 시작하여 데이터를 수집하고 데이터가 주장과 일치하는지 판단합니다.

**설정:**

```
귀무가설(H0):        기본 가정, 일반적으로 "효과 없음"
대립가설(H1):       입증하려는 것

예시:
  H0: 모델 A와 모델 B의 정확도는 같다
  H1: 모델 B의 정확도가 모델 A보다 높다
```

**p-값**은 H0이 참일 때 관측된 데이터만큼 극단적인 데이터를 볼 확률입니다. H0이 참일 확률이 아닙니다. 이는 통계학에서 가장 흔한 오해입니다.

```
p-값 = P(데이터가 이만큼 극단적 | H0이 참)

p-값 < 알파(일반적으로 0.05):
    H0 기각. 결과는 "통계적으로 유의미"합니다.
p-값 >= 알파:
    H0 기각 실패. 증거가 부족합니다.
    이는 H0이 참임을 의미하지 않습니다.
```

**신뢰 구간**은 매개변수의 가능한 값의 범위를 제공합니다.

```
평균에 대한 95% 신뢰 구간:
    x_bar +/- z * (s / sqrt(n))

여기서 z = 95% 신뢰도에 대해 1.96

해석: 이 실험을 여러 번 반복하면 계산된 구간 중 95%는
진짜 평균을 포함할 것입니다. 이는 특정 구간에 진짜 평균이
있을 확률이 95%라는 의미가 아닙니다.
```

신뢰 구간의 너비는 정밀도를 나타냅니다. 넓은 구간은 높은 불확실성을 의미합니다. 좁은 구간은 데이터가 편향되지 않았다면 정확한 추정치를 의미합니다.

### t-검정

t-검정은 평균을 비교합니다. 여러 종류가 있습니다.

**일표본 t-검정:** 모집단 평균이 가설값과 다른가?

```
t = (x_bar - mu_0) / (s / sqrt(n))

자유도 = n - 1
```

**이표본 t-검정(독립):** 두 그룹 평균이 다른가?

```
t = (x_bar_1 - x_bar_2) / sqrt(s1^2/n1 + s2^2/n2)

이는 등분산을 가정하지 않는 웰치의 t-검정입니다.
등분산에 대한 특별한 이유가 없다면 항상 웰치를 사용합니다.
```

**대응표본 t-검정:** 측정값이 쌍으로 존재하는 경우(동일한 데이터 분할에서 평가된 동일한 모델):

```
각 쌍에 대해 d_i = x_i - y_i 계산
d_i 값에 대해 일표본 t-검정 수행 (mu_0 = 0)
```

ML에서는 대응표본 t-검정이 일반적입니다: 동일한 10개의 교차 검증 폴드에서 두 모델을 실행하고 점수를 쌍으로 비교합니다.

### 카이제곱 검정

카이제곱 검정은 관측된 빈도가 기대 빈도와 일치하는지 확인합니다. 범주형 데이터에 유용합니다.

```
chi^2 = sum((관측값 - 기대값)^2 / 기대값)

예시: 언어 모델의 출력 분포가 훈련 분포와 범주별로 일치하는가?

범주    관측값   기대값
긍정       120        100
부정        80        100
chi^2 = (120-100)^2/100 + (80-100)^2/100 = 4 + 4 = 8

자유도 1에서 chi^2 = 8은 p < 0.005.
차이는 유의미합니다.
```

### ML 모델을 위한 A/B 테스트

ML에서의 A/B 테스트는 웹 A/B 테스트와 다릅니다. 모델 비교에는 특정 도전 과제가 있습니다.

```
1. 동일한 테스트 세트:    두 모델은 동일한 데이터에서 평가되어야 합니다.
                     다른 테스트 세트는 비교를 무의미하게 만듭니다.

2. 여러 지표: 정확도만으로는 부족합니다. 정밀도,
                     재현율, F1, 지연 시간, 공정성 지표가 필요합니다.

3. 분산:         교차 검증이나 부트스트랩을 사용하여
                     각 지표의 분산을 추정해야 합니다. 점 추정치만으로는 부족합니다.

4. 데이터 누출:     테스트 세트가 모델 선택 중에 사용되었다면,
                     비교는 편향됩니다. 최종 테스트 세트를 별도로 보관합니다.
```

**절차:**

```
1. 지표와 유의 수준(알파 = 0.05) 정의
2. 동일한 k-폴드 교차 검증 분할에서 두 모델 실행
3. 쌍별 점수 수집: [(a1, b1), (a2, b2), ..., (ak, bk)]
4. 차이 계산: d_i = b_i - a_i
5. 차이에 대해 대응표본 t-검정 수행
6. 확인: 평균 차이가 0과 유의미하게 다른가?
7. 평균 차이에 대한 신뢰 구간 계산
8. 효과 크기(Cohen's d) 계산으로 실질적 유의성 판단
```

### 통계적 유의성 vs 실질적 유의성

결과는 통계적으로 유의미하지만 실질적으로는 무의미할 수 있습니다. 충분한 데이터가 있으면 사소한 차이도 통계적으로 유의미해집니다.

```
예시:
  모델 A 정확도: 0.9234
  모델 B 정확도: 0.9237
  n = 1,000,000 테스트 샘플
  p-값 = 0.001

통계적으로 유의미한가? 예.
실질적으로 유의미한가? 0.03% 개선은 새 모델 배포의
엔지니어링 비용을 정당화할 수 없습니다.
```

**효과 크기**는 표본 크기와 독립적으로 차이의 크기를 정량화합니다.

```
Cohen's d = (mean_1 - mean_2) / pooled_std

d = 0.2:  작은 효과
d = 0.5:  중간 효과
d = 0.8:  큰 효과
```

항상 p-값과 효과 크기를 함께 보고합니다. p-값은 차이가 실제인지 알려줍니다. 효과 크기는 중요한지 알려줍니다.

### 다중 비교 문제

많은 가설을 검정할 때 일부는 우연히 "유의미"해질 수 있습니다. 알파 = 0.05에서 20가지를 검정하면 아무것도 없을 때도 1개의 거짓 양성이 기대됩니다.

```
P(적어도 하나의 거짓 양성) = 1 - (1 - alpha)^m

m = 20 검정, alpha = 0.05:
P(거짓 양성) = 1 - 0.95^20 = 0.64

적어도 하나의 거짓 양성 가능성이 64%입니다.
```

**보니페로니 보정:** 알파를 검정 횟수로 나눕니다.

```
조정된 알파 = alpha / m = 0.05 / 20 = 0.0025

p-값 < 0.0025일 때만 H0 기각.
보수적이지만 간단합니다. 검정이 독립적일 때 작동합니다.
```

ML에서는 여러 지표, 하이퍼파라미터 구성, 여러 데이터셋에 걸쳐 모델을 비교할 때 중요합니다.

### 부트스트랩 방법

부트스트랩은 데이터를 복원 추출하여 통계량의 표본 분포를 추정합니다. 기본 분포에 대한 가정이 필요 없습니다.

**알고리즘:**

```
1. n개의 데이터 포인트 보유
2. n개의 샘플을 복원 추출하여 선택(일부 포인트는 여러 번,
   일부는 전혀 선택되지 않음)
3. 이 부트스트랩 샘플에서 통계량 계산
4. B번 반복(일반적으로 B = 1000~10000)
5. 부트스트랩 통계량의 분포는 표본 분포를 근사
```

**부트스트랩 신뢰 구간(백분위수 방법):**

```
B개의 부트스트랩 통계량 정렬
95% CI = [2.5번째 백분위수, 97.5번째 백분위수]
```

**ML에서 부트스트랩의 중요성:**

```
- 테스트 세트 정확도는 점 추정치입니다. 부트스트랩은 신뢰 구간을 제공합니다.
- 지표 분포가 정규 분포를 따른다고 가정할 수 없습니다(특히 AUC, F1, 상위 k개 정밀도).
- 부트스트랩은 모든 통계량에 적용 가능: 중앙값, 두 평균의 비율,
  두 모델 간 AUC 차이.
- 닫힌 형식의 공식이 필요 없습니다.
```

**모델 비교를 위한 부트스트랩:**

```
1. 동일한 테스트 세트에서 모델 A와 B의 예측값 보유
2. 각 부트스트랩 반복:
   a. 테스트 인덱스를 복원 추출하여 선택
   b. 재추출된 세트에서 metric_A와 metric_B 계산
   c. diff = metric_B - metric_A 저장
3. 차이에 대한 95% CI:
   [diff의 2.5번째 백분위수, 97.5번째 백분위수]
4. CI가 0을 포함하지 않으면 차이가 유의미함
```

이는 분포 가정을 하지 않기 때문에 대응표본 t-검정보다 더 강건합니다.

### 모수적 vs 비모수적 검정

**모수적 검정**은 특정 분포(일반적으로 정규 분포)를 가정합니다.

```
t-검정:         정규 분포 데이터(또는 큰 n에 의한 중심 극한 정리) 가정
ANOVA:          정규성과 등분산성 가정
피어슨 r:      이변량 정규성 가정
```

**비모수적 검정**은 분포 가정을 하지 않습니다.

```
Mann-Whitney U:     두 그룹 비교(독립 t-검정 대체)
Wilcoxon 부호 순위: 대응 데이터 비교(대응표본 t-검정 대체)
스피어만 rho:       순위 기반 상관(피어슨 대체)
Kruskal-Wallis:     여러 그룹 비교(ANOVA 대체)
```

**비모수적 검정 사용 시기:**

```
- 작은 표본 크기(n < 30)이고 데이터가 명확히 비정규
- 순서형 데이터(평점, 순위)
- 제거할 수 없는 심한 이상치
- 왜곡된 분포
```

**모수적 검정 사용 시기:**

```
- 큰 표본 크기(중심 극한 정리로 검정 통계량이 대략 정규 분포)
- 데이터가 극단 이상치 없이 대략 대칭
- 더 높은 통계적 검정력(실제 차이를 더 잘 탐지)
```

ML 실험에서는 일반적으로 작은 n(5~10 교차 검증 폴드)을 가지므로 t-검정보다 Wilcoxon 부호 순위 검정이 더 적합할 수 있습니다.

### 중심 극한 정리: 실용적 함의

중심 극한 정리는 표본 평균의 분포가 n이 커질수록 정규 분포에 가까워짐을 말합니다. 이는 모집단 분포와 무관합니다.

```
X_1, X_2, ..., X_n이 iid이고 평균 mu, 분산 sigma^2를 가질 때:

    X_bar ~ Normal(mu, sigma^2 / n)    n -> infinity

대부분의 경우 n >= 30에서 작동.
고도로 왜곡된 분포의 경우 n >= 100이 필요할 수 있음.
```

**ML에 중요한 이유:**

```
1. 집계된 지표에 대한 신뢰 구간과 t-검정을 정당화
2. 교차 검증 폴드 간 평균이 개별 폴드 변동이 크더라도
   안정적인 추정치를 제공하는 이유 설명
3. 미니배치 경사 하강법이 작동하는 이유(배치 평균 기울기가
   진짜 기울기를 근사)
4. 앙상블 방법: 여러 모델 예측의 평균은 단일 모델보다
   더 안정적인 출력을 제공
```

**중심 극한 정리가 하지 않는 것:**

```
- 데이터를 정규 분포로 만들지 않습니다. 표본 평균의 분포를 정규 분포로 만듭니다.
- 분산이 무한대인 분포(카이제곱 분포)에는 적용되지 않습니다.
- 종속 데이터(수정이 없는 시계열)에는 적용되지 않습니다.
```

### ML 논문에서 흔한 통계적 실수

1. **훈련 세트에서 테스트.** 과적합을 보장합니다. 훈련 중 모델이 절대 보지 못한 데이터를 항상 보관합니다.

2. **신뢰 구간 없음.** 불확실성 없이 단일 정확도 수치를 보고하면 결과를 재현 불가능하고 검증할 수 없습니다.

3. **다중 비교 무시.** 50개 구성을 테스트하고 보정 없이 최고를 보고하면 거짓 양성률이 증가합니다.

4. **통계적 유의성과 실질적 유의성 혼동.** 0.01% 정확도 개선에 대한 p-값 0.001은 의미가 없습니다.

5. **불균형 데이터에서 정확도 사용.** 99% 음성 클래스 데이터에서 99% 정확도는 모델이 아무것도 학습하지 않았음을 의미합니다. 정밀도, 재현율, F1, AUC를 사용합니다.

6. **지표 선택적 보고.** 모델이 이기는 지표만 보고. 정직한 평가는 모든 관련 지표를 보고합니다.

7. **훈련/테스트 분할 간 정보 누출.** 분할 전 정규화, 과거 예측을 위해 미래 데이터 사용.

8. **분산 추정 없는 작은 테스트 세트.** 100개 샘플에서 2% 개선은 신호가 아닌 노이즈입니다.

9. **독립성 가정 위반.** 동일한 환자의 의료 영상, 동일한 문서의 여러 문장. 그룹 내 관측값은 상관되어 있습니다.

10. **p-해킹.** p < 0.05가 나올 때까지 다른 테스트, 부분 집합, 제외 기준을 시도. 결과는 검색의 인공물입니다.

## 구현 방법

다음 항목들을 직접 구현합니다:

1. **기초 통계량 계산** (평균(mean), 중앙값(median), 최빈값(mode), 표준편차(standard deviation), 백분위수(percentiles), 사분위 범위(IQR))
2. **상관 함수** (피어슨(Pearson) 및 스피어만(Spearman) 상관계수, 공분산 행렬(covariance matrix) 포함)
3. **가설 검정** (단일 표본 t-검정(one-sample t-test), 두 표본 t-검정(two-sample t-test), 카이제곱 검정(chi-squared test))
4. **부트스트랩 신뢰구간** (어떤 통계량에 대해서도 가정 없이 계산 가능)
5. **A/B 테스트 시뮬레이터** (데이터 생성, 검정 수행, 제1종/제2종 오류 확인)
6. **통계적 유의성 vs 실질적 유의성 데모** (큰 표본(n)에서는 모든 것이 "유의미"해지는 현상 보여주기)

모든 구현은 `math`와 `random` 모듈만 사용하여 직접 코딩합니다. NumPy, SciPy 등 외부 라이브러리는 사용하지 않습니다.

## 주요 용어

| 용어 | 정의 |
|---|---|
| 평균(Mean) | 값의 합을 개수로 나눈 값. 이상치(outlier)에 민감함. |
| 중앙값(Median) | 정렬된 데이터의 중간 값. 이상치에 강건함. |
| 표준편차(Standard deviation) | 분산(variance)의 제곱근. 원본 단위에서 퍼진 정도를 측정함. |
| 백분위수(Percentile) | 주어진 백분율의 데이터가 해당 값보다 작은 값. |
| IQR(Interquartile range) | 사분위 범위. Q3에서 Q1을 뺀 값. 중간 50% 데이터의 퍼진 정도. |
| 피어슨 상관계수(Pearson correlation) | 두 변수 간 선형 연관성을 측정함. 범위 [-1, 1]. |
| 스피어만 상관계수(Spearman correlation) | 순위를 사용한 단조 연관성(monotonic association)을 측정함. |
| 공분산 행렬(Covariance matrix) | 모든 특성(feature) 간 쌍별 공분산(pairwise covariance)으로 구성된 행렬. |
| 귀무가설(Null hypothesis) | 효과나 차이가 없다는 기본 가정. |
| p-값(p-value) | 귀무가설이 참일 때 관측된 데이터만큼 극단적인 데이터가 나올 확률. |
| 신뢰구간(Confidence interval) | 주어진 신뢰 수준에서 매개변수의 가능한 값 범위. |
| t-검정(t-test) | 평균이 유의미하게 다른지 검정함. t-분포를 사용함. |
| 카이제곱 검정(Chi-squared test) | 관측된 빈도가 기대 빈도와 다른지 검정함. |
| 효과 크기(Effect size) | 표본 크기와 무관한 차이의 크기. 코헨의 d(Cohen's d)가 일반적임. |
| 본페로니 보정(Bonferroni correction) | 거짓 양성(false positive)을 통제하기 위해 유의 수준을 검정 횟수로 나눔. |
| 부트스트랩(Bootstrap) | 복원 추출(resampling with replacement)을 사용해 표본 분포를 추정함. |
| 제1종 오류(Type I error) | 거짓 양성(false positive). 귀무가설이 참일 때 기각함. |
| 제2종 오류(Type II error) | 거짓 음성(false negative). 귀무가설이 거짓일 때 기각하지 못함. |
| 통계적 검정력(Statistical power) | 거짓인 귀무가설을 올바르게 기각할 확률. 검정력 = 1 - 제2종 오류율. |
| 중심 극한 정리(Central limit theorem) | 표본 크기가 커질수록 표본 평균이 정규 분포에 수렴함. |
| 모수적 검정(Parametric test) | 데이터에 대해 특정 분포(일반적으로 정규 분포)를 가정함. |
| 비모수적 검정(Non-parametric test) | 분포 가정을 하지 않음. 순위(rank)나 부호(sign)를 사용함.
