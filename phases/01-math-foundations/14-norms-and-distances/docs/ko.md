# 노름(Norms)과 거리(Distances)

> 거리 함수는 "유사함"의 의미를 정의합니다. 잘못 선택하면 모든 후속 작업이 무너집니다.

**유형:** 빌드  
**언어:** Python  
**선수 지식:** 1단계, 레슨 01 (선형 대수 직관), 02 (벡터, 행렬 및 연산)  
**소요 시간:** ~90분

## 학습 목표

- L1, L2, 코사인(cosine), 마할라노비스(Mahalanobis), 자카드(Jaccard), 편집 거리(edit distance) 함수를 처음부터 구현
- 주어진 ML 작업에 적합한 거리 메트릭을 선택하고, 다른 메트릭이 실패하는 이유 설명
- L1 및 L2 노름(norm)을 LASSO(라쏘) 및 Ridge(릿지) 정규화와 연결하고, 이들의 기하학적 제약 영역 설명
- 동일한 데이터셋이 다른 메트릭 하에서 다른 최근접 이웃(nearest neighbors)을 생성하는 방식 시연

## 문제 정의

두 벡터가 있습니다. 단어 임베딩일 수도 있고, 사용자 프로필일 수도 있으며, 픽셀 배열일 수도 있습니다. 중요한 것은 이 두 벡터가 얼마나 가까운지 알아내는 것입니다.

답은 전적으로 선택한 거리 함수에 달려 있습니다. 하나의 메트릭에서는 가장 가까운 이웃이었던 데이터 포인트가 다른 메트릭에서는 멀리 떨어져 있을 수 있습니다. KNN 분류기, 추천 엔진, 벡터 데이터베이스, 클러스터링 알고리즘, 손실 함수 등 모든 것이 이 선택에 의존합니다. 잘못 선택하면 모델이 잘못된 목표를 최적화하게 됩니다.

보편적으로 최고의 거리는 없습니다. L2 거리(L2 distance)는 공간 데이터에 적합합니다. 코사인 유사도(cosine similarity)는 NLP 분야에서 주로 사용됩니다. 자카드 거리(Jaccard distance)는 집합(set)을 처리하며, 편집 거리(edit distance)는 문자열에 적용됩니다. 마할라노비스 거리(Mahalanobis distance)는 상관관계를 고려하고, 바셰슈타인 거리(Wasserstein distance)는 확률 질량(probability mass)을 이동시킵니다. 각각의 거리 함수는 "유사성(similar)"에 대한 서로 다른 가정을 내포하고 있습니다.

이 강의에서는 모든 주요 거리 함수를 처음부터 구현하고, 각 함수가 적합한 상황을 보여주며, 동일한 데이터라도 어떤 메트릭을 사용하느냐에 따라 완전히 다른 최근접 이웃(nearest neighbors)이 생성되는 것을 시연합니다.

## 개념

### 노름: 벡터 크기 측정

노름은 벡터의 "크기"를 측정합니다. 두 벡터 사이의 모든 거리 함수는 그 차이의 노름으로 표현할 수 있습니다: d(a, b) = ||a - b||. 따라서 노름을 이해하는 것은 거리를 이해하는 것입니다.

### L1 노름 (맨해튼 거리)

L1 노름은 모든 성분의 절댓값을 합산합니다.

```
||x||_1 = |x_1| + |x_2| + ... + |x_n|
```

맨해튼 거리라고 불리는 이유는 축 방향으로만 이동할 수 있는 도시 격자에서 얼마나 걸어야 하는지를 측정하기 때문입니다. 대각선은 없습니다.

```
점 A = (1, 1)
점 B = (4, 5)

L1 거리 = |4-1| + |5-1| = 3 + 4 = 7

격자에서 동쪽으로 3블록, 북쪽으로 4블록을 걷습니다.
```

L1 사용 시기:
- 고차원 희소 데이터 (텍스트 특징, 원-핫 인코딩)
- 이상치에 강건한 경우 (단일 큰 차이가 지배하지 않음)
- 특징 선택 문제 (L1 정규화는 희소성 유도)

L1 정규화(Lasso)와의 연결: 손실 함수에 ||w||_1을 추가하면 절댓값 가중치 합이 페널티로 적용됩니다. 이는 작은 가중치를 정확히 0으로 만들어 자동 특징 선택을 수행합니다. L1 페널티는 가중치 공간에 다이아몬드 형태의 제약 영역을 생성하며, 다이아몬드의 모서리는 일부 가중치가 0인 축에 위치합니다.

손실 함수와의 연결: 평균 절댓값 오차(MAE)는 예측과 타겟 사이의 평균 L1 거리입니다. 모든 오차를 선형적으로 페널티하여 MSE에 비해 이상치에 강건합니다.

### L2 노름 (유클리드 거리)

L2 노름은 직선 거리입니다. 제곱 성분의 합의 제곱근입니다.

```
||x||_2 = sqrt(x_1^2 + x_2^2 + ... + x_n^2)
```

이것은 기하학 수업에서 배운 거리입니다. n차원에서의 피타고라스 정리입니다.

```
점 A = (1, 1)
점 B = (4, 5)

L2 거리 = sqrt((4-1)^2 + (5-1)^2) = sqrt(9 + 16) = sqrt(25) = 5.0

격자를 대각선으로 가로지르는 직선입니다.
```

L2 사용 시기:
- 저~중차원 연속 데이터
- 특징 스케일이 비슷할 때
- 물리적 거리 (공간 데이터, 센서 측정값)
- 픽셀 수준의 이미지 유사도

L2 정규화(Ridge)와의 연결: 손실 함수에 ||w||_2^2을 추가하면 큰 가중치가 페널티됩니다. L1과 달리 가중치를 0으로 만들지 않습니다. 모든 가중치를 0으로 비례적으로 축소합니다. L2 페널티는 원형 제약 영역을 생성하므로 축에 모서리가 없습니다. 가중치는 작아지지만 거의 정확히 0이 되지 않습니다.

손실 함수와의 연결: 평균 제곱 오차(MSE)는 L2 거리의 제곱의 평균입니다. 제곱은 큰 오차를 작은 오차보다 더 강하게 페널티합니다.

```
MAE (L1 손실):  |y - y_hat|         선형 페널티. 이상치에 강건.
MSE (L2 손실):  (y - y_hat)^2       이차 페널티. 이상치에 민감.
```

### Lp 노름: 일반적인 패밀리

L1과 L2는 Lp 노름의 특수한 경우입니다.

```
||x||_p = (|x_1|^p + |x_2|^p + ... + |x_n|^p)^(1/p)
```

p 값이 다르면 다른 형태의 "단위 구"(원점에서 거리 1인 모든 점의 집합)가 생성됩니다.

```
p=1:    다이아몬드 형태      (모서리가 축에 위치)
p=2:    원/구면      (일반적인 둥근 구)
p=3:    초타원      (둥근 사각형)
p=inf:  사각형/초입방체   (축을 따라 평평한 면)
```

### L-무한대 노름 (체비셰프 거리)

p가 무한대로 갈 때, Lp 노름은 최대 절댓값 성분으로 수렴합니다.

```
||x||_inf = max(|x_1|, |x_2|, ..., |x_n|)
```

두 점 사이의 거리는 가장 큰 차이가 나는 단일 차원에 의해 결정됩니다. 다른 차원은 무시됩니다.

```
점 A = (1, 1)
점 B = (4, 5)

L-무한대 거리 = max(|4-1|, |5-1|) = max(3, 4) = 4
```

L-무한대 사용 시기:
- 어떤 단일 차원에서의 최악의 편차가 중요할 때
- 게임 보드 (체스 킹의 이동: 어떤 방향으로든 한 칸 이동은 1)
- 제조 허용 오차 (모든 차원이 규격 내에 있어야 함)

### 코사인 유사도와 코사인 거리

코사인 유사도는 두 벡터 사이의 각도를 측정하며, 크기는 무시합니다.

```
cos_sim(a, b) = (a . b) / (||a||_2 * ||b||_2)
```

-1(반대 방향)에서 +1(같은 방향)까지 범위를 가집니다. 수직인 벡터는 코사인 유사도가 0입니다.

코사인 거리는 이를 거리로 변환합니다: cosine_distance = 1 - cosine_similarity. 0(동일한 방향)에서 2(반대 방향)까지 범위를 가집니다.

```
a = (1, 0)    b = (1, 1)

cos_sim = (1*1 + 0*1) / (1 * sqrt(2)) = 1/sqrt(2) = 0.707
cos_dist = 1 - 0.707 = 0.293
```

NLP와 임베딩에서 코사인이 지배적인 이유: 텍스트에서 문서 길이는 유사도에 영향을 주지 않아야 합니다. 고양이에 관한 문서가 다른 문서보다 두 배 길더라도 여전히 "유사"해야 합니다. 코사인 유사도는 크기(길이)를 무시하고 방향만 고려합니다. 같은 단어 분포를 가지지만 길이가 다른 두 문서는 같은 방향을 가리키며 코사인 유사도 1.0을 얻습니다.

코사인 유사도 사용 시기:
- 텍스트 유사도 (TF-IDF 벡터, 단어 임베딩, 문장 임베딩)
- 크기는 노이즈이고 방향이 신호인 모든 도메인
- 추천 시스템 (사용자 선호도 벡터)
- 임베딩 검색 (벡터 데이터베이스는 거의 항상 코사인이나 내적 사용)

### 내적 유사도 vs 코사인 유사도

두 벡터의 내적은 다음과 같습니다.

```
a . b = a_1*b_1 + a_2*b_2 + ... + a_n*b_n
      = ||a|| * ||b|| * cos(각도)
```

코사인 유사도는 내적을 두 크기로 정규화한 것입니다. 두 벡터가 이미 단위 정규화되었다면(크기 = 1), 내적과 코사인 유사도는 동일합니다.

```
||a|| = 1이고 ||b|| = 1일 때:
    a . b = cos(a와 b 사이의 각도)
```

다를 때: 내적은 크기 정보를 포함합니다. 더 큰 크기를 가진 벡터는 더 높은 내적 점수를 얻습니다. 이는 "인기" 있는 항목이 더 높은 순위를 차지하도록 하는 검색 시스템에서 중요합니다. 크기는 암묵적인 품질이나 중요도 신호로 작용합니다.

```
a = (3, 0)    b = (1, 0)    c = (0, 1)

dot(a, b) = 3     dot(a, c) = 0
cos(a, b) = 1.0   cos(a, c) = 0.0

둘 다 방향에 동의하지만, 내적은 크기도 반영합니다.
```

실제 적용:
- 순수한 방향 유사도가 필요할 때 코사인 유사도 사용
- 크기가 의미 있는 정보를 가질 때 내적 사용
- 많은 벡터 데이터베이스(Pinecone, Weaviate, Qdrant)는 둘 중 선택 가능
- 임베딩이 L2 정규화되었다면 선택은 중요하지 않음

### 마할라노비스 거리

유클리드 거리는 모든 차원을 동일하게 취급합니다. 하지만 특징들이 상관관계가 있거나 스케일이 다르면 L2는 오해의 소지가 있는 결과를 줍니다.

마할라노비스 거리는 데이터의 공분산 구조를 고려합니다.

```
d_M(x, y) = sqrt((x - y)^T * S^(-1) * (x - y))
```

여기서 S는 데이터의 공분산 행렬입니다.

직관적으로: 마할라노비스 거리는 먼저 데이터를 비상관화 및 정규화(화이트닝)한 후, 변환된 공간에서 L2 거리를 계산합니다. S가 단위 행렬(비가상관, 단위 분산 특징)이면 마할라노비스 거리는 유클리드 거리로 축소됩니다.

```
예시: 키와 몸무게는 상관관계가 있습니다.
188cm에 82kg은 이상하지 않습니다.
152cm에 82kg은 이상합니다.

유클리드 거리는 둘 다 평균에서 같은 거리라고 할 수 있습니다.
마할라노비스 거리는 두 번째를 이상치로 올바르게 식별합니다
왜냐하면 키-몸무게 상관관계를 고려하기 때문입니다.
```

마할라노비스 거리 사용 시기:
- 이상치 탐지 (평균에서 마할라노비스 거리가 큰 점은 이상치)
- 특징 스케일과 상관관계가 다른 분류 문제
- 신뢰할 수 있는 공분산 행렬을 추정할 수 있을 만큼 충분한 데이터가 있을 때
- 제조 품질 관리 (다변량 공정 모니터링)

### 자카드 유사도 (집합용)

자카드 유사도는 두 집합 사이의 중복을 측정합니다.

```
J(A, B) = |A ∩ B| / |A ∪ B|
```

0(중복 없음)에서 1(동일한 집합)까지 범위를 가집니다. 자카드 거리 = 1 - 자카드 유사도.

```
A = {고양이, 개, 물고기}
B = {고양이, 새, 물고기, 뱀}

교집합 = {고양이, 물고기}         크기 = 2
합집합 = {고양이, 개, 물고기, 새, 뱀}  크기 = 5

자카드 유사도 = 2/5 = 0.4
자카드 거리 = 0.6
```

자카드 사용 시기:
- 태그, 카테고리, 특징 집합 비교
- 단어 존재 기반 문서 유사도 (빈도수 아님)
- 거의 중복 검출 (자카드의 MinHash 근사)
- 이진 특징 벡터 비교 (존재/부재 데이터)
- 분할 모델 평가 (교집합 대 합집합 = 자카드)

### 편집 거리 (레벤슈타인 거리)

편집 거리는 한 문자열을 다른 문자열로 변환하는 데 필요한 최소 단일 문자 작업 수를 셉니다. 작업은 삽입, 삭제, 치환입니다.

```
"kitten" -> "sitting"

kitten -> sitten  (k -> s 치환)
sitten -> sittin  (e -> i 치환)
sittin -> sitting (g 삽입)

편집 거리 = 3
```

동적 프로그래밍을 사용하여 계산됩니다. 행렬을 채우는데, (i, j) 항목은 문자열 A의 첫 i 문자와 문자열 B의 첫 j 문자 사이의 편집 거리입니다.

```
        ""  s  i  t  t  i  n  g
    ""   0  1  2  3  4  5  6  7
    k    1  1  2  3  4  5  6  7
    i    2  2  1  2  3  4  5  6
    t    3  3  2  1  2  3  4  5
    t    4  4  3  2  1  2  3  4
    e    5  5  4  3  2  2  3  4
    n    6  6  5  4  3  3  2  3
```

편집 거리 사용 시기:
- 맞춤법 검사 및 수정
- DNA 서열 정렬 (가중치 작업 포함)
- 퍼지 문자열 매칭
- 지저분한 텍스트 데이터 중복 제거

### KL 발산 (거리는 아니지만 거리처럼 사용)

KL 발산은 한 확률 분포가 다른 분포와 어떻게 다른지를 측정합니다. 레슨 09에서 다루지만, 사람들이 거리처럼 사용하기 때문에 이 논의에 포함됩니다. 하지만 실제 거리는 아닙니다.

```
D_KL(P || Q) = sum(p(x) * log(p(x) / q(x)))
```

중요한 성질: KL 발산은 대칭적이지 않습니다.

```
D_KL(P || Q) != D_KL(Q || P)
```

이는 거리 척도의 기본 요건을 충족하지 못합니다. 또한 삼각형 부등식도 만족하지 않습니다. 발산은 거리입니다.

전방 KL(D_KL(P || Q))은 "평균 추구": Q는 P의 모든 모드를 커버하려고 합니다.
역방향 KL(D_KL(Q || P))은 "모드 추구": Q는 P의 단일 모드에 집중합니다.

KL 발산을 볼 때:
- VAE (ELBO의 KL 항은 잠재 분포를 사전 분포로 밀어냄)
- 지식 증류 (학생이 교사의 분포를 맞추려고 함)
- RLHF (KL 페널티는 미세 조정된 모델을 기본 모델에 가깝게 유지)
- 정책 경사 방법 (정책 업데이트 제약)

### 바셰르슈타인 거리 (지구 이동 거리)

바셰르슈타인 거리는 한 확률 분포를 다른 분포로 변환하는 데 필요한 최소 "작업량"을 측정합니다. 한 분포가 흙더미이고 다른 분포가 구덩이라면, 얼마나 많은 흙을 얼마나 멀리 이동시켜야 하는지를 생각합니다.

```
W(P, Q) = 모든 수송 계획 감마에 대한 inf E[d(x, y)]
```

1차원 분포의 경우 누적 분포 함수의 절댓값 차이의 적분으로 단순화됩니다.

```
W_1(P, Q) = integral |CDF_P(x) - CDF_Q(x)| dx
```

바셰르슈타인 거리의 중요성:
- 진정한 거리 척도(대칭적, 삼각형 부등식 만족)
- 분포가 겹치지 않아도 기울기 제공 (KL 발산은 무한대로 발산)
- 이 성질로 인해 바셰르슈타인 GAN(WGAN)에서 원래 GAN의 훈련 불안정성을 해결

```
겹치지 않는 분포:

P: [1, 0, 0, 0, 0]    Q: [0, 0, 0, 0, 1]

KL 발산: 무한대 (0의 로그)
바셰르슈타인: 4 (모든 질량을 4칸 이동)

바셰르슈타인은 의미 있는 기울기를 제공합니다. KL은 제공하지 않습니다.
```

바셰르슈타인 사용 시기:
- GAN 훈련 (WGAN, WGAN-GP)
- 겹치지 않을 수 있는 분포 비교
- 최적 수송 문제
- 이미지 검색 (색상 히스토그램 비교)

### 다른 작업에 다른 거리가 필요한 이유

| 작업 | 최적 거리 | 이유 |
|------|--------------|-----|
| 텍스트 유사도 | 코사인 | 크기는 노이즈, 방향은 의미 |
| 이미지 픽셀 비교 | L2 | 공간 관계 중요, 특징 스케일 비슷 |
| 희소 고차원 특징 | L1 | 강건함, 드문 큰 차이 증폭하지 않음 |
| 집합 중복 (태그, 카테고리) | 자카드 | 데이터는 본질적으로 집합 값, 벡터 아님 |
| 문자열 매칭 | 편집 거리 | 작업은 인간의 편집 직관과 매핑 |
| 이상치 탐지 | 마할라노비스 | 특징 상관관계와 스케일 고려 |
| 분포 비교 | KL 발산 | Q 대신 P를 사용할 때 손실 정보 측정 |
| GAN 훈련 | 바셰르슈타인 | 분포가 겹치지 않아도 기울기 제공 |
| 임베딩 (벡터 DB) | 코사인 또는 내적 | 임베딩은 방향에 의미를 인코딩하도록 훈련 |
| 추천 | 내적 | 크기는 인기 또는 신뢰도 인코딩 가능 |
| DNA 서열 | 가중치 편집 거리 | 치환 비용은 뉴클레오타이드 쌍에 따라 다름 |
| 제조 QC | L-무한대 | 어떤 차원에서의 최악의 편차 중요 |

### 손실 함수와의 연결

손실 함수는 예측과 타겟에 적용된 거리 함수입니다.

```
손실 함수       사용하는 거리       동작
MSE                 L2 제곱             큰 오차에 강한 페널티
MAE                 L1                     모든 오차에 동일한 페널티
Huber 손실          큰 오차는 L1,        둘 다의 장점: 이상치에 강건,
                    작은 오차는 L2       0 근처에서 매끄러운 기울기
교차 엔트로피       KL 발산          분포 불일치 측정
Hinge 손실          max(0, margin - d)     마진 아래에서만 페널티
Triplet 손실        일반적으로 L2         긍정은 가깝게, 부정은 멀리
대조 손실           L2                     유사 쌍은 가깝게, 비유사
                                           쌍은 마진 이상으로 멀리
```

### 정규화와의 연결

정규화는 손실 함수에 가중치에 대한 노름 페널티를 추가합니다.

```
L1 정규화 (Lasso):   loss + lambda * ||w||_1
  -> 희소 가중치. 일부 가중치는 정확히 0이 됨.
  -> 자동 특징 선택.
  -> 해는 모서리를 가짐 (0에서 미분 불가능).

L2 정규화 (Ridge):   loss + lambda * ||w||_2^2
  -> 작은 가중치. 모든 가중치는 0으로 축소.
  -> 특징 선택 없음 (정확히 0으로 가지 않음).
  -> 모든 곳에서 매끄러운 해.

Elastic Net:                  loss + lambda_1 * ||w||_1 + lambda_2 * ||w||_2^2
  -> L1의 희소성과 L2의 안정성 결합.
  -> 상관관계가 있는 특징 그룹은 함께 유지되거나 제거됨.
```

L1은 희소성을 생성하지만 L2는 그렇지 않은 이유: 2D 가중치 공간에서 제약 영역을 상상해 보세요. L1은 다이아몬드, L2는 원입니다. 손실 함수의 등고선(타원)은 다이아몬드 모서리에서 접촉할 가능성이 높아 한 가중치가 0이 됩니다. 원에서는 매끄러운 점에서 접촉하여 두 가중치 모두 0이 아닙니다.

### 최근접 이웃 검색

모든 거리 함수는 최근접 이웃 검색 문제를 암시합니다: 쿼리 포인트가 주어졌을 때, 데이터셋에서 가장 가까운 포인트를 찾습니다.

정확한 최근접 이웃 검색은 d차원 n개 포인트 데이터셋에서 쿼리당 O(n * d)입니다. 대규모 데이터셋에서는 너무 느립니다.

근사 최근접 이웃(ANN) 알고리즘은 작은 정확도를 희생하여 엄청난 속도 향상을 제공합니다.

```
알고리즘         접근 방식                      사용처
KD-트리          축 정렬 공간 분할   scikit-learn (저차원)
Ball 트리        중첩 초구            scikit-learn (중간차원)
LSH               무작위 해시 투영        거의 중복 검출
HNSW              계층적 탐색 가능         FAISS, Qdrant, Weaviate
                  소규모 세계 그래프
IVF               클러스터 기반 검색과       FAISS (십억 규모)
                  역파일 인덱스
곱의 양자화      벡터 압축, 압축 공간       FAISS (메모리 제약)
                  검색
```

HNSW(Hierarchical Navigable Small World)는 현대 벡터 데이터베이스의 주요 알고리즘입니다. 각 노드가 근사 최근접 이웃에 연결되는 다층 그래프를 구축합니다. 검색은 상위 계층(희소, 긴 점프)에서 시작하여 하위 계층(밀집, 짧은 점프)으로 내려갑니다.

## 구축 방법

### 단계 1: 모든 노름(norm) 및 거리 함수

완전한 구현은 `code/distances.py`를 참조하세요. 모든 함수는 기본 Python 수학만을 사용하여 처음부터 구축되었습니다.

### 단계 2: 동일한 데이터, 다른 거리, 다른 이웃

`distances.py`의 데모는 데이터셋을 생성하고 쿼리 포인트를 선택한 후 거리 메트릭에 따라 가장 가까운 이웃이 어떻게 변하는지 보여줍니다. L1에서 "가장 가까운" 포인트가 L2 또는 코사인(cosine)에서는 가장 가까운 것이 아닐 수 있습니다.

### 단계 3: 임베딩(embedding) 유사도 검색

코드에는 코사인 유사도(cosine similarity) 대 L2 거리를 사용하여 쿼리와 가장 유사한 "문서"를 찾는 모의 임베딩 유사도 검색이 포함되어 있으며, 순위가 달라질 수 있음을 보여줍니다.

## 사용 방법

가장 일반적인 실제 활용 사례: 벡터 데이터베이스에서 유사 항목 찾기.

```python
import numpy as np

def cosine_similarity_matrix(X):
    norms = np.linalg.norm(X, axis=1, keepdims=True)
    norms = np.where(norms == 0, 1, norms)
    X_normalized = X / norms
    return X_normalized @ X_normalized.T

embeddings = np.random.randn(1000, 768)

sim_matrix = cosine_similarity_matrix(embeddings)

query_idx = 0
similarities = sim_matrix[query_idx]
top_k = np.argsort(similarities)[::-1][1:6]
print(f"항목 0과 가장 유사한 상위 5개: {top_k}")
print(f"유사도: {similarities[top_k]}")
```

`model.encode(text)`를 호출한 후 벡터 데이터베이스를 검색할 때, 내부적으로 다음과 같은 과정이 발생합니다. 임베딩 모델은 텍스트를 벡터로 매핑합니다. 벡터 데이터베이스는 ANN(Approximate Nearest Neighbor) 알고리즘을 사용하여 모든 항목을 검사하지 않고도 쿼리 벡터와 저장된 모든 벡터 간의 코사인 유사도(또는 내적)를 계산합니다.

| 코사인 유사도(Cosine similarity) | "벡터 간 각도" | 두 벡터의 내적을 각각의 크기로 정규화한 값. -1에서 +1 사이. 벡터 길이 무시 |
| 코사인 거리(Cosine distance) | "1에서 코사인 유사도를 뺀 값" | 코사인 유사도를 거리로 변환. 0에서 2 사이 |
| 내적(Dot product) | "정규화되지 않은 코사인" | 성분별 곱의 합. 코사인 유사도에 두 크기를 곱한 값 |
| 마할라노비스 거리(Mahalanobis distance) | "상관관계 인식 거리" | 데이터 공분산 행렬로 화이트닝(탈상관화 및 정규화)된 공간에서의 L2 거리 |

| L1 정규화(L1 regularization) | "라소" | 손실에 가중치의 L1 노름 추가. 가중치를 0으로 유도(희소성) |
| L2 정규화(L2 regularization) | "릿지" 또는 "가중치 감소" | 손실에 가중치 제곱의 L2 노름 추가. 희소성은 없지만 가중치를 0으로 축소 |
| 엘라스틱 넷(Elastic Net) | "L1 + L2" | L1과 L2 정규화 결합. 상관된 특징 그룹을 단독으로 사용할 때보다 더 잘 처리 |
