# 다중 에이전트 논쟁과 협업

> Du et al. (ICML 2024, "Society of Minds")는 N개의 모델 인스턴스를 실행하여 독립적으로 답변을 제안한 후, R 라운드에 걸쳐 서로 비평하며 수렴합니다. 사실성, 규칙 준수, 추론 능력을 향상시킵니다. 희소 토폴로지가 전체 메시 대비 토큰 비용에서 우수합니다.

**유형:** 학습 + 구축  
**언어:** Python (표준 라이브러리)  
**선수 지식:** 14단계 · 12 (워크플로우 패턴), 14단계 · 05 (자기 개선 및 CRITIC)  
**소요 시간:** ~60분

## 학습 목표

- 토론 프로토콜 설명: N명의 제안자, R라운드, 공유된 답변에 수렴.
- 토론이 사실성, 규칙 준수, 추론을 개선하는 이유 설명.
- 희소 토폴로지 설명: 모든 토론자가 다른 모든 토론자를 볼 필요 없음.
- 전체 메시(full-mesh) 및 희소(sparse) 변형과 함께 스크립티드 LLM에서 stdlib 토론 구현; 토큰 비용 대 정확도 측정.

## 문제 정의

Self-Refine(레슨 05)은 단일 모델이 자기 자신을 비평하는 방식으로 — 집단사고(groupthink) 위험이 존재합니다. CRITIC(레슨 05)은 외부 도구에 기반한 비평을 수행하지만 — 이러한 도구가 항상 사용 가능한 것은 아닙니다. 디베이트(Debate)는 세 번째 방식을 도입합니다: 여러 모델 인스턴스, 상호 비평(cross-critique), 의견 불일치를 통한 합의 도출.

## 개념

### 마음의 사회 (Du et al., ICML 2024)

- N개의 모델 인스턴스가 동일한 질문에 대해 독립적으로 답변을 제안합니다.
- R 라운드 동안 각 모델은 다른 모델의 제안을 읽고 이를 비평합니다.
- 모델은 비평에 기반해 답변을 업데이트합니다.
- R 라운드 후 수렴된 답변을 반환합니다.

원본 실험은 비용 문제로 N=3, R=2를 사용했습니다. 어려운 문제(MMLU, GSM8K, 체스 이동 유효성, 전기 생성)에서 더 많은 에이전트와 라운드가 정확도를 향상시킵니다.

모델 간 조합은 단일 모델 토론보다 우수합니다: ChatGPT + Bard 함께 > 각각 단독.

### 희소 토폴로지

"희소 통신 토폴로지를 통한 다중 에이전트 토론 개선" (arXiv:2406.11776, 2024-2025)은 완전 메시(full-mesh) 토론이 항상 최적이 아님을 보였습니다. 희소 토폴로지(스타, 링, 허브-앤-스포크)는 더 낮은 토큰 비용으로 정확도를 맞출 수 있습니다. 각 토론자는 일부 동료만 참조합니다.

시사점:

- 완전 메시 N=5, R=3 = 15개 제안, 각 모델은 4개 동료를 읽음 = 60개 비평 작업.
- 스타 N=5, R=3 (1개 허브 + 4개 스포크) = 15개 제안, 스포크는 허브만 읽음 = 12개 비평 작업.

### 토론이 도움이 되는 경우

- **사실성.** N개의 독립적 제안, 상호 검증으로 환각(hallucination) 감소.
- **규칙 준수.** 체스 이동 유효성 — 한 모델이 규칙을 놓치면 다른 모델이 포착.
- **개방형 추론.** 여러 접근 방식이 정답으로 수렴.

### 토론이 해로운 경우

- **지연 시간 민감 UX.** N × R 직렬 라운드는 허용되지 않는 지연 시간 발생.
- **비용 민감 확장.** 질문당 N × R 토큰 비용.
- **단순 사실 조회.** 5회 토론보다 1회 조회가 더 저렴.

### 2026년 실용적 구현

- **Anthropic 오케스트레이터-워커** (레슨 12) — 종합 단계를 포함한 토론의 한 변형.
- **LangGraph 감독자** (레슨 13) — 중앙 라우터 + 전문가 에이전트가 토론 노드를 구현.
- **OpenAI 에이전트 SDK** (레슨 16) — 에이전트 간 반복적 비평을 위한 핸드오프.
- **다중 에이전트 평가** — 토론 + 평가자-최적화기 조합으로 평가 신호 생성.

### 이 패턴이 실패하는 경우

- **수렴 붕괴.** 모든 에이전트가 첫 번째 잘못된 답변으로 수렴. 필수 불일치 라운드로 완화.
- **허브 실패.** 스타 토폴로지에서 나쁜 허브가 모두를 오염. 회전 또는 다중 허브 사용.
- **프롬프트 동질화.** 모든 에이전트가 동일한 프롬프트를 사용; 동일한 답변 생성. 다양한 프롬프트 및/또는 모델 사용.

## 빌드 및 실행

`code/main.py`는 표준 라이브러리 토론(debate)을 구현합니다:

- `Debater` 클래스 (스크립트된 LLM으로, 토론자별 의견 변동(drift) 포함).
- `FullMeshDebate` 및 `SparseDebate` 실행기(runner).
- 세 가지 질문: 사실 확인형(factual) 1개, 규칙 기반(rule-based) 1개, 추론형(reasoning) 1개.
- 평가 지표: 수렴된 답변(convergent answer), 수렴까지 소요된 라운드(rounds to convergence), 총 비판 작업량(total critique ops).

실행 방법:

```
python3 code/main.py
```

출력 결과: 프로토콜별 정확도(accuracy)와 비용(cost); 희소 토론(sparse)이 전체 메시 토론(full mesh)과 3개 질문 중 2개에서 동일한 성능을 보이면서도 더 낮은 비용으로 실행됨.

## 사용 방법

- **Anthropic orchestrator-workers**를 사용한 간단한 2-3개 작업자(worker) 간 토론.
- **LangGraph**를 활용한 상태(state) 유지가 가능한 다라운드 토론 및 체크포인팅(checkpointing).
- **커스텀** 구현을 통한 연구 또는 특수한 정확성 보장(correctness guarantees) 요구 사항.

## Ship It

`outputs/skill-debate.md`는 구성 가능한 토폴로지, N, R 및 수렴 규칙을 갖춘 다중 에이전트 토론을 위한 스캐폴드를 제공합니다.

## 연습 문제

1. "강제 불일치" 규칙 구현: 1라운드에서 모든 토론자는 서로 다른 제안을 생성해야 함. 수렴 속도에 미치는 영향 측정.
2. 신뢰도 가중치 집계 추가: 토론자는 (답변, 신뢰도)를 반환; 집계기는 신뢰도에 따라 가중치 부여. 도움이 되는가?
3. "에이전트" 하나를 다른 의견을 가진 스크립트 LLM으로 교체. 이질성이 정확도를 향상시키는가?
4. 전체 메시(mesh) 대 희소(sparse) 연결에 대한 토큰 비용 측정. 3개 질문에 대해 비용 대 정확도 그래프 작성.
5. "Society of Minds" 논문 읽기. N=5, R=3으로 토이 모델 확장. 어떤 부분이 고장나고, 어떤 부분이 개선되는가?
