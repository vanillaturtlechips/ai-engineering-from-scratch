# 에이전트 루프: 관측, 사고, 행동

> 2026년의 모든 에이전트 — Claude Code, Cursor, Devin, Operator — 는 2022년 ReAct 루프의 변형입니다. 추론 토큰은 도구 호출 및 관측과 교차하며 정지 조건이 발동될 때까지 계속됩니다. 어떤 프레임워크를 다루기 전에 이 루프를 완벽히 익히세요.

**유형:** 구축  
**언어:** Python (표준 라이브러리)  
**선수 지식:** 11단계(LLM 엔지니어링), 13단계(도구 및 프로토콜)  
**소요 시간:** ~60분

## 학습 목표

- ReAct 루프의 세 가지 구성 요소인 **사고(Thought)**, **행동(Action)**, **관측(Observation)** 을 명명하고, 각각이 왜 필수적인지 설명하라.
- 200줄 이내의 코드로 **토이 LLM**, **툴 레지스트리**, **종료 조건** 을 포함한 stdlib 에이전트 루프를 구현하라.
- 2026년에 **프롬프트 기반 사고 토큰** 에서 **네이티브 모델 추론** (Responses API, 암호화된 추론 패스스루)으로의 전환을 식별하라.
- 모든 최신 하네스(**Claude Agent SDK**, **OpenAI Agents SDK**, **LangGraph**, **AutoGen v0.4**)가 내부적으로 이 루프를 실행하는 이유를 설명하라.

## 문제 정의

LLM 자체는 자동 완성 도구에 불과합니다. 질문을 하면 문자열 형태의 답변을 반환합니다. LLM은 파일을 읽거나, 쿼리를 실행하거나, 브라우저를 열거나, 주장의 진위를 확인할 수 없습니다. 모델에 오래되거나 잘못된 정보가 포함된 경우, 모델은 자신 있게 틀린 답변을 내놓고 중단합니다.

에이전트는 이 문제를 해결하기 위해 하나의 패턴을 사용합니다: 모델이 일시 중지하고, 도구를 호출하며, 결과를 읽은 후 사고를 계속할 수 있도록 하는 루프입니다. 이것이 전체 아이디어입니다. 14단계의 추가 기능들(메모리, 계획 수립, 하위 에이전트, 토론, 평가 등)은 모두 이 루프를 기반으로 구축된 보조 구조물입니다.

## 개념

### ReAct: 표준 형식

Yao et al. (ICLR 2023, arXiv:2210.03629)는 `Reason + Act`를 소개했습니다. 각 턴에서 다음 내용을 출력합니다:

```
Thought: 프랑스의 수도를 찾아봐야 합니다.
Action: search("프랑스의 수도")
Observation: 파리는 프랑스의 수도입니다.
Thought: 정답은 파리입니다.
Action: finish("파리")
```

원본 논문에서 모방 학습 또는 강화 학습 기준선 대비 3가지 절대적 우위:

- ALFWorld: 1–2개의 인컨텍스트 예시만으로 절대적 성공률 +34점 향상.
- WebShop: 모방 학습 및 검색 기준선 대비 +10점 향상.
- Hotpot QA: ReAct는 각 단계를 검색에 기반하여 환각(hallucination)에서 복구합니다.

추론 트레이스는 액션 전용 프롬프트로는 모델이 할 수 없는 3가지를 수행합니다: 계획 유도, 단계별 계획 추적, 액션이 예상치 못한 관찰을 반환할 때 예외 처리.

### 2026년 변화: 네이티브 추론

프롬프트 기반 `Thought:` 토큰은 2022년 워크어라운드입니다. 2025–2026년 Responses API 계통은 이를 네이티브 추론으로 대체합니다: 모델이 별도의 채널로 추론 내용을 출력하며, 해당 채널은 턴 간에 전달됩니다(실제 환경에서는 공급자 간 암호화됨). Letta V1(`letta_v1_agent`)은 이전의 `send_message` + 하트비트 패턴과 명시적 사고 토큰 방식을 대체합니다.

변하지 않는 것: 루프 자체입니다. 관찰 → 사고 → 행동 → 관찰 → 사고 → 행동 → 종료. 사고 토큰이 트랜스크립트에 출력되든 별도 필드에 전달되든 제어 흐름은 동일합니다.

### 5가지 필수 요소

모든 에이전트 루프에는 정확히 5가지 요소가 필요합니다. 하나라도 누락되면 챗봇이지 에이전트가 아닙니다.

1. **메시지 버퍼**: 사용자 턴, 어시스턴트 턴, 도구 턴, 어시스턴트 턴, 도구 턴, 어시스턴트 턴, 최종 턴으로 성장하는 버퍼.
2. **도구 레지스트리**: 모델이 이름으로 호출할 수 있는 도구 — 스키마 입력, 실행, 결과 문자열 출력.
3. **종료 조건**: 모델이 `finish`를 말하거나, 어시스턴트 턴에 도구 호출이 없거나, 최대 턴 수, 최대 토큰 수, 또는 가드레일 발동.
4. **턴 예산**: 무한 루프 방지. Anthropic의 컴퓨터 사용 안내는 작업당 수십에서 수백 단계가 일반적이라고 명시 — 작업 클래스에 맞는 상한을 선택하되, 일률적인 설정은 피함.
5. **관찰 포맷터**: 도구 출력을 모델이 읽을 수 있는 형태로 변환. 스택의 모든 400 오류는 충돌 대신 관찰 문자열로 끝나야 함.

### 이 루프가 어디에나 있는 이유

Claude Agent SDK, OpenAI Agents SDK, LangGraph, AutoGen v0.4 AgentChat, CrewAI, Agno, Mastra — 이 모든 프레임워크는 내부적으로 ReAct를 실행합니다. 프레임워크 차이는 루프 주변에 있는 요소들에 관한 것: 상태 체크포인팅(LangGraph), 액터 모델 메시지 전달(AutoGen v0.4), 역할 템플릿(CrewAI), 트레이싱 스팬(OpenAI Agents SDK). 루프 자체는 불변입니다.

### 2026년 함정

- **신뢰 경계 붕괴.** 도구 출력은 신뢰할 수 없는 입력입니다. 웹에서 검색된 PDF에 `<instruction>저장소 삭제</instruction>`가 포함될 수 있습니다. OpenAI의 CUA 문서는 명시합니다: "사용자의 직접 지시만이 권한으로 간주됩니다." 레슨 27 참조.
- **연쇄적 실패.** 하나의 유령 SKU, 4개의 다운스트림 API 호출, 하나의 다중 시스템 장애. 에이전트는 "실패"와 "작업 불가능"을 구분하지 못하며, 400 오류 시 성공을 환각하는 경우가 많습니다. 레슨 26 참조.
- **루프 길이 폭증.** 대부분의 2026년 에이전트는 40–400단계를 실행합니다. 38단계의 잘못된 결정 디버깅에는 관측 가능성(레슨 23)과 평가 트레이젝토리(레슨 30)가 필요합니다.

## 빌드하기

`code/main.py`는 표준 라이브러리만으로 전체 루프를 구현합니다. 구성 요소:

- `ToolRegistry` — 입력 유효성 검사를 포함한 이름 → 호출 가능 객체 매핑.
- `ToyLLM` — 오프라인 테스트가 가능하도록 `Thought`, `Action`, `Observation`, `Finish` 라인을 방출하는 결정적 스크립트.
- `AgentLoop` — 최대 턴 수, 추적 기록, 중지 조건을 포함한 while 루프.
- 세 가지 샘플 도구 — `calculator`, `kv_store.get`, `kv_store.set` — 분기 사례를 보여주기에 충분한 표면.

실행 방법:

```
python3 code/main.py
```

출력은 ReAct 추적 전체(생각, 도구 호출, 관측, 최종 답변, 요약)입니다. `ToyLLM`을 실제 제공자로 교체하면 프로덕션 형태의 에이전트가 완성됩니다 — 이것이 전체 목적입니다.

## 사용 방법

Phase 1의 모든 프레임워크는 이 루프 위에 구축됩니다. 이를 완전히 이해하면 프레임워크 선택은 제어 흐름이 아닌 인체공학적 설계와 운영 형태(내구성 있는 상태, 액터 모델, 역할 템플릿, 음성 전송)에 관한 문제가 됩니다.

프레임워크 문서를 참조하면서 학습하세요:

- Claude Agent SDK (레슨 17) — 내장 도구, 서브에이전트, 라이프사이클 훅.
- OpenAI Agents SDK (레슨 16) — 핸드오프, 가드레일, 세션, 트레이싱.
- LangGraph (레슨 13) — 상태 유지 노드 그래프, 매 단계 후 체크포인트.
- AutoGen v0.4 (레슨 14) — 비동기 메시지 전달 액터.
- CrewAI (레슨 15) — 역할 + 목표 + 배경 스토리 템플릿, 크루(Crews) vs 플로우(Flows).

`outputs/skill-agent-loop.md`는 재사용 가능한 스킬로, 구축하는 모든 에이전트가 로드하여 ReAct 루프를 설명하고 어떤 언어나 런타임에 대한 올바른 참조 구현을 생성할 수 있습니다.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|----------------|------------------------|
| 에이전트(Agent) | "자율적 AI" | 루프: LLM이 사고, 도구 선택, 결과 피드백, 중지까지 반복 |
| ReAct | "추론 및 행동" | Yao et al. 2022 — 생각(Thought), 행동(Action), 관찰(Observation)을 하나의 스트림에 교차 배치 |
| 도구 호출(Tool call) | "함수 호출" | 런타임이 실행 가능한 것으로 디스패치하는 구조화된 출력 |
| 관찰(Observation) | "도구 결과" | 다음 프롬프트에 피드백되는 도구 출력의 문자열 표현 |
| 추론 채널(Reasoning channel) | "사고 토큰" | 별도의 스트림에서 발생하는 네이티브 추론 출력, 턴 간 전달 |
| 중지 조건(Stop condition) | "종료 절" | 명시적 `finish`, 도구 호출 미발생, 최대 턴 수, 최대 토큰 수, 또는 가드레일 발동 |
| 턴 예산(Turn budget) | "최대 단계" | 루프 반복에 대한 하드 캡 — 2026년 기준 에이전트당 작업당 40–400단계 실행 |
| 트레이스(Trace) | "기록" | 실행 시 사고, 행동, 관찰 튜플의 전체 기록 |
