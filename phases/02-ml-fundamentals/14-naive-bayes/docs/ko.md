# 나이브 베이즈

> "순진한" 가정은 틀렸지만, 그래도 작동한다. 그것이 바로 나이브 베이즈의 아름다움이다.

**유형:** 구축(Build)
**언어:** Python
**선수 지식:** 2단계, 레슨 01-07 (분류, 베이즈 정리)
**소요 시간:** ~75분

## 학습 목표

- 텍스트 분류를 위해 라플라스 스무딩(Laplace smoothing)을 적용한 다항 나이브 베이즈(Multinomial Naive Bayes)를 직접 구현
- 순진 독립성 가정(naive independence assumption)이 수학적으로는 틀렸지만 실제로는 올바른 클래스 순위(class rankings)를 생성하는 이유 설명
- 다항(Multinomial), 베르누이(Bernoulli), 가우시안(Gaussian) 나이브 베이즈 변형(variants) 비교 및 주어진 특성(feature) 유형에 적합한 모델 선택
- 고차원 희소 데이터(high-dimensional sparse data)에서 나이브 베이즈와 로지스틱 회귀(logistic regression) 성능 비교 및 작용하는 편향-분산 트레이드오프(bias-variance tradeoff) 설명

## 문제 정의

텍스트를 분류해야 합니다. 이메일을 스팸 또는 비스팸으로, 고객 리뷰를 긍정 또는 부정으로, 지원 티켓을 카테고리로 분류해야 합니다. 수천 개의 특징(단어당 하나씩)이 있고 훈련 데이터는 제한적입니다.

대부분의 분류기는 여기서 실패합니다. 로지스틱 회귀는 수천 개의 가중치를 안정적으로 추정하기 위해 충분한 샘플이 필요합니다. 결정 트리는 한 번에 하나의 단어만 기준으로 분할하며 심하게 과적합됩니다. 10,000차원에서 KNN은 모든 점이 서로 동일한 거리에 있기 때문에 의미가 없습니다.

나이브 베이즈는 이 문제를 처리합니다. 수학적으로 잘못된 가정(클래스 주어진 모든 특징이 서로 독립적이라는 가정)을 하지만, 특히 작은 훈련 세트에서 텍스트 분류에서 "더 똑똑한" 모델들을 능가합니다. 데이터를 한 번 통과시켜 훈련합니다. 수백만 개의 특징까지 확장 가능합니다. 확률 추정치를 생성합니다(비록 독립성 가정으로 인해 종종 보정이 잘 안 될 수 있지만).

잘못된 가정이 좋은 예측으로 이어지는 이유를 이해하면 머신러닝에 대한 근본적인 통찰을 얻을 수 있습니다: 가장 좋은 모델은 가장 정확한 모델이 아니라, 데이터에 대한 최적의 편향-분산 균형을 가진 모델입니다.

## 개념

### 베이즈 정리 (빠른 복습)

베이즈 정리는 조건부 확률을 뒤집습니다:

```
P(클래스 | 특징) = P(특징 | 클래스) * P(클래스) / P(특징)
```

우리는 `P(클래스 | 특징)`을 원합니다 -- 문서에 있는 단어들을 보고 해당 문서가 클래스에 속할 확률입니다. 다음으로부터 이를 계산할 수 있습니다:
- `P(특징 | 클래스)` -- 이 클래스의 문서에서 이 단어들을 볼 가능성
- `P(클래스)` -- 클래스의 사전 확률 (스팸은 일반적으로 얼마나 흔한가?)
- `P(특징)` -- 모든 클래스에 대해 동일한 증거이므로 비교 시 무시할 수 있습니다

`P(클래스 | 특징)`이 가장 높은 클래스가 승리합니다.

### 순진한 독립성 가정

`P(특징 | 클래스)`를 정확히 계산하려면 모든 특징의 결합 확률을 추정해야 합니다. 10,000개의 단어로 이루어진 어휘에서는 2^10,000개의 가능한 조합에 대한 분포를 추정해야 합니다. 불가능합니다.

순진한 가정: 모든 특징은 클래스 주어졌을 때 조건부 독립입니다.

```
P(w1, w2, ..., wn | 클래스) = P(w1 | 클래스) * P(w2 | 클래스) * ... * P(wn | 클래스)
```

불가능한 하나의 결합 분포 대신, n개의 간단한 특징별 분포를 추정합니다. 각각은 카운트만 필요합니다.

이 가정은 명백히 잘못되었습니다. "machine"과 "learning"이라는 단어는 어떤 문서에서도 독립적이지 않습니다. 하지만 분류기는 정확한 확률 추정이 필요하지 않습니다. 정확한 순위가 필요합니다 -- 어떤 클래스의 확률이 가장 높은지. 독립성 가정은 체계적인 오류를 도입하지만, 그 오류는 모든 클래스에 비슷하게 영향을 미치므로 순위는 올바르게 유지됩니다.

### 여전히 작동하는 이유

세 가지 이유:

1. **보정보다 순위.** 분류는 최상위 순위 클래스만 정확하면 됩니다. 실제 확률이 0.7일 때 P(스팸) = 0.99999라도 분류기는 여전히 스팸을 올바르게 선택합니다. 정확한 확률이 필요하지 않습니다. 올바른 승자만 필요합니다.

2. **고편향, 저분산.** 독립성 가정은 강력한 사전 확률입니다. 모델을 크게 제약하여 과적합을 방지합니다. 제한된 훈련 데이터에서는 약간 틀렸지만 안정적인 모델이 이론적으로 맞지만 매우 불안정한 모델보다 낫습니다. 이는 편향-분산 트레이드오프의 예시입니다.

3. **특징 중복이 상쇄.** 상관된 특징은 중복된 증거를 제공합니다. 분류기는 이 증거를 두 번 계산하지만, 올바른 클래스에 대해서도 두 번 계산합니다. "machine"과 "learning"이 항상 함께 나타난다면, 둘 다 "tech" 클래스에 대한 증거를 제공합니다. NB는 이를 두 번 계산하지만, 올바른 클래스에 대해 두 번 계산합니다.

네 번째 실용적인 이유: 나이브 베이즈는 매우 빠릅니다. 훈련은 데이터를 한 번 순회하며 빈도를 계산하는 것입니다. 예측은 행렬 곱셈입니다. 수백만 개의 문서를 몇 초 안에 훈련할 수 있습니다. 이 속도 덕분에 더 빠르게 반복하고, 더 많은 특징 집합을 시도하며, 느린 모델보다 더 많은 실험을 실행할 수 있습니다.

### 수학적 단계별 설명

구체적인 예시를 따라가 보겠습니다. 두 클래스가 있습니다: 스팸과 비스팸. 어휘에는 세 단어가 있습니다: "free", "money", "meeting".

훈련 데이터:
- 스팸 이메일은 "free"를 80번, "money"를 60번, "meeting"을 10번 언급 (총 150단어)
- 비스팸 이메일은 "free"를 5번, "money"를 10번, "meeting"을 100번 언급 (총 115단어)
- 이메일의 40%는 스팸, 60%는 비스팸

라플라스 스무딩(alpha=1) 적용:

```
P(free | 스팸)    = (80 + 1) / (150 + 3) = 81/153 = 0.529
P(money | 스팸)   = (60 + 1) / (150 + 3) = 61/153 = 0.399
P(meeting | 스팸) = (10 + 1) / (150 + 3) = 11/153 = 0.072

P(free | 비스팸)    = (5 + 1) / (115 + 3) = 6/118 = 0.051
P(money | 비스팸)   = (10 + 1) / (115 + 3) = 11/118 = 0.093
P(meeting | 비스팸) = (100 + 1) / (115 + 3) = 101/118 = 0.856
```

새 이메일 내용: "free" (2회), "money" (1회), "meeting" (0회).

```
log P(스팸 | 이메일) = log(0.4) + 2*log(0.529) + 1*log(0.399) + 0*log(0.072)
                    = -0.916 + 2*(-0.637) + (-0.919) + 0
                    = -3.109

log P(비스팸 | 이메일) = log(0.6) + 2*log(0.051) + 1*log(0.093) + 0*log(0.856)
                        = -0.511 + 2*(-2.976) + (-2.375) + 0
                        = -8.838
```

스팸이 큰 차이로 승리합니다. "free"가 두 번 나타나는 것은 스팸에 대한 강력한 증거입니다. "meeting"이 나타나지 않는 것은 두 로그 합에 0을 기여합니다(0 * log(P)) -- 다항 NB에서는 없는 단어가 영향을 미치지 않습니다. 단어 부재를 명시적으로 모델링하는 것은 베르누이 NB입니다.

### 세 가지 변형

나이브 베이즈는 세 가지 변형이 있습니다. 각각은 `P(특징 | 클래스)`를 다르게 모델링합니다.


#### 다항 나이브 베이즈

각 특징을 카운트로 모델링합니다. 특징이 단어 빈도나 TF-IDF 값인 텍스트 데이터에 가장 적합합니다.

```
P(단어_i | 클래스) = (클래스 내 단어_i 카운트 + alpha) / (클래스 내 총 단어 수 + alpha * 어휘 크기)
```

`alpha`는 라플라스 스무딩입니다(아래 설명). 이 변형은 텍스트 분류의 핵심 모델입니다.


#### 가우시안 나이브 베이즈

각 특징을 정규 분포로 모델링합니다. 연속형 특징에 가장 적합합니다.

```
P(x_i | 클래스) = (1 / sqrt(2 * pi * 분산)) * exp(-(x_i - 평균)^2 / (2 * 분산))
```

각 클래스는 특징별로 고유한 평균과 분산을 가집니다. 특징이 각 클래스 내에서 종 모양의 분포를 따를 때 잘 작동합니다.


#### 베르누이 나이브 베이즈

각 특징을 이진(존재/부재)으로 모델링합니다. 짧은 텍스트나 이진 특징 벡터에 가장 적합합니다.

```
P(단어_i | 클래스) = (단어_i를 포함하는 클래스 내 문서 수 + alpha) / (클래스 내 총 문서 수 + 2 * alpha)
```

다항과 달리, 베르누리는 단어 부재를 명시적으로 페널티로 적용합니다. "free"가 일반적으로 스팸에 나타나지만 이 이메일에는 없다면, 베르누리는 이를 스팸에 대한 반대 증거로 간주합니다.

### 각 변형 사용 시기

| 변형 | 특징 유형 | 가장 적합한 경우 | 예시 |
|---------|-------------|----------|---------|
| 다항 | 카운트 또는 빈도 | 텍스트 분류, 단어 가방 | 이메일 스팸, 주제 분류 |
| 가우시안 | 연속 값 | 정규 분포와 유사한 특징을 가진 표 형식 데이터 | 아이리스 분류, 센서 데이터 |
| 베르누이 | 이진(0/1) | 짧은 텍스트, 이진 특징 벡터 | SMS 스팸, 존재/부재 특징 |

### 라플라스 스무딩

테스트 데이터에 있는 단어가 특정 클래스의 훈련 데이터에 전혀 나타나지 않으면 어떻게 될까요?

스무딩 없을 때: `P(단어 | 클래스) = 0/N = 0`. 하나의 0이 전체 곱을 0으로 만들어 `P(클래스 | 특징) = 0`이 됩니다. 다른 모든 증거와 상관없이, 한 번도 보지 못한 단어가 전체 예측을 파괴합니다.

라플라스 스무딩은 모든 특징 카운트에 작은 값 `alpha`(보통 1)를 더합니다:

```
P(단어_i | 클래스) = (단어_i 카운트 + alpha) / (클래스 내 총 단어 수 + alpha * 어휘 크기)
```

alpha=1이면 모든 단어가 최소한 작은 확률을 가집니다. 테스트 이메일에 "discombobulate"가 나타나도 스팸 확률이 0이 되지 않습니다. 스무딩은 베이지안 해석이 있습니다: 단어 분포에 균일 디리클레 사전 확률을 적용하는 것과 동일합니다.

높은 alpha는 더 강한 스무딩(더 균일한 분포)을 의미합니다. 낮은 alpha는 모델이 데이터를 더 신뢰함을 의미합니다. alpha는 튜닝하는 하이퍼파라미터입니다.

alpha의 효과:

| Alpha | 효과 | 사용 시기 |
|-------|--------|-------------|
| 0.001 | 거의 스무딩 없음, 데이터 신뢰 | 매우 큰 훈련 세트, 미확인 특징 예상되지 않음 |
| 0.1 | 가벼운 스무딩 | 큰 훈련 세트 |
| 1.0 | 표준 라플라스 스무딩 | 기본 시작점 |
| 10.0 | 강한 스무딩, 분포 평탄화 | 매우 작은 훈련 세트, 많은 미확인 특징 예상 |

### 로그 공간 계산

수백 개의 확률(각각 1 미만)을 곱하면 부동소수점 언더플로가 발생합니다. 실제 값은 매우 작은 양수이지만, 부동소수점에서는 0이 됩니다.

해결책: 로그 공간에서 작업합니다. 확률을 곱하는 대신 로그를 더합니다:

```
log P(클래스 | x1, x2, ..., xn) = log P(클래스) + sum_i log P(xi | 클래스)
```

이를 통해 예측을 내적로 변환합니다:

```
log_scores = X @ log_feature_probs.T + log_class_priors
예측 = argmax(log_scores)
```

행렬 곱셈입니다. 나이브 베이즈 예측이 매우 빠른 이유입니다 -- 단일 계층 선형 모델과 동일한 연산입니다.

### 나이브 베이즈 vs 로지스틱 회귀

둘 다 텍스트를 위한 선형 분류기입니다. 차이점은 모델링하는 대상에 있습니다.

| 측면 | 나이브 베이즈 | 로지스틱 회귀 |
|--------|------------|-------------------|
| 유형 | 생성적 (P(X\|Y) 모델링) | 판별적 (P(Y\|X) 모델링) |
| 훈련 | 빈도 계산 | 손실 함수 최적화 |
| 소량 데이터 | 더 좋음 (강한 사전 확률 도움) | 더 나쁨 (가중치 추정에 부족) |
| 대량 데이터 | 더 나쁨 (잘못된 가정 영향) | 더 좋음 (유연한 경계) |
| 특징 | 독립성 가정 | 상관관계 처리 |
| 속도 | 단일 순회, 매우 빠름 | 반복적 최적화 |
| 보정 | 낮은 확률 정확도 | 더 나은 확률 정확도 |

경험적 규칙: 나이브 베이즈로 시작하세요. 충분한 데이터가 있고 NB가 정체되면 로지스틱 회귀로 전환하세요.

실제로 부동소수점 언더플로를 피하기 위해 로그 공간에서 작업합니다. 많은 작은 확률을 곱하는 대신 로그를 더합니다:

```
log P(클래스 | 특징) = log P(클래스) + sum_i log P(특징_i | 클래스)
```

## 구현

`code/naive_bayes.py`의 코드는 MultinomialNB와 GaussianNB를 처음부터 구현합니다.

### MultinomialNB

처음부터 구현한 버전:

1. **fit(X, y)**: 각 클래스에 대해 각 피처의 빈도를 계산합니다. 라플라스 스무딩을 추가합니다. 로그 확률을 계산합니다. 클래스 사전 확률(클래스 빈도의 로그)을 저장합니다.

2. **predict_log_proba(X)**: 각 샘플에 대해 모든 클래스에 대해 log P(class) + sum of log P(feature_i | class)를 계산합니다. 이는 행렬 곱셈: X @ log_probs.T + log_priors입니다.

3. **predict(X)**: 가장 높은 로그 확률을 가진 클래스를 반환합니다.

```python
class MultinomialNB:
    def __init__(self, alpha=1.0):
        self.alpha = alpha

    def fit(self, X, y):
        classes = np.unique(y)
        n_classes = len(classes)
        n_features = X.shape[1]

        self.classes_ = classes
        self.class_log_prior_ = np.zeros(n_classes)
        self.feature_log_prob_ = np.zeros((n_classes, n_features))

        for i, c in enumerate(classes):
            X_c = X[y == c]
            self.class_log_prior_[i] = np.log(X_c.shape[0] / X.shape[0])
            counts = X_c.sum(axis=0) + self.alpha
            self.feature_log_prob_[i] = np.log(counts / counts.sum())

        return self
```

핵심 통찰: 학습 후 예측은 행렬 곱셈에 바이어스를 더한 것과 같습니다. 이것이 나이브 베이즈가 매우 빠른 이유입니다.

### GaussianNB

연속형 피처의 경우, 클래스별 피처별 평균과 분산을 추정합니다:

```python
class GaussianNB:
    def __init__(self):
        pass

    def fit(self, X, y):
        classes = np.unique(y)
        self.classes_ = classes
        self.means_ = np.zeros((len(classes), X.shape[1]))
        self.vars_ = np.zeros((len(classes), X.shape[1]))
        self.priors_ = np.zeros(len(classes))

        for i, c in enumerate(classes):
            X_c = X[y == c]
            self.means_[i] = X_c.mean(axis=0)
            self.vars_[i] = X_c.var(axis=0) + 1e-9
            self.priors_[i] = X_c.shape[0] / X.shape[0]

        return self
```

예측은 피처별 가우시안 PDF를 사용하며, 로그 공간에서 곱셈을 덧셈으로 변환합니다.

### 데모: 텍스트 분류

코드는 두 클래스(기술 기사 vs 스포츠 기사)를 시뮬레이션하는 합성 백오브워즈 데이터를 생성합니다. 각 클래스는 서로 다른 단어 빈도 분포를 가집니다. MultinomialNB는 단어 카운트를 사용하여 분류합니다.

합성 데이터는 다음과 같이 작동합니다: 200개의 "단어"(피처 열)를 생성합니다. 단어 0-39는 기술 기사에서 높은 빈도를, 스포츠 기사에서 낮은 빈도를 가집니다. 단어 80-119는 스포츠 기사에서 높은 빈도를, 기술 기사에서 낮은 빈도를 가집니다. 단어 40-79는 두 클래스에서 중간 빈도를 가집니다. 이는 일부 단어가 강력한 클래스 지표이고 다른 단어는 노이즈인 현실적인 시나리오를 만듭니다.

### 데모: 연속형 피처

코드는 Iris와 유사한 데이터(3개 클래스, 4개 피처, 가우시안 클러스터)를 생성합니다. GaussianNB는 클래스별 평균과 분산을 사용하여 분류합니다. 각 클래스는 다른 중심(평균 벡터)과 다른 분산(퍼짐)을 가지며, 이는 카테고리별로 측정값이 체계적으로 다른 실제 데이터를 모방합니다.

코드는 또한 다음을 보여줍니다:
- **스무딩 비교:** 다양한 alpha 값으로 MultinomialNB를 학습시켜 스무딩 강도가 정확도에 미치는 영향을 보여줍니다.
- **학습 데이터 크기 실험:** NB 정확도가 20개에서 1600개 샘플로 증가함에 따라 어떻게 개선되는지 보여줍니다. NB는 매우 적은 샘플로도 괜찮은 정확도에 도달합니다. 이것이 주요 장점입니다.
- **혼동 행렬:** 클래스별 정밀도, 재현율, F1 점수를 통해 NB가 어디서 오류를 만드는지 보여줍니다.

### 예측 속도

나이브 베이즈 예측은 행렬 곱셈입니다. n개 샘플, d개 피처, k개 클래스에 대해:
- MultinomialNB: 하나의 행렬 곱셈 (n x d) @ (d x k) = O(n * d * k)
- GaussianNB: n * k개의 가우시안 PDF 평가, 각각 d개 피처에 대해 = O(n * d * k)

둘 다 모든 차원에서 선형입니다. 모든 학습 포인트와의 거리 계산이 필요한 KNN이나 모든 서포트 벡터와의 커널 평가가 필요한 RBF 커널 SVM과 비교해 보세요. NB는 예측 시간에서 훨씬 빠릅니다.

```python
from sklearn.naive_bayes import GaussianNB, MultinomialNB

gnb = GaussianNB()
gnb.fit(X_train, y_train)
print(f"GaussianNB 정확도: {gnb.score(X_test, y_test):.3f}")

mnb = MultinomialNB(alpha=1.0)
mnb.fit(X_train_counts, y_train)
print(f"MultinomialNB 정확도: {mnb.score(X_test_counts, y_test):.3f}")
```

sklearn을 이용한 텍스트 분류:

```python
from sklearn.feature_extraction.text import CountVectorizer
from sklearn.naive_bayes import MultinomialNB
from sklearn.pipeline import Pipeline

text_clf = Pipeline([
    ("vectorizer", CountVectorizer()),

text_clf.fit(train_texts, train_labels)

```

`naive_bayes.py` 코드의 구현체는 동일 데이터에 대해 sklearn과 직접 비교하여 정확성을 검증합니다.

### TF-IDF와 나이브 베이즈

원시 단어 빈도는 모든 단어에 발생 횟수에 따른 동일한 가중치를 부여합니다. 하지만 "the"나 "is" 같은 일반 단어는 모든 클래스에서 자주 나타나며 -- 정보를 제공하지 않습니다. TF-IDF(Term Frequency - Inverse Document Frequency)는 일반 단어를 감소시키고 희귀하고 판별력 있는 단어를 강조합니다.

```python
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.naive_bayes import MultinomialNB
from sklearn.pipeline import Pipeline

text_clf = Pipeline([
    ("tfidf", TfidfVectorizer()),
    ("classifier", MultinomialNB(alpha=0.1)),
])
```

TF-IDF 값은 음수가 아니므로 MultinomialNB와 함께 사용할 수 있습니다. TF-IDF + MultinomialNB 조합은 텍스트 분류를 위한 가장 강력한 베이스라인 중 하나입니다. 10,000개 미만의 훈련 샘플을 가진 데이터셋에서 더 복잡한 모델을 자주 능가합니다.

### 짧은 텍스트를 위한 BernoulliNB

짧은 텍스트(트윗, SMS, 채팅 메시지)의 경우 BernoulliNB가 MultinomialNB보다 더 나은 성능을 보일 수 있습니다. 짧은 텍스트는 단어 빈도가 낮아 MultinomialNB가 의존하는 빈도 정보가 노이즈가 많습니다. BernoulliNB는 단어의 존재 여부만 고려하므로 짧은 텍스트에서 더 신뢰할 수 있습니다.

```python
from sklearn.naive_bayes import BernoulliNB
from sklearn.feature_extraction.text import CountVectorizer

text_clf = Pipeline([
    ("vectorizer", CountVectorizer(binary=True)),
    ("classifier", BernoulliNB(alpha=1.0)),
])
```

CountVectorizer의 `binary=True` 플래그는 모든 빈도를 0/1로 변환합니다. 이 플래그가 없으면 BernoulliNB는 설계되지 않은 빈도 값을 보게 됩니다.

### NB 확률 보정

NB 확률은 보정이 잘 되지 않습니다. NB가 P(spam) = 0.95라고 해도 실제 확률은 0.7일 수 있습니다. 신뢰할 수 있는 확률 추정치가 필요한 경우(예: 임계값 설정 또는 다른 모델과 결합) sklearn의 CalibratedClassifierCV를 사용하세요:

```python
from sklearn.calibration import CalibratedClassifierCV

calibrated_nb = CalibratedClassifierCV(MultinomialNB(), cv=5, method="sigmoid")
calibrated_nb.fit(X_train, y_train)
proba = calibrated_nb.predict_proba(X_test)
```

이것은 교차 검증을 사용하여 NB의 원시 점수에 로지스틱 회귀를 적합합니다. 결과 확률은 실제 클래스 빈도에 훨씬 더 가깝습니다.

### 일반적인 주의 사항

1. **음수 특성 값.** MultinomialNB는 음수가 아닌 특성 값을 요구합니다. 음수 값(예: 특정 설정의 TF-IDF 또는 표준화된 특성)이 있는 경우 GaussianNB를 사용하거나 특성 값을 양수로 이동시키세요.

2. **분산 0 특성.** GaussianNB는 분산으로 나눕니다. 특정 클래스에 대해 분산이 0인 특성(모든 값이 동일)이 있으면 확률 계산이 실패합니다. 코드는 모든 분산에 작은 스무딩 항(1e-9)을 추가하여 이를 방지합니다.

3. **클래스 불균형.** 이메일의 99%가 스팸이 아닌 경우, 사전 확률 P(not-spam) = 0.99가 우도 증거를 압도할 수 있습니다. 클래스 사전 확률을 수동으로 설정하거나 sklearn의 class_prior 매개변수를 사용할 수 있습니다.

4. **특성 스케일링.** MultinomialNB는 스케일링이 필요 없습니다(빈도에서 작동). GaussianNB도 스케일링이 필요 없습니다(특성별 통계를 추정). 이는 특성 스케일에 민감한 로지스틱 회귀 및 SVM에 비해 장점입니다.

### 나이브 베이즈가 실패하는 경우

NB는 독립성 가정이 잘못된 순위(단순 확률 오류가 아닌)를 유발할 때 실패합니다. 이는 다음 경우에 발생합니다:

1. **강한 특징 상호작용.** 클래스가 두 특징의 조합에 의존하지만 각각 단독으로는 의존하지 않는 경우(XOR 유사 패턴), NB는 이를 완전히 놓칩니다. 각 특징 단독으로는 증거를 제공하지 않으며, NB는 비선형적으로 결합할 수 없습니다.

2. **상반된 증거를 가진 높은 상관관계 특징.** 특징 A가 "스팸"을 나타내고 특징 B가 "스팸 아님"을 나타내지만, A와 B가 완벽하게 상관되어 있는 경우(실제로는 항상 동의), NB는 존재하지 않는 상충되는 증거를 보게 됩니다.

3. **매우 큰 훈련 데이터 세트.** 충분한 데이터가 있으면 로지스틱 회귀와 같은 판별 모델이 실제 결정 경계를 학습하고 NB를 능가합니다. 소량의 데이터에 도움이 되었던 독립성 가정이 이제 모델을 제한합니다.

실제로 텍스트 분류에서는 이러한 실패 모드가 드뭅니다. 텍스트 특징은 많고, 개별적으로 약하며, 독립성 가정의 오류가 상쇄되는 경향이 있습니다. 강하게 상관된 소수의 특징을 가진 표 형식 데이터의 경우, 먼저 로지스틱 회귀 또는 트리 기반 모델을 고려하세요.

## 주요 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|----------------|----------------------|
| 나이브 베이즈(Naive Bayes) | "간단한 확률적 분류기" | 클래스 주어진 조건에서 특징들이 조건부 독립이라는 가정 하에 베이즈 정리를 적용하는 분류기 |
| 조건부 독립(Conditional independence) | "특징들이 서로 영향을 주지 않음" | P(A, B \| C) = P(A \| C) * P(B \| C) -- C를 알면 B가 A에 대해 새로운 정보를 제공하지 않음 |
| 라플라스 평활화(Laplace smoothing) | "Add-one smoothing" | 예측에서 0 확률이 지배적으로 작용하는 것을 방지하기 위해 모든 특징에 작은 카운트를 추가 |
| 사전 확률(Prior) | "데이터를 보기 전에 믿었던 것" | P(class) -- 어떤 특징도 관측하기 전 각 클래스의 확률 |
| 가능도(Likelihood) | "데이터가 얼마나 잘 맞는지" | P(features \| class) -- 클래스가 주어졌을 때 이 특징들을 관측할 확률 |
| 사후 확률(Posterior) | "데이터를 본 후 믿는 것" | P(class \| features) -- 특징들을 관측한 후 업데이트된 클래스 확률 |
| 생성 모델(Generative model) | "데이터가 어떻게 생성되는지 모델링" | P(X \| Y)와 P(Y)를 학습한 후 베이즈 정리를 사용해 P(Y \| X)를 얻는 모델 |
| 판별 모델(Discriminative model) | "결정 경계를 모델링" | X가 어떻게 생성되는지 모델링하지 않고 직접 P(Y \| X)를 학습하는 모델 |
| 로그 확률(Log probability) | "언더플로 방지" | 많은 작은 수의 곱이 부동소수점에서 0이 되는 것을 방지하기 위해 P 대신 log P를 사용 |
