# 비디오 생성

> 이미지는 2-D 텐서입니다. 비디오는 3-D 텐서입니다. 이론은 동일하지만 계산량은 10-100배 더 어렵습니다. OpenAI의 Sora(2024년 2월)는 그것이 가능함을 입증했습니다. 2026년까지 Veo 2, Kling 1.5, Runway Gen-3, Pika 2.0, WAN 2.2는 텍스트로부터 1080p 해상도의 프로덕션 비디오를 출시할 것이며, 오픈소스 가중치 스택(CogVideoX, HunyuanVideo, Mochi-1, WAN 2.2)은 12개월 뒤처져 있습니다.

**유형:** 구축(Build)
**언어:** Python
**선수 지식:** Phase 8 · 07 (잠재 확산), Phase 7 · 09 (ViT), Phase 8 · 06 (DDPM)
**소요 시간:** ~45분

## 문제 정의

24fps의 10초 1080p 비디오는 1920×1080×3 픽셀 크기의 프레임 240개로 구성됩니다. 이는 클립당 약 1.5GB의 원시 데이터에 해당합니다. 픽셀 공간에서의 확산(diffusion)은 실행 불가능합니다. 다음 요구사항이 필요합니다:

1. **시공간 압축(Spatiotemporal compression).** 프레임이 아닌 비디오를 시공간 패치 시퀀스로 인코딩하는 VAE(Variational Autoencoder).
2. **시간적 일관성(Temporal coherence).** 프레임들은 수 초 동안 콘텐츠, 조명, 객체 식별성을 공유해야 합니다. 네트워크는 모션을 모델링할 수 있어야 합니다.
3. **계산 예산(Compute budget).** 동일한 모델 크기 기준으로 비디오 훈련은 이미지 대비 10-100배 더 비용이 듭니다.
4. **조건부 입력(Conditioning).** 텍스트, 이미지(첫 프레임), 오디오 또는 다른 비디오. 대부분의 프로덕션 모델은 이 네 가지 모두를 허용합니다.

이 문제를 해결한 아키텍처는 **시공간 패치에 적용된 확산 트랜스포머(Diffusion Transformer, DiT)**로, 대규모 (프롬프트, 캡션, 비디오) 데이터셋으로 훈련됩니다. 레슨 06과 동일한 확산 손실(diffusion loss)을 사용합니다.

## 개념

![비디오 디퓨전: 패치화, DiT, 디코딩](../assets/video-generation.svg)

### 패치화(Patchify)

3D VAE(학습된 시공간 압축)를 사용하여 비디오를 인코딩합니다. 잠재 공간(latent)의 형태는 `[T_latent, H_latent, W_latent, C_latent]`입니다. 크기 `[t_p, h_p, w_p]`의 패치로 분할합니다. Sora 스타일 모델의 경우 `t_p = 1`(프레임별 패치) 또는 `t_p = 2`(2프레임 간격)입니다. 10초 분량의 1080p 비디오는 약 20,000~100,000개의 패치로 압축됩니다.

### 시공간 DiT(Spatiotemporal DiT)

트랜스포머가 평탄화된(flat) 패치 시퀀스를 처리합니다. 각 패치에는 3D 위치 임베딩(시간 + y + x)이 부여됩니다. 어텐션은 일반적으로 다음과 같이 분해됩니다:

- **공간 어텐션(Spatial attention)**: 각 프레임 내 패치 간.
- **시간 어텐션(Temporal attention)**: 동일 공간 위치의 프레임 간.
- **전체 3D 어텐션(Full 3D attention)**: 16~100배 더 비싸며, 저해상도 또는 연구용으로만 사용됩니다.

### 텍스트 조건화(Text conditioning)

대규모 텍스트 인코더(T5-XXL for Sora, CogVideoX-5B도 T5-XXL 사용)와의 크로스 어텐션. 긴 프롬프트가 중요합니다 — Sora의 학습 데이터에는 클립당 평균 200토큰의 GPT 생성 밀집 재캡션이 포함되었습니다.

### 학습(Training)

시공간 잠재 공간에 대한 표준 디퓨전 손실(ε 또는 v 예측). 데이터: 웹 비디오 + ~1억 개의 선별된 클립 + 합성 텍스트 캡션. 컴퓨팅: 소규모 연구 실행에도 10,000+ GPU 시간 소요; Sora 규모는 100,000+입니다.

## 빌드하기

`code/main.py`는 핵심 시공간 DiT 아이디어를 시뮬레이션합니다: 작은 합성 비디오를 패치화하고, 패치별 위치 임베딩을 추가한 후, 패치 간 트랜스포머 스타일 어텐션을 통해 전체 시퀀스를 디노이즈합니다. NumPy 없이 순수 Python으로 구현되었습니다. 인접 프레임 패치가 디노이저와 위치 임베딩을 공유할 때 1차원에서도 시간적 일관성이 나타남을 보여줍니다.

### 단계 1: 합성 1-D "비디오" 패치화

```python
def make_video(T_frames=8, rng=None):
    # "비디오"는 부드러운 궤적을 따르는 1-D 값들의 시퀀스입니다
    base = rng.gauss(0, 1)
    return [base + 0.3 * t + rng.gauss(0, 0.1) for t in range(T_frames)]
```

### 단계 2: 프레임별 위치 임베딩

```python
def pos_embed(t, dim):
    return sinusoidal(t, dim)
```

### 단계 3: 디노이저가 전체 시퀀스 처리

각 프레임을 독립적으로 디노이즈하는 대신, 이 소형 네트워크는 모든 프레임 값 + 해당 위치 임베딩을 연결(concatenate)하고 모든 프레임에 대한 노이즈를 공동으로 예측합니다.

### 단계 4: 시간적 일관성 테스트

훈련 후 비디오를 샘플링합니다. 프레임 간 델타(delta)를 측정합니다. 모델이 시간적 구조를 학습했다면, 델타는 각 프레임을 독립적으로 샘플링한 경우보다 작게 유지됩니다.

## 주의 사항

- **프레임별 독립 샘플링 = 깜빡임.** 각 프레임을 별도로 이미지 디퓨전에 적용하면 프레임별 노이즈가 독립적이기 때문에 출력이 깜빡입니다. 비디오 디퓨전은 어텐션 또는 공유 노이즈를 통해 프레임을 결합함으로써 이 문제를 해결합니다.
- **순진한 3D 어텐션 = 메모리 부족(OOM).** 10초 길이의 1080p 잠재 공간에 대한 전체 3D 어텐션은 수백 조 번의 연산이 필요합니다. 공간 + 시간 차원으로 분해(factorize)해야 합니다.
- **데이터 캡셔닝은 양보다 질이 중요.** Sora의 주요 개선점은 이전 연구 대비 ~10배 더 상세한 캡션(GPT-4가 클립 재라벨링)으로 학습한 것입니다. OpenAI 기술 보고서에서도 이를 명시적으로 언급했습니다.
- **첫 프레임 조건화.** 대부분의 프로덕션 모델은 첫 프레임으로 이미지를 입력받기도 합니다. 이는 "이미지-투-비디오" 모드이며, 학습 시 이 변형(variant)이 포함됩니다.
- **물리 법칙 드리프트.** 긴 클립(>10초)은 미묘한 불일치가 누적됩니다. 슬라이딩 윈도우 생성 + 키프레임 앵커링이 도움이 됩니다.

## 프로덕션 노트: 비디오 잠재 공간은 메모리 대역폭 문제

10초 길이의 1080p 클립(24fps)은 240프레임 × 1920 × 1080 × 3 ≈ 1.5GB의 원시 픽셀로 구성됩니다. 4× 비디오 VAE 압축(`2 × 공간 × 2 × 시간`) 후 잠재 공간은 요청당 약 100MB가 됩니다. 이 데이터를 배치 크기 1로 30단계 동안 시공간 DiT에 통과시키면 HBM을 통해 ~3GB/step의 데이터가 이동하게 됩니다. 이때 병목 현상은 FLOPs가 아닌 메모리 대역폭입니다.

프로덕션 추론 문헌에서 직접 가져온 세 가지 핵심 조절 장치:

- **DiT에 걸친 TP(Tensor Parallelism).** 텍스트-비디오 모델은 일반적으로 ≥10B 파라미터를 가집니다. 4개의 H100에 TP=4를 적용하는 것이 표준이며, 405B급 모델의 경우 PP=2 × TP=2를 사용합니다. 단계별 지연 시간은 TP 증가에 따라 대략 선형적으로 감소하지만, all-reduce 통신 오버헤드가 한계 요인입니다.
- **프레임 배치 처리 = 연속 배치 처리.** 생성 단계에서 비디오는 어텐션으로 연결된 프레임 배치로 개념화됩니다. 연속 배치 처리(비행 중 스케줄링)가 적용 가능합니다: 모델 아키텍처가 슬라이딩 윈도우 생성을 지원하는 경우, 프레임 `t-1`이 반환되는 동안 프레임 `t+1` 렌더링을 시작할 수 있습니다.
- **클립 수준 프리필 캐시.** 이미지-비디오 생성에서 첫 프레임 조건화는 LLM의 프롬프트 프리필과 유사합니다: 한 번 계산한 후 시간 디코더 패스에서 재사용합니다. 이는 비디오를 위한 KV-캐시와 동일한 효과를 가집니다.
