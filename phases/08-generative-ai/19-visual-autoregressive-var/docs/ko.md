# 시각적 자기회귀 모델링(VAR: Visual Autoregressive Modeling): 다음 스케일 예측

> 확산 모델(diffusion models)은 시간적으로 반복적으로 샘플링합니다(노이즈 제거 단계). VAR은 스케일 단위로 반복적으로 샘플링합니다 — 1x1 토큰을 예측한 다음 2x2, 4x4로 최종 해상도까지 예측하며, 각 스케일은 이전 스케일에 조건부로 의존합니다. 2024년 논문에서는 VAR이 이미지 생성에 대해 GPT 스타일의 스케일링 법칙(scaling laws)을 따르며, 동일한 계산 예산에서 DiT를 능가함을 보였습니다. 이 레슨에서는 핵심 메커니즘을 구축합니다.

**유형:** 구축(Build)
**언어:** Python (PyTorch 사용)
**선수 지식:** 7단계 레슨 03(멀티헤드 어텐션), 8단계 레슨 06(DDPM)
**소요 시간:** ~90분

## 문제 정의

자기회귀 생성은 예측 가능한 확장성으로 인해 언어 모델링을 지배했습니다: 더 많은 계산 자원, 더 많은 파라미터, 더 낮은 퍼플렉서티(perplexity), 더 나은 출력. 이미지 생성 분야에서는 2024년 이전에 두 가지 주요 AR 시도가 있었습니다: 픽셀 단위(PixelRNN/PixelCNN) 및 VQ-VAE 코드 토큰 단위(DALL-E 1 / Parti / MuseGAN) 접근 방식.

둘 다 생성 순서 문제로 어려움을 겪었습니다. 픽셀과 토큰은 2D 그리드로 배열되어 있지만, AR 모델은 이를 1D 래스터 순서로 방문해야 합니다. 초기 코너 픽셀은 이미지가 최종적으로 어떻게 될지 전혀 알 수 없습니다. 생성 품질은 GPT-on-text보다 확장성이 떨어졌으며, 동일한 계산 자원으로는 확산 모델(diffusion model) 품질에 도달하지 못했습니다.

VAR은 생성하는 대상을 변경함으로써 생성 순서 문제를 해결합니다. 공간적으로 하나씩 이미지 토큰을 예측하는 대신, VAR은 점진적으로 해상도를 높이며 전체 이미지를 예측합니다. 1단계: 1x1 토큰(전체 이미지 "요약") 예측. 2단계: 2x2 그리드 토큰(더 거친 특징) 예측. 3단계: 4x4 그리드 예측. K단계: 최종 (H/8)x(W/8) 그리드 예측.

각 스케일은 이전 모든 스케일("스케일 순서"에서 인과적으로)에 어텐션(attention)을 수행하며, 동일 스케일 내에서는 병렬 처리됩니다. 순서 문제가 사라집니다: 스케일 k의 전체 이미지는 단일 트랜스포머(Transformer) 패스로 생성됩니다.

## 개념

### VQ-VAE 다중 스케일 토크나이저

VAR는 **다중 스케일 이산 토크나이저**가 필요합니다. 이미지 x에 대해 점진적으로 해상도가 높아지는 토큰 그리드 시퀀스를 생성합니다:

```
x -> 인코더 -> 잠재 표현 f
f -> 1x1 토크나이징: (1, 1) 형태의 토큰 그리드 z_1
f -> 2x2 토크나이징: (2, 2) 형태의 토큰 그리드 z_2
...
f -> (H/p)x(W/p) 토크나이징: (H/p, W/p) 형태의 토큰 그리드 z_K
```

각 z_k는 동일한 코드북(일반적인 크기 4096-16384)을 사용합니다. 각 스케일의 토크나이징은 독립적이지 않으며, 모든 스케일에서 잔차를 합산하여 f를 재구성하도록 훈련됩니다:

```
f ≈ upsample(embed(z_1), target_size) + ... + upsample(embed(z_K), target_size)
```

이것은 **잔차 VQ** 변형입니다. 스케일 k는 스케일 1..k-1이 놓친 부분을 포착합니다. 디코더는 모든 스케일 임베딩의 합을 입력으로 받아 이미지를 생성합니다.

다중 스케일 VQ 토크나이저는 한 번 훈련된 후 고정됩니다(VQGAN과 유사). 모든 생성 작업은 상위 오토리그레시브 모델에서 수행됩니다.

### 다음 스케일 예측

생성 모델은 모든 이전 스케일의 토큰을 보고 다음 스케일의 토큰을 예측하는 트랜스포머입니다.

입력 시퀀스 구조:
```
[START, z_1 토큰, z_2 토큰, z_3 토큰, ..., z_K 토큰]
```

위치 임베딩은 스케일 인덱스와 스케일 내 공간 위치를 모두 인코딩합니다. 어텐션은 스케일 순서에서 인과적입니다: 스케일 k, 위치 (i, j)의 토큰은 스케일 1..k의 모든 토큰과 스케일 k 내에서 더 일찍 등장하는 토큰에 어텐션할 수 있습니다(VAR은 스케일 내 인과성이 없는 고정 위치 어텐션을 사용하며, 스케일 내 모든 위치는 병렬로 예측됩니다).

훈련 손실: 각 스케일 k에서 이전 스케일 토큰을 조건으로 z_k 토큰을 예측합니다. 이산 VQ 코드에 대한 교차 엔트로피 손실을 사용합니다. GPT와 동일한 구조이지만 "시퀀스"가 스케일 구조로 구성됩니다.

### 생성

추론 시:
```
z_1 = p(z_1)에서 샘플링                    # 1개 토큰
z_2 = p(z_2 | z_1)에서 샘플링              # 4개 토큰 병렬 생성
z_3 = p(z_3 | z_1, z_2)에서 샘플링         # 16개 토큰 병렬 생성
...
디코딩: f = 스케일 1..K의 임베딩 및 업샘플링 합
이미지 = VAE_디코더(f)
```

K = 10 스케일일 때 생성에는 10번의 트랜스포머 순방향 패스가 필요합니다. 각 패스는 해당 스케일의 모든 토큰을 병렬로 생성합니다 — 스케일 내 토큰별 오토리그레시브가 없습니다. 256x256 이미지의 경우 DiT의 28-50패스 대비 약 10패스로 효율적입니다.

### 다음 스케일 예측이 다음 토큰 예측을 능가하는 이유

세 가지 구조적 장점:
1. **거친 것에서 세밀한 것으로의 접근은 자연스러운 이미지 통계와 일치합니다.** 인간의 시각 인지와 이미지 데이터셋은 모두 스케일 의존적 규칙성을 나타냅니다: 저주파 구조는 안정적이고 예측 가능하며, 고주파 세부 사항은 저주파 콘텐츠에 조건적입니다. 다음 스케일 예측은 이를 활용합니다.
2. **스케일 내 병렬 생성.** GPT 스타일의 토큰 오토리그레시브와 달리 VAR은 한 단계에서 스케일의 모든 토큰을 생성합니다. 효과적인 생성 길이는 선형이 아닌 로그 스케일입니다.
3. **생성 순서 편향 없음.** 스케일 k의 토큰은 스케일 k-1의 모든 토큰을 참조합니다; "왼쪽" 또는 "위쪽" 편향이 없어 초기 토큰이 늦은 컨텍스트 없이 커밋할 필요가 없습니다.

### 스케일링 법칙

Tian 등은 VAR이 ImageNet에서 FID에 대해 GPT의 퍼플렉서티와 마찬가지로 멱법칙 스케일링 곡선을 따름을 입증했습니다. 파라미터나 계산량을 두 배로 늘리면 오류가 신뢰성 있게 절반으로 감소합니다. 이는 언어 모델만큼 깔끔하게 스케일링 동작을 보이는 최초의 이미지 생성 모델입니다. 결과적으로 VAR-스케일 예측은 경험적 추측이 아닌 계산량으로부터 예측 가능해집니다.

### 확산 모델과의 관계

VAR과 확산 모델은 동일한 데이터 압축 스토리를 공유합니다: 둘 다 생성 문제를 더 쉬운 하위 문제들의 시퀀스로 분해합니다.

- 확산 모델: 점진적으로 노이즈를 추가하고 한 단계를 되돌리는 방법을 학습합니다.
- VAR: 점진적으로 해상도를 추가하고 다음 스케일을 예측하는 방법을 학습합니다.

이들은 문제를 해결하는 서로 다른 축입니다. 둘 다 처리 가능한 조건부 분포를 제공합니다. 경험적으로 VAR은 추론 시 더 빠르며(더 적은 패스, 스케일 내 모든 병렬 처리) 클래스 조건부 ImageNet에서 DiT를 일치하거나 능가합니다. 텍스트 조건부 VAR(VARclip, HART)은 활발한 연구 분야입니다.

## 구축 단계

`code/main.py`에서는 다음을 수행합니다:
1. 합성 "이미지" 데이터(2D 가우시안 링) 위에 **다중 스케일 VQ 토크나이저**를 구축합니다.
2. **VAR 스타일 트랜스포머**를 훈련시켜 토큰의 다음 스케일을 예측합니다.
3. 트랜스포머를 4회 호출(4개 스케일)하여 샘플링하고 디코딩합니다.
4. 스케일 순서 훈련이 스케일 내에서의 병렬 생성을 가능하게 하는지 검증합니다.

이것은 장난감 구현입니다. 핵심은 스케일 구조화된 어텐션 마스크와 스케일 내에서의 병렬 생성이 실제로 작동하는 것을 확인하는 것입니다.

## Ship It

이 레슨은 `outputs/skill-var-tokenizer-designer.md`를 생성합니다 — 다중 스케일 토크나이저 설계를 위한 스킬: 스케일 수, 스케일 비율, 코드북 크기, 잔차 공유, 디코더 아키텍처.

## 연습 문제

1. **스케일 수 감소 실험.** 4, 6, 8, 10개의 스케일을 사용하여 VAR을 학습시킵니다. 재구성 품질 대 자기회귀적 패스 수를 측정합니다. 스케일이 많을수록 잔차가 더 세밀해져 품질은 향상되지만 패스 수가 증가합니다.

2. **코드북 크기.** 코드북 크기 512, 4096, 16384로 토크나이저를 학습시킵니다. 더 큰 코드북은 재구성 품질을 향상시키지만 예측이 더 어려워집니다. 최적의 균형점(knee)을 찾습니다.

3. **스케일 내 병렬성 검증.** 학습된 VAR에 대해 어텐션 패턴을 명시적으로 측정합니다. 스케일 k 내에서 모델이 스케일 간 위치에는 어텐션하지만 스케일 내 위치에는 어텐션하지 않는지 확인합니다. 마스크 구현을 검증합니다.

4. **VAR vs DiT 확장성 비교.** 동일한 ImageNet 클래스 조건부 작업에 대해, 파라미터 예산을 일치시킨(예: 33M, 130M, 458M) VAR과 DiT를 학습시킵니다. FID 대 계산 비용을 그래프로 그립니다. 각 크기에서 VAR이 DiT를 앞지르는지 확인합니다 — 소규모에서 논문의 결과를 재현합니다.

5. **텍스트 조건화.** VAR을 확장하여 CLIP 풀링된 텍스트 임베딩을 추가 조건 입력(adaLN 경로)으로 받도록 합니다. 이는 HART 레시피입니다. 텍스트 정렬 샘플링에서 FID가 얼마나 향상되는지 측정합니다.

## 핵심 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|----------------|----------------------|
| VAR | "Visual AutoRegressive" | VQ 토큰 그리드 피라미드를 통한 다음 스케일 예측 기반 이미지 생성 |
| 다음 스케일 예측 | "더 거친 스케일 예측 후 더 세밀한 스케일 예측" | 모델이 모든 이전 스케일을 조건으로 하여 증가하는 해상도 스케일에서 토큰을 예측 |
| 멀티 스케일 VQ 토크나이저 | "잔차 VQ" | K개의 증가하는 해상도 토큰 그리드를 생성하는 VQ-VAE, 디코더는 모든 스케일을 합산 |
| 스케일 k | "피라미드 레벨 k" | K개의 해상도 레벨 중 하나, k=1에서 1x1부터 k=K에서 (H/p)x(W/p)까지 |
| 스케일 내 병렬 처리 | "스케일당 한 번의 순전파" | 스케일 k의 모든 토큰은 자동회귀적이지 않고 하나의 트랜스포머 패스에서 예측 |
| 스케일 간 인과성 | "스케일 순서 어텐션" | 스케일 k의 토큰은 스케일 1..k에 어텐션할 수 있지만 스케일 k+1..K에는 어텐션 불가 |
| 잔차 VQ | "가산적 토크나이징" | 각 스케일의 토큰은 하위 스케일에서 남은 잔차를 인코딩, 디코더는 모든 스케일 임베딩을 합산 |
| VAR 스케일링 법칙 | "이미지 GPT 스케일링" | FID는 언어 모델의 퍼플렉시티와 같이 계산량에 따라 예측 가능한 멱법칙을 따름 |
| HART | "하이브리드 VAR + 텍스트" | VAR의 스케일 구조와 MaskGIT 스타일 반복적 디코딩을 결합한 텍스트 조건부 VAR 변형 |
| 스케일 위치 임베딩 | "(스케일, 행, 열) 트리플" | 위치 인코딩은 스케일 인덱스와 스케일 내 공간 좌표를 모두 포함 |
