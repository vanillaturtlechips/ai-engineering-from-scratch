# 확산 모델 — DDPM 처음부터 구현하기

> Ho, Jain, Abbeel (2020)은 이 분야가 포기할 수 없는 레시피를 제공했습니다. 천 번의 작은 단계를 거쳐 데이터에 노이즈를 추가합니다. 하나의 신경망을 훈련시켜 노이즈를 예측하게 합니다. 추론 시 이 과정을 역으로 실행합니다. 오늘날 모든 주류 이미지, 비디오, 3D, 음악 모델은 이 루프를 기반으로 실행되며, 경우에 따라 플로우 매칭이나 일관성 기법이 추가됩니다.

**유형:** 구축
**언어:** Python
**선수 지식:** Phase 3 · 02 (역전파), Phase 8 · 02 (VAE)
**소요 시간:** ~75분

## 문제 정의

`p_data(x)`에 대한 샘플러를 원합니다. GAN은 종종 발산하는 최소최대 게임을 수행합니다. VAE는 가우시안 디코더로부터 흐릿한 샘플을 생성합니다. 실제로 원하는 것은 (a) 단일 안정 손실(새들 포인트 없음, 최소최대 없음), (b) `log p(x)`의 하한(가능도 확보), (c) SOTA 품질과 일치하는 샘플을 제공하는 훈련 목적 함수입니다.

Sohl-Dickstein et al. (2015)은 이론적 해답을 제시했습니다: 가우시안 노이즈를 점진적으로 추가하는 마르코프 체인 `q(x_t | x_{t-1})`을 정의하고, 노이즈를 제거하는 역방향 체인 `p_θ(x_{t-1} | x_t)`를 훈련시킵니다. Ho, Jain, Abbeel (2020)은 손실 함수를 노이즈 예측이라는 한 줄로 단순화하고 수학적 정리를 완성했습니다. 2020년에는 이 방법이 단순한 호기심 대상이었습니다. 2021년에는 최첨단(SOTA) 샘플을 생성했습니다. 2022년에는 Stable Diffusion이 되었습니다. 2026년에는 기반이 되는 기술이 되었습니다.

## 개념

![DDPM: 순방향 노이즈, 역방향 디노이즈](../assets/ddpm.svg)

**순방향 과정 `q`.** `T`개의 작은 단계로 가우시안 노이즈를 추가합니다. 누적 단계가 가우시안 분포를 따른다는 닫힌 형태(closed form)가 수학적 계산의 핵심입니다:

```
q(x_t | x_0) = N( sqrt(α̅_t) · x_0,  (1 - α̅_t) · I )
```

여기서 `α̅_t = ∏_{s=1..t} (1 - β_s)`이며, `β_t`는 1e-4에서 0.02까지 선형적으로 증가하는 스케줄입니다. T=1000 단계로 설정하면 `x_T`는 대략 `N(0, I)`에 근접합니다.

**역방향 과정 `p_θ`.** 추가된 노이즈를 예측하는 신경망 `ε_θ(x_t, t)`를 학습합니다. `x_t`가 주어졌을 때 디노이징은 다음 식으로 수행됩니다:

```
x_{t-1} = (1 / sqrt(α_t)) · ( x_t - (β_t / sqrt(1 - α̅_t)) · ε_θ(x_t, t) )  +  σ_t · z
```

여기서 `σ_t`는 `sqrt(β_t)`이거나 학습된 분산입니다. 수식은 복잡해 보이지만, 사후 분포 `q(x_{t-1} | x_t, x_0)`를 풀고 `x_0`를 노이즈 예측값으로 치환한 대수적 과정입니다.

**훈련 손실.**

```
L_simple = E_{x_0, t, ε} [ || ε - ε_θ( sqrt(α̅_t) · x_0 + sqrt(1 - α̅_t) · ε,  t ) ||² ]
```

데이터에서 `x_0`를 샘플링하고, 무작위 `t`를 선택한 후 `ε ~ N(0, I)`를 샘플링합니다. 닫힌 형태를 통해 한 번에 노이즈가 추가된 `x_t`를 계산하고, 노이즈를 회귀 예측합니다. 단일 손실 함수이며, minimax, KL 발산, 재매개변수화 기법이 필요 없습니다.

**샘플링.** `x_T ~ N(0, I)`로 시작합니다. `t = T`부터 `1`까지 역방향 단계를 반복합니다. 완료.

## 왜 작동하는가

세 가지 직관:

1. **노이즈 제거는 쉽고, 생성은 어렵다.** `t=T`에서 데이터는 순수한 노이즈 — 네트워크는 사소한 문제만 해결하면 된다. `t=0`에서 네트워크는 몇 개의 픽셀만 정리하면 된다. 중간 `t`에서는 문제가 어렵지만 네트워크는 모든 노이즈 레벨에서 동일한 가중치를 통해 많은 그래디언트를 얻는다.

2. **변장한 스코어 매칭.** Vincent (2011)은 노이즈 예측이 `∇_x log q(x_t | x_0)`, 즉 *스코어* 추정과 동등함을 증명했다. 역방향 SDE는 이 스코어를 사용해 밀도 기울기를 따라 걷는다 — 높은 확률 영역을 향한 유도된 랜덤 워크이다.

3. **ELBO는 단순한 MSE로 축소된다.** 전체 변분 하한(ELBO)은 타임스텝마다 KL 항을 가진다. DDPM의 파라미터화에서는 이 KL 항들이 특정 계수를 가진 노이즈 예측에 대한 MSE로 단순화된다; Ho는 계수를 생략("단순" 손실이라 부름)했고 품질이 *향상*되었다.

## 구축 방법

`code/main.py`는 1-D DDPM을 구현합니다. 데이터는 두 모드 혼합입니다. "net"은 `(x_t, t)`를 입력으로 받아 예측 노이즈를 출력하는 소형 MLP입니다. 훈련은 한 줄짜리 손실 함수로 이루어집니다. 샘플링은 역방향 체인을 반복합니다.

### 단계 1: 순방향 스케줄 (닫힌 형식)

```python
betas = [1e-4 + (0.02 - 1e-4) * t / (T - 1) for t in range(T)]
alphas = [1 - b for b in betas]
alpha_bars = []
cum = 1.0
for a in alphas:
    cum *= a
    alpha_bars.append(cum)
```

### 단계 2: 한 번에 `x_t` 샘플링

```python
def forward_sample(x0, t, alpha_bars, rng):
    a_bar = alpha_bars[t]
    eps = rng.gauss(0, 1)
    x_t = math.sqrt(a_bar) * x0 + math.sqrt(1 - a_bar) * eps
    return x_t, eps
```

### 단계 3: 한 번의 훈련 단계

```python
def train_step(x0, model, alpha_bars, rng):
    t = rng.randrange(T)
    x_t, eps = forward_sample(x0, t, alpha_bars, rng)
    eps_hat = model_forward(model, x_t, t)
    loss = (eps - eps_hat) ** 2
    return loss, gradient_step(model, ...)
```

### 단계 4: 역방향 샘플링

```python
def sample(model, alpha_bars, T, rng):
    x = rng.gauss(0, 1)
    for t in range(T - 1, -1, -1):
        eps_hat = model_forward(model, x, t)
        beta_t = 1 - alphas[t]
        x = (x - beta_t / math.sqrt(1 - alpha_bars[t]) * eps_hat) / math.sqrt(alphas[t])
        if t > 0:
            x += math.sqrt(beta_t) * rng.gauss(0, 1)
    return x
```

40 타임스텝과 24유닛 MLP를 사용한 1-D 문제에서 이 방법은 약 200 에포크 만에 두 모드 혼합을 학습합니다.

## 시간 조건화

네트는 어떤 타임스텝을 디노이징하고 있는지 알아야 합니다. 두 가지 표준 옵션:

- **사인 임베딩(sinusoidal embedding).** 트랜스포머 위치 인코딩과 유사. `embed(t) = [sin(t/ω_0), cos(t/ω_0), sin(t/ω_1), ...]`. MLP를 통과시킨 후 네트에 브로드캐스트.
- **FiLM / 그룹 정규화 조건화.** 각 블록에서 임베딩을 채널별 스케일/바이어스(FiLM)로 투영.

토이 코드는 사인 임베딩 → 연결(concat)을 사용합니다. 프로덕션 U-Net은 FiLM을 사용합니다.

## 함정(Pitfalls)

- **스케줄(schedule)이 매우 중요.** 선형 `β`는 DDPM의 기본 설정이지만 코사인 스케줄(Nichol & Dhariwal, 2021)이 동일한 계산 비용으로 더 나은 FID 점수를 제공. 품질이 정체되면 스케줄을 변경.
- **타임스텝 임베딩(timestep embedding)이 취약.** 원시 `t`를 부동소수점으로 전달하는 것은 장난감 1-D 예제에서는 작동하지만 이미지에서는 실패. 항상 적절한 임베딩(embedding)을 사용.
- **V-예측(v-prediction) vs ε-예측(ε-prediction).** 좁은 영역(매우 작거나 매우 큰 t)에서 `ε`는 신호 대 잡음비(signal-to-noise ratio)가 낮음. V-예측(`v = α·ε - σ·x`)이 더 안정적. SDXL, SD3, Flux에서 사용.
- **분류기 없는 가이드(classifier-free guidance).** 추론(inference) 시 조건부(conditional)와 비조건부(unconditional) `ε`를 모두 계산한 후, `ε_cfg = (1 + w) · ε_cond - w · ε_uncond`를 적용(`w ≈ 3-7`). 레슨 08에서 다룸.
- **1000단계는 과도함.** 프로덕션에서는 DDIM(20-50단계), DPM-Solver(10-20단계), 또는 증류(distillation, 1-4단계)를 사용. 레슨 12 참조.

| DDIM | "결정적 샘플러" | 비-마르코프 샘플러, 20-50단계, 동일한 학습 목적. |
| 분류기-프리 가이던스 | "CFG" | 조건부 및 무조건부 노이즈 예측을 혼합하여 조건화를 증폭. |

## 프로덕션 노트: 확산 모델 추론은 단계 수 문제

DDPM 논문은 T=1000 역방향 단계를 실행합니다. 프로덕션 환경에서는 아무도 그렇게 하지 않습니다. 모든 실제 추론 스택은 다음 세 가지 전략 중 하나를 선택하며, 각각은 "지연 시간이 어디서 발생하는가?"라는 프로덕션 프레임에 깔끔하게 매핑됩니다:

1. **더 빠른 샘플러, 동일한 모델.** DDIM (20-50 단계), DPM-Solver++ (10-20), UniPC (8-16). 역방향 루프를 직접 교체하며, 학습된 `ε_θ` 가중치는 변경되지 않습니다. 지연 시간을 20-50배 줄입니다.
2. **지식 증류.** 더 적은 단계로 교사 모델을 모방하도록 학생 모델을 학습: Progressive Distillation (2 → 1), Consistency Models (임의의 단계 → 1-4), LCM, SDXL-Turbo, SD3-Turbo. 지연 시간을 추가로 5-10배 줄이지만 재학습이 필요합니다.
3. **캐싱 및 컴파일.** `torch.compile(unet, mode="reduce-overhead")`, TensorRT-LLM의 확산 백엔드, `xformers`/SDPA 어텐션, bf16 가중치. 단계별 지연 시간을 ~2배 줄입니다. (1)과 (2)와 함께 사용 가능합니다.

프로덕션 확산 모델 서버의 예산 논의는 LLM에 대해 프로덕션 문헌에서 설명하는 것과 동일합니다: 지연 시간은 `num_steps × step_cost + VAE_decode`이고, 처리량은 `batch_size × (num_steps × step_cost)^-1`입니다. TTFT(첫 토큰 대기 시간)는 작습니다(한 단계). TPOT(전체 처리 시간)에 상응하는 것은 이미지 생성이 사용자 관점에서 "한 번에" 이루어지기 때문에 전체 응답 시간입니다.
