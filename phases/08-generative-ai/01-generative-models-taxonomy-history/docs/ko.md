# 생성 모델 — 분류 체계 및 역사

> 모든 이미지 모델, 텍스트 모델, 비디오 모델, 3D 모델은 다섯 가지 범주 중 하나에 속합니다. 잘못된 범주를 선택하면 수 주 동안 수학적 문제와 씨름하게 될 것입니다. 올바른 범주를 선택하면 해당 분야의 지난 12년 간의 발전이 머릿속에 깔끔하게 정리될 것입니다.

**유형:** 학습
**언어:** Python
**선수 지식:** 2단계(ML 기초), 3단계(딥러닝 핵심), 7·14단계(트랜스포머)
**소요 시간:** ~45분

## 문제 정의

생성 모델은 하나의 작업을 수행합니다: 알려지지 않은 분포 `p_data(x)`에서 추출된 훈련 샘플을 기반으로, 동일한 분포에서 나온 것처럼 보이는 새로운 샘플을 출력합니다. 얼굴, 문장, MIDI 파일, 단백질 구조 등 — 모든 문제는 자세히 보면 동일합니다.

문제는 `p_data`가 수백만 차원의 공간에 존재한다는 점입니다(512x512 RGB 이미지는 약 786k 차원). 샘플들은 그 공간 내부의 얇은 매니폴드(manifold) 위에 놓여 있으며, 사용 가능한 예시는 약 10M개 정도입니다. 밀도를 무차별적으로 추정하는 것은 불가능합니다. 모든 생성 모델은 하나의 어려운 문제를 약간 덜 어려운 문제로 대체하는 절충안입니다.

지난 12년 동안 살아남은 5가지 모델 계열이 있습니다. 각 계열이 어떤 절충을 하는지 알면, 왜 특정 작업에서는 우수한 성능을 내지만 다른 작업에서는 실패하는지를 이해할 수 있습니다.

## 개념

![생성 모델의 5가지 계열 — 모델링 대상에 따른 분류](../assets/taxonomy.svg)

**1. 명시적 밀도, 계산 가능.** `log p(x)`를 실제로 평가할 수 있는 합으로 표현. 오토리그레시브 모델(PixelCNN, WaveNet, GPT)은 `p(x) = ∏ p(x_i | x_<i)`로 분해. 정규화 흐름(RealNVP, Glow)은 간단한 기저의 가역적 변환으로 `p(x)`를 구축. 장점: 정확한 우도, 명확한 훈련 손실. 단점: 오토리그레시브 추론은 순차적(긴 시퀀스에서 느림), 흐름은 가역적 아키텍처 필요(구조적 제약).

**2. 명시적 밀도, 근사.** `log p(x)`를 하한(ELBO)으로 바운드하고 이 바운드를 최적화. VAE(Kingma 2013)는 변분 사후분포를 사용하는 인코더-디코더 구조. 확산 모델(DDPM, Ho 2020)은 가중 ELBO를 암묵적으로 최적화하는 노이즈 제거기를 훈련. 확산 모델은 2026년 기준 이미지, 비디오, 3D 분야의 핵심 백본.

**3. 암시적 밀도.** 밀도 함수 자체를 생략하고 샘플 생성기 `G(z)`와 진위 판별기 `D(x)`를 학습. GAN(Goodfellow 2014). 추론 시 빠름(단일 순전파)이지만 훈련 시 불안정성으로 악명. StyleGAN 1/2/3는 2026년에도 고정 도메인(예: 얼굴, 침실) 사진 현실감 분야에서 최정상.

**4. 스코어 기반 / 연속 시간.** 로그 밀도의 그래디언트 `∇_x log p(x)`(스코어)를 직접 학습. Song & Ermon(2019)은 스코어 매칭이 확산 모델을 SDE로 일반화함을 증명. 플로우 매칭(Lipman 2023)은 2024-2026년 핵심 트렌드: 시뮬레이션 없는 훈련, 직선적 경로, DDPM 대비 4-10배 빠른 샘플링. Stable Diffusion 3, Flux, AudioCraft 2가 플로우 매칭 사용.

**5. 이산 코드 기반 오토리그레시브.** VQ-VAE 또는 잔차 양자화기로 고차원 데이터를 짧은 이산 토큰 시퀀스로 압축한 후, 트랜스포머로 토큰 시퀀스를 모델링. Parti, MuseNet, AudioLM, VALL-E, Sora의 패치 토크나이저가 이 방식 사용. 이는 버킷 1에 학습형 토크나이저를 추가한 형태.

## 5문항 트라이아지

새로운 생성 모델 논문이 나왔을 때, 방법론 섹션을 읽기 전에 이 5가지 질문에 답해 보세요.

1. **무엇을 모델링하는가?** 픽셀, 잠재 변수(latents), 이산 토큰(discrete tokens), 3D 가우시안(3D Gaussians), 메시(meshes), 파형(waveforms)?
2. **밀도(density)는 명시적(explicit)인가 암시적(implicit)인가?** `log p(x)`를 직접 작성하는가?
3. **샘플링: 원샷(one-shot)인가 반복적(iterative)인가?** 반복적은 추론(inference)이 느리며, 원샷은 일반적으로 적대적(adversarial) 또는 증류(distilled) 방식을 의미합니다.
4. **조건(conditioning): 무조건(unconditional), 클래스(class), 텍스트(text), 이미지(image), 포즈(pose)인가?** 이는 손실 함수(loss function)와 아키텍처 스캐폴딩(architecture scaffolding)을 결정합니다.
5. **평가: FID, CLIP 점수, IS(Inception Score), 인간 선호도(human preference), 작업 정확도(task accuracy)인가?** 각각 알려진 실패 모드가 있습니다(레슨 14 참조).

이 단계의 모든 레슨에서 이 5가지 질문을 다시 답하게 될 것입니다. 끝까지 반복하면 자동적으로 답할 수 있게 될 것입니다.

## 빌드하기

이 강의의 코드는 경량 시각화 도구입니다: 세 가지 장난감 접근법(커널 밀도 추정, 이산 히스토그램, 최근접 샘플 "GAN-ish" 생성기)을 사용하여 샘플에서 1차원 가우시안 혼합 모델을 피팅하여 한 화면에 출력할 수 있는 문제에서 명시적 vs 암시적 밀도 차이를 확인할 수 있습니다.

`code/main.py`를 실행하세요. 이 코드는 두 모드 가우시안 혼합에서 2000개의 샘플을 생성한 후 다음을 출력합니다:

```
명시적 밀도 (히스토그램): p(x in [-0.5, 0.5]) ≈ 0.38
근사 밀도 (KDE): p(x in [-0.5, 0.5]) ≈ 0.41
암시적 (최근접 샘플 생성기): 20개의 새 샘플 출력, p(x) 없음
```

주의: 처음 두 방법은 "이 점의 가능성은 얼마나 되는가?"라고 질문할 수 있습니다. 세 번째 방법은 불가능합니다. 이것이 모든 향후 강의에서 중요해질 *명시적 vs 암시적* 구분입니다.

## 연습 문제

1. **쉬움.** 다음 다섯 가지 제품(ChatGPT image, Midjourney v7, Sora, Runway Gen-3, ElevenLabs)에 대해 각각 **패밀리(family)**와 **백본(backbone)**을 식별하시오. 증거는 공개 기술 보고서에서 가져와야 합니다.  
   - **패밀리** 예시: Diffusion, Autoregressive, GAN 등  
   - **백본** 예시: Transformer, U-Net, ResNet 등  

2. **중간.** 내일 읽을 논문에서 **확산(diffusion) 대비 100배 빠른 샘플링**을 주장합니다. 이 속도 향상이 **조건화(conditioning)**와 **고해상도(high resolution)**에서도 유지되는지 확인하기 위한 세 가지 질문을 작성하시오.  
   - 조건화 예시: 텍스트/이미지 프롬프트, 클래스 레이블 등  
   - 고해상도 예시: 512x512 이상, 비디오/3D 생성 등  

3. **어려움.** 관심 있는 도메인(예: 단백질 구조, CAD, 분자, 궤적) 하나를 선택하고, 해당 분야의 **현재 SOTA(State-of-the-Art) 모델**에 대해 다음 **5단계 트리아지(triage) 질문**을 답하시오. 또한 더 나은 모델이 어떤 변화를 가져올지 간략히 스케치하시오.  
   - **5단계 트리아지 질문**:  
     1. 이 모델은 어떤 문제를 해결하나?  
     2. 현재 접근법의 한계는 무엇인가?  
     3. 어떤 평가 지표로 성공/실패를 판단하나?  
     4. 어떤 데이터/계산 자원이 필요한가?  
     5. 실제 적용 시 주요 장애물은 무엇인가?  

> **참고**: "트리아지(triage)"는 문제 해결을 위해 핵심 질문을 체계적으로 분류하는 과정을 의미합니다.

## 주요 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|-------------------|-----------|
| 생성 모델(Generative model) | "새로운 것을 만든다" | `p_data(x)`에 대한 샘플러를 학습하며, 선택적으로 `log p(x)`를 노출한다. |
| 명시적 밀도(Explicit density) | "평가할 수 있다" | 모델이 닫힌 형식 또는 계산 가능한 `log p(x)`를 제공한다. |
| 암시적 밀도(Implicit density) | "GAN 스타일" | 샘플러만 존재 — 주어진 점의 `p(x)`를 평가할 방법이 없다. |
|
ELBO(Evidence lower bound) | "증거 하한" | `log p(x)`에 대한 계산 가능한 하한; VAE와 확산 모델이 이를 최적화한다. |
| 스코어(Score) | "로그 밀도의 그래디언트" | `∇_x log p(x)`; 확산 및 SDE 모델은 이 필드를 학습한다. |
| 매니폴드 가설(Manifold hypothesis) | "데이터는 표면 위에 존재한다" | 고차원 데이터는 저차원 매니폴드에 집중된다; 차원 축소가 작동하는 이유. |
| 자기회귀적(Autoregressive) | "다음 조각을 예측한다" | 결합 분포를 조건부 분포의 곱으로 분해한다. |
| 잠재(Latent) | "압축된 코드" | 디코더가 입력을 재구성할 수 있는 저차원 표현. |

## 프로덕션 노트: 다섯 가지 패밀리, 다섯 가지 추론 형태

각 패밀리는 서로 다른 추론-서버 비용 곡선에 매핑됩니다. 프로덕션 추론 문헌은 LLM 추론을 프리필(prefill) + 디코드(decode)로 프레임화하며, 동일한 분해가 여기에도 적용됩니다:

- **자기회귀(버킷 1과 5).** 순차적 디코드가 지연 시간을 지배합니다. KV-캐시, 연속 배치(continuous batching), 추측 디코딩(speculative decoding)이 모두 직접 적용됩니다.
- **VAE / 디퓨전 / 플로우 매칭(버킷 2와 4).** LLM 의미에서의 디코드는 없습니다. 비용 = `num_steps × step_cost`이며, `step_cost`는 전체 잠재 해상도에서의 트랜스포머 또는 U-Net 순전파(forward)입니다. 프로덕션 조절 장치는 스텝 수(DDIM / DPM-Solver / 증류), 배치 크기, 정밀도(bf16 / fp8 / int4)입니다.
- **GAN(버킷 3).** 단일 순전파. 스케줄도, KV-캐시도 없습니다. TTFT(첫 토큰 시간) ≈ 총 지연 시간. 이것이 StyleGAN이 좁은 도메인 UX에서 여전히 우위를 점하는 이유입니다.

논문 초록에서 "디퓨전보다 빠르다"는 표현을 보면, "더 적은 스텝 × 동일한 스텝 비용" 또는 "동일한 스텝 × 더 저렴한 스텝 비용"으로 번역하세요. 그 외는 모두 마케팅입니다.
