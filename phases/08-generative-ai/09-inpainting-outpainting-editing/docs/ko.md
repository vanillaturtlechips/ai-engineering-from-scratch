# 인페인팅, 아웃페인팅 & 이미지 편집

> 텍스트-이미지 모델은 새로운 것을 만듭니다. 인페인팅은 기존 것을 수정합니다. 실제 작업 환경에서 청구 가능한 이미지 작업의 70%는 편집 작업입니다 — 배경을 교체하거나, 로고를 제거하거나, 캔버스를 확장하거나, 손을 재생성하는 작업 등입니다. 인페인팅은 확산 모델이 진가를 발휘하는 분야입니다.

**유형:** 구축(Build)
**언어:** Python
**사전 요구 사항:** Phase 8 · 07 (잠재 확산), Phase 8 · 08 (ControlNet & LoRA)
**소요 시간:** ~75분

## 문제 정의

클라이언트가 배경에 주의를 분산시키는 간판이 있는 완벽한 제품 사진을 보냅니다. 간판만 지우고 나머지 부분은 픽셀 단위로 동일하게 유지하고 싶습니다. 처음부터 텍스트-이미지 생성을 실행할 수는 없습니다. 그렇게 하면 색상, 조명, 제품 각도가 달라지기 때문입니다. 마스킹된 영역만 재생성하고, 재생성 시 주변 컨텍스트를 고려하기를 원합니다.

이것이 인페인팅(inpainting)입니다. 변형 사례:

- **인페인팅(Inpainting).** 마스크 내부를 재생성하고 외부 픽셀은 유지합니다.
- **아웃페인팅(Outpainting).** 마스크 외부(또는 캔버스 너머)를 재생성하고 내부는 유지합니다.
- **이미지 편집(Image editing).** 전체 이미지를 재생성하되 원본과의 의미적/구조적 일관성을 유지합니다(SDEdit, InstructPix2Pix).

2026년의 모든 디퓨전 파이프라인은 인페인팅 모드를 탑재합니다. Flux.1-Fill, Stable Diffusion Inpaint, SDXL-Inpaint, DALL-E 3 Edit 등이 있으며, 모두 동일한 원리로 작동합니다.

## 개념

![Inpainting: 마스크 인식 노이즈 제거와 컨텍스트 보존 재주입](../assets/inpainting.svg)

### 순진한 접근법(그리고 왜 잘못되었는지)

마스크를 사용하여 표준 텍스트-이미지 생성을 실행합니다. 각 샘플링 단계에서 노이즈가 있는 잠재 공간의 마스크되지 않은 영역을 순방향 확산된 깨끗한 이미지로 대체합니다. 이 방법은... 잘 작동하지 않습니다. 모델이 마스크된 영역에 무엇이 있는지 알 수 없기 때문에 경계 아티팩트가 번집니다.

### 올바른 인페인팅 모델

4개 대신 9개의 입력 채널을 사용하는 수정된 U-Net을 훈련시킵니다:

```
input = concat([ noisy_latent (4ch), encoded_image (4ch), mask (1ch) ], dim=channel)
```

추가 채널은 VAE로 인코딩된 소스 이미지 복사본과 단일 채널 마스크입니다. 훈련 시 이미지의 무작위 영역을 마스크하고, 마스크된 영역만 노이즈를 제거하도록 모델을 훈련시킵니다. 이때 마스크되지 않은 영역은 깨끗한 조건부 신호로 제공됩니다. 추론 시 모델은 마스크된 영역 주변을 "보고" 일관된 완성을 생성합니다.

SD-Inpaint, SDXL-Inpaint, Flux-Fill은 모두 이 9채널(또는 유사) 입력을 사용합니다. Diffusers의 `StableDiffusionInpaintPipeline`, `FluxFillPipeline`.

### SDEdit (Meng et al., 2022) — 자유로운 편집

소스 이미지에 중간 `t`까지 노이즈를 추가한 후, `t`에서 0까지 새로운 프롬프트로 역과정을 실행합니다. 재훈련이 필요 없습니다. 시작 `t` 선택은 충실도와 창의적 자유 사이의 균형을 조절합니다:

- `t/T = 0.3` → 소스와 거의 동일, 작은 스타일 변경
- `t/T = 0.6` → 중간 편집, 거친 구조 유지
- `t/T = 0.9` → 노이즈 근처에서 생성, 소스 보존 최소화

### InstructPix2Pix (Brooks et al., 2023)

`(입력 이미지, 지시문, 출력 이미지)` 삼중항에 대해 확산 모델을 파인튜닝합니다. 추론 시 입력 이미지와 텍스트 지시문("일몰로 만들기", "용 추가하기") 모두에 조건화합니다. 두 개의 CFG 스케일: 이미지 스케일과 텍스트 스케일.

### RePaint (Lugmayr et al., 2022)

표준 비조건부 확산 모델을 유지합니다. 각 역과정에서 재샘플링 — 가끔 더 노이즈가 많은 상태로 돌아가 재생성합니다. 경계 아티팩트를 방지합니다. 훈련된 인페인팅 모델이 없을 때 사용됩니다.

## 구축 방법

`code/main.py`는 5차원 데이터에 대한 장난감 1-D 인페인팅 방식을 구현합니다. 두 클러스터 중 하나에서 나온 5개의 부동 소수점 값으로 구성된 5-D 혼합 데이터에 DDPM을 훈련시킵니다. 추론 시 5개 차원 중 2개를 "마스킹"하고, 각 단계에서 마스킹되지 않은 3개의 노이즈가 추가된 버전을 주입한 후 마스킹된 차원만 재생성합니다.

### 1단계: 5-D DDPM 데이터

```python
def sample_data(rng):
    cluster = rng.choice([0, 1])
    center = [-1.0] * 5 if cluster == 0 else [1.0] * 5
    return [c + rng.gauss(0, 0.2) for c in center], cluster
```

### 2단계: 5개 차원 전체에 대한 디노이저 훈련

표준 DDPM. 네트워크는 5-D 노이즈 입력에 대해 5-D 노이즈 예측을 출력합니다.

### 3단계: 추론 시 마스킹 인식 역과정

```python
def inpaint_step(x_t, mask, clean_image, alpha_bars, t, rng):
    # 마스킹되지 않은 차원을 깨끗한 소스의 새로 노이즈가 추가된 버전으로 교체
    a_bar = alpha_bars[t]
    for i in range(len(x_t)):
        if not mask[i]:
            x_t[i] = math.sqrt(a_bar) * clean_image[i] + math.sqrt(1 - a_bar) * rng.gauss(0, 1)
    # ...그런 다음 x_t에 대해 일반적인 역과정 단계 실행
```

이것은 순진한 접근법이며 장난감 1-D 데이터에서 작동합니다. 실제 이미지 인페인팅은 텍스처 일관성이 더 중요하기 때문에 9채널 입력을 사용합니다.

### 4단계: 아웃페인팅

아웃페인팅은 마스크를 반전시킨 인페인팅입니다: 새로운(이전에 존재하지 않았던) 캔버스를 마스킹하고 나머지는 원본으로 채웁니다. 동일한 훈련 목적 함수를 사용합니다.

## 함정(Pitfalls)

- **경계(Seams).** 순진한 접근 방식은 마스크 경계에서 눈에 보이는 경계를 남깁니다. 왜냐하면 그래디언트 정보가 마스크를 가로질러 흐르지 않기 때문입니다. 해결 방법: 마스크를 8-16픽셀 확장(dilate)하거나 적절한 인페인팅(inpainting) 모델을 사용합니다.
- **마스크 누출(Mask leakage).** 조건 이미지의 마스크되지 않은 영역이 저품질이거나 노이즈가 있는 경우, 마스크 내부 생성에 악영향을 줍니다. 디노이징(denoise)하거나 약간 블러(blur) 처리합니다.
- **CFG와 마스크 크기의 상호작용.** 작은 마스크에 높은 CFG(Classifier-Free Guidance)를 적용하면 포화(saturated)된 패치가 생성됩니다. 작은 편집 시 CFG를 줄입니다.
- **SDEdit 충실도 급락.** `t/T = 0.5`에서 `t/T = 0.6`으로 변경하면 대상의 정체성이 손실될 수 있습니다. 스위핑(sweep) 후 체크포인트(checkpoint)를 설정합니다.
- **프롬프트 불일치.** 프롬프트는 새 콘텐츠뿐만 아니라 *전체* 이미지를 설명해야 합니다. "의자에 앉은 고양이"가 아니라 "의자에 앉은 고양이"로 작성합니다.

## 사용 방법

| 작업 | 파이프라인 |
|------|----------|
| 객체 제거, 작은 마스크 | SD-Inpaint 또는 Flux-Fill, 표준 프롬프트 |
| 하늘 교체 | SD-Inpaint + "해질녘의 푸른 하늘" |
| 캔버스 확장 | SDXL 아웃페인트 모드(8px 페더) 또는 아웃페인트 마스크가 있는 Flux-Fill |
| 손/얼굴 재생성 | 주제 재설명한 프롬프트 + ControlNet-Openpose를 사용한 SD-Inpaint |
| 한 영역의 스타일 변경 | 마스크 영역에서 `t/T=0.5`로 SDEdit 적용 |
| "해질녘으로 만들기" | InstructPix2Pix 또는 Flux-Kontext |
| 배경 교체 | SAM 마스크 → SD-Inpaint |
| 초고해상도 | 가장 어려운 경우 Flux-Fill 또는 GPT-Image(호스팅) |

SAM(Meta의 Segment Anything, 2023) + 디퓨전 인페인트 조합이 2026년 배경 제거 파이프라인입니다. SAM 2(2024)는 비디오에서 작동합니다.

## 연습 문제

1. **쉬움.** `code/main.py`에서 마스킹된 차원의 비율을 0.2에서 0.8까지 변화시킵니다. 어떤 비율에서 인페인팅 품질(마스킹된 차원의 잔차)이 무조건 생성(unconditional generation)과 동일해집니까?
2. **중간.** RePaint를 구현합니다: 매 10번째 역단계(reverse step)마다 5단계 뒤로 돌아가(노이즈 추가) 다시 노이즈 제거(denoise)합니다. 마스크 경계에서 경계 잔차(boundary residual)가 감소하는지 측정합니다.
3. **어려움.** Hugging Face diffusers를 사용하여 비교합니다: SD 1.5 Inpaint + ControlNet-Openpose vs Flux.1-Fill을 20개의 얼굴 재생성 작업에서 평가합니다. 포즈 준수(pose adherence)와 신원 보존(identity preservation)을 별도로 점수화합니다.

## 주요 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|-------------------|-----------|
| 인페인팅(Inpainting) | "구멍을 메꿔라" | 마스크 내부를 재생성; 외부 픽셀은 유지. |
| 아웃페인팅(Outpainting) | "캔버스를 확장하라" | 캔버스 외부를 재생성; 내부는 유지. |
| 9-채널 U-Net | "제대로 된 인페인팅 모델" | `노이즈(noisy) | 인코딩된 소스(encoded-source) | 마스크(mask)`를 입력으로 받는 U-Net. |
| SDEdit | "노이즈 수준 기반 이미지 변환(Img2img)" | 노이즈를 시간 `t`로 변환, 새 프롬프트로 노이즈 제거. |
| 인스트럭트픽스투픽스(InstructPix2Pix) | "텍스트 기반 편집" | (이미지, 지시문, 출력) 삼중항에 대해 미세 조정된 확산 모델. |
| 리페인트(RePaint) | "재학습 불필요" | 역과정 중 주기적으로 재노이즈 적용하여 이음새 감소. |
| SAM | "모든 것 분할(Segment Anything)" | 클릭 또는 박스로 마스크 생성; 인페인팅과 연동. |
| 플럭스-콘텍스트(Flux-Kontext) | "컨텍스트 기반 편집" | 참조 이미지 + 지시문을 입력으로 받는 플럭스 변형 모델.>

## 프로덕션 참고 사항: 편집 파이프라인은 지연 시간에 민감함

이미지를 편집하는 사용자는 5초 미만의 왕복 시간을 기대합니다. 1024² 해상도의 30단계 SDXL-Inpaint는 L4에서 3-4초가 소요되며, SAM 마스크 생성(~200ms)과 VAE 인코딩/디코딩(~500ms 합산)이 추가됩니다. 프로덕션 환경에서는 처리량보다 TTFT(첫 번째 토큰 대기 시간)가 중요합니다. 배치 크기 1, 낮은 동시성, 모든 단계 최소화:

- **SAM-H가 가장 느립니다.** 1024² 해상도에서 SAM-H는 ~200ms가 소요되며, SAM-ViT-B는 ~40ms로 약간의 품질 손실이 발생합니다. SAM 2(비디오)는 시간적 오버헤드를 추가하므로 단일 이미지 편집에는 사용하지 마세요.
- **가능한 경우 인코딩을 건너뜁니다.** `pipe.image_processor.preprocess(img)`는 잠재 공간으로 인코딩합니다. 이전 생성에서 얻은 잠재 공간이 있다면(반복적 편집 UI에서 일반적), `latents=...`를 통해 직접 전달하여 VAE 인코딩 단계를 건너뛸 수 있습니다.
- **마스크 확장은 처리량에도 영향을 미칩니다.** 작은 마스크는 U-Net 순전파 연산의 대부분이 낭비됨을 의미합니다(마스크되지 않은 픽셀은 어차피 클램핑됨). `diffusers`의 `StableDiffusionInpaintPipeline`은 전체 U-Net을 실행하지만, 9채널 전용 인페인트 변형만 마스크된 연산을 활용합니다.
- **Flux-Kontext는 2025년 솔루션입니다.** `(source_image, instruction)`에 대한 단일 순전파 — 별도의 마스크나 SDEdit 노이즈 스윕이 필요 없습니다. H100에서 ~1.5초 만에 편집을 완료합니다. 아키텍처 교훈: 단계를 축소하세요.
