# T5, BART — 인코더-디코더 모델

> 인코더는 이해한다. 디코더는 생성한다. 둘을 다시 결합하면 입력 → 출력 작업을 위해 구축된 모델이 된다: 번역, 요약, 재구성, 전사.

**유형:** 학습
**언어:** Python
**선수 지식:** Phase 7 · 05 (풀 트랜스포머), Phase 7 · 06 (BERT), Phase 7 · 07 (GPT)
**소요 시간:** ~45분

## 문제 정의

디코더 전용 GPT와 인코더 전용 BERT는 각각 2017년 아키텍처를 다른 목표로 간소화했습니다. 하지만 많은 작업들은 자연스럽게 입력-출력 형태를 띱니다:

- 번역: 영어 → 프랑스어.
- 요약: 5,000토큰 기사 → 200토큰 요약.
- 음성 인식: 오디오 토큰 → 텍스트 토큰.
- 구조화된 추출: 산문 → JSON.

이러한 작업에는 인코더-디코더 구조가 가장 깔끔하게 맞습니다. 인코더는 소스의 밀집 표현을 생성하고, 디코더는 매 단계마다 해당 표현에 교차 어텐션(cross-attention)하며 출력을 생성합니다. 훈련은 출력 측에서 시프트-바이-원(shift-by-one) 방식으로 이루어집니다. GPT와 동일한 손실 함수를 사용하지만, 인코더 출력에 조건부로 적용됩니다.

두 논문이 현대 방법론을 정의했습니다:

1. **T5** (Raffel et al. 2019). "Text-to-Text Transfer Transformer." 모든 NLP 작업을 텍스트 입력-텍스트 출력으로 재구성. 단일 아키텍처, 단일 어휘, 단일 손실 함수. 마스킹된 스팬 예측(입력에서 스팬을 손상시키고 출력에서 복원)으로 사전 훈련.
2. **BART** (Lewis et al. 2019). "Bidirectional and Auto-Regressive Transformer." 노이즈 제거 오토인코더: 입력을 여러 방식(섞기, 마스킹, 삭제, 회전)으로 손상시킨 후 디코더가 원본을 재구성하도록 훈련.

2026년에도 인코더-디코더 형식은 입력 구조가 중요한 영역에서 계속 사용됩니다:

- Whisper (음성 → 텍스트).
- Google의 번역 스택.
- 명확한 컨텍스트-수정 구조를 가진 일부 코드 완성/수정 모델.
- 구조화된 추론 작업을 위한 Flan-T5 및 변형 모델.

디코더 전용 모델이 주목을 받았지만, 인코더-디코더는 결코 사라지지 않았습니다.

## 개념

![인코더-디코더 with 크로스 어텐션](../assets/encoder-decoder.svg)

### 순방향 루프

```
소스 토큰 ─▶ 인코더 ─▶ (N_src, d_model)  ──┐
                                                 │
타겟 토큰 ─▶ 디코더 블록                   │
                 ├─▶ 마스크드 셀프 어텐션       │
                 ├─▶ 크로스 어텐션 ◀───────────┘
                 └─▶ FFN
                ↓
              다음 토큰 로짓
```

중요하게, 인코더는 입력당 한 번 실행됩니다. 디코더는 자기회귀적으로 실행되지만 모든 단계에서 *동일한* 인코더 출력에 크로스 어텐션합니다. 인코더 출력을 캐싱하는 것은 긴 입력에 대한 무료 속도 향상입니다.

### T5 사전학습 — 스팬 손상

입력의 무작위 스팬(평균 길이 3토큰, 전체 15%)을 선택합니다. 각 스팬을 고유 센티넬로 대체: `<extra_id_0>`, `<extra_id_1>` 등. 디코더는 센티넬 접두사와 함께 손상된 스팬만 출력합니다:

```
소스: The quick <extra_id_0> fox jumps <extra_id_1> dog
타겟: <extra_id_0> brown <extra_id_1> over the lazy
```

전체 시퀀스 예측보다 저렴한 신호입니다. T5 논문의 실험에서 MLM(BERT) 및 접두사-LM(UniLM)과 경쟁력 있습니다.

### BART 사전학습 — 다중 노이즈 제거

BART는 5가지 노이즈 함수를 시도합니다:

1. 토큰 마스킹.
2. 토큰 삭제.
3. 텍스트 인필링(스팬 마스킹, 디코더가 올바른 길이 삽입).
4. 문장 순열.
5. 문서 회전.

텍스트 인필링 + 문장 순열 조합이 최고의 다운스트림 성능을 냈습니다. 디코더는 항상 원본을 재구성합니다. BART의 출력은 손상된 스팬이 아닌 전체 시퀀스이므로 사전학습 계산 비용이 T5보다 높습니다.

### 추론

GPT와 동일한 자기회귀 생성 방식을 사용합니다. 탐욕적 / 빔 / 탑-p 샘플링이 적용됩니다. 번역 및 요약에는 빔 탐색(너비 4–5)이 표준입니다. 출력 분포가 채팅보다 좁기 때문입니다.

### 2026년 각 변형 선택 시기

| 작업 | 인코더-디코더? | 이유 |
|------|------------------|-----|
| 번역 | 예, 일반적으로 | 명확한 소스 시퀀스; 고정된 출력 분포; 빔 탐색 작동 |
| 음성-텍스트 변환 | 예 (Whisper) | 입력 모달리티가 출력과 다름; 인코더가 오디오 특징 형성 |
| 채팅 / 추론 | 아니오, 디코더 전용 | 지속적인 "입력" 없음 — 대화가 시퀀스임 |
| 코드 완성 | 일반적으로 아니오 | 긴 컨텍스트를 가진 디코더 전용이 우세; Qwen 2.5 Coder 같은 코드 모델은 디코더 전용 |
| 요약 | 둘 다 가능 | BART, PEGASUS가 이전 디코더 전용 베이스라인 능가; 현대 디코더 전용 LLM도 동등 |
| 구조화된 추출 | 둘 다 | T5는 "텍스트 → 텍스트"가 모든 출력 형식을 흡수하기 때문에 깔끔함 |

~2022년 이후 추세: (a) 명령어 튜닝된 디코더 전용 LLM이 프롬프팅으로 모든 작업에 일반화, (b) 하나의 아키텍처가 두 개보다 확장 용이, (c) RLHF가 디코더를 가정하기 때문에 인코더-디코더가 소유하던 작업을 디코더 전용이 대체. 인코더-디코더는 입력 모달리티가 다른 경우(음성, 이미지) 또는 빔 탐색 품질이 중요한 경우에 유지.

## 구축 방법

`code/main.py`를 참조하세요. 우리는 장난감 코퍼스(텍스트 데이터)에 대해 T5 스타일의 스팬 손상(span corruption)을 구현합니다. 이 방법은 모든 인코더-디코더 사전 훈련 레시피에서 사용되기 때문에 이 레슨에서 가장 유용한 단일 구성 요소입니다.

### 1단계: 스팬 손상

```python
def corrupt_spans(tokens, mask_rate=0.15, mean_span=3.0, rng=None):
    """토큰의 ~mask_rate에 해당하는 스팬을 선택합니다. (손상된 입력, 타겟)을 반환합니다."""
    n = len(tokens)
    n_mask = max(1, int(n * mask_rate))
    n_spans = max(1, int(round(n_mask / mean_span)))
    ...
```

타겟 형식은 T5 규칙입니다: `<sent0> span0 <sent1> span1 ...`. 손상된 입력은 변경되지 않은 토큰과 스팬 위치의 센티넬 토큰을 교차 배치합니다.

### 2단계: 왕복 검증

손상된 입력과 타겟을 사용하여 원본 문장을 재구성합니다. 손상이 가역적이라면 순전파(forward pass)가 잘 정의된 것입니다. 이는 간단한 검증 단계이며, 실제 훈련에서는 수행되지 않지만 스팬 관리에서의 오프-바이-원(off-by-one) 버그를 저렴하게 잡아낼 수 있습니다.

### 3단계: BART 노이즈 추가

다섯 가지 함수: `token_mask`, `token_delete`, `text_infill`, `sentence_permute`, `document_rotate`. 이 중 두 가지를 조합하여 결과를 보여줍니다.

## 사용 방법

HuggingFace 참고:

```python
from transformers import T5ForConditionalGeneration, T5Tokenizer
tok = T5Tokenizer.from_pretrained("google/flan-t5-base")
model = T5ForConditionalGeneration.from_pretrained("google/flan-t5-base")

inputs = tok("translate English to French: Attention is all you need.", return_tensors="pt")
out = model.generate(**inputs, max_new_tokens=32)
print(tok.decode(out[0], skip_special_tokens=True))
```

T5의 특징: 작업 이름이 입력 텍스트에 포함됩니다. 각 작업이 텍스트 입력-텍스트 출력 방식이기 때문에 동일한 모델이 수십 가지 작업을 처리할 수 있습니다. 2026년에는 이 패턴이 명령어 튜닝된 디코더 전용 모델들에 의해 일반화되었지만, T5가 최초로 이를 정립했습니다.

## 연습 문제

1. **쉬움.** `code/main.py`를 실행하고, 30토큰 문장에 스팬 손상(span corruption)을 적용한 후, 비센티넬(non-sentinel) 소스 토큰과 디코딩된 타겟 스팬을 연결(concatenate)했을 때 원본 문장이 재현되는지 확인하세요.
2. **중간.** BART의 `text_infill` 노이즈를 구현하세요: 무작위 스팬을 단일 `<mask>` 토큰으로 대체하고, 디코더가 올바른 스팬 길이 및 내용을 추론하도록 합니다. 예시 하나를 제시하세요.
3. **어려움.** `flan-t5-small`을 소규모 영어 → 피그-라틴(pig-Latin) 코퍼스(200개 쌍)로 파인튜닝(fine-tuning)하세요. 50개 쌍의 홀드아웃(held-out) 세트에서 BLEU 점수를 측정하세요. 동일한 데이터와 동일한 컴퓨팅 자원으로 `Llama-3.2-1B`을 파인튜닝한 결과와 비교하세요.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|---------------------|-----------|
| 인코더-디코더(Encoder-decoder) | "Seq2seq 트랜스포머" | 두 개의 스택: 입력을 위한 양방향 인코더, 출력을 위한 교차 어텐션이 있는 인과적 디코더. |
| 교차 어텐션(Cross-attention) | "소스가 타겟과 대화하는 곳" | 디코더의 Q × 인코더의 K/V. 인코더 정보가 디코더에 들어가는 유일한 지점. |
| 스팬 손상(Span corruption) | "T5의 사전 학습 트릭" | 무작위 스팬을 센티넬 토큰으로 대체; 디코더는 스팬을 출력합니다. |
| 노이즈 제거 목적 함수(Denoising objective) | "BART의 게임" | 입력에 노이즈 함수를 적용하고, 디코더가 깨끗한 시퀀스를 재구성하도록 훈련. |
| 센티넬 토큰(Sentinel token) | "`<extra_id_N>` 플레이스홀더" | 소스의 손상된 스팬을 태그하고 타겟에서 다시 태그하는 특수 토큰. |
| 플란(Flan) | "지시문 튜닝된 T5" | 1,800개 이상의 태스크로 미세 조정된 T5; 인코더-디코더를 지시문 따르기 작업에서 경쟁력 있게 만듦. |
| 빔 서치(Beam search) | "디코딩 전략" | 각 단계에서 상위 k개의 부분 시퀀스를 유지; 번역/요약에 표준적으로 사용. |
| 티처 포싱(Teacher forcing) | "훈련 시간 입력" | 훈련 중 샘플링된 토큰이 아닌 실제 이전 출력 토큰을 디코더에 제공.
