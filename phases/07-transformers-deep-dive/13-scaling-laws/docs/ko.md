# 스케일링 법칙

> 2020년 Kaplan 논문은 다음과 같이 말했습니다: 더 큰 모델, 더 낮은 손실. 2022년 Hoffmann 논문은 다음과 같이 말했습니다: 당신은 과소 훈련하고 있었습니다. 컴퓨팅은 두 가지 범주로 나뉩니다 — 파라미터와 토큰 — 그리고 이 분할은 명확하지 않습니다.

**유형:** 학습  
**언어:** Python  
**선수 지식:** Phase 7 · 05 (풀 트랜스포머), Phase 7 · 07 (GPT)  
**소요 시간:** ~45분

## 문제 정의

C FLOPs의 훈련 계산 자원을 가지고 최고의 모델을 만들 때, 두 가지 조절 가능한 요소(knob)에 직면합니다:

1. **매개변수 수(N)?** 더 큰 모델은 더 높은 용량을 가집니다.
2. **훈련 토큰 수(D)?** 더 많은 데이터는 용량을 더 잘 활용합니다.

FLOPs는 대략 `6 × N × D`로 확장됩니다. N을 높이고 D를 낮추거나, D를 높이고 N을 낮출 수 있습니다. 어떤 것이 더 좋을까요?

2022년 이전에는 "N을 강하게 밀어붙여라"가 정답이었습니다. GPT-3(2020)는 1750억 개의 매개변수를 약 3000억 개의 토큰으로 훈련했으며, 매개변수당 약 1.7토큰의 비율을 가졌습니다. Kaplan의 스케일링 법칙도 이를 뒷받침했습니다.

Hoffmann et al. (2022)는 Chinchilla라는 소규모 모델 패밀리를 훈련하면서 다른 결과를 발견했습니다: 최적의 비율은 **매개변수당 20토큰**에 더 가깝습니다. GPT-3는 10배 과소 훈련되었습니다. Chinchilla(700억 매개변수, 1.4조 토큰)는 모든 벤치마크에서 GPT-3(1750억, 3000억 토큰)를 2.5배 적은 추론 비용으로 능가했습니다.

2026년은 Chinchilla의 세계입니다 — 단, 중요한 반전이 있습니다. Llama 3 8B는 15조 개의 토큰으로 훈련되었으며, 매개변수당 1,875토큰의 비율을 가집니다. Chinchilla 최적 비율보다 94배 더 많습니다. 대규모로 사용될 모델의 경우 훈련 비용보다 추론 비용이 더 중요하므로, 더 작은 배포 가능한 풋프린트를 위해 Chinchilla를 넘어서는 과훈련(over-training)이 2026년의 기본 전략입니다.

## 개념

![Chinchilla 곡선: 다양한 N/D 비율에서의 손실 대 계산량](../assets/scaling-laws.svg)

### 호프만 법칙

Chinchilla 논문에서 손실은 다음을 따릅니다:

```
L(N, D) = A / N^α + B / D^β + E
```

- `N` = 파라미터(임베딩 제외).
- `D` = 학습 토큰.
- `α ≈ 0.34`, `β ≈ 0.28` (대략 대칭적).
- `E ≈ 1.69`, 감소 불가능한 손실 상한선.
- `A ≈ 406`, `B ≈ 411`.

두 항은 확장 시 서로 트레이드오프됩니다. 고정된 계산량(C = 6ND)에서 `N`에 대한 미분을 구하고 풀면:

```
N_opt ≈ 0.6 × (C/6)^0.5
D_opt ≈ 0.6 × (C/6)^0.5
D_opt / N_opt ≈ 20
```

계산량 최적: 파라미터당 20 토큰.

### 그럼에도 과적합하는 이유

Chinchilla-최적은 학습 FLOP당 학습 손실을 최소화합니다. 하지만 학습 비용은 한 번만 지불하고, 추론 비용은 영원히 지불합니다.

월간 1조 토큰을 처리하는 챗봇의 경우 추론 비용이 총 비용을 지배합니다. Llama의 접근 방식: 더 작게, 더 오래 학습. 8B 파라미터에 15T 토큰은 추론에 깊게 최적화되었습니다:

- 소비자 GPU에 적합.
- 지연 시간이 70B Chinchilla-최적의 일부.
- 대부분의 작업에서 품질이 충분히 근접.

DeepMind의 2024년 논문("과적합이 새로운 최적")은 이를 공식화했습니다. 추론 중심 워크로드의 경우, 올바른 비율은 제공량에 따라 파라미터당 100–500 토큰에 더 가깝습니다.

### 출현 vs 부드러움

주장: 특정 능력(산술, 다단계 추론, 사고 사슬 따르기)은 어떤 규모에서 갑자기 "출현"합니다.

Schaeffer et al. (2023)은 이것이 측정 오류라고 주장했습니다: 출현 지표는 불연속 평가(정확한 일치, 임계값 정확도)를 사용하여 기본 로짓의 부드러운 개선을 숨깁니다. 연속 지표(교차 엔트로피)는 부드러운 곡선을 보여줍니다.

2026년 합의: 연속 손실을 통한 예측은 신뢰할 수 있습니다. 벤치마크 점프는 종종 평가자 오류입니다. 연속 지표에 따라 예산을 계획하세요.

### 2026년 상황

스케일링 법칙은 여전히 작동하지만:

| 요소 | 변화 방식 |
|------|-----------|
| 데이터 품질 | "좋은" 토큰 선별(Phi 스타일)은 곡선을 2배 이상 유효 계산량 이동 |
| MoE | 총 파라미터는 활성 FLOPs와 분리; 활성 FLOPs당 스케일링 법칙 |
| 사후 학습 | 일부 능력(지시 따르기, 코드)은 사전 학습보다 SFT+RLHF로 더 많이 이동 |
| 멀티모달 | 이미지 + 텍스트 토큰이 함께 확장; 모달리티별 별도 곡선 |
| 합성 데이터 | 모델이 학습 데이터 생성; 유효 계산량 복리 효과 가능 |

Muon 옵티마이저(Kimi Moonlight, 2024)는 AdamW와 동일한 데이터에서 ~2배 유효 계산량 향상을 보였습니다. 일부 2026 학습 실행은 기본적으로 Muon을 사용합니다. 스케일링 법칙의 절대 상수는 변경되지만 형태는 유지됩니다.

## 빌드하기

`code/main.py`를 참조하세요. Chinchilla 손실 방정식을 구현하고 여러 계산 예산에서 계산-최적 `(N, D)`를 구합니다.

### 1단계: Chinchilla 손실

```python
def chinchilla_loss(N, D, A=406.4, B=410.7, alpha=0.34, beta=0.28, E=1.69):
    return A / N ** alpha + B / D ** beta + E
```

고정된 `C = 6ND`에서 `(N, D)`에 대한 등고선으로 `L`을 플롯하고 최소값을 찾습니다.

### 2단계: 계산-최적 프론티어

1e17에서 1e25 FLOPs까지의 계산 예산에 대해, `6ND = C` 제약 하에서 손실을 최소화하는 `(N, D)`를 찾습니다. `D/N ≈ 20` 비율을 검증하세요.

### 3단계: 과적합 비용

10배 작은 모델(최적 N의 1/10, 최적 D의 10배)을 훈련시킬 때 발생하는 추가 손실을 계산합니다. 추론 FLOPs 절감 효과(N에 비례)를 보고하세요.

### 4단계: 실제 모델과 비교

GPT-3, Chinchilla, Llama 3 8B, DeepSeek-V3(활성 파라미터)에 대한 알려진 `(N, D)` 쌍을 입력하고 예측된 손실과 보고된 손실을 비교합니다.

## 사용 방법

프론티어 모델을 직접 훈련시킬 가능성은 낮습니다. 하지만 스케일링 법칙은 다음을 알려줍니다:

1. **파인튜닝에 충분한 데이터가 있는지 여부.** 작업별 데이터가 기본 모델의 파라미터당 20토큰 미만이라면 특정 손실 수준에서 포화 상태가 될 것으로 예상합니다.
2. **더 큰 기본 모델 선택 여부.** 예산을 모두 추론에 사용하고 있다면, 더 작고 더 오래 훈련된 모델을 선호합니다.
3. **수익 감소 지점.** Chinchilla 최적 대비 1000×를 넘어서면 로그-손실 변화는 노이즈 수준이 됩니다.

**2026년 연구 전망:**

- **데이터 제약 영역.** 웹에는 필터링 후 약 5~10조 개의 고품질 영어 토큰이 존재합니다. 프론티어 사전훈련은 이 한계에 근접하고 있습니다. 합성 데이터, 다국어, 멀티모달, RLHF 확장 파인튜닝이 다음 주요 수단입니다.
- **계산 승수 기법.** 뮤온 옵티마이저, MoE, 더 나은 데이터 큐레이션 — 각각 절대 상수를 변경하지만 점근적 특성은 변경하지 않습니다.
- **RL을 위한 스케일링 법칙.** 미해결 문제. 초기 증거는 RL 샘플에서 멱법칙이 나타나지만 사전훈련과 매우 다른 지수를 가짐을 시사합니다.

## 주요 용어

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|---------------------|-----------|
| 파라미터(N) | "모델 크기" | 임베딩이 아닌 가중치 수; 모델의 용량(capacity)을 결정 |
| 토큰(D) | "학습 데이터" | 학습 과정에서 본 토큰 수; 파라미터가 얼마나 잘 활용되는지 결정 |
| 계산량(C) | "소모된 FLOPs" | 표준 트랜스포머의 경우 약 `6 × N × D` |
| 친칠라 최적 | "D/N ≈ 20" | 사전 학습 FLOP당 손실(loss)을 최소화하는 비율 |
| 과적합 학습 | "친칠라 이후" | 추론 FLOPs를 절약하기 위해 추가 학습 FLOPs를 소모; D/N >> 20 |
| 감소 불가능한 손실 | "바닥값" | 스케일링 법칙의 `E` 항; 데이터 자체의 엔트로피 |
| 출현 능력 | "규모에 따른 갑작스러운 점프" | 종종 평가자(scorer) 오류; 연속적인 손실 곡선은 매끄러움 |
| 유효 계산량 | "학습 효율성 승수" | 더 나은 데이터/옵티마이저/아키텍처는 FLOP당 학습 효과를 배가시킴 |
