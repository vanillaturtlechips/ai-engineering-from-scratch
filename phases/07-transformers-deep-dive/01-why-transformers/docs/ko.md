# 트랜스포머가 필요한 이유 — RNN의 문제점

> RNN은 토큰을 하나씩 처리합니다. 트랜스포머는 모든 토큰을 한 번에 처리합니다. 이 단일 아키텍처 선택이 2017년 이후 딥러닝의 모든 확장 곡선을 바꿨습니다.

**유형:** 학습
**언어:** Python
**선수 지식:** 3단계(딥러닝 코어), 5단계·09(시퀀스-투-시퀀스), 5단계·10(어텐션 메커니즘)
**소요 시간:** ~45분

## 문제 정의

2017년 이전에는 언어, 번역, 음성 등 모든 최첨단 시퀀스 모델이 순환 신경망(RNN)이었습니다. LSTM과 GRU는 5년 동안 ImageNet에 상응하는 번역 벤치마크에서 우승했으며, 당시 유일한 도구였습니다.

이 모델들은 세 가지 치명적인 약점을 가지고 있었습니다. 순차적 계산으로 인해 시간 축을 따라 병렬화가 불가능했습니다. 토큰 `t+1`은 토큰 `t`의 은닉 상태를 필요로 했습니다. 1,024토큰 시퀀스는 1사이클당 1,000,000개의 부동소수점 연산을 처리할 수 있는 GPU에서도 1,024번의 직렬 단계를 의미했습니다. 훈련 벽시계 시간은 병렬화를 위해 설계된 하드웨어에서 시퀀스 길이에 선형적으로 비례했습니다.

기울기 소실(vanishing gradient) 문제는 50토큰 전의 정보가 이미 50번의 비선형 변환을 거쳐 압축되었음을 의미했습니다. 게이트 순환 유닛(LSTM, GRU)은 이 문제를 완화했지만 완전히 제거하지는 못했습니다. 장거리 의존성 — "지난 여름 교토로 가는 비행기에서 읽은 책은…" — 은 종종 실패했습니다.

고정 크기의 은닉 상태는 인코더가 디코더가 아무것도 보기 전에 전체 소스 시퀀스를 단일 벡터로 압축해야 함을 의미했습니다. 소스가 5토큰이든 500토큰이든 병목 현상은 동일한 형태였습니다.

2017년 논문 "Attention Is All You Need"는 급진적인 제안을 했습니다. 순환을 완전히 제거하고 모든 위치가 병렬로 다른 모든 위치에 어텐션(attention)할 수 있게 하는 것입니다. 1,024번의 순차적 행렬 곱셈 대신 하나의 큰 행렬 곱셈으로 훈련하는 것입니다.

이 결과는 2026년까지 모든 분야에서 지배적이 되었습니다. 언어(GPT-5, Claude 4, Llama 4), 비전(ViT, DINOv2, SAM 3), 오디오(Whisper), 생물학(AlphaFold 3), 로봇공학(RT-2) 등. 동일한 블록, 다른 입력입니다.

## 개념

![RNN 순차적 계산 vs 트랜스포머 병렬 어텐션](../assets/rnn-vs-transformer.svg)

**병목 현상으로서의 순환(Recurrence).** RNN은 `h_t = f(h_{t-1}, x_t)`를 계산합니다. 각 단계는 이전 단계에 의존합니다. `h_4` 없이 `h_5`를 계산할 수 없습니다. 10,000개 이상의 병렬 코어를 가진 현대 GPU에서 긴 시퀀스는 실리콘의 99%를 낭비합니다.

**브로드캐스트로서의 어텐션(Attention).** 셀프 어텐션은 모든 쌍 `(i, j)`에 대해 `output_i = sum_j(a_ij * v_j)`를 동시에 계산합니다. 전체 N×N 어텐션 행렬은 한 번의 배치 행렬 곱셈으로 채워집니다. 어떤 단계도 다른 단계에 의존하지 않습니다. GPU는 이를 좋아합니다.

**속도 향상은 상수가 아닙니다.** 이는 `O(N)` 직렬 깊이와 `O(1)` 직렬 깊이의 차이입니다. 실제로 트랜스포머는 N=512에서 동일한 하드웨어로 에포크당 5–10배 빠르게 훈련되며, 어텐션의 `O(N²)` 메모리 한계에 도달할 때까지 시퀀스 길이에 따라 격차가 커집니다(이후 Flash Attention이 해결 — 레슨 12 참조).

**트랜스포머의 비용.** 어텐션 메모리는 `O(N²)`로 확장됩니다. 2K 컨텍스트는 괜찮지만, 128K 컨텍스트에는 슬라이딩 윈도우, RoPE 외삽법, Flash Attention 틸링 또는 선형 어텐션 변형이 필요합니다. 순환은 시간과 메모리 모두 `O(N)`이었지만, 트랜스포머는 시간을 메모리와 교환한 후 병렬성을 통해 시간을 되찾습니다.

**귀납적 편향의 변화.** RNN은 지역성(locality)과 최신성(recency)을 가정합니다. 트랜스포머는 아무것도 가정하지 않습니다 — 모든 쌍이 어텐션 후보입니다. 그래서 트랜스포머는 잘 훈련되려면 더 많은 데이터가 필요하지만, 일단 데이터를 확보하면 더 멀리 확장됩니다. Chinchilla(2022)는 이를 공식화했습니다: 충분한 토큰이 주어지면 동일한 매개변수 수의 RNN보다 트랜스포머가 항상 우수합니다.

## 구축 방법

여기서는 신경망을 사용하지 않습니다 — 핵심 병목 현상을 수치적으로 시뮬레이션하여 노트북에서 그 차이를 직접 체감할 수 있도록 합니다.

### 1단계: 순차적 깊이 측정

`code/main.py`를 참조하세요. 두 개의 함수를 구현합니다. 하나는 시퀀스를 덧셈 체인(순환 신경망(RNN)처럼 순차적)으로 인코딩하고, 다른 하나는 병렬 감소(어텐션(attention)처럼 브로드캐스트)로 인코딩합니다. 수학은 동일하지만 의존성 그래프가 다릅니다.

```python
def rnn_style(xs):
    h = 0.0
    for x in xs:
        h = 0.9 * h + x   # 병렬화 불가: h는 이전 h에 의존
    return h

def attention_style(xs):
    return sum(xs) / len(xs)  # 모든 x는 독립적
```

100,000개 요소까지 시퀀스 길이를 늘려 두 방식을 시간 측정합니다. RNN 버전은 O(N)이며 단일 CPU 파이프라인으로 실행됩니다. 순수 파이썬에서도 어텐션 스타일 감소가 길이 ≥ 1,000에서 승리합니다. 파이썬의 `sum()`은 C로 구현되어 있으며 단계마다 인터프리터 오버헤드 없이 반복하기 때문입니다.

### 2단계: 이론적 연산 횟수 계산

두 알고리즘 모두 N번 덧셈을 수행합니다. 차이는 *의존성 깊이*입니다. 즉, 다음 연산이 시작되기 전에 순차적으로 발생해야 하는 연산 수입니다. RNN 깊이 = N. 어텐션 깊이 = 트리 감소 시 log(N) 또는 병렬 스캔 시 1. 연산 횟수가 아닌 깊이가 GPU 시간을 결정합니다.

### 3단계: 긴 시퀀스에서의 경험적 확장

O(N) 차이를 보여주는 타이밍 테이블을 출력합니다. 2026년형 맥북 노트북에서 1,000개 미만 요소는 측정이 어려울 정도로 빠릅니다. 100,000개 시퀀스는 선형 스캔을 명확하게 보여줍니다. 이를 16,384토큰 트랜스포머와 12층 LSTM 동등 모델로 확장하면 2016년 당시 훈련 벽시계가 왜 문제였는지 알 수 있습니다.

## 사용 시기

2026년에도 RNN을 선택해야 하는 경우:

| 상황 | 선택 |
|-----------|------|
| 스트리밍 추론, 한 번에 한 토큰씩, 일정한 메모리 | RNN 또는 상태-공간 모델(Mamba, RWKV) |
| 매우 긴 시퀀스(>1M 토큰)에서 어텐션 메모리 폭발 | 선형 어텐션(linear attention), Mamba 2, Hyena |
| 행렬 곱셈(matmul) 가속기가 없는 엣지 장치 | FLOPs/와트 측면에서 여전히 깊이별 분리 가능 RNN(depthwise-separable RNN)이 우세 |
| 그 외 모든 경우(훈련, 배치 추론, 최대 128K 컨텍스트) | 트랜스포머(Transformer) |

Mamba와 같은 상태-공간 모델(SSM)은 구조화된 파라미터화를 통해 RNN의 장점을 유지하면서 `O(N)` 스캔 메모리와 선택적 스캔(selective scan)을 통한 병렬 훈련이 가능한 모델입니다. 이들은 더 나은 장기 컨텍스트 확장성을 가지며 트랜스포머 품질의 90%를 회복합니다. 2026년에는 대부분의 최전방 연구실이 하이브리드 SSM+트랜스포머 모델(예: Jamba, Samba)을 훈련합니다. 순환 구조는 사라지지 않았으며, 하나의 구성 요소로 남아 있습니다.

## 주요 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|-------------------|-----------|
| 재귀(Recurrence) | "RNN은 순차적이야" | 단계 `t`가 단계 `t-1`에 의존하는 계산; 시간 축을 따라 직렬 실행을 강제함. |
| 직렬 깊이(Serial depth) | "그래프의 깊이" | 가장 긴 종속 연산 체인; 무한한 하드웨어에서도 벽시계 시간을 제한함. |
| 어텐션(Attention) | "토큰들이 서로 보게 한다" | 가중치 합 `sum_j a_ij v_j`; 여기서 `a_ij`는 위치 i와 j 간 유사도 점수에서 나옴. |
| 컨텍스트 윈도우(Context window) | "모델이 보는 범위" | 어텐션 레이어가 입력으로 받을 수 있는 위치 수; 여기서 메모리 비용이 제곱으로 증가함. |
| 귀납적 편향(Inductive bias) | "아키텍처에 내재된 가정" | 데이터 형태에 대한 사전 지식; CNN은 이동 불변성을, RNN은 최신성(recency)을 가정함. |
| 상태-공간 모델(State-space model) | "수학이 뒷받침된 RNN" | 구조화된 상태-공간 행렬을 통해 병렬 훈련이 가능한 재귀 파라미터화. |
| 이차 병목(Quadratic bottleneck) | "컨텍스트 비용이 큰 이유" | 어텐션 메모리 = 시퀀스 길이에 대한 `O(N²)`; Flash Attention은 상수를 숨기지만 확장성은 그대로임.
