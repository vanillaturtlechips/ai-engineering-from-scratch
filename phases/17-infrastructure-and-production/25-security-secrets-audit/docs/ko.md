# 보안 — 시크릿, API 키 순환, 감사 로그, 가드레일

> 중앙 집중식 볼트(HashiCorp Vault, AWS Secrets Manager, Azure Key Vault)를 통해 시크릿 난립을 제거하세요. 자격 증명을 구성 파일, VCS의 환경 파일, 스프레드시트에 저장하지 마세요. 정적 키 대신 IAM 역할을 사용하고, CI/CD에는 OIDC를 적용하세요. AI-게이트웨이 패턴은 2026년 솔루션입니다: 앱 → 게이트웨이 → 모델 공급자, 게이트웨이는 런타임에 볼트에서 자격 증명을 가져옵니다. 볼트에서 키를 순환하면 모든 앱이 몇 분 내에 새 키를 획득합니다 — 재배포, "새 키를 가진 사람은 누구인가요?"라는 Slack 메시지가 필요 없습니다. 순환 정책은 90일 이하; TruffleHog/GitGuardian/Gitleaks로 커밋 시마다 스캔하세요. 제로 트러스트: MFA, SSO, RBAC/ABAC, 단기 토큰, 디바이스 상태 확인. PII 스크러빙은 엔티티 인식을 사용해 PHI/PII를 마스킹한 후 전달하며, 일관된 토큰화(Mesh 접근법)는 민감한 값을 안정적인 플레이스홀더로 매핑하여 LLM이 코드/관계 의미를 보존하도록 합니다. 네트워크 이그레스: 전용 VPC/VNet 서브넷의 LLM 서비스는 `api.openai.com`, `api.anthropic.com` 등만 화이트리스트에 등록하고, 다른 모든 아웃바운드를 차단하세요. 2026년 사고 원인: Vercel 공급망 공격으로 손상된 CI/CD 자격 증명이 수천 고객 배포에 걸쳐 환경 변수를 유출했습니다.

**유형:** 학습  
**언어:** Python (표준 라이브러리, 장난감 PII 스크러버 + 감사 로그 작성기)  
**선수 지식:** 17단계 · 19항목 (AI 게이트웨이), 17단계 · 13항목 (관측 가능성)  
**소요 시간:** ~60분

## 학습 목표

- 4가지 시크릿 관리 안티패턴(VCS 내 구성 파일, 하드코딩된 환경 변수, 스프레드시트, 정적 키)을 열거하고 각각의 대체 방안을 제시한다.
- 2026년 프로덕션 표준으로 예상되는 **AI-gateway-pulls-from-vault** 패턴을 설명한다.
- 의미론적 정보가 보존되도록 **일관된 토큰화**(동일 값 → 동일 플레이스홀더)를 적용한 PII 스크러버를 구현한다.
- 2026년 **Vercel 공급망 사고**의 명칭과 CI/CD 자격 증명 위생 관리에 대한 교훈을 설명한다.

## 문제

인턴이 API 키가 포함된 `.env` 파일을 커밋합니다. 그들은 이를 빠르게 삭제합니다. 그러나 키는 이미 git 히스토리에 남아 있습니다 — GitGuardian 스캔에서 이를 감지했고, 키 순환 프로세스는 "팀에 Slack으로 알림, 40개의 설정 파일 업데이트, 모든 서비스 재배포"입니다. 8시간 후, 서비스의 절반은 가동 중이고 나머지 절반은 배포 창을 기다리고 있습니다.

별개로, 사용자 프롬프트에 "내 SSN은 123-45-6789입니다."라는 내용이 포함됩니다. 이 프롬프트는 OpenAI로 전송됩니다. BAA(Business Associate Agreement)는 있지만, 내부 정책에 따라 PII(개인 식별 정보)를 마스킹한 후 전달해야 합니다. 그러나 이를 수행하지 않았습니다.

별개로, EKS 클러스터의 LLM 파드는 모든 인터넷 호스트에 접근할 수 있습니다. 누군가가 공격자가 제어하는 도메인으로 DNS 조회를 통해 데이터를 유출합니다. 이를 차단한 것은 아무것도 없었습니다.

LLM 서비스 보안은 세 가지 벡터(벡터) 모두를 해결해야 합니다. Vault 기반 자격 증명. PII 스크러빙. 네트워크 이그레스 필터링. 감사 로그.

## 개념

### 중앙 집중식 볼트 + IAM 역할 풀

**볼트**: HashiCorp Vault, AWS Secrets Manager, Azure Key Vault, GCP Secret Manager. 단일 소스 오브 트루스(SSOT).

**IAM 역할**: 앱/게이트웨이는 정적 키가 아닌 IAM 신원을 통해 인증. 볼트는 토큰 수명 동안 시크릿을 반환.

**AI-게이트웨이 패턴**: 게이트웨이는 요청 시점에 볼트에서 `OPENAI_API_KEY`를 가져옴. 볼트 내에서 키를 순환하면 다음 요청 시 새 키를 획득. 재배포 불필요.

### 순환 정책 ≤ 90일

모든 API 키, 볼트 루트 토큰, CI/CD 자격 증명. 가능한 경우 자동 순환. 수동 순환은 로그 기록 및 추적.

### 시크릿 스캐닝

- **TruffleHog** — 커밋에 대한 정규식 + 엔트로피 분석.
- **GitGuardian** — 상용, 높은 정확도.
- **Gitleaks** — 오픈소스, CI에서 실행.

모든 커밋 시 실행. 새 시크릿 감지 시 PR 차단.

### 제로 트러스트 정책

- 모든 계정에서 MFA 필수.
- SAML/OIDC 기반 SSO.
- 세분화된 접근을 위한 RBAC(역할 기반) 또는 ABAC(속성 기반).
- 단기 토큰(시간 단위, 일 단위 아님).
- 장치 상태 확인 — 디스크 암호화가 적용된 기업 장치만 허용.

### PII / PHI 제거

프롬프트가 인프라를 떠나기 전:

1. 엔티티 인식(spaCy NER, Presidio, 상용 솔루션).
2. 일치한 엔티티 마스킹: `"My SSN is 123-45-6789"` → `"My SSN is [SSN_TOKEN_A3F]"`.
3. 일관된 토큰화(Mesh 접근법): 동일한 값은 동일한 플레이스홀더로 매핑되어 LLM이 관계성을 유지.
4. LLM 응답에 대한 선택적 역매핑.

정적 정규식 필터는 기본 패턴 포착, NER은 더 많은 패턴 포착. 둘 모두 사용.

### 입력 + 출력 가드레일

입력: 알려진 탈옥 시도, 금지 주제 차단; 사용자별 요청 제한.

출력: 유출된 시크릿(거부 컨텍스트 내 API 키 패턴, 이메일 패턴) 정규식 검사, 정책 위반 분류기.

### 네트워크 아웃바운드 화이트리스트

전용 서브넷 내 LLM 서비스:
- 화이트리스트: `api.openai.com`, `api.anthropic.com`, 벡터 DB 엔드포인트, 볼트 엔드포인트.
- 그 외 모든 트래픽: 차단.
- DNS는 허용 목록 전용 리졸버 사용(DNS 터널링 유출 방지).

### 감사 로그

모든 LLM 호출에 대한 불변 로그 포함:
- 타임스탬프.
- 사용자/테넌트.
- 프롬프트 해시(개인정보 보호를 위해 원본 프롬프트가 아닌 해시값).
- 모델 + 버전.
- 토큰 수.
- 비용.
- 응답 해시.
- 가드레일 위반 사항.

규제 요구사항에 따라 보관(SOC 2 1년, HIPAA 6년).

### 2026년 Vercel 사고

공급망 공격: 손상된 CI/CD 자격 증명을 통해 수천 고객 배포 환경에서 환경 변수 유출. 교훈: CI/CD 자격 증명은 프로덕션과 동등한 중요도로 관리. 볼트에 저장. 범위 최소화. 적극적으로 순환.

### 기억해야 할 숫자

- 순환 정책: ≤ 90일.
- 커밋 시 스캔: TruffleHog / GitGuardian / Gitleaks.
- Vercel 2026: CI/CD 자격 증명 유출 → 수천 고객 환경 변수 유출.
- 감사 로그 보관: SOC 2 = 1년, HIPAA = 6년.

## 연습 문제

1. `code/main.py`를 실행하세요. 동일한 SSN을 참조하는 두 개의 프롬프트를 전송하고, 둘 다 동일한 플레이스홀더를 받는지 확인하세요.
2. OpenAI + Anthropic + Weaviate를 호출하는 vLLM-on-EKS 배포를 위한 네트워크 이그레스(egress) 정책을 설계하세요.
3. git 히스토리(2년 전)에서 키를 발견했습니다. 올바른 대응은 키 순환(rotate the key), 히스토리 스크러빙(scrub history), 또는 둘 다인가요? 근거를 제시하세요.
4. 감사 로그가 하루에 10GB씩 증가합니다. 보존 계층(핫 30일, 웜 12개월, 콜드 6년)을 설계하세요.
5. 역토큰화(LLM 응답에 실제 값 재삽입)가 플레이스홀더 유지 대비 복잡성 증가에도 가치가 있는지 논하세요.
