# 추론 메트릭 — TTFT, TPOT, ITL, Goodput, P99

> 추론 배포가 작동하는지 결정하는 네 가지 메트릭이 있습니다. TTFT는 프리필(prefill) + 큐 대기 + 네트워크 시간입니다. TPOT(동등하게 ITL)는 토큰당 메모리 바운드 디코딩 비용입니다. 엔드투엔드 지연 시간은 TTFT에 TPOT을 출력 길이로 곱한 값을 더한 것입니다. 처리량(throughput)은 플릿(fleet) 전체에서 집계된 초당 토큰 수입니다. 하지만 제품에서 중요한 것은 Goodput입니다. Goodput은 모든 SLO를 동시에 충족한 요청의 비율입니다. 낮은 Goodput에서 높은 처리량은 사용자에게 제시간에 도달하지 못하는 토큰을 처리한다는 의미입니다. 2026년 TRT-LLM에서 Llama-3.1-8B-Instruct의 참조 수치: 평균 TTFT 162ms, 평균 TPOT 7.33ms, 평균 E2E 1,093ms. 항상 P50, P90, P99를 보고하세요. 평균만 보고하지 마세요. 그리고 측정 함정을 주의하세요: GenAI-Perf는 ITL 계산에서 TTFT를 제외하지만, LLMPerf는 포함합니다. 두 도구는 동일한 실행에 대해 TPOT 값이 다를 수 있습니다.

**유형:** 학습
**언어:** Python (표준 라이브러리, 간단한 백분위수 계산기 및 Goodput 리포터)
**선수 지식:** 17단계 · 04 (vLLM 서빙 내부 구조)
**소요 시간:** ~60분

## 학습 목표

- TTFT(Thought Time to First Token), TPOT(Token Per Output Token), ITL(Idle Time Latency), E2E(End-to-End), 처리량(throughput), 유효 처리량(goodput)을 정확히 정의하고 각각이 측정하는 구성 요소를 명시한다.
- LLM 서빙에서 평균(mean)이 잘못된 통계인 이유와 P50/P90/P99를 해석하는 방법을 설명한다.
- SLO 다중 제약 조건(예: TTFT<500 ms AND TPOT<15 ms AND E2E<2 s)을 구성하고 이에 대한 유효 처리량(goodput)을 계산한다.
- 동일한 실행에서 TPOT에 대해 서로 다른 결과를 보이는 두 가지 벤치마크 도구를 제시하고 그 이유를 설명한다.

## 문제

"초당 처리량이 15,000토큰입니다." 그래서 어쨌다는 거죠? 요청의 40%가 엔드투엔드 2초를 초과했다면 사용자는 세션을 포기했습니다. 처리량만으로는 제품이 제대로 작동하는지 알 수 없습니다.

추론에는 여러 차원의 지연 시간이 있으며 각각 다른 방식으로 실패합니다. 프리필(prefill)은 계산 집약적이며 프롬프트 길이에 따라 확장됩니다. 디코드(decode)는 메모리 집약적이며 배치 크기에 따라 확장됩니다. 큐잉 지연은 운영상의 문제입니다. 네트워크는 물리적 거리 문제입니다. 각각에 대한 별도의 지표가 필요하며, 백분위수(percentile)가 필요하고, "사용자가 기대한 것을 얻었는가"를 나타내는 단일 복합 지표(goodput)가 필요합니다.

## 개념

### TTFT — 첫 토큰까지의 시간

`TTFT = 큐 대기 시간 + 네트워크 요청 + 프리필 시간`

프롬프트가 길 경우 프리필 시간이 지배적입니다. H100에서 Llama-3.3-70B FP8 모델의 경우 32k 프롬프트에 순수 프리필 시간이 약 800ms 소요됩니다. 큐 대기 시간은 부하 하에서의 스케줄러 동작입니다. 네트워크 요청은 TLS를 포함한 전송 시간입니다. TTFT는 사용자에게 첫 토큰이 스트리밍되기 전에 보이는 지연 시간입니다.

### TPOT / ITL — 토큰 간 지연 시간

동일한 지표를 가리키는 여러 이름입니다. `TPOT`(출력 토큰당 시간), `ITL`(토큰 간 지연 시간), `토큰당 디코딩 지연 시간` — 모두 같은 의미입니다. 첫 토큰 이후 스트리밍되는 연속 토큰 사이의 시간입니다.

`TPOT = (디코딩 순전파 시간 + 스케줄러 오버헤드) / 생성된 토큰 수`

청크 프리필이 적용된 동일한 Llama-3.3-70B H100 스택에서 TPOT 평균은 약 7ms입니다. 청크 프리필이 없을 경우, 인접 시퀀스에서 긴 프리필이 진행 중일 때 TPOT은 50ms까지 치솟을 수 있습니다. 평균이 아닌 P99를 주시하세요.

### E2E 지연 시간

`E2E = TTFT + TPOT * 출력 토큰 수 + 네트워크 응답`

긴 출력(500토큰 이상)의 경우 E2E는 TPOT이 지배적입니다. 긴 프롬프트와 짧은 출력의 경우 E2E는 TTFT이 지배적입니다. 출력 길이에 따른 E2E를 보고하세요.

### 처리량

`처리량 = 총 출력 토큰 수 / 경과 시간`

집계 지표입니다. 플릿 효율성을 알려줍니다. 개별 요청 상태는 알 수 없습니다.

### 굿풋 — 실제로 관심 있는 지표

`굿풋 = (TTFT <= a) AND (TPOT <= b) AND (E2E <= c)를 만족하는 요청 비율`

SLO는 다중 제약 조건입니다. 모든 제약 조건을 만족하는 요청만 "양호"로 간주됩니다. 굿풋은 해당 비율입니다. 60% 굿풋에서 높은 처리량은 실패입니다. 99% 굿풋에서 낮은 처리량이 목표입니다.

2026년에는 MLPerf Inference v6.0 제출 및 AI 플랫폼 제공업체의 내부 SLA 추적에서 굿풋을 사용합니다.

### 평균이 잘못된 통계인 이유

LLM 지연 시간 분포는 오른쪽으로 치우칩니다. 한 디코딩 배치에서 긴 프리필 이웃이 있는 경우 TPOT ~7ms로 500토큰, TPOT ~60ms로 20토큰을 전송할 수 있습니다. 평균 TPOT은 9ms입니다. P99 TPOT은 65ms입니다. 사용자는 정기적으로 P99를 경험합니다 — 그래서 이탈합니다.

항상 (P50, P90, P99) 삼중값을 보고하세요. 사용자 경험 최적화를 위해 P99가 중요합니다.

### 참조 수치 — TRT-LLM의 Llama-3.1-8B-Instruct, 2026

- 평균 TTFT: 162ms
- 평균 TPOT: 7.33ms
- 평균 E2E: 1,093ms
- P99 TPOT: 청크 프리필 구성에 따라 10-25ms 변동.

이것은 공개된 NVIDIA 기준점입니다. 모델 크기(70B는 3-5배), 하드웨어(H100 vs B200 ~3배), 부하에 따라 변경됩니다.

### 측정 함정

가장 많이 사용되는 2026 벤치마크 도구 2개는 동일 실행에 대해 TPOT을 다르게 보고합니다:

- **NVIDIA GenAI-Perf**: ITL 계산에서 TTFT를 제외합니다. ITL은 토큰 2부터 시작합니다.
- **LLMPerf**: TTFT를 포함합니다. ITL은 토큰 1부터 시작합니다.

TTFT 500ms, 총 디코딩 시간 700ms(100출력 토큰)인 요청의 경우, GenAI-Perf는 `ITL = 700/99 = 7.07ms`, LLMPerf는 `ITL = 1200/100 = 12.00ms`로 보고합니다. 도구 선택에 따라 수치가 달라집니다.

항상 사용한 도구를 명시하세요. 정의를 반드시 공개하세요.

### SLO 구성

2026년 70B 채팅 모델의 합리적인 소비자 대상 SLO:

- TTFT P99 <= 800ms.
- TPOT P99 <= 25ms.
- 300토큰 미만 출력의 E2E P99 <= 3초.
- 굿풋 목표 >= 99%.

엔터프라이즈 SLO는 TTFT(200-400ms)를 강화하고 E2E를 완화합니다. 핵심은 세 가지를 모두 명시하고 측정하며, 굿풋을 단일 복합 지표로 추적하는 것입니다.

### 측정 방법

- 실제 트래픽 또는 현실적인 합성 트래픽(LLMPerf `--mean-input-tokens 800 --stddev-input-tokens 300 --mean-output-tokens 150`)을 실행합니다.
- 벤치마크 실행 시 피크 동시성의 2배를 목표로 합니다.
- 30-50회 반복 실행 후, 통합 샘플의 백분위수를 취합니다.
- 도구 이름, 버전, 모델, 하드웨어, 동시성, 프롬프트 분포와 함께 공개합니다.

## 사용 방법

`code/main.py`는 장난감(간단한) 처리량(goodput) 계산기입니다. 합성 지연 시간 분포를 생성하고, SLO(서비스 수준 목표)를 적용한 후 처리량을 계산합니다. 또한 동일한 트레이스(trace)에서 GenAI-Perf와 LLMPerf TPOT의 차이점을 보여줍니다.

## 주요 용어

| 용어 | 사람들이 말하는 것 | 실제 의미 |
|------|----------------|------------------------|
| TTFT | "첫 토큰까지의 시간" | 큐 대기 + 네트워크 + 프리필; 긴 프롬프트에서는 프리필이 지배적 |
| TPOT | "출력 토큰당 시간" | 첫 토큰 이후 토큰당 메모리 바운드 디코딩 비용 |
| ITL | "토큰 간 지연" | 대부분의 도구에서 TPOT과 동일(모두는 아님 — GenAI-Perf 참조) |
| E2E | "엔드 투 엔드" | TTFT + TPOT * 출력_길이; 응답 측 네트워크 추가 |
| 처리량 | "tok/s" | 플릿 효율성; 지연 백분위 수 없으면 무의미 |
| 굿풋 | "SLO 충족률" | 모든 SLO 제약 조건을 동시에 충족하는 요청의 비율 |
| P99 | "꼬리" | 100개 중 최악의 1개 지연; 사용자 경험 지표 |
| SLO 다중 제약 | "결합" | 세 가지 지연 한계의 AND; 하나라도 위반 시 요청 실패 |
| GenAI-Perf vs LLMPerf | "도구 함정" | ITL에 TTFT 포함 여부에 대해 도구 간 불일치 |
