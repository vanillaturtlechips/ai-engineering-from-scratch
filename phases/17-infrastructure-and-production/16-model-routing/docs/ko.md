# 비용 절감 기법으로서의 모델 라우팅

> 동적 브로커는 모든 요청(작업 유형, 토큰 길이, 임베딩 유사도, 신뢰도)을 평가하고 간단한 쿼리는 저렴한 모델로 보내며 복잡한 쿼리는 최첨단 모델로 에스컬레이션합니다. 모델 캐스케이딩이라고도 합니다. 프로덕션 사례 연구에 따르면 미국/영국/유럽 배포에서 동일한 품질 기준으로 20-60% 비용 절감이 가능하며, 고부하 SaaS에서 30% 라우팅 효율성 개선은 연간 5자리 수 절감으로 이어집니다. 2026년 컨텍스트는 LLM 추론 가격이 연간 ~10배 하락했다는 점입니다. GPT-4급 토큰은 2022년 말부터 2026년까지 $20/M에서 ~$0.40/M로 떨어졌습니다. 이 하락의 대부분은 하드웨어가 아닌 더 나은 서빙 스택(Phase 17 · 04-09) 덕분입니다. 라우팅은 제품 성능 저하 없이 이 가격 하락을 마진으로 전환하는 방법입니다. 실패 모드는 저렴한 모델의 드리프트입니다. 라우팅이 40%를 약한 모델로 푸시하면 추론 작업에서 품질이 3-5% 하락하지만, 분기 동안 아무도 눈치채지 못합니다. 오프라인 평가 세트가 아닌 온라인 품질 지표로 게이트 라우팅을 수행하세요.

**유형:** 학습  
**언어:** Python (표준 라이브러리, 장난감 캐스케이딩 라우터 시뮬레이터)  
**선수 지식:** Phase 17 · 01 (관리형 LLM 플랫폼), Phase 17 · 19 (AI 게이트웨이)  
**소요 시간:** ~60분

## 학습 목표

- **모델 캐스케이딩** 설명: 저렴한 모델 우선 실행 + 신뢰도 검증, 낮은 신뢰도 시 고비용 모델로 전환.
- **4가지 라우팅 신호** 열거:  
  (1) 작업 분류(task classification),  
  (2) 프롬프트 길이(prompt length),  
  (3) 알려진 어려운 샘플 집합과의 임베딩 유사도(embedding similarity to known-hard set),  
  (4) 1차 모델 통과 시 자기 신뢰도(self-confidence from first-pass).
- **목표 라우팅 분할 지점**에서 예상 혼합 비용(expected blended cost) 및 품질 손실 허용치(quality loss tolerance) 계산.
- **드리프트 모니터링 지표**(온라인 품질 게이트) 명칭: **저렴한 모델 성능 저하(cheap-model creep)** 감지.

## 문제 정의

귀사의 서비스는 GPT-5 사용으로 월 $80,000의 비용이 발생합니다. 분석 결과 70%의 쿼리는 "파리 시간은 몇 시인가요?" "이 문장을 다른 방식으로 표현해 주세요"와 같은 단순 작업인 것으로 나타났습니다. 하이쿠(Haiku)급 모델은 이러한 작업을 완벽하게 처리하며 비용은 3%에 불과합니다. 나머지 30%는 코딩, 수학 문제, 다단계 계획 수립 등 GPT-5의 추론 능력이 필요한 작업입니다.

70%의 쿼리를 저비용 모델로, 30%를 고성능 모델로 라우팅하면 동일한 제품 품질을 유지하면서 비용을 약 65% 절감할 수 있습니다. 이것이 바로 라우팅입니다. 핵심은 품질을 저하시키지 않으면서 브로커 시스템을 구축하는 것입니다.

## 개념

### 네 가지 라우팅 신호

1. **작업 분류**: 단순/복잡/코드 생성/수학/채팅. 규칙 기반 분류기, 소형 LLM(Haiku-class, $0.25/M), 또는 레이블된 버킷과의 임베딩 유사도로 구현 가능. 출력: 경로 = 저가형/균형형/프론티어형.

2. **프롬프트 길이**: 4K 토큰 이상의 프롬프트는 일관성 유지를 위해 프론티어형이 필요한 경우가 많음. 500 토큰 미만의 프롬프트는 일반적으로 그렇지 않음.

3. **알려진 어려운 작업과의 임베딩 유사도**: 쿼리가 알려진 어려운 버킷과 유사도(cosine > 0.88)가 높으면 프론티어형으로 직접 에스컬레이션.

4. **첫 번째 시도에서의 자체 신뢰도**: 저가형 모델로 전송. 모델의 로그 확률이 낮은 신뢰도를 보이거나 거부하거나 회피적 언어를 출력할 경우 프론티어형에서 재시도. 트래픽의 ~10%에서 P95 지연 시간 추가되지만 나머지 90%에서 50%+ 비용 절감.

### 세 가지 패턴

**사전 라우팅**(분류기 선행): ~5-10ms 지연 시간 추가. 전체적으로 가장 빠름.

**캐스케이드**(저가형 우선, 낮은 신뢰도 시 에스컬레이션): ~1.2x 중간 지연 시간(저가형 실행 + 검증), 에스컬레이션 시 ~2x. 최고 품질 보장.

**앙상블 라우팅**(샘플에 대해 저가형과 프론티어형을 병렬 실행, 보상 모델로 선택): 최고 품질, 최고 비용. 중요한 A/B 테스트에만 사용.

### 구현

AI 게이트웨이(Phase 17 · 19)에서 라우팅 노출. LiteLLM은 `router` 설정에 폴백 및 비용 라우팅 기능 제공. Portkey는 가드 + 라우팅 기능 보유. Kong AI 게이트웨이는 플러그인 기반 라우팅 지원. OpenRouter의 모델 마켓플레이스는 추천 API 노출.

오픈소스: RouteLLM(LMSYS), Not Diamond(상업용), Prompt Mule.

### 2026년 가격 곡선

| 모델 클래스 | 2022년 말 | 2026년 | 변화 |
|-------------|-----------|------|--------|
| GPT-4 수준 품질 | ~$20/M | ~$0.40/M | 50배 저렴 |
| 프론티어(GPT-5, Claude 4) | — | ~$3-10/M | 새로운 등급 |

개선 사항의 대부분은 서빙 효율성 향상 때문. Phase 17 · 04-09의 핵심 교훈이 공급자 측 비용 절감으로 이어짐. 라우팅을 통해 모든 사용자가 저가형으로 마이그레이션할 때까지 기다리지 않고 애플리케이션 계층에서 이러한 이점을 활용할 수 있음.

### 드리프트가 실제 위험

라우터가 40%의 트래픽을 저가형 모델로 전송. 6개월 후 작업 분포가 변화(사용자가 더 정교해지고 긴 질문 요청). 라우터는 분류기가 Q1 데이터로 학습되었기 때문에 변화를 감지하지 못함. 품질이 조용히 저하. 아무도 충분히 불평하지 않음. 경쟁사 벤치마크에서 패배한 후에야 발견.

온라인 품질 지표로 게이트 라우팅:

- 경로별 사용자 추천/비추천.
- 경로별 보유 샘플(5%)에 대한 자동화된 LLM-판관.
- 에스컬레이션 비율: 캐스케이드가 30% 이상 상위 경로로 에스컬레이션하면 저가형 모델이 과도하게 라우팅되고 있음.
- 경로별 거부율.

### 기억해야 할 숫자

- 2026년 동일 품질 기준 라우팅 절감액: 사례 연구 기준 20-60%.
- 2022-2026년 LLM 가격 하락: 연간 ~10배(집계 기준).
- 2022년 GPT-4 수준 vs 2026년: ~$20/M → ~$0.40/M.
- 캐스케이드 지연 시간 영향: 중간값 ~1.2x, 에스컬레이션 시 ~2x(트래픽의 ~10%).

## 사용 방법

`code/main.py`는 혼합 워크로드에서 사전 경로 설정(pre-route), 캐스케이드(cascade), 앙상블(ensemble)을 시뮬레이션합니다. 혼합 비용(blended cost), 품질 손실(quality loss), 에스컬레이션 비율(escalation rate)을 보고합니다.

## 연습 문제

1. `code/main.py`를 실행해 보세요. 어떤 정확도 수준에서 캐스케이드가 사전 라우팅을 능가하나요?  
2. 사용자 기반은 30%가 엔터프라이즈(복잡한 쿼리), 70%가 무료 티어(간단한 쿼리)입니다. 라우팅 분할을 설계해 보세요. 어떤 온라인 메트릭이 이를 결정하나요?  
3. 특정 라우트가 품질을 2% 저하시키지만 비용을 40% 절감합니다. 이를 출시할 수 있을까요? 제품에 따라 다릅니다. 양쪽 입장을 모두 주장해 보세요.  
4. OpenAI/Anthropic API의 `logprobs`를 활용해 신뢰도 검사를 구현해 보세요. 초기 임계값은 어떻게 설정하나요?  
5. 6개월 동안 에스컬레이션 비율이 8%에서 22%로 증가했습니다. 세 가지 원인과 각각의 해결 방안을 진단해 보세요.  

> **번역 규칙 적용 사항**  
> - "cascade", "pre-route", "escalation rate" 등 기술 용어는 원어(영어)를 괄호 안에 병기하지 않고 한국어 표현만 사용 (예: "캐스케이드", "사전 라우팅", "에스컬레이션 비율")  
> - "logprobs"는 API 응답 필드명으로 간주해 번역하지 않음  
> - "ship"은 제품 출시 컨텍스트에서 "출시"로 번역

| 용어 | 사람들이 말하는 표현 | 실제 의미 |
|------|----------------|------------------------|
| 모델 라우팅 | "cost broker" | 요청별 동적 모델 선택 |
| 모델 캐스케이드 | "cheap-first escalate" | 저렴한 모델 실행, 낮은 신뢰도 시 프론티어 모델로 전환 |
